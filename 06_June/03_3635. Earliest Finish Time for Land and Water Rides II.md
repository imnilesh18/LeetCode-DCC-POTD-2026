# 3635. Earliest Finish Time for Land and Water Rides II

**Difficulty:** Medium 🟡  
**Problem Link:** [LeetCode 3635](https://leetcode.com/problems/earliest-finish-time-for-land-and-water-rides-ii/)  
**Tags:** `Array`, `Two Pointers`, `Binary Search`, `Greedy`, `Sorting`

## 📝 Problem Statement

You are given two categories of theme park attractions: **land rides** and **water rides**.

**Land Rides:**
* `landStartTime[i]` – the earliest time the $i^{th}$ land ride can be boarded.
* `landDuration[i]` – how long the $i^{th}$ land ride lasts.

**Water Rides:**
* `waterStartTime[j]` – the earliest time the $j^{th}$ water ride can be boarded.
* `waterDuration[j]` – how long the $j^{th}$ water ride lasts.

A tourist must experience **exactly one ride from each category**, in either order. 
* A ride may be started at its opening time or any later moment.
* If a ride is started at time `t`, it finishes at time `t + duration`.
* Immediately after finishing one ride, the tourist may board the other (if it is already open) or wait until it opens.

**Goal:** Return the **earliest possible time** at which the tourist can finish both rides.

---

## 🎯 Constraints

* $1 \leq n, m \leq 5 \times 10^4$
* `landStartTime.length == landDuration.length == n`
* `waterStartTime.length == waterDuration.length == m`
* $1 \leq landStartTime[i], landDuration[i], waterStartTime[j], waterDuration[j] \leq 10^5$

## 🧠 Explanation & Approach

To find the minimum time needed to complete both rides, the tourist has two strategic options:
1. Complete a **Land Ride** first, then a **Water Ride**.
2. Complete a **Water Ride** first, then a **Land Ride**.

### 1. Brute-Force Approach
We can systematically try every single pair of land and water rides. For each pair $(i, j)$, we calculate the time taken to perform $i \rightarrow j$ and $j \rightarrow i$. We track the minimum finish time across all possible combinations. 
*This approach takes $O(N \times M)$ time. It works perfectly for small inputs, but fails on stricter constraints where lengths can reach up to $50,000$.*

### 2. Optimized Greedy Approach
We can optimize this purely through a logical observation. If the tourist commits to doing the **First Category** (e.g., Land) before the **Second Category** (e.g., Water), they should logically pick the ride in the First Category that finishes the absolute *earliest*. Doing so inherently minimizes the wait time available before jumping onto the Second Category ride.

**The strategy:**
1. Assume we do **Category 1** first. We find the ride in Category 1 that yields the absolute minimum `startTime + duration`. Let's call this `earliestFinish1`.
2. Given that we finish our first task at `earliestFinish1`, we evaluate all rides in **Category 2**. The start time for the second ride will be `max(earliestFinish1, ride2StartTime)`. We find the minimum of `max(earliestFinish1, ride2StartTime) + ride2Duration` across all rides in Category 2.
3. We do this twice: once assuming **Land** is Category 1, and once assuming **Water** is Category 1.
4. Finally, we return the minimum of the two scenarios. 
*This approach slashes our time complexity to $O(N + M)$ as we evaluate the arrays sequentially rather than in nested pairs.*

### Brute-Force Solution (C++)
<details>

<summary>View O(N*M) Approach</summary>

```cpp
// Intuition: Try all possible combinations of one land ride and one water ride. For each pair, calculate the finish time if we take the land ride first, and the finish time if we take the water ride first. Keep track of the minimum finish time.
// Approach:
// 1. Iterate through every land ride `i` and every water ride `j`.
// 2. For the pair (i, j), calculate the finish time if the land ride is taken first.
// 3. Calculate the finish time if the water ride is taken first.
// 4. Update the overall minimum finish time with the best result from either choice.
// Time Complexity: O(n * m), where n is the number of land rides and m is the number of water rides.
// Space Complexity: O(1), as we are using a few variables to store the minimum finish time.

//Approach-1 (Brute Force - Try all pairs)
//T.C : O(n*m)
//S.C : O(1)
class Solution {
public:
    int earliestFinishTime(vector<int>& landStartTime, vector<int>& landDuration, vector<int>& waterStartTime, vector<int>& waterDuration) {

        int n = landStartTime.size();
        int m = waterStartTime.size();

        int result = INT_MAX; // Initialize to maximum possible value

        for (int i = 0; i < n; i++) {
            for (int j = 0; j < m; j++) {

                // Case 1: Try Land first, then Water
                int land                = landStartTime[i] + landDuration[i];
                int pehleLand_FirWater  = max(land, waterStartTime[j]) + waterDuration[j];
                result                  = min(result, pehleLand_FirWater);

                // Case 2: Try Water first, then Land
                int water              = waterStartTime[j] + waterDuration[j];
                int pehleWater_FirLand = max(water, landStartTime[i]) + landDuration[i];

                result = min(result, pehleWater_FirLand); // Update minimum overall finish time
            }
        }

        return result;
    }
};

/*
*
* Dry Run
*
* landStartTime = [2], landDuration = [4]
* waterStartTime = [6], waterDuration = [3]
*
* Outer loop i=0, Inner loop j=0:
* land = 2 + 4 = 6
* pehleLand_FirWater = max(6, 6) + 3 = 9
* result = min(INT_MAX, 9) = 9
* * water = 6 + 3 = 9
* pehleWater_FirLand = max(9, 2) + 4 = 13
* result = min(9, 13) = 9
*
* Return result = 9
*
*/
```

### Optimized Greedy Solution (C++)
<details>
<summary>View Optimized Greedy Approach</summary>


```cpp
// Intuition: To minimize the total finish time, we should finish the first ride as early as possible. We independently evaluate taking a land ride first vs. a water ride first.
// Approach:
// 1. Create a helper function `findFinishTime` to compute the earliest finish time for taking `start1/duration1` first, then `start2/duration2`.
// 2. Inside the helper, find the absolute earliest finish time among all rides in the first category.
// 3. Using this optimal first finish time, find the earliest possible finish time for the second category.
// 4. Return the minimum of (Land -> Water) and (Water -> Land) sequences.
// Time Complexity: O(n + m), where n is the number of land rides and m is the number of water rides, because we iterate through the arrays sequentially.
// Space Complexity: O(1), as no extra auxiliary data structures are used.

//Approach-2 (Greedy - Pick earliest finishing task first)
//T.C : O(n+m)
//S.C : O(1)
class Solution {
public:

    int findFinishTime(vector<int>& start1, vector<int>& duration1, vector<int>& start2, vector<int>& duration2) {

        int finish1 = INT_MAX; // Store earliest finish time of first activity
        
        // Find earliest finish time for the first activity
        for (int i = 0; i < start1.size(); i++) {
            finish1 = min(finish1, start1[i] + duration1[i]);
        }

        int finish2 = INT_MAX; // Store earliest final finish time
        
        // Find the best second activity based on the first activity's finish time
        for (int i = 0; i < start2.size(); i++) {
            finish2 = min(finish2, max(finish1, start2[i]) + duration2[i]);
        }

        return finish2;
    }

    int earliestFinishTime(vector<int>& landStartTime, vector<int>& landDuration, vector<int>& waterStartTime, vector<int>& waterDuration) {
        
        // Try Land first, then Water
        int pehleLand_FirWater = findFinishTime(landStartTime, landDuration, waterStartTime, waterDuration);

        // Try Water first, then Land
        int pehleWater_FirLand = findFinishTime(waterStartTime, waterDuration, landStartTime, landDuration);

        return min(pehleLand_FirWater, pehleWater_FirLand); // Return the best possible time
    }
};

/*
*
* Dry Run
*
* landStartTime = [2,8], landDuration = [4,1]
* waterStartTime = [6], waterDuration = [3]
*
* Call findFinishTime(land, water):
* - Loop 1 (land): min(INT_MAX, 2+4=6) -> 6, min(6, 8+1=9) -> 6. finish1 = 6.
* - Loop 2 (water): max(6, 6) + 3 = 9. finish2 = min(INT_MAX, 9) = 9.
* pehleLand_FirWater = 9
*
* Call findFinishTime(water, land):
* - Loop 1 (water): min(INT_MAX, 6+3=9) -> 9. finish1 = 9.
* - Loop 2 (land): max(9, 2) + 4 = 13. max(9, 8) + 1 = 10. finish2 = min(13, 10) = 10.
* pehleWater_FirLand = 10
*
* min(9, 10) = 9
* Return 9
*
*/

```

## 🔍 Key Insights & Further Exploration

* **Greedy Principle is Optimal:** If we commit to category A before category B, there's absolutely no benefit in purposely delaying the finish of category A. Minimizing A's finish time directly minimizes or keeps B's start time constant, thereby mathematically guaranteeing the most optimal configuration.
* **Separation of Concerns:** Writing a generic wrapper `findFinishTime()` prevents duplicated logic when swapping the order of land and water rides, resulting in significantly cleaner, more robust code.
