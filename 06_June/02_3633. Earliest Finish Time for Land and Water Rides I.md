## 📝 Problem Statement

You are given two categories of theme park attractions: **land rides** and **water rides**.

* **Land rides:**
  * `landStartTime[i]` – the earliest time the $i^{th}$ land ride can be boarded.
  * `landDuration[i]` – how long the $i^{th}$ land ride lasts.
* **Water rides:**
  * `waterStartTime[j]` – the earliest time the $j^{th}$ water ride can be boarded.
  * `waterDuration[j]` – how long the $j^{th}$ water ride lasts.

A tourist must experience **exactly one ride from each category**, in either order. 
* A ride may be started at its opening time or any later moment.
* If a ride is started at time $t$, it finishes at time $t + \text{duration}$.
* Immediately after finishing one ride, the tourist may board the other (if it is already open) or wait until it opens.

**Objective:** Return the earliest possible time at which the tourist can finish both rides.

## ⚙️ Constraints

- $1 \le n, m \le 100$
- `landStartTime.length == landDuration.length == n`
- `waterStartTime.length == waterDuration.length == m`
- $1 \le \text{landStartTime[i]}, \text{landDuration[i]}, \text{waterStartTime[j]}, \text{waterDuration[j]} \le 1000$

---

## 🧠 Approach & Explanation

The problem asks us to find the absolute minimum time required to complete exactly one land ride and one water ride. The sequence of rides matters, meaning the tourist can choose to take a **Land Ride first and then a Water Ride**, or a **Water Ride first and then a Land Ride**.

### The Core Intuition
If a tourist decides to take a Land Ride first, what is the best strategy? To minimize the overall time, the tourist should greedily pick the land ride that **finishes the earliest**. Once that first ride is finished, the tourist looks at all available water rides. For each water ride, the start time will either be right after the land ride finishes (if the water ride has already opened) or after some waiting time (if the water ride opens later). By evaluating all water rides against this earliest land finish time, we find the best (Land $\rightarrow$ Water) combination.

We apply the exact same logic for the reverse sequence: taking a Water Ride first (picking the one that finishes earliest) and then evaluating all Land Rides. The final answer is simply the minimum of the two scenarios.

## 💻 Solution Code

### 1️⃣ Brute-Force Solution

<summary>Click to view Brute-Force Approach</summary>

```cpp
// Intuition: Try every possible pair of land and water rides to find the absolute minimum finish time.
// Approach: Use nested loops to pair every land ride with every water ride. For each pair, calculate the total time if we do the land ride first, and the total time if we do the water ride first. Keep track of the global minimum.
// Time Complexity: O(n * m) because we iterate through all pairs of land and water rides.
// Space Complexity: O(1) as we only use a few variables to store the minimum time.

class Solution {
public:
    int earliestFinishTime(vector<int>& landStartTime, vector<int>& landDuration, vector<int>& waterStartTime, vector<int>& waterDuration) {

        int n = landStartTime.size();
        int m = waterStartTime.size();

        // Track global minimum time
        int result = INT_MAX;

        // Iterate through all pairs of land and water rides
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < m; j++) {

                // Scenario 1: Land first, then Water
                int land = landStartTime[i] + landDuration[i]; // Finish time of land ride
                int pehleLand_FirWater = max(land, waterStartTime[j]) + waterDuration[j]; // Wait if needed, add water duration
                result = min(result, pehleLand_FirWater);

                // Scenario 2: Water first, then Land
                int water = waterStartTime[j] + waterDuration[j]; // Finish time of water ride
                int pehleWater_FirLand = max(water, landStartTime[i]) + landDuration[i]; // Wait if needed, add land duration
                result = min(result, pehleWater_FirLand);
            }
        }

        return result;
    }
};

/*
*
* Dry Run
*
* Input: landStart=[2,8], landDur=[4,1], waterStart=[6], waterDur=[3]
* * Pair i=0, j=0:
* Land 0 -> Water 0: land = 2+4=6. waterStart is 6. max(6,6)+3 = 9.
* Water 0 -> Land 0: water = 6+3=9. landStart is 2. max(9,2)+4 = 13.
* result = min(INT_MAX, 9, 13) = 9
*
* Pair i=1, j=0:
* Land 1 -> Water 0: land = 8+1=9. waterStart is 6. max(9,6)+3 = 12.
* Water 0 -> Land 1: water = 6+3=9. landStart is 8. max(9,8)+1 = 10.
* result = min(9, 12, 10) = 9
*
* Return 9.
*
*/
```

### 2️⃣ Optimized Solution (Greedy)

<details open>
<summary>Click to view Optimized Approach</summary>

```cpp
// Intuition: To finish both rides as early as possible, the first ride should be completed at the earliest possible time. Once the first ride is done, pick the second ride that gives the earliest overall finish time.
// Approach: 
// 1. Calculate the earliest finish time for the first category.
// 2. Using that finish time, evaluate all rides in the second category and find the minimum possible final finish time (taking waiting time into account).
// 3. Do this for both (Land -> Water) and (Water -> Land) sequences and return the minimum.
// Time Complexity: O(n + m) because we iterate through both arrays separately rather than using nested loops.
// Space Complexity: O(1) as no extra auxiliary space is required.

class Solution {
public:

    // Helper function to calculate the earliest finish time for a specific sequence (Act 1 -> Act 2)
    int findFinishTime(vector<int>& start1, vector<int>& duration1, vector<int>& start2, vector<int>& duration2) {

        // Step 1: Find the absolute earliest finish time for the first activity
        int finish1 = INT_MAX;
        for (int i = 0; i < start1.size(); i++) {
            finish1 = min(finish1, start1[i] + duration1[i]);
        }

        // Step 2: Use the earliest finish time of Act 1 to find the best Act 2
        int finish2 = INT_MAX;
        for (int i = 0; i < start2.size(); i++) {
            // max(finish1, start2[i]) accounts for any waiting time required before Act 2 opens
            finish2 = min(finish2, max(finish1, start2[i]) + duration2[i]);
        }

        return finish2;
    }

    int earliestFinishTime(vector<int>& landStartTime, vector<int>& landDuration, vector<int>& waterStartTime, vector<int>& waterDuration) {
        
        // Scenario 1: Land Ride first, followed by Water Ride
        int pehleLand_FirWater = findFinishTime(landStartTime, landDuration, waterStartTime, waterDuration);

        // Scenario 2: Water Ride first, followed by Land Ride
        int pehleWater_FirLand = findFinishTime(waterStartTime, waterDuration, landStartTime, landDuration);

        // Return the minimum of both possible sequences
        return min(pehleLand_FirWater, pehleWater_FirLand);
    }
};

/*
*
* Dry Run
*
* Input: landStart=[2,8], landDur=[4,1], waterStart=[6], waterDur=[3]
*
* Calling findFinishTime(Land, Water):
* Step 1 (Land): 
* i=0 -> 2+4=6
* i=1 -> 8+1=9
* finish1 = min(6, 9) = 6
* Step 2 (Water):
* i=0 -> max(6, 6) + 3 = 9
* finish2 = 9
* Returns 9
*
* Calling findFinishTime(Water, Land):
* Step 1 (Water):
* i=0 -> 6+3=9
* finish1 = 9
* Step 2 (Land):
* i=0 -> max(9, 2) + 4 = 13
* i=1 -> max(9, 8) + 1 = 10
* finish2 = min(13, 10) = 10
* Returns 10
*
* Result: min(9, 10) = 9
*
*/
```

## 📌 Notes & Key Insights

* **Independence of Choices:** The problem heavily relies on the fact that if we fix the first ride type, picking the one that gets us out the fastest (earliest finish time) will never negatively impact our choices for the second ride type. An earlier finish time for the first ride means we have strictly *more* or *equal* options and less waiting time for the second ride.
* **Greedy Principle:** By decoupling the nested dependency $O(N \times M)$ into two sequential $O(N)$ and $O(M)$ operations, we optimize the solution perfectly. This approach naturally scales up to the limits present in Part II of the problem.
