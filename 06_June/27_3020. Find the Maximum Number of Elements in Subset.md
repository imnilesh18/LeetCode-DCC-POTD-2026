## 📝 Problem Statement

You are given an array of positive integers `nums`. You need to select a subset of `nums` which satisfies the following condition:

You can place the selected elements in a 0-indexed array such that it follows the pattern: 
$[x, x^2, x^4, \dots, x^{k/2}, x^k, x^{k/2}, \dots, x^4, x^2, x]$ 
*(Note that $k$ can be any non-negative power of $2$)*. 

For example, `[2, 4, 16, 4, 2]` and `[3, 9, 3]` follow the pattern while `[2, 4, 8, 4, 2]` does not.

Return the **maximum number of elements in a subset** that satisfies these conditions.

---

## 🚦 Constraints

*   $2 \le \text{nums.length} \le 10^5$
*   $1 \le \text{nums}[i] \le 10^9$

## 🧠 Approach

The core pattern demands a sequence that strictly squares upwards to a "peak" element and then square-roots downwards symmetrically. This requires that every element in the sequence (except the peak) must appear **at least twice** in our initial array. The peak element only needs to appear **at least once**.

1. **Frequency Mapping**: Use a Hash Map to store the frequency of every element in the array. This allows us to easily check if we have enough copies of a number to form the sides of our pattern.
2. **Handling the Edge Case for `1`**: The number `1` is a special case because $1^2 = 1$. If we process it dynamically, it will cause an infinite loop. 
   * If the frequency of `1` is odd (e.g., 3), we can use all of them: `[1, 1, 1]`. 
   * If the frequency is even (e.g., 4), we must leave one out to maintain the single-peak pattern: `[1, 1, 1]`. 
   * Therefore, for `1`, if its frequency is odd, the length is `freq`. If even, the length is `freq - 1`.
3. **Building the Pattern**: Iterate through the unique elements in the map. For each number $x$ (where $x \neq 1$):
   * Check if its frequency is $> 1$. If so, we can use two copies (one for the left side, one for the right side) and add $2$ to our sequence length.
   * Move to the next required number in the pattern by squaring the current number (`curr = curr * curr`).
   * Continue this until we hit a number that does not have a frequency $> 1$.
4. **Validating the Peak**: Once the loop breaks, we must ensure the sequence has a valid peak. 
   * If the last calculated square exists in the map with a frequency of exactly $1$ (or more), it becomes our peak. We add $1$ to our length.
   * If it does **not** exist in the map at all, the last "pair" we added cannot form a complete valid pattern (a pattern cannot end without a single peak). We must subtract $1$ from our length to correct this.

## 💻 Solution Code (Brute Force & Optimized)

### Brute-force Solution (Conceptual)
A brute-force approach would involve generating all possible subsets of the array, sorting them, and verifying if they match the required square/square-root pattern. Given the constraint $N \le 10^5$, generating $2^N$ subsets is completely impractical and will result in a Time Limit Exceeded (TLE).

### Optimized Solution (C++)

```cpp
// Intuition: The problem requires us to find a symmetrical subset pattern that squares upwards to a peak and then square-roots downwards. This means every element before the peak must appear at least twice in our array, while the peak element needs to appear at least once.
// Approach:
// 1. Store the frequencies of all elements in a hash map.
// 2. Handle the edge case for the number `1` separately. Since 1 squared is 1, it forms a pattern with itself. If the count of 1 is odd, we can use all of them. If even, we can use `count - 1`.
// 3. Iterate through the unique elements in the map. For each element (excluding 1), continuously check if its square exists with a frequency > 1.
// 4. If it does, we add 2 to our length (one for the ascending part, one for the descending part) and move to the square.
// 5. When the loop breaks, check if the last square exists with a frequency of at least 1 to act as the peak. If it does, add 1 to the length. If not, subtract 1 from the length because our last added pair doesn't have a valid peak to close the pattern.
// 6. Keep track of the maximum length found across all possible starting numbers.
// Time Complexity: O(N * log(log(M))), where N is the number of elements and M is the maximum element. The inner loop squares the number, growing exponentially, so it runs at most log(log(M)) times.
// Space Complexity: O(N) to store the frequencies of the unique elements in the hash map.

class Solution {
public:
    typedef long long ll;

    int maximumLength(vector<int>& nums) {
        unordered_map<ll, int> mp;

        // Populate the frequency map
        for(int &num : nums) {
            mp[num]++;
        }

        int result = 0;

        // Handle the special edge case for 1s
        if(mp[1] % 2) { 
            result = mp[1]; // Odd frequency: use all 1s
        } else { 
            result = mp[1] - 1; // Even frequency: leave one out to maintain the peak
        }

        // Iterate through all unique numbers in the map
        for(auto &[num, _] : mp) {
            if(num == 1) continue; // Skip 1 as it is already handled

            ll curr = num;
            int len = 0;

            // Build the sequence sides as long as we have pairs available
            while(mp.count(curr) && mp[curr] > 1) {
                len += 2;
                curr = curr * curr;
            }

            // Check if a valid peak element exists
            if(mp.count(curr) == 1) {
                len += 1; // Valid peak found
            } else {
                len -= 1; // No valid peak, discard one element from the last pair
            }

            // Track the maximum length subset found
            result = max(result, len);
        }

        return result;
    }
};

/*
*
* Dry Run
*
* Input: nums = [5,4,1,2,2]
* 
* 1. Frequency Map: {5:1, 4:1, 1:1, 2:2}
* 2. Handle 1s: mp[1] is 1 (odd). result = 1.
* 3. Loop through map elements:
*    - For num = 5:
*      curr = 5, len = 0.
*      mp[5] is 1 (not > 1). While loop skipped.
*      mp[5] is 1, so len += 1 -> len = 1.
*      result = max(1, 1) = 1.
*    - For num = 4:
*      curr = 4, len = 0.
*      mp[4] is 1 (not > 1). While loop skipped.
*      mp[4] is 1, so len += 1 -> len = 1.
*      result = max(1, 1) = 1.
*    - For num = 2:
*      curr = 2, len = 0.
*      mp[2] is 2 (> 1). While loop executes!
*      len += 2 -> len = 2. curr = 2 * 2 = 4.
*      mp[4] is 1 (not > 1). While loop breaks.
*      mp[4] is 1. Valid peak! len += 1 -> len = 3.
*      result = max(1, 3) = 3.
* 
* 4. Final result = 3.
*
*/
```

## 📌 Notes
* Be mindful of integer overflow when continuously squaring the `curr` value. Using `long long` is crucial here because squaring numbers up to $10^9$ will comfortably exceed the standard 32-bit signed integer limit.
* The number `1` is a notorious edge case in problems involving consecutive exponentiation. Always isolate and handle `1` mathematically before entering loops that apply multiplication or powers to avoid infinite execution.

## 🔍 Key Insights
The time complexity might appear as $O(N^2)$ at first glance due to the nested `while` loop inside the `for` loop. However, mathematically, a number $x \ge 2$ being continuously squared ($x \to x^2 \to x^4 \dots$) grows incredibly fast. 
If the maximum number in the array is $M$, the sequence reaches $M$ in $k$ steps where $x^{(2^k)} \le M$. 
Taking $\log_2$ on both sides yields $2^k \le \log_2 M$, and taking it again yields $k \le \log_2(\log_2 M)$. 
Therefore, the inner `while` loop runs at most $O(\log \log M)$ times, making the overall optimal time complexity a highly efficient $O(N \log \log M)$.
