## 📝 Problem Statement

In a linked list of size `n`, where `n` is **even**, the $i^{th}$ node (0-indexed) of the linked list is known as the **twin** of the $(n-1-i)^{th}$ node, if $0 \le i \le (n / 2) - 1$.

For example, if `n = 4`, then node `0` is the twin of node `3`, and node `1` is the twin of node `2`. These are the only nodes with twins for `n = 4`.

The **twin sum** is defined as the sum of a node and its twin.

Given the `head` of a linked list with even length, return the **maximum twin sum** of the linked list.

---

## ⛔ Constraints

* The number of nodes in the list is an **even** integer in the range $[2, 10^5]$.
* $1 \le \text{Node.val} \le 10^5$

## 🚀 Approach

We can solve this problem using three progressive approaches, optimizing for space complexity as we move from Brute Force to the Optimal solution.

<details>
<summary><b>1. Vector/Array Approach (Brute Force)</b></summary>
<br>
Because linked lists do not have random access, we can simply traverse the list and dump all node values into a dynamic array (vector). Once stored, we can easily use a two-pointer technique—one pointer at the start `i = 0` and one at the end `j = n - 1`—to calculate the sum of pairs and capture the maximum.
</details>

<details>
<summary><b>2. Stack Approach (Better Space Utilization)</b></summary>
<br>
A stack naturally reverses the order of elements (LIFO). We can traverse the entire linked list and push every value into a stack. Once populated, we can traverse the linked list again from the `head`, but this time only up to the halfway mark (`N/2`). During this traversal, we pop elements from the stack (which gives us the twins from the end) and sum them with our current pointer to find the maximum. 
</details>

<details>
<summary><b>3. Reversing the Second Half (Optimized Approach)</b></summary>
<br>
To achieve $O(1)$ space complexity, we must perform the operation in-place. 
1. **Find the Middle:** Use the Tortoise and Hare (Slow and Fast pointers) algorithm to find the start of the second half of the linked list.
2. **Reverse the Second Half:** Invert the pointers of the second half of the linked list so it points backwards.
3. **Calculate Max Twin Sum:** Use one pointer at the original `head` and another pointer at the start of the newly reversed second half. Iterate them simultaneously, adding their values to track the maximum twin sum.
</details>

## 💻 Solution Code

### Approach 1: Brute-force Solution (Using Vector)

```cpp
// Intuition: The simplest way to access elements from the end of a linked list is to store them in an array-like structure where we have random access.
// Approach: Traverse the linked list and store all node values in a vector. Use two pointers, one at the start and one at the end, to calculate the sum of twins and find the maximum.
// Time Complexity: O(N) where N is the number of nodes in the linked list.
// Space Complexity: O(N) to store the values in the vector.
class Solution {
public:
    int pairSum(ListNode* head) {
        vector<int> vec; // Store linked list values
        
        // Extract all values
        while(head) {
            vec.push_back(head->val);
            head = head->next;
        }
        
        int i = 0, j = vec.size()-1; // Two pointers for twins
        int result = 0; 
        
        // Find max twin sum from outside inwards
        while(i < j) {
            result = max(result, vec[i] + vec[j]);
            i++;
            j--;
        }
        
        return result;
    }
};

/*
*
* Dry Run
* Input: head = [5,4,2,1]
* Vector becomes [5, 4, 2, 1]
* i = 0, j = 3: vec[0] + vec[3] = 5 + 1 = 6. Max = 6
* i = 1, j = 2: vec[1] + vec[2] = 4 + 2 = 6. Max = 6
* Loop ends. Output 6.
*
*/
```

### Approach 2: Better Solution (Using Stack)

```cpp
// Intuition: Since twins pair the first elements with the last elements, a stack's LIFO (Last-In-First-Out) property perfectly matches the requirement of pairing from the back.
// Approach: Push all node values onto a stack. Traverse the first half of the linked list while simultaneously popping from the stack to add the current node with its twin from the end.
// Time Complexity: O(N) for traversing the list twice (once for pushing, half for calculating).
// Space Complexity: O(N) to store elements in the stack.
class Solution {
public:
    int pairSum(ListNode* head) {
        stack<int> st; 
        ListNode* curr = head;
        
        // Push all node values into stack
        while(curr) {
            st.push(curr->val);
            curr = curr->next;
        }
        
        curr = head; // Reset to start
        int count = 1;
        int N = st.size();
        int result = 0;
        
        // Traverse first half of the list
        while(count <= N/2) {
            // curr->val is from start, st.top() is twin from end
            result = max(result, curr->val + st.top());
            curr = curr->next;
            st.pop();
            count++;
        }
        
        return result;
    }
};

/*
*
* Dry Run
* Input: head = [4,2,2,3]
* Stack: [4, 2, 2, 3] (Top is 3)
* count=1: max(0, 4 + 3) = 7. curr moves to 2, stack pops 3
* count=2: max(7, 2 + 2) = 7. curr moves to 2, stack pops 2
* Loop ends (count reaches 3 > N/2). Output 7.
*
*/
```

### Approach 3: Optimized Solution (Reversing Second Half)

```cpp
// Intuition: We can optimize space complexity by modifying the linked list in place. Reversing the second half allows us to traverse both halves forward simultaneously to find the twin sums.
// Approach: Use slow and fast pointers to find the middle. Reverse the second half of the list. Iterate through the first half and the reversed second half, summing pairs to find the maximum.
// Time Complexity: O(N) as we traverse the list to find mid, reverse half, and iterate through half.
// Space Complexity: O(1) as we modify pointers in place without extra data structures.
class Solution {
public:
    int pairSum(ListNode* head) {
        ListNode* mid = NULL;
        
        ListNode* slow = head;
        ListNode* fast = head;
        
        // Find middle of the linked list
        while(fast && fast->next) {
            slow = slow->next;
            fast = fast->next->next;
        }
        mid = slow; 
        
        // Reversing the second half
        ListNode* nextNode = NULL;
        ListNode* prev = NULL;
        while(mid) {
            nextNode = mid->next; // Store next node
            mid->next = prev;     // Reverse pointer
            prev = mid;           // Move prev forward
            mid = nextNode;       // Move mid forward
        }
        
        ListNode* curr = head;
        int result = 0;
        
        // Traverse both halves for max sum
        while(prev) {
            result = max(result, curr->val + prev->val);
            curr = curr->next;
            prev = prev->next;
        }
        
        return result;
        
    }
};

/*
*
* Dry Run
* Input: head = [5,4,2,1]
* Find Mid: slow stops at 2 (mid).
* Reverse from Mid: [2,1] becomes [1,2]. prev points to 1.
* Calculate: 
*   - curr=5, prev=1. Sum=6. Max=6.
*   - curr=4, prev=2. Sum=6. Max=6.
* Loop ends. Output 6.
*
*/
```

## 📌 Notes
- This problem is an excellent test of foundational Linked List manipulation techniques, specifically finding the middle node and reversing a sub-section in-place.
- Be mindful of edge cases, though the constraints gracefully state the list length is always even and has a minimum size of 2.

## 🧠 Key Insights
- **The Twin Definition:** The twin of index $i$ is exactly symmetrically opposite to it from the back. Thus, accessing twins means accessing the list sequentially from the front and sequentially from the back at the same time.
- **Why $O(1)$ space is superior:** Relying on extra arrays or stacks allocates dynamic memory, which causes slight overhead. Manipulating pointers directly utilizes existing memory and proves deeper mastery of list mechanics.
