## 📜 Problem Statement

There is an undirected tree with `n` nodes labeled from `1` to `n`, rooted at node `1`. The tree is represented by a 2D integer array `edges` of length `n - 1`, where `edges[i] = [ui, vi]` indicates that there is an edge between nodes `ui` and `vi`.

Initially, all edges have a weight of `0`. You must assign each edge a weight of either `1` or `2`. 
The **cost** of a path between any two nodes `u` and `v` is the total weight of all edges in the path connecting them.

Select any one node `x` at the **maximum depth**. Return the number of ways to assign edge weights in the path from node `1` to `x` such that its total cost is **odd**.

Since the answer may be large, return it modulo $10^9 + 7$.

> **Note:** Ignore all edges not in the path from node `1` to `x`.

---

## 🚦 Constraints

* $2 \le n \le 10^5$
* `edges.length == n - 1`
* `edges[i] == [ui, vi]`
* $1 \le ui, vi \le n$
* `edges` represents a valid tree.

## 🧠 Key Insights & Approach

<details>
<summary><b>Click to expand the logical derivation</b></summary>

1.  **Identify the Core Task:** 
    The problem asks us to find the number of valid weight assignments on a path from the root (Node 1) to a node at the maximum depth, such that the sum of the weights is odd. The first step is to calculate this maximum depth, let's call it $d$.

2.  **Graph Traversal for Depth ($d$):**
    Since we are dealing with an undirected tree (meaning nodes can have multiple neighbors, not just left/right children), we can't use standard binary tree depth functions directly. We must build an Adjacency List. We run a Depth First Search (DFS) starting from Node 1. To prevent traversing backward to the parent node and causing an infinite loop, we track the `parent` node in our DFS state.

3.  **Combinatorics and Symmetries:**
    Once we know the maximum depth $d$, we know there are exactly $d$ edges in the targeted path. 
    *   Each edge can take one of 2 values: `1` or `2`.
    *   Total possible weight combinations for the path = $2^d$.

    Now, out of these $2^d$ combinations, how many yield an *odd* sum? 
    *   Consider any combination of weights that gives an **even** sum. If you change exactly one weight in that path (e.g., flip a `1` to a `2` or a `2` to a `1`), the overall sum immediately changes from even to odd.
    *   Similarly, flipping one edge in an odd-sum path makes it even.
    *   Because every even-sum combination can be uniquely mapped to an odd-sum combination by flipping the first edge, there is a perfect 1-to-1 symmetry.
    *   Exactly half of the $2^d$ combinations will be even, and exactly half will be odd!

4.  **Final Formula:**
    The number of odd combinations is exactly $\frac{2^d}{2} = 2^{d-1}$.

5.  **Handling Large Numbers:**
    Because $d$ can be up to $10^5$, computing $2^{d-1}$ directly will cause integer overflow. We use **Binary Exponentiation** to compute $2^{d-1} \pmod{10^9 + 7}$ efficiently in $O(\log d)$ time.

</details>

## 💻 Solution Code

```cpp
// Intuition: The problem requires finding the number of weight assignments (using 1 or 2) on a path of length 'd' such that the sum is odd. Since flipping any single edge weight changes the parity of the total sum, exactly half of all possible 2^d assignments will have an odd sum. Thus, the answer is always 2^(d-1).
// Approach: 
// 1. Build an adjacency list from the given undirected edges.
// 2. Perform a DFS from the root (node 1) to find the maximum depth 'd'. Pass the parent node to avoid backward traversal.
// 3. Return the result of 2^(d-1) modulo 10^9 + 7 using binary exponentiation for efficiency.
// Time Complexity: O(n) for DFS tree traversal to find max depth + O(log n) for binary exponentiation. Overall Time Complexity is O(n).
// Space Complexity: O(n) to store the adjacency list representing the tree and O(n) for the recursion stack in the worst-case skewed tree. Overall Space Complexity is O(n).

class Solution {
public:
    int M = 1e9 + 7;
    typedef long long ll;

    // Binary Exponentiation to handle large powers under modulo
    ll power(ll base, ll exponent) {
        if(exponent == 0) {
            return 1;
        }

        // Recursively calculate half power
        ll half = power(base, exponent / 2);
        
        // Square the half result and apply modulo
        ll result = (half * half) % M;

        // If exponent is odd, multiply by base one more time
        if(exponent % 2 == 1) { 
            result = (result * base) % M;
        }
        
        return result;
    }

    // DFS to find the maximum depth of the tree
    int getMaxDepth(unordered_map<int, vector<int>> &adj, int node, int parent) {
        int depth = 0;
        
        // Traverse all neighbors of the current node
        for(int &ngbr : adj[node]) {
            // Prevent going backward to the parent node
            if(ngbr == parent) continue;

            // Recursively find depth of children and track the maximum
            depth = max(depth, getMaxDepth(adj, ngbr, node) + 1);
        }

        return depth; 
    }

    int assignEdgeWeights(vector<vector<int>>& edges) {
        // Build Adjacency List for the undirected tree
        unordered_map<int, vector<int>> adj; 

        for(auto &edge : edges) {
            int u = edge[0];
            int v = edge[1];

            // Add undirected edges
            adj[u].push_back(v);
            adj[v].push_back(u);
        }

        // Find max depth starting from root node 1. Root has no parent (-1)
        int d = getMaxDepth(adj, 1, -1); 

        // Result is 2^(d-1) % (10^9 + 7)
        return power(2, d - 1); 
    }
};

/*
 *
 * Dry Run
 * 
 * Input: edges = [[1,2],[1,3],[3,4],[3,5]]
 * 
 * 1. Build Adjacency List:
 *    1 -> [2, 3]
 *    2 -> [1]
 *    3 -> [1, 4, 5]
 *    4 -> [3]
 *    5 -> [3]
 * 
 * 2. Calculate Max Depth (Start DFS at node 1, parent -1):
 *    - node 1 visits neighbor 2:
 *      - node 2 has no valid children (1 is parent). Returns 0.
 *      - depth from 2 = 0 + 1 = 1.
 *    - node 1 visits neighbor 3:
 *      - node 3 visits neighbor 4:
 *        - node 4 has no valid children (3 is parent). Returns 0.
 *        - depth from 4 = 0 + 1 = 1.
 *      - node 3 visits neighbor 5:
 *        - node 5 has no valid children (3 is parent). Returns 0.
 *        - depth from 5 = 0 + 1 = 1.
 *      - Max depth from node 3 is max(1, 1) = 1. 
 *      - depth from 3 to 1 = 1 + 1 = 2.
 *    - Max depth at node 1 'd' = max(1, 2) = 2.
 * 
 * 3. Calculate 2^(d-1) % (10^9 + 7):
 *    - d = 2
 *    - power(2, 2-1) -> power(2, 1)
 *    - returns 2 % 1000000007 = 2
 * 
 * Output: 2
 *
 */

```

## 📌 Notes & Further Exploration

* **Parity Properties:** This problem is an elegant demonstration of how combinatorial properties and parity (even/odd) symmetries can instantly collapse an exponential problem $O(2^n)$ down to a simple mathematical formula $O(1)$ math computation.
* **Tree Characteristics:** Unlike binary trees which have strictly defined `left` and `right` pointers, standard graphs/trees represented by edge lists require adjacency lists to traverse. Passing the `parent` reference in the recursive function is the standard, $O(1)$ space method to avoid infinite recursion without maintaining a `visited` array.
