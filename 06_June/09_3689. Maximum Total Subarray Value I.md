# 3689. Maximum Total Subarray Value I

**Difficulty:** Medium  
**Tags:** `Array`, `Greedy`, `Weekly Contest 468`  

---

## 📖 Problem Statement

You are given an integer array `nums` of length `n` and an integer `k`.

You need to choose exactly `k` non-empty subarrays `nums[l..r]` of `nums`. Subarrays may overlap, and the exact same subarray (same `l` and `r`) can be chosen more than once.

The value of a subarray `nums[l..r]` is defined as:
**`max(nums[l..r]) - min(nums[l..r])`**

The total value is the sum of the values of all chosen subarrays. Return the maximum possible total value you can achieve.

---

## 🎯 Examples

| Input | Output | Explanation |
| :--- | :--- | :--- |
| `nums = [1,3,2], k = 2` | `4` | One optimal approach is:<br>Choose `nums[0..1] = [1, 3]`. Value: `3 - 1 = 2`.<br>Choose `nums[0..2] = [1, 3, 2]`. Value: `3 - 1 = 2`.<br>Adding these gives `2 + 2 = 4`. |
| `nums = [4,2,5,1], k = 3` | `12` | One optimal approach is:<br>Choose `nums[0..3] = [4, 2, 5, 1]`. Value: `5 - 1 = 4`.<br>Choose `nums[0..3] = [4, 2, 5, 1]`. Value: `4`.<br>Choose `nums[2..3] = [5, 1]`. Value: `4`.<br>Adding these gives `4 + 4 + 4 = 12`. |

---

## 🚦 Constraints

* `1 <= n == nums.length <= 5 * 10^4`
* `0 <= nums[i] <= 10^9`
* `1 <= k <= 10^5`

---

## 🧠 Approach

The problem contains a crucial, slightly hidden hint in its description: *"Subarrays may overlap, and the exact same subarray can be chosen more than once."* 

We are asked to maximize the total sum of `max - min` across `k` subarrays. To maximize the difference `max(subarray) - min(subarray)`, we need the largest possible maximum and the smallest possible minimum. 

If we consider the **entire array** as our single subarray, it is guaranteed to contain the absolute global maximum and the absolute global minimum of all elements. Therefore, the difference between the maximum and minimum elements of the entire array will give us the maximum possible value any subarray could possibly yield.

Because the problem explicitly allows us to pick the *exact same subarray* multiple times, the most optimal strategy is to simply select the entire array `k` times. 

**Steps:**
1. Traverse the array to find the global maximum element (`maxEl`).
2. Traverse the array to find the global minimum element (`minEl`).
3. Calculate the max possible difference for one subarray: `maxEl - minEl`.
4. Multiply this difference by `k` to account for picking it `k` times.
5. Return the result (make sure to cast to a 64-bit integer, e.g., `long long` in C++, to avoid overflow since $10^9 \times 10^5 = 10^{14}$).

<summary><b>💻 Brute-Force Solution (C++)</b></summary>

```cpp
// Intuition: Generate all possible subarrays, calculate their values, and greedily pick the top 'k' values.
// Approach: Use nested loops to generate every subarray, find the max and min for each, store the differences, sort them descending, and sum the top 'k'.
// Time Complexity: O(n^3) or O(n^2) depending on optimization, which will Time Limit Exceed (TLE).
// Space Complexity: O(n^2) to store subarray values.

class Solution {
public:
    long long maxTotalValue(vector<int>& nums, int k) {
        vector<long long> values;
        int n = nums.size();
        
        // Generate all subarrays
        for (int i = 0; i < n; i++) {
            int currentMax = nums[i];
            int currentMin = nums[i];
            for (int j = i; j < n; j++) {
                currentMax = max(currentMax, nums[j]);
                currentMin = min(currentMin, nums[j]);
                values.push_back(currentMax - currentMin);
            }
        }
        
        // Sort to get the maximum values
        sort(values.rbegin(), values.rend());
        
        long long totalValue = 0;
        // The problem allows picking the same subarray multiple times,
        // so we actually just take the top 1 value and multiply by k.
        totalValue = values[0] * k; 
        
        return totalValue;
    }
};

/*
*
* Dry Run
*
* Input: nums = [1, 3, 2], k = 2
* Subarrays generated and their (max - min) values:
* [1] -> 0
* [1, 3] -> 2
* [1, 3, 2] -> 2
* [3] -> 0
* [3, 2] -> 1
* [2] -> 0
* Maximum value is 2. Multiply by k (2) -> 4.
* Output: 4
*
*/

```

## 🚀 Optimized Solution Code

```cpp
// Intuition: The maximum difference between any two elements in a subarray is bounded by the global max and global min of the entire array. Reusing the whole array 'k' times yields the absolute maximum sum.
// Approach: Iterate through the array exactly once to find both the global maximum and the global minimum. Then, subtract the minimum from the maximum and multiply the result by 'k'.
// Time Complexity: O(n) because we traverse the array only once to find the minimum and maximum.
// Space Complexity: O(1) as we only use a few integer variables for tracking.

class Solution {
public:
    long long maxTotalValue(vector<int>& nums, int k) {
        int maxEl = INT_MIN; // Store global maximum
        int minEl = INT_MAX; // Store global minimum

        for(int &num : nums) {
            maxEl = max(maxEl, num); // Update max
            minEl = min(minEl, num); // Update min
        }

        // Multiply by 1LL to ensure 64-bit integer calculation
        return 1LL*(maxEl - minEl)*k;
    }
};

/*
*
* Dry Run
*
* Input: nums = [4, 2, 5, 1], k = 3
*
* maxEl = INT_MIN, minEl = INT_MAX
*
* num = 4: maxEl = 4, minEl = 4
* num = 2: maxEl = 4, minEl = 2
* num = 5: maxEl = 5, minEl = 2
* num = 1: maxEl = 5, minEl = 1
*
* Loop finishes. maxEl = 5, minEl = 1
* Calculation: 1LL * (5 - 1) * 3
* 1LL * 4 * 3 = 12
*
* Output: 12
*
*/

```

## 💡 Key Insights

* The phrase "exact same subarray can be chosen more than once" reduces this from a complex Dynamic Programming or Sliding Window problem to a pure Greedy/Math observation problem. Always read constraints and special conditions carefully!
* The highest possible difference in any subset/subarray of a collection will always be defined by the absolute maximum and absolute minimum elements of that collection.
