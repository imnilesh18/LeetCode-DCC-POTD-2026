## 📝 Problem Statement

**Problem Link**: [Partition Array According to Given Pivot](https://leetcode.com/problems/partition-array-according-to-given-pivot/)

You are given a **0-indexed** integer array `nums` and an integer `pivot`. Rearrange `nums` such that the following conditions are satisfied:

1. Every element less than `pivot` appears **before** every element greater than `pivot`.
2. Every element equal to `pivot` appears **in between** the elements less than and greater than `pivot`.
3. The **relative order** of the elements less than `pivot` and the elements greater than `pivot` is maintained.
   * More formally, consider every $p_i, p_j$ where $p_i$ is the new position of the $i^{th}$ element and $p_j$ is the new position of the $j^{th}$ element. If $i < j$ and both elements are smaller (or larger) than `pivot`, then $p_i < p_j$.

Return `nums` after the rearrangement.

---

## 🎯 Constraints

* `1 <= nums.length <= 10^5`
* `-10^6 <= nums[i] <= 10^6`
* `pivot` equals to an element of `nums`.

## 🧠 Approach

Based on the problem requirements, we need to maintain the relative order of the elements (making the partition stable) while sorting them into three distinct groups: `< pivot`, `== pivot`, and `> pivot`.

Here are three incremental approaches to solve this:

### 1️⃣ Brute Force (Using Extra Space of 3 Vectors)
The most beginner-friendly and intuitive approach is to explicitly group the elements.
* Create three separate arrays (or vectors): `lessPivot`, `equalPivot`, and `greaterPivot`.
* Iterate over the `nums` array and push each element into its respective bucket based on its relation to the `pivot`.
* Finally, concatenate all three buckets together to form the result.
* **Drawback**: This uses three extra vectors which increases memory overhead, even though it runs in linear time.

### 2️⃣ Better (Using Counting & Pre-calculated Indices)
We can optimize the space to just the single required `result` array.
* First, iterate through the array to find the exact count of elements that are strictly less than the `pivot` (`countLess`) and equal to the `pivot` (`countEqual`).
* With these counts, we know exactly where each group should start in the new array:
  * Elements `< pivot` will start at index `0`.
  * Elements `== pivot` will start at index `countLess`.
  * Elements `> pivot` will start at index `countLess + countEqual`.
* We initialize three pointers (`i`, `j`, `k`) pointing to these starting positions.
* Iterate through `nums` a second time, placing each element into the `result` array using the corresponding pointer and incrementing it.

### 3️⃣ Optimized (Two Pointers from Both Ends)
We can fill the `result` array from the outside in, avoiding the need for initial counting.
* Create a `result` array of the same size.
* Use a pointer `i` traversing from left-to-right (`0` to `n-1`) to find elements `< pivot`. We place these at the beginning of the `result` array using a write-pointer `i_`.
* Simultaneously, use a pointer `j` traversing from right-to-left (`n-1` to `0`) to find elements `> pivot`. We place these at the end of the `result` array (starting from the back) using a write-pointer `j_`.
* After finishing the traversal, the unwritten middle portion of the `result` array naturally belongs to the elements `== pivot`. We just fill the remaining gap between `i_` and `j_` with the `pivot` value.

## 💻 Solution Code

<details>
<summary><b>1️⃣ Brute Force Solution (C++)</b></summary>

```cpp
// Intuition: The simplest way to group elements is to separate them into different buckets based on their relation to the pivot and then combine them.
// Approach:
// 1. Create three separate arrays: lessPivot, equalPivot, and greaterPivot.
// 2. Iterate through the input array and place each element into the appropriate bucket.
// 3. Append the equal and greater arrays to the less array.
// 4. Return the combined less array.
// Time Complexity: O(n) - We iterate through the array a few times, which simplifies to O(n) time.
// Space Complexity: O(n) - We use three additional arrays to store the elements, requiring O(n) extra space.
class Solution {
public:
    vector<int> pivotArray(vector<int>& nums, int pivot) {
        int n = nums.size();

        // Separate vectors to hold grouped elements
        vector<int> lessPivot;
        vector<int> equalPivot;
        vector<int> greaterPivot;

        // Pass 1: Segregate elements into 3 buckets
        for(int i = 0; i < n; i++) {
            if(nums[i] < pivot) {
                lessPivot.push_back(nums[i]);
            } else if(nums[i] == pivot) {
                equalPivot.push_back(nums[i]);
            } else {
                greaterPivot.push_back(nums[i]);
            }
        }

        // Pass 2: Merge the buckets into 'lessPivot'
        for(int &num : equalPivot) {
            lessPivot.push_back(num);
        }

        for(int &num : greaterPivot) {
            lessPivot.push_back(num);
        }

        return lessPivot;

    }
};

/*
*
* Dry Run
* nums = [9,12,5,10,14,3,10], pivot = 10
*
* 1st pass (separate):
* lessPivot = [9, 5, 3]
* equalPivot = [10, 10]
* greaterPivot = [12, 14]
*
* 2nd pass (combine):
* lessPivot + equalPivot = [9, 5, 3, 10, 10]
* lessPivot + greaterPivot = [9, 5, 3, 10, 10, 12, 14]
*
* Result: [9, 5, 3, 10, 10, 12, 14]
*
*/
```

<summary><b>2️⃣ Better Solution (Pointers & Counting) (C++)</b></summary>

```cpp
// Intuition: We can avoid extra arrays by counting elements first. Knowing the counts allows us to pre-calculate the exact starting index for each category in the result array.
// Approach:
// 1. Iterate through the array to count elements strictly less than and exactly equal to the pivot.
// 2. Calculate the starting indices: 'less' starts at 0, 'equal' starts after all 'less' elements, 'greater' starts after 'less' + 'equal'.
// 3. Iterate again, placing each element into its correctly calculated index in the result array and incrementing the respective pointer.
// Time Complexity: O(n) - Two passes through the array.
// Space Complexity: O(n) - For the result array (O(1) auxiliary space if the returned array is not counted).
class Solution {
public:
    vector<int> pivotArray(vector<int>& nums, int pivot) {
        int n = nums.size();

        int countless  = 0;
        int countequal = 0;

        // Pass 1: Count elements less than and equal to pivot
        for(int &num : nums) {
            if(num < pivot)
                countless++;
            else if(num == pivot)
                countequal++;
        }

        // Set starting indices for the 3 categories
        int i = 0; //less than pivot
        int j = countless; //equal to pivot
        int k = countless + countequal; //more than pivot
        vector<int> result(n);

        // Pass 2: Place elements directly into their calculated positions
        for(int &num : nums) {
            if(num < pivot) {
                result[i] = num;
                i++;
            } else if(num == pivot) {
                result[j] = num;
                j++;
            } else {
                result[k] = num;
                k++;
            }
        }
        
        return result;

    }
};

/*
*
* Dry Run
* nums = [9,12,5,10,14,3,10], pivot = 10
*
* Pass 1:
* countless = 3 (for 9, 5, 3)
* countequal = 2 (for 10, 10)
*
* Pointers Setup:
* i (less) = 0
* j (equal) = 3
* k (greater) = 5
*
* Pass 2:
* 9 (<10) -> result[0]=9, i=1
* 12 (>10) -> result[5]=12, k=6
* 5 (<10) -> result[1]=5, i=2
* 10 (==10) -> result[3]=10, j=4
* 14 (>10) -> result[6]=14, k=7
* 3 (<10) -> result[2]=3, i=3
* 10 (==10) -> result[4]=10, j=5
*
* Result: [9, 5, 3, 10, 10, 12, 14]
*
*/
```

<summary><b>3️⃣ Optimized Solution (Two Pointers from Ends) (C++)</b></summary>

```cpp
// Intuition: Elements less than the pivot naturally belong on the left, and elements greater belong on the right. We can place them simultaneously using pointers from both ends.
// Approach:
// 1. Initialize two pointers for reading `nums` (start and end) and two pointers for writing to `result` (start and end).
// 2. Iterate while the left pointer is within bounds. If left element < pivot, place it at the left write pointer. If right element > pivot, place it at the right write pointer.
// 3. The remaining middle section in `result` inherently belongs to elements equal to the pivot. Fill them in.
// Time Complexity: O(n) - Single pass to place smaller/larger elements, and a loop to fill the middle, resulting in linear time.
// Space Complexity: O(n) - For the result array (O(1) auxiliary space).
class Solution {
public:
    vector<int> pivotArray(vector<int>& nums, int pivot) {
        int n = nums.size();

        vector<int> result(n);

        //original nums read pointers
        int i = 0;
        int j = n-1;

        //result write pointers
        int i_ = 0;
        int j_ = n-1;

        // Pass 1: Fill left with smaller elements, right with larger elements
        while(i < n && j >= 0) {
            // Write elements strictly less than pivot to the left side
            if(nums[i] < pivot) {
                result[i_] = nums[i];
                i_++;
            }

            // Write elements strictly greater than pivot to the right side
            if(nums[j] > pivot) {
                result[j_] = nums[j];
                j_--;
            }

            i++;
            j--;
        }

        //i_ to j_ -> fill the pivot
        // Pass 2: Fill the remaining middle section with the pivot element
        while(i_ <= j_) {
            result[i_] = pivot;
            i_++;
        }

        return result;

    }
};

/*
*
* Dry Run
* nums = [9,12,5,10,14,3,10], pivot = 10, n = 7
* result = [0,0,0,0,0,0,0]
*
* Pass 1:
* i=0, j=6: nums[0]=9 (<10) -> result[0]=9. nums[6]=10 (not >10). i_=1, j_=6
* i=1, j=5: nums[1]=12 (not <10). nums[5]=3 (not >10). i_=1, j_=6
* i=2, j=4: nums[2]=5 (<10) -> result[1]=5. nums[4]=14 (>10) -> result[6]=14. i_=2, j_=5
* i=3, j=3: nums[3]=10. nums[3]=10. i_=2, j_=5
* i=4, j=2: nums[4]=14. nums[2]=5. i_=2, j_=5
* i=5, j=1: nums[5]=3 (<10) -> result[2]=3. nums[1]=12 (>10) -> result[5]=12. i_=3, j_=4
* i=6, j=0: nums[6]=10. nums[0]=9. i_=3, j_=4
*
* Pass 2 (Fill Pivot):
* i_=3, j_=4
* Fill result[3] and result[4] with 10.
*
* Result: [9, 5, 3, 10, 10, 12, 14]
*
*/
```

## 🔑 Key Insights

- **Stable Partitioning**: Unlike the standard QuickSort partition algorithm which swaps elements in-place and destroys relative ordering, this problem requires a *stable* partition. 
- **Space-Time Tradeoff**: Achieving an in-place (O(1) extra space), stable partition in O(n) time is complex. By allowing O(n) extra space for a `result` array, the logic simplifies drastically.
- **Two Pointers Mastery**: The 3rd approach showcases a brilliant use of dual two-pointers (one pair for reading `i, j` and one pair for writing `i_, j_`). Reading from opposite ends simultaneously ensures that elements `< pivot` populate cleanly from left-to-right, and elements `> pivot` populate backwards right-to-left while inherently preserving their relative orders!
