## 📝 Problem Statement

**Problem Link:** [Left and Right Sum Differences](https://leetcode.com/problems/left-and-right-sum-differences/description/)

Given a **0-indexed** integer array `nums` of size `n`, define two arrays `leftSum` and `rightSum` where:

*   `leftSum[i]` is the sum of elements to the left of the index `i` in the array `nums`. If there is no such element, `leftSum[i] = 0`.
*   `rightSum[i]` is the sum of elements to the right of the index `i` in the array `nums`. If there is no such element, `rightSum[i] = 0`.

Return an integer array `answer` of size `n` where `answer[i] = |leftSum[i] - rightSum[i]|`.

---

## 🛑 Constraints

*   `1 <= nums.length <= 1000`
*   `1 <= nums[i] <= 10^5`

## 💡 Approach

### 1. Brute Force (Simulation)
The problem explicitly asks us to calculate the left sum and the right sum for every index. A straightforward simulation involves creating two arrays of size `n` (`leftSum` and `rightSum`). We populate `leftSum` by iterating from left to right, taking cumulative sums. We populate `rightSum` by iterating from right to left. Finally, we loop through and find the absolute difference. This takes **O(N) Time** and **O(N) Space**.

### 2. Optimized (Constant Space)
As explained in the video walkthrough, we can optimize the auxiliary space down to **O(1)**. 
Instead of maintaining two separate arrays, we can do the following:
1. Find the `total_sum` of the entire array.
2. Maintain a `leftSum` variable initialized to `0`.
3. Traverse the array. Since the total sum represents all elements, subtracting the current element (`nums[i]`) from the `total_sum` dynamically turns `total_sum` into the `rightSum` for the current index.
4. Calculate the absolute difference between `leftSum` and the newly updated `total_sum` (which acts as our right sum).
5. Add the current element `nums[i]` to `leftSum` so it is ready for the next iteration.

## 💻 Solution Code

<details>
<summary><b>Brute-force Solution (C++)</b></summary>

```cpp
// Time Complexity: O(N)
// Space Complexity: O(N)
class Solution {
public:
    vector<int> leftRightDifference(vector<int>& nums) {
        int n = nums.size();
        vector<int> leftSum(n, 0);
        vector<int> rightSum(n, 0);
        vector<int> ans(n);
        
        // Calculate prefix sums
        for (int i = 1; i < n; i++) {
            leftSum[i] = leftSum[i - 1] + nums[i - 1];
        }
        
        // Calculate suffix sums
        for (int i = n - 2; i >= 0; i--) {
            rightSum[i] = rightSum[i + 1] + nums[i + 1];
        }
        
        // Calculate absolute difference
        for (int i = 0; i < n; i++) {
            ans[i] = abs(leftSum[i] - rightSum[i]);
        }
        
        return ans;
    }
};
```

**Optimized Solution (C++)**

```cpp
// Intuition: To find the absolute difference between left and right sums without using extra space, we can precalculate the total sum. Subtracting elements on the fly dynamically tracks the right sum, while a running variable tracks the left sum.
// Approach:
// 1. Calculate the overall sum of the array.
// 2. Initialize `leftSum` to 0.
// 3. Iterate through the array. For each element, subtract it from `sum` (which updates `sum` to act as the `rightSum`).
// 4. Calculate the absolute difference between `leftSum` and the updated `sum`, storing it in the result array.
// 5. Add the current element to `leftSum` to prep for the next iteration.
// Time Complexity: O(N) where N is the length of the array, since we iterate through the array twice (once for accumulate, once for the loop).
// Space Complexity: O(1) auxiliary space. We only use a few integer variables. The output array is excluded from extra space complexity.

// Approach (Linear Scan with running prefix and suffix sums)
// T.C : O(n)
// S.C : O(1) , excluding the output array
class Solution {
public:
    vector<int> leftRightDifference(vector<int>& nums) {
        int n = nums.size(); // Get the size of the array

        // Calculate the total sum of all elements in the array
        int sum = accumulate(begin(nums), end(nums), 0);

        vector<int> result(n); // Vector to store the absolute differences
        int leftSum = 0;       // Running sum of elements to the left

        for(int i = 0; i < n; i++) {
            // Deduct current element to make 'sum' act as the rightSum
            sum -= nums[i];

            // Store absolute difference of left and right sums
            result[i] = abs(leftSum - sum);

            // Add current element to leftSum for the next iteration
            leftSum += nums[i];
        }

        return result;
    }
};

/*
*
* Dry Run
* 
* Input: nums = [10, 4, 8, 3]
* Initialization:
* sum = 10 + 4 + 8 + 3 = 25
* leftSum = 0
* 
* Iteration 1 (i = 0, nums[0] = 10):
* sum = 25 - 10 = 15 (Right sum for index 0)
* result[0] = abs(0 - 15) = 15
* leftSum = 0 + 10 = 10
* 
* Iteration 2 (i = 1, nums[1] = 4):
* sum = 15 - 4 = 11 (Right sum for index 1)
* result[1] = abs(10 - 11) = 1
* leftSum = 10 + 4 = 14
* 
* Iteration 3 (i = 2, nums[2] = 8):
* sum = 11 - 8 = 3 (Right sum for index 2)
* result[2] = abs(14 - 3) = 11
* leftSum = 14 + 8 = 22
* 
* Iteration 4 (i = 3, nums[3] = 3):
* sum = 3 - 3 = 0 (Right sum for index 3)
* result[3] = abs(22 - 0) = 22
* leftSum = 22 + 3 = 25
* 
* Output: [15, 1, 11, 22]
*
*/
```

## 🔍 Key Insights
* **Running State:** We don't always need to store calculated values in entirely new arrays. Often, traversing the array while maintaining a "running state" (like `leftSum` and `rightSum`) eliminates the need for `O(N)` auxiliary space.
* **Math Trick:** `Total Sum = Left Sum + Current Element + Right Sum`. Using this mathematical identity allows us to dynamically infer the `Right Sum` simply via `Total Sum - Current Element - Left Sum`. In our optimized code, we cleverly repurposed the initial `total_sum` variable to serve as the shrinking `Right Sum`.

## 📓 Notes
* Keep in mind that the returned array does not count toward your algorithmic space complexity. Your interviewer will look for the `O(1)` auxiliary space approach as the optimal answer.
* Be cautious with edge cases such as an array of size 1. The optimized logic naturally handles `n=1` because `sum` reduces to 0, `leftSum` is 0, and the difference is correctly calculated as `0`.
