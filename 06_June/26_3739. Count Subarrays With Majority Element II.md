## 📝 Problem Statement

You are given an integer array `nums` and an integer `target`.

Return the number of subarrays of `nums` in which `target` is the **majority element**.

> The **majority element** of a subarray is the element that appears strictly more than half of the times in that subarray.

### Constraints:
- `1 <= nums.length <= 10^5`
- `1 <= nums[i] <= 10^9`
- `1 <= target <= 10^9`

## 🧠 Explanation & Key Insights

This problem is the Hard version of Part I, increasing the constraint of `N` from `1,000` to `100,000`. This strictly requires an **O(N)** approach, meaning nested loops (O(N²)) or map-based overhead operations might result in a Time Limit Exceeded (TLE).

### 1. The Transformation (+1 / -1)
A subarray has the `target` as its majority element if the frequency of `target` is strictly greater than `size / 2`. 
To simplify this mathematical check, we map the elements:
- If `nums[i] == target`, treat it as **`+1`**.
- If `nums[i] != target`, treat it as **`-1`**.

If the sum of this mapped subarray is **strictly greater than 0**, the `target` outnumbers all other elements combined, making it the absolute majority.

### 2. The Prefix Sum Observation
For a subarray starting at index `i` and ending at `j`, its sum is:
`prefixSum[j] - prefixSum[i-1] > 0`

Rewriting this, we get:
**`prefixSum[i-1] < prefixSum[j]`**

This means at any index `j`, the number of valid subarrays ending at `j` is exactly equal to the number of times we have seen a `prefixSum` strictly smaller than our current `prefixSum[j]` in the past.

### 3. The Optimal O(N) Strategy (Avoiding Map Overhead)
Instead of searching backwards for smaller prefix sums (which takes O(N²)), we can maintain a running count called **`validLeftPoints`**:
- **When we see `target` (`+1`)**: Our `currSum` increases. All previous occurrences of our *old* `currSum` are now strictly smaller than our *new* `currSum`. Thus, we **add** the frequency of the old `currSum` to `validLeftPoints`.
- **When we see non-`target` (`-1`)**: Our `currSum` decreases. Some previous prefix sums that were strictly smaller are no longer smaller (they are now equal to our new `currSum`). Thus, we **subtract** the frequency of the new `currSum` from `validLeftPoints`.

**Crucial Optimization for Part II:** Instead of an `unordered_map` (which has expensive hash lookups and can cause TLE for `10^5`), we use an **array of size `2N + 1`**. Because our running sum can only swing between `-N` and `+N`, we offset our array index by `N` to handle negative numbers in `O(1)` contiguous memory!

## 🚀 Solution Codes

### <details><summary>Brute-force Solution (TLE for Part II)</summary>

```cpp
// Intuition: We can transform the array into +1 for the target and -1 for others. Then, the problem reduces to finding subarrays with a sum strictly greater than 0.
// Approach: Transform the array in-place, calculate the prefix sums, and then use two nested loops to check every possible (l, r) subarray bound.
// Time Complexity: O(n^2) due to checking every pair of boundaries. This will give TLE for N = 10^5.
// Space Complexity: O(1) auxiliary space as we modify the given array directly.
class Solution {
public:
    long long countMajoritySubarrays(vector<int>& nums, int target) {
        int n = nums.size();

        // Step 1: Transform array to +1 and -1
        for(int i = 0; i < n; i++) {
            if(nums[i] == target)
                nums[i] = 1;
            else
                nums[i] = -1;
        }

        // Step 2: Compute cumulative prefix sums
        for(int i = 1; i < n; i++) {
            nums[i] += nums[i-1];
        }

        long long ans = 0;

        // Step 3: Check all possible subarrays
        for (int l = 0; l < n; l++) {
            for (int r = l; r < n; r++) {
                int sum = nums[r] - ((l > 0) ? nums[l-1] : 0);

                if(sum > 0)
                    ans++;
            }
        }

        return ans;
    }
};

/*
*
* Dry Run
* Input: nums = [1,2,2], target = 2
* Transformation: [-1, 1, 1]
* Prefix Sums: [-1, 0, 1]
* -> Evaluates every pair (l, r) sequentially
* -> ans updates on valid > 0 sums
* -> Returns 4
*
*/

```

### Optimized Solution (O(N) Shifted Array)

```cpp
// Intuition: We need to count how many previous prefix sums are strictly smaller than the current one in O(1) time per step.
// Approach: We track `validLeftPoints` which dynamically stores the count of strictly smaller past prefix sums. We use an offset array `freq` instead of a hash map to guarantee pure O(1) insertions and lookups without hashing overhead. Max bounds for the sum are [-N, N], so an array of size 2N+1 offset by N handles everything perfectly.
// Time Complexity: O(n) as we traverse the array exactly once and all operations inside the loop take O(1) time.
// Space Complexity: O(n) for the frequency array of size 2*N + 1.
class Solution {
public:
    long long countMajoritySubarrays(vector<int>& nums, int target) {
        int n = nums.size();
        
        // We use an array instead of unordered_map to prevent TLE.
        // Prefix sum bounds are [-n, +n], total 2*n + 1 possibilities.
        // We offset by 'n' to store negative sums at positive indices.
        vector<int> freq(2 * n + 1, 0);
        
        int currSum = 0;
        freq[n + 0] = 1; // Base case: cumulative sum 0 has occurred once
        
        long long validLeftPoints = 0;
        long long result = 0;
        
        for (int x : nums) {
            if (x == target) {
                // Sum increases. The old sum becomes strictly smaller than our new sum.
                // Therefore, all past occurrences of our old sum are now valid.
                validLeftPoints += freq[n + currSum];
                currSum++;
            } else {
                // Sum decreases. Our new sum is no longer strictly smaller than itself.
                // We must subtract its frequency from our valid count.
                currSum--;
                validLeftPoints -= freq[n + currSum];
            }
            
            // Register the newly formed prefix sum
            freq[n + currSum]++;
            
            // Accumulate valid subarrays ending at the current element
            result += validLeftPoints;
        }
        
        return result;
    }
};

/*
*
* Dry Run
* Input: nums = [1,2,2,3], target = 2
* n = 4, freq array size = 9, offset = 4.
* Initial state: freq[4+0] = 1, currSum = 0, validLeftPoints = 0
* * Iteration 1 (x = 1):
* - x != 2, so currSum decreases to -1
* - validLeftPoints -= freq[4-1] (which is 0) -> validLeftPoints = 0
* - freq[3] becomes 1
* - result += 0 -> result = 0
*
* Iteration 2 (x = 2):
* - x == 2, validLeftPoints += freq[4-1] (1) -> validLeftPoints = 1
* - currSum increases to 0
* - freq[4] becomes 2
* - result += 1 -> result = 1
*
* Iteration 3 (x = 2):
* - x == 2, validLeftPoints += freq[4+0] (2) -> validLeftPoints = 3
* - currSum increases to 1
* - freq[5] becomes 1
* - result += 3 -> result = 4
*
* Iteration 4 (x = 3):
* - x != 2, currSum decreases to 0
* - validLeftPoints -= freq[4+0] (2) -> validLeftPoints = 1
* - freq[4] becomes 3
* - result += 1 -> result = 5
*
* Output = 5
*
*/

```
