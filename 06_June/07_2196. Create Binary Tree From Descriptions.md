# 2196. Create Binary Tree From Descriptions

**Difficulty:** 🟡 Medium  
**Topics:** `Array`, `Hash Table`, `Tree`, `Binary Tree`  
**Problem Link:** [LeetCode 2196](https://leetcode.com/problems/create-binary-tree-from-descriptions/)

## 📖 Problem Statement

You are given a 2D integer array `descriptions` where `descriptions[i] = [parent_i, child_i, isLeft_i]` indicates that `parent_i` is the parent of `child_i` in a binary tree of unique values. Furthermore:
* If `isLeft_i == 1`, then `child_i` is the left child of `parent_i`.
* If `isLeft_i == 0`, then `child_i` is the right child of `parent_i`.

Construct the binary tree described by `descriptions` and return its root.

*The test cases will be generated such that the binary tree is valid.*

## ⚠️ Constraints

* $1 \le \text{descriptions.length} \le 10^4$
* $\text{descriptions}[i].\text{length} == 3$
* $1 \le \text{parent}_i, \text{child}_i \le 10^5$
* $0 \le \text{isLeft}_i \le 1$
* The binary tree described by `descriptions` is valid.

---

## 💡 Approach & Key Insights

To construct the tree and find its root, we can break the problem down into two simple steps using Hash Maps and Hash Sets:

1.  **Constructing the Nodes and Edges:**
    *   We don't know the root initially, and nodes can be given in any order.
    *   We use a Hash Map (`unordered_map`) to store the mapping between a node's integer value and its actual `TreeNode*` in memory.
    *   As we iterate over the `descriptions`, we check if the parent and child nodes exist. If they don't, we dynamically allocate them and add them to our map.
    *   Based on the `isLeft` flag, we connect the child to the parent's left or right pointer.

2.  **Finding the Root:**
    *   A fundamental property of a tree's root is that **it is never a child of any node**.
    *   We can maintain a Hash Set (`unordered_set`) to keep track of every node that appears as a `child`.
    *   Once the tree is built, we iterate through the `descriptions` one more time. The `parent` that is *not* found in our `childSet` is guaranteed to be the root of our tree.

## 💻 Solution Code

```cpp
// Intuition: The root of a binary tree is the only node that is never a child. We can use a map to create/store nodes and a set to track children.
// Approach:
// 1. Traverse descriptions to build the tree. Create nodes and link them based on the isLeft flag.
// 2. Track all child nodes in a Hash Set.
// 3. The node value that appears as a parent but never as a child in the set is our root.
// 4. Return the root node from our map.
// Time Complexity: O(n), where n is the size of descriptions. We iterate through the array twice.
// Space Complexity: O(n), for storing the map of nodes and the set of children.

class Solution {
public:
    TreeNode* createBinaryTree(vector<vector<int>>& descriptions) {
        unordered_map<int, TreeNode*> mp;
        unordered_set<int> childSet; // Tracks nodes that have a parent

        // Step 1: Construct the tree
        for(vector<int>& vec : descriptions) {
            int parent = vec[0];
            int child  = vec[1];
            int isLeft = vec[2];

            // Create parent node if it doesn't exist
            if(mp.find(parent) == mp.end()) {
                mp[parent] = new TreeNode(parent);
            }

            // Create child node if it doesn't exist
            if(mp.find(child) == mp.end()) {
                mp[child] = new TreeNode(child);
            }

            // Link the child to the parent
            if(isLeft == 1) { 
                mp[parent]->left = mp[child];
            } else {
                mp[parent]->right = mp[child];
            }

            // Mark as a child
            childSet.insert(child);
        }

        // Step 2: Find the root
        for(vector<int>& vec : descriptions) { 
            int parent = vec[0];
            // The root is the only node not present in the childSet
            if(childSet.find(parent) == childSet.end()) { 
                return mp[parent];
            }
        }

        return NULL; // Fallback, logically unreachable given valid tree inputs
    }
};

/*
*
* Dry Run
*
* Input: descriptions = [[20,15,1], [20,17,0], [50,20,1]]
* 
* Initialization:
* mp = {}
* childSet = {}
* 
* Iteration 1 (Building Tree):
* 1. vec = [20,15,1]
*    - Create Node 20 and Node 15.
*    - Link 20 -> left = 15.
*    - childSet = {15}
*    - mp = {20: Node(20), 15: Node(15)}
* 
* 2. vec = [20,17,0]
*    - Node 20 exists, create Node 17.
*    - Link 20 -> right = 17.
*    - childSet = {15, 17}
*    - mp = {20, 15, 17}
* 
* 3. vec = [50,20,1]
*    - Create Node 50, Node 20 exists.
*    - Link 50 -> left = 20.
*    - childSet = {15, 17, 20}
*    - mp = {20, 15, 17, 50}
* 
* Iteration 2 (Finding Root):
* 1. vec = [20,15,1] -> Is 20 in childSet? Yes.
* 2. vec = [20,17,0] -> Is 20 in childSet? Yes.
* 3. vec = [50,20,1] -> Is 50 in childSet? No. 
* 
* 50 is not anyone's child. It is the root!
* Return mp[50].
*
*/

```
