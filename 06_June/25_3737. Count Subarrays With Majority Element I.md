## 📝 Problem Statement

You are given an integer array `nums` and an integer `target`.

Return the number of subarrays of `nums` in which `target` is the **majority element**.

> The **majority element** of a subarray is the element that appears strictly more than half of the times in that subarray.

### Constraints:
- `1 <= nums.length <= 1000`
- `1 <= nums[i] <= 10^9`
- `1 <= target <= 10^9`

## 🧠 Key Insights & Approach

### 1. The Core Transformation (+1 / -1)
A subarray has the `target` as its majority element if the frequency of `target` is strictly greater than `size / 2`. 
Instead of tracking frequencies and dividing the size, we can transform the problem:
- Assign **`+1`** whenever we see the `target`.
- Assign **`-1`** whenever we see a non-`target` element.

If the sum of this transformed subarray is **strictly greater than 0**, it means the `target` has offset all other elements, making it the majority element.

### 2. Prefix Sum Logic
For any subarray from index `i` to `j`, its sum is given by:
`prefixSum[j] - prefixSum[i-1] > 0`

Rewriting this inequality:
`prefixSum[i-1] < prefixSum[j]`

This tells us: at any index `j`, the number of valid subarrays ending at `j` is exactly equal to the number of times we have seen a `prefixSum` strictly smaller than the current `prefixSum[j]` in the past!

### 3. The Optimal Strategy (O(n) Map)
Instead of searching backward every time, we maintain a running `currSum` and keep a count of **valid left points**:
- If `nums[i] == target`, `currSum` increases by 1. Previous prefix sums that were equal to our old `currSum` are now strictly smaller than our new `currSum`. Thus, we add `map[currSum]` to our `validLeftPoints`.
- If `nums[i] != target`, `currSum` decreases by 1. Some previous sums that were strictly smaller might no longer be strictly smaller. Thus, we subtract `map[currSum]` (the new sum) from `validLeftPoints`.
- We add `validLeftPoints` to our total `result` and increment the frequency of our new `currSum` in the map.

## 🚀 Solution Codes

### <details><summary>Approach 1: Brute Force Counting</summary>

```cpp
// Intuition: The simplest way is to check all possible subarrays. We can fix a starting point and iterate through all possible ending points, calculating the majority count on the fly.
// Approach: Use two nested loops. The outer loop fixes the start index 'i'. The inner loop expands the subarray to 'j', incrementing the count if we see 'target', else decrementing. If the count is > 0, it means 'target' is the majority.
// Time Complexity: O(n^2) where n is the size of the array. We evaluate every possible subarray.
// Space Complexity: O(1) as we only use a few variables for counting.
class Solution {
public:
    int countMajoritySubarrays(vector<int>& nums, int target) {
        int n = nums.size();
        int result = 0;

        // Iterate over all possible starting points
        for (int i = 0; i < n; ++i) {
            int count = 0; // Tracks the majority score (+1 for target, -1 for others)

            // Iterate over all possible ending points
            for (int j = i; j < n; ++j) {
                // Update score on the fly
                count += (nums[j] == target ? 1 : -1);

                // If score > 0, target is strictly in the majority
                if (count > 0) {
                    result++;
                }

            }
        }

        return result;
    }
};

/*
*
* Dry Run
* Input: nums = [1,2,2], target = 2
* * i = 0 (val 1)
* j = 0 (val 1): count = -1 (result = 0)
* j = 1 (val 2): count = 0  (result = 0)
* j = 2 (val 2): count = 1  (result = 1) -> valid subarray [1,2,2]
* * i = 1 (val 2)
* j = 1 (val 2): count = 1  (result = 2) -> valid subarray [2]
* j = 2 (val 2): count = 2  (result = 3) -> valid subarray [2,2]
* * i = 2 (val 2)
* j = 2 (val 2): count = 1  (result = 4) -> valid subarray [2]
* * Total valid subarrays = 4
*
*/

```

### Approach 2: Prefix Sum Brute Force

```cpp
// Intuition: We can transform the array into +1 for the target and -1 for others. Then, the problem reduces to finding subarrays with a sum strictly greater than 0.
// Approach: First, transform the array elements to 1 (if target) and -1 (otherwise). Compute the prefix sum of this array. Finally, use nested loops to find the sum of any subarray [l..r] in O(1) time and count if it's > 0.
// Time Complexity: O(n^2) to iterate through all pairs of (l, r).
// Space Complexity: O(1) if we modify the given array in-place.
class Solution {
public:
    int countMajoritySubarrays(vector<int>& nums, int target) {
        int n = nums.size();

        // Transform the array: 1 if target, -1 otherwise
        for(int i = 0; i < n; i++) {
            if(nums[i] == target)
                nums[i] = 1;
            else
                nums[i] = -1;
        }

        // Compute cumulative prefix sums in-place
        for(int i = 1; i < n; i++) {
            nums[i] += nums[i-1];
        }

        long long ans = 0;

        // Check sum of all subarrays using the prefix sums
        for (int l = 0; l < n; l++) {
            for (int r = l; r < n; r++) {
                // Subarray sum formula: prefix[r] - prefix[l-1]
                int sum = nums[r] - ((l > 0) ? nums[l-1] : 0);

                if(sum > 0)
                    ans++;
            }
        }

        return ans;

    }
};

/*
*
* Dry Run
* Input: nums = [1,2,2], target = 2
* Transformation: nums -> [-1, 1, 1]
* Prefix Sums:    nums -> [-1, 0, 1]
* * l = 0
* r = 0: sum = nums[0] - 0 = -1 (not > 0)
* r = 1: sum = nums[1] - 0 = 0 (not > 0)
* r = 2: sum = nums[2] - 0 = 1 (> 0) -> ans = 1
* l = 1
* r = 1: sum = nums[1] - nums[0] = 0 - (-1) = 1 (> 0) -> ans = 2
* r = 2: sum = nums[2] - nums[0] = 1 - (-1) = 2 (> 0) -> ans = 3
* l = 2
* r = 2: sum = nums[2] - nums[1] = 1 - 0 = 1 (> 0) -> ans = 4
* * Output = 4
*
*/

```

### Approach 3: Optimal Using Map

```cpp
// Intuition: Using the +1/-1 transformation, we need to find how many previous prefix sums are strictly less than the current prefix sum. Maintaining a running count of such valid left points avoids nested loops.
// Approach: Use a map to store the frequency of each cumulative sum. Track 'validLeftPoints', which is the count of prefix sums strictly smaller than 'currSum'. If we see 'target', 'currSum' increases, and previous sums equal to the old 'currSum' become valid. If not 'target', 'currSum' decreases, and we lose some valid points. Update 'result' with 'validLeftPoints'.
// Time Complexity: O(n) as we traverse the array once and unordered_map operations are O(1) on average.
// Space Complexity: O(n) in the worst case to store the frequencies in the map.
class Solution {
public:
    long long countMajoritySubarrays(vector<int>& nums, int target) {
        //Instead of map you can use an array of size 2*n+1 as well because we can have cumulaive sum from -n to +n)
        unordered_map<int,int> mp;   // mp[cumSum] = frequency of how many times cumSum has occurred
        
        int currSum = 0;
        
        mp[0] = 1; //we have seen cumSum = 0 in the beginning once

        long long validLeftPoints = 0;
        long long result = 0;

        for (int x : nums) {
            // Target seen: sum increases
            if (x == target) {
                // The old sum becomes strictly smaller than the new sum
                validLeftPoints += mp[currSum];
                currSum++;
            } 
            // Non-target seen: sum decreases
            else {
                currSum--;
                // The new sum is no longer strictly smaller than itself
                validLeftPoints -= mp[currSum];
            }

            // Register the new cumulative sum
            mp[currSum]++;

            // Accumulate valid subarrays ending at the current element
            result += validLeftPoints;
        }

        return result;
    }
};

/*
*
* Dry Run
* Input: nums = [1,2,2], target = 2
* Initial: mp = {0: 1}, currSum = 0, validLeftPoints = 0, result = 0
* * Iteration 1 (x = 1): 
* x != target, so currSum-- -> currSum = -1
* validLeftPoints -= mp[-1] (0) -> validLeftPoints = 0
* mp[-1]++ -> mp = {0:1, -1:1}
* result += 0 -> result = 0
* * Iteration 2 (x = 2):
* x == target, validLeftPoints += mp[-1] (1) -> validLeftPoints = 1
* currSum++ -> currSum = 0
* mp[0]++ -> mp = {0:2, -1:1}
* result += 1 -> result = 1
* * Iteration 3 (x = 2):
* x == target, validLeftPoints += mp[0] (2) -> validLeftPoints = 3
* currSum++ -> currSum = 1
* mp[1]++ -> mp = {0:2, -1:1, 1:1}
* result += 3 -> result = 4
* * Output = 4
*
*/

```
