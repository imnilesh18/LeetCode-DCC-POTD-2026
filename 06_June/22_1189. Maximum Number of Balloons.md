## 📌 Problem Statement

Given a string `text`, you want to use the characters of `text` to form as many instances of the word **"balloon"** as possible.

You can use each character in `text` at most once. Return the maximum number of instances that can be formed.

*(Note: This question is the same as 2287: Rearrange Characters to Make Target String.)*

---

## 🚦 Constraints

- `1 <= text.length <= 10^4`
- `text` consists of lower case English letters only.

---

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :---: | :--- |
| **1** | `text = "nlaebolko"` | `1` | We can form one instance of "balloon" using the available characters. |
| **2** | `text = "loonbalxballpoon"` | `2` | We can form two complete instances of "balloon". |
| **3** | `text = "leetcode"` | `0` | Not enough characters to form even a single "balloon". |

---

## 🧠 Approach

**Intuition:**
The word "balloon" requires a specific composition of characters: 1 `b`, 1 `a`, 2 `l`'s, 2 `o`'s, and 1 `n`. The core logic revolves around identifying the limiting factor (bottleneck). We count how many of each required letter we have in the text. Because `l` and `o` are required twice per word, their available counts must be halved. The minimum value among these counts will dictate the maximum number of times we can construct the word. 

**Steps:**
1. Create a frequency array to count the occurrences of all lowercase English letters in the given `text`.
2. Extract the counts for the specific characters: `'b'`, `'a'`, `'l'`, `'o'`, and `'n'`.
3. Divide the counts of `'l'` and `'o'` by 2, since each instance of the word "balloon" consumes two of these characters.
4. The maximum instances we can form is simply the minimum count among these five essential characters.

---

## 💻 Solution Code

```cpp
// Intuition: The maximum number of times we can form "balloon" is limited by the scarcest required character.
// Approach: Count all character frequencies, halve the counts of 'l' and 'o' (since they appear twice), and find the minimum count among the required letters.
// Time Complexity: O(n), where n is the length of the string `text`. We traverse the string exactly once.
// Space Complexity: O(1), since we only use a fixed-size array of 26 integers regardless of input size.

class Solution {
public:
    int maxNumberOfBalloons(string text) { 
        // Array to store frequency of each lowercase letter
        int t[26] = {0};
        
        // Count occurrences of every character in the text
        for(const char& ch : text) {
            t[ch-'a']++;
        }
        
        int count = INT_MAX;
        
        // Find the bottleneck character count limiting the word formation
        count = min(count, t['b'-'a']);     // 'b' requires 1 char
        count = min(count, t['a'-'a']);     // 'a' requires 1 char
        count = min(count, t['l'-'a']/2);   // 'l' requires 2 chars
        count = min(count, t['o'-'a']/2);   // 'o' requires 2 chars
        count = min(count, t['n'-'a']);     // 'n' requires 1 char
        
        return count;
    }
};

/*
*
* Dry Run
* * Input: text = "nlaebolko"
* * Frequency Array Building:
* 'b': 1, 'a': 1, 'l': 2, 'o': 2, 'n': 1, 'e': 1, 'k': 1
* * Bottleneck Calculation:
* count = INT_MAX
* check 'b': min(INT_MAX, 1) = 1
* check 'a': min(1, 1) = 1
* check 'l': min(1, 2/2) = min(1, 1) = 1
* check 'o': min(1, 2/2) = min(1, 1) = 1
* check 'n': min(1, 1) = 1
* * Final count = 1
* Output: 1
*
*/
```

## 📝 Notes

- Using a fixed size array (`int t[26]`) is optimal and much faster than using a dynamic Hash Map structure like `std::unordered_map`.
- Initializing `count` to `INT_MAX` is a standard way to find the minimum out of a sequence of bounds smoothly.

## 🔍 Key Insights

The problem perfectly illustrates the **Bottleneck Principle** in competitive programming. Just like a chemical reaction is limited by the limiting reagent, string formation queries are restricted by the rarest required character (taking into account the quantity of the character needed per word).
