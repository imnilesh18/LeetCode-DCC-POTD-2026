## 📝 Problem Statement

A shop is selling candies at a discount. For every two candies sold, the shop gives a **third candy for free**.

The customer can choose any candy to take away for free as long as the cost of the chosen candy is **less than or equal to the minimum cost of the two candies bought**.

For example, if there are `4` candies with costs `1, 2, 3, and 4`, and the customer buys candies with costs `2` and `3`, they can take the candy with cost `1` for free, but not the candy with cost `4`.

Given a **0-indexed** integer array `cost`, where `cost[i]` denotes the cost of the $i^{th}$ candy, return the **minimum cost** of buying all the candies.

---

### 🛡️ Constraints
- `1 <= cost.length <= 100`
- `1 <= cost[i] <= 100`

---

### 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `cost = [1,2,3]` | `5` | We buy candies with costs `2` and `3`, taking cost `1` for free. Total: `2 + 3 = 5`. |
| **2** | `cost = [6,5,7,9,2,2]` | `23` | Buy `9` & `7` (Free `6`). Buy `5` & `2` (Free `2`). Total: `9 + 7 + 5 + 2 = 23`. |
| **3** | `cost = [5,5]` | `10` | Only 2 candies exist. We buy both. Total: `5 + 5 = 10`. |

---

## 🧠 Approach & Explanation

To achieve the **minimum cost**, we need to get the **maximum possible discount**. The discount rule allows us to get a free candy as long as its cost is less than or equal to the minimum of the two candies we just purchased.

If we buy the two *most expensive* candies, their minimum will be as high as possible, giving us the freedom to claim the *next most expensive* candy completely for free. 

**Steps:**
1. **Sort the array in descending order**: This guarantees that the largest elements come first. 
2. **Greedy Selection**: Pick the first two largest values (buy them), and take the third largest value (free). Continue this pattern.
3. **Mathematical Check**: While iterating through the sorted array, any candy at an index where `index % 3 == 2` represents our 3rd candy (the free one). We simply skip adding its cost to our total. 

---

## 💻 Solution Code

```cpp
// Intuition: To maximize savings, pair the most expensive candies together. This allows us to claim the next most expensive candy for free.
// Approach: 
// 1. Sort the array in descending order.
// 2. Iterate through the array and accumulate costs.
// 3. Skip every 3rd candy (index % 3 == 2) since it represents the free candy we are eligible for.
// Time Complexity: O(n log n) due to the sorting operation. Traversing the array takes O(n).
// Space Complexity: O(1) as we are only using an integer to keep track of the total cost (ignoring the space used by the sorting algorithm).

class Solution {
public:
    int minimumCost(vector<int>& cost) {
        int n = cost.size();
        
        // Sort the array in descending order
        sort(begin(cost), end(cost), greater<int>());

        int total = 0;

        // Iterate through the sorted candies
        for (int i = 0; i < n; i++) {
            // Add cost if it's NOT the 3rd candy (indices 2, 5, 8...)
            if (i % 3 != 2)
                total += cost[i];
        }
        
        return total;
    }
};

/*
*
* Dry Run
* Input: cost = [6, 5, 7, 9, 2, 2]
* * After descending sort: [9, 7, 6, 5, 2, 2]
* * Iteration:
* i = 0 | cost[0] = 9 | 0 % 3 != 2 (Buy)  | total = 0 + 9 = 9
* i = 1 | cost[1] = 7 | 1 % 3 != 2 (Buy)  | total = 9 + 7 = 16
* i = 2 | cost[2] = 6 | 2 % 3 == 2 (Free) | total remains 16
* i = 3 | cost[3] = 5 | 3 % 3 != 2 (Buy)  | total = 16 + 5 = 21
* i = 4 | cost[4] = 2 | 4 % 3 != 2 (Buy)  | total = 21 + 2 = 23
* i = 5 | cost[5] = 2 | 5 % 3 == 2 (Free) | total remains 23
* * Final Output: 23
*
*/
```
