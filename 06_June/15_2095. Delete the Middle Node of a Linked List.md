## 📝 Problem Statement

You are given the `head` of a linked list. Delete the **middle node**, and return the `head` of the modified linked list.

The middle node of a linked list of size `n` is the `⌊n / 2⌋`th node from the start using **0-based indexing**, where `⌊x⌋` denotes the largest integer less than or equal to `x`.

> **Note:** For `n = 1, 2, 3, 4, and 5`, the middle nodes are `0, 1, 1, 2, and 2`, respectively.

## 💡 Approach

<details>
<summary><b>Click here to expand the approach explanation</b></summary>

Finding the middle of a linked list is a classic problem perfectly suited for the **Tortoise and Hare (Two Pointers)** algorithm. 

1. **Standard Two Pointer Approach (Tracking Previous):** 
   We initialize two pointers, `slow` and `fast`, both starting at the `head`. `slow` moves one step at a time while `fast` moves two steps. By the time `fast` reaches the end of the list, `slow` will be exactly at the middle node. Since we need to *delete* the middle node, we also maintain a `prevSlow` pointer that stays one step behind `slow`. Once the middle is found, we simply link `prevSlow->next` to `slow->next` and safely delete `slow`.

2. **Optimized Two Pointer Approach (Head Start):**
   Instead of using a third `prevSlow` variable, we can simply give the `fast` pointer a "head start". By initializing `slow` at `head` and `fast` at `head->next->next`, the `slow` pointer will finish its traversal exactly one node *before* the middle node. This allows us to directly execute `slow->next = slow->next->next` to skip the target node.
</details>

## 💻 Solution Code

### 🐌 Brute Force / Standard Approach (Using Previous Pointer)

```cpp
// Intuition: The "Tortoise and Hare" algorithm allows us to find the middle of a linked list in a single pass. By keeping track of the node just before the middle, we can easily bypass the middle node to delete it.
// Approach: Initialize `slow` and `fast` pointers at `head`, and track the previous node with `prevSlow`. Move `slow` by 1 and `fast` by 2 until `fast` hits the end. Delete the node at `slow` by adjusting `prevSlow->next`.
// Time Complexity: O(N), where N is the number of nodes. We traverse the linked list exactly once.
// Space Complexity: O(1), as we are only allocating three extra pointers regardless of list size.
class Solution {
public:
    ListNode* deleteMiddle(ListNode* head) {
        ListNode* prevSlow  = NULL; // Tracks node just before slow
        ListNode* slow      = head; // Moves 1 step
        ListNode* fast      = head; // Moves 2 steps
        
        // Traverse to find the middle node
        while(fast && fast->next) {
            prevSlow = slow;
            slow     = slow->next;
            
            fast     = fast->next->next;
        }
        
        // Base case: list had only 1 node
        if(prevSlow == NULL) {
            return NULL;
        }
        
        // Bypass the middle node
        prevSlow->next = slow->next;
        delete(slow); // Free the memory
        
        return head;
    }
};

/*
* Dry Run
* Input: [1, 3, 4, 7, 1, 2, 6]
* Init: prevSlow=NULL, slow=1, fast=1
* Loop 1: prevSlow=1, slow=3, fast=4
* Loop 2: prevSlow=3, slow=4, fast=1
* Loop 3: prevSlow=4, slow=7, fast=6
* End Loop: fast->next is NULL. slow is at 7 (middle).
* Update: prevSlow->next (4->next) = slow->next (1).
* Return: [1, 3, 4, 1, 2, 6]
*/
```

### 🚀 Optimized Approach (No Previous Pointer)

```cpp
// Intuition: We can eliminate the need for a `prev` pointer by giving the `fast` pointer a head start. This guarantees the `slow` pointer stops exactly one node *before* the middle.
// Approach: Handle base cases first. Initialize `slow` at `head` and `fast` at `head->next->next`. Move them at their respective speeds. Once `fast` reaches the end, `slow` sits right before the target, allowing direct deletion.
// Time Complexity: O(N), making a single linear scan through the nodes.
// Space Complexity: O(1), requiring strictly constant extra space.
class Solution {
public:
    ListNode* deleteMiddle(ListNode* head) {
        // Base case: if 0 or 1 node, return NULL
        if(!head || !head->next)
            return NULL;
        
        ListNode* slow      = head;             // Will stop right before the middle
        ListNode* fast      = head->next->next; // Given a head start
        
        // Traverse the list
        while(fast && fast->next) {
            slow     = slow->next;
            
            fast     = fast->next->next;
        }
        
        // slow is now exactly before the middle node
        ListNode* toDelete = slow->next;
        slow->next = slow->next->next; // Bypass the middle node
        delete(toDelete);              // Safely free the memory
        
        return head;
    }
};

/*
* Dry Run
* Input: [1, 2, 3, 4]
* Init: slow=1, fast=3
* Loop 1: slow=2, fast=NULL
* End Loop: fast is NULL. slow is at 2 (node right before middle 3).
* Update: toDelete = 3. slow->next (2->next) = slow->next->next (4).
* Return: [1, 2, 4]
*/
```

## 🧠 Key Insights

* **Pointer manipulation is paramount:** In linked lists, safely maintaining reference chains before deletion ensures no memory leaks or dangling pointers.
* **The "Head Start" trick:** Adjusting the starting position of the fast pointer by one or two nodes is a powerful technique to shift where your slow pointer ultimately lands. 
* **Edge Cases Matter:** Always ensure your code gracefully handles lists of size 1 and 2, which are the most common trapdoors in linked list problems.
