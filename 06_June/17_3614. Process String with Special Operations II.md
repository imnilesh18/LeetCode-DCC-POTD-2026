## 📝 Problem Statement

You are given a string `s` consisting of lowercase English letters and the special characters: `'*'`, `'#'`, and `'%'`. You are also given an integer `k`.

Build a new string `result` by processing `s` according to the following rules from left to right:
1. If the letter is a lowercase English letter, append it to `result`.
2. A `'*'` removes the last character from `result`, if it exists.
3. A `'#'` duplicates the current `result` and appends it to itself.
4. A `'%'` reverses the current `result`.

Return the `k`-th character of the final string `result`. If `k` is out of the bounds of `result`, return `'.'`.

---

## 🚦 Constraints

- `1 <= s.length <= 10^5`
- `s` consists of only lowercase English letters and special characters `'*'`, `'#'`, and `'%'`.
- `0 <= k <= 10^15`
- The length of `result` after processing `s` will not exceed `10^15`.

## 🧠 Approach

### Why Brute Force Fails
A naive simulation where you sequentially build the string string character by character will fail because the length of the string can exponentially grow up to $10^{15}$ (due to the `#` duplicate operation). This will immediately result in **Time Limit Exceeded (TLE)** and **Memory Limit Exceeded (MLE)**.

### Optimized Approach: Reverse Simulation
Since building the string forward is impossible, we can trace the origin of the $k$-th character by moving **backwards**!



1. **Calculate Final Length:** First, we iterate left-to-right to find out the total length $L$ of the completely processed string. 
   - Normal character: `L++`
   - `*`: `L = max(0, L - 1)`
   - `#`: `L *= 2`
   - `%`: `L` stays the same.
   
2. **Bounds Check:** If the queried index $k \ge L$, we return `'.'`.

3. **Trace Backward (Right to Left):**
   We iterate from the end of the string to the beginning and undo the operations, simultaneously updating $L$ and the target index $k$.
   - **Undo `*`**: The string used to be 1 character longer, so `L++`. (No change to $k$).
   - **Undo `%`**: The string was reversed. To find where $k$ was before the reversal, we reflect it: `k = L - k - 1`.
   - **Undo `#`**: The string was doubled. So we half the length: `L /= 2`. If $k$ was in the right half ($k \ge L$), it means it was a copy of the left half, so we map it back to the left: `k = k - L`.
   - **Undo Normal Character**: The length was smaller by 1, so `L--`. At this point, if `k == L`, it means the current character we just un-did *is* the exact character that ended up at our target index! Return this character.

## 💻 Brute-force Solution (Conceptual)
```cpp
// Intuition: Simulate the exact steps mentioned in the problem.
// Approach: Iteratively build the string using standard string operations.
// Time Complexity: O(2^N) - Length grows exponentially, causing TLE.
// Space Complexity: O(2^N) - Requires massive memory, causing MLE.

class Solution {
public:
    char processStr(string s, long long k) {
        string res = "";
        for(char ch : s) {
            if(ch == '*') {
                if(!res.empty()) res.pop_back();
            } else if (ch == '#') {
                res += res;
            } else if (ch == '%') {
                reverse(res.begin(), res.end());
            } else {
                res += ch;
            }
        }
        return k < res.length() ? res[k] : '.';
    }
};

/*
* Dry Run
* TLE for constraints where string length reaches 10^15
*/
```

## 🚀 Optimized Solution (C++)
```cpp
// Intuition:
// Building the string explicitly will result in a Time Limit Exceeded (TLE) error because the string length can grow up to 10^15. 
// Instead of building the string, we can use "Reverse Simulation". We first find the final length of the string. 
// Then, we iterate backward to map the requested index 'k' back to its original character in the input string.
//
// Approach:
// 1. First Pass (Left to Right): Calculate the final length 'L' of the processed string.
//    - Normal char: L++
//    - '*': L--
//    - '#': L *= 2
//    - '%': L remains unchanged
// 2. Base Case: If k >= L, return '.' since it's out of bounds.
// 3. Second Pass (Right to Left): Undo the operations to trace the original character at index k.
//    - '*': L++ (Undo length reduction)
//    - '%': k = L - k - 1 (Undo reversal by mapping index to its pre-reversed position)
//    - '#': L = L / 2 (Undo duplication). If k >= L (it was in the duplicated half), map it back: k = k - L.
//    - Normal char: L--. If the current k equals this new L, the current character is the answer!
//
// Time Complexity: O(N) where N is the length of the string 's'. We do two passes over the string.
// Space Complexity: O(1) as we only use a few variables for counting length and tracking indices.

class Solution {
public:
    char processStr(string s, long long k) {
        int n = s.length();
        long long L = 0; // Tracks the length of the built string

        // First Pass: Determine the final length L
        for(char &ch : s) {
            if(ch == '*') {
                if(L > 0) L--; // '*' removes the last character
            } else if(ch == '#') {
                L *= 2; // '#' duplicates the string
            } else if(ch == '%') {
                continue; // '%' reverses the string, no length change
            } else { 
                L++; // 'a' to 'z' appends a character
            }
        }

        // If k is out of bounds
        if(k >= L) {
            return '.';
        }

        // Second Pass: Reverse simulation to trace the origin of k
        for(int i = n - 1; i >= 0; i--) {
            if(s[i] == '*') {
                L++; // Undo deletion, no change to k
            } else if(s[i] == '%') {
                k = L - k - 1; // Undo reversal by mapping index to opposite end
            } else if(s[i] == '#') {
                L = L / 2; // Undo duplication
                k = (k >= L) ? k - L : k; // Map k back to original half if it was in the copy
            } else { 
                L--; // Undo character append
            }

            // If k matches the length right before this character was appended, this is our character
            if(k == L) {
                return s[i];
            }
        }

        return '.';
    }
};

/*
*
* Dry Run
* s = "ab%#", k = 2
* * 1. First Pass (Find Final L)
* 'a': L = 1
* 'b': L = 2
* '%': L = 2
* '#': L = 4
* Final L = 4. Since k=2 < 4, it is valid.
*
* 2. Second Pass (Trace backward from i = 3 to 0)
* i=3 (s[3]='#')
* Undo '#': L = 4 / 2 = 2
* k = 2. Since k >= L (2 >= 2), k = 2 - 2 = 0
* i=2 (s[2]='%')
* Undo '%': k = L - k - 1 = 2 - 0 - 1 = 1
* i=1 (s[1]='b')
* Undo 'b': L-- => L = 1
* Check: k == L? (1 == 1) -> TRUE! Return 'b'.
*
*/
```

## 📌 Notes & Key Insights
- **Think Backwards:** Whenever a problem simulates a string explosion (operations like duplicate or replace that grow exponentially), standard iteration fails. Reversing the operations starting from the final target is a classic competitive programming pattern.
- **Index Reflection:** When reversing an array of length $L$, the new position $k$ of an element originally at index $j$ is given by $k = L - j - 1$. Reversing this gives the exact same formula: $j = L - k - 1$.
- **Mapping Duplicates:** Duplicating a string of length $X$ means indices $0$ to $X-1$ are copied exactly to $X$ to $2X-1$. So any query $k \ge X$ is just asking for $k - X$.
