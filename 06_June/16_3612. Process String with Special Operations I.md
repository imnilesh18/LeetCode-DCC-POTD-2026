## 📝 Problem Statement

You are given a string `s` consisting of lowercase English letters and the special characters: `*`, `#`, and `%`.

Build a new string `result` by processing `s` according to the following rules from left to right:

1. If the letter is a **lowercase English letter**, append it to `result`.
2. A `*` **removes the last character** from `result`, if it exists.
3. A `#` **duplicates** the current `result` and appends it to itself.
4. A `%` **reverses** the current `result`.

Return the final string `result` after processing all characters in `s`.

---

## 🚦 Constraints

- `1 <= s.length <= 20`
- `s` consists of only lowercase English letters and special characters `*`, `#`, and `%`.

---

## 💡 Examples

### Example 1

**Input:** `s = "a#b%*"`  
**Output:** `"ba"`  

**Explanation:**
| `i` | `s[i]` | Operation | Current `result` |
| :---: | :---: | :--- | :--- |
| `0` | `'a'` | Append `'a'` | `"a"` |
| `1` | `'#'` | Duplicate result | `"aa"` |
| `2` | `'b'` | Append `'b'` | `"aab"` |
| `3` | `'%'` | Reverse result | `"baa"` |
| `4` | `'*'` | Remove the last character | `"ba"` |

### Example 2

**Input:** `s = "z*#"`  
**Output:** `""`  

**Explanation:**
| `i` | `s[i]` | Operation | Current `result` |
| :---: | :---: | :--- | :--- |
| `0` | `'z'` | Append `'z'` | `"z"` |
| `1` | `'*'` | Remove the last character | `""` |
| `2` | `'#'` | Duplicate the string | `""` |

## 🧠 Approach

This is a classic **Simulation** problem. The problem statement gives us a direct set of rules to follow. Since the input constraint is extremely small (`s.length <= 20`), simulating the exact process left-to-right is the optimal way to proceed.

1. Create a `result` string, initially empty.
2. Iterate through each character `ch` in the input string `s`.
3. Check the conditions:
   - If it's a `*`, pop the last character from `result` (making sure `result` is not empty first!).
   - If it's a `#`, duplicate `result` by doing `result += result`.
   - If it's a `%`, completely reverse `result`.
   - If it's a lowercase letter, just append it.
4. Return `result` after loop completion.

## 💻 Solution Code

```cpp
// Intuition: The problem specifies a straightforward set of string operations. Since the constraints are very small (N=20), a direct left-to-right simulation works perfectly. The key is mapping each character to its distinct operation.
// Approach:
// 1. Initialize an empty string `result`.
// 2. Iterate through each character `ch` in the input string `s`.
// 3. If `ch` is '*', remove the last character from `result` (ensure it's not empty to avoid errors).
// 4. If `ch` is '#', append `result` to itself (duplicate).
// 5. If `ch` is '%', reverse all the characters in `result`.
// 6. Otherwise (lowercase letters), append `ch` to `result`.
// 7. Return the final `result`.
// Time Complexity: O(2^n) - In the worst case (e.g., repeating '#'), the string length doubles at each step. Reversing or duplicating this string takes time proportional to its length. The sum of lengths forms a geometric progression (1 + 2 + 4 + ... + 2^n) which simplifies to an upper bound of O(2^n).
// Space Complexity: O(2^n) - The resulting string can grow exponentially, up to O(2^n) characters in length due to the '#' duplication operations.

class Solution {
public:
    string processStr(string s) {
        string result = ""; // Stores the final processed string

        for(char &ch : s) { // Iterate through each character in the string
            if(ch == '*') {
                // Remove the last character, but only if the result is not empty
                if(result.length() > 0) result.pop_back();
            } else if(ch == '#') {
                // Duplicate the string by appending it to itself
                result += result;
            } else if(ch == '%') {
                // Reverse the entire current result string
                reverse(begin(result), end(result));
            } else {
                // Append the lowercase English letter
                result += ch;
            }
        }

        return result;
    }
};

/*
*
* Dry Run
* Input: s = "a#b%*"
*
* Step 1: ch = 'a'
* - Is letter -> append 'a'
* - result = "a"
*
* Step 2: ch = '#'
* - Is '#' -> duplicate result
* - result = "a" + "a" = "aa"
*
* Step 3: ch = 'b'
* - Is letter -> append 'b'
* - result = "aab"
*
* Step 4: ch = '%'
* - Is '%' -> reverse result
* - result = "baa"
*
* Step 5: ch = '*'
* - Is '*' -> remove last character
* - result = "ba"
*
* Final Output: "ba"
*
*/

```

## 📌 Notes

* Make sure to handle edge cases properly, such as hitting a `*` when the `result` string is completely empty. `std::string::pop_back()` on an empty string causes undefined behavior in C++, which leads to crashes.
* The `std::reverse` function takes linear time with respect to the string's length at that moment.

---

## 🚀 Key Insights

The reason the Time Complexity bounds strictly towards $O(2^n)$ is because of the geometric growth caused by `#`.
If our input looks like `a####...`, after `k` hashes, the string length becomes $1 \times 2^k$. If we do a reversal `%` at the end, flipping the array will cost operations directly proportional to that exponentially huge size. Due to the constraint `1 <= s.length <= 20`, even $2^{20}$ is around $10^6$ operations, which easily passes within the 1-second limit of modern competitive programming environments.

## 🔍 Further Exploration

* What if the constraints were higher (e.g., `s.length <= 10^5`)? Direct simulation would fail and lead to Memory/Time Limit Exceeded. You would need to build an implicit abstract syntax tree (AST) or linked list approach to keep track of repeated segments and reversals conceptually without explicitly allocating memory for them.
