# 1358. Number of Substrings Containing All Three Characters

**Problem Link:** [Number of Substrings Containing All Three Characters](https://leetcode.com/problems/number-of-substrings-containing-all-three-characters/description/)
**Difficulty:** Medium
**Topics:** `Hash Table`, `String`, `Sliding Window`

---

## 📝 Problem Statement

Given a string `s` consisting only of characters `'a'`, `'b'`, and `'c'`. 

Return the number of substrings containing **at least one** occurrence of all these characters `'a'`, `'b'`, and `'c'`.

---

## 🚦 Constraints

* `3 <= s.length <= 5 * 10^4`
* `s` only consists of `'a'`, `'b'`, or `'c'` characters.

---

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `s = "abcabc"` | `10` | The substrings containing at least one occurrence of the characters a, b and c are "abc", "abca", "abcab", "abcabc", "bca", "bcab", "bcabc", "cab", "cabc" and "abc" (again). |
| **2** | `s = "aaacb"` | `3` | The substrings containing at least one occurrence of the characters a, b and c are "aaacb", "aacb" and "acb". |
| **3** | `s = "abc"` | `1` | There is only 1 valid substring. |

### The Brute-Force Idea
The most intuitive approach is to generate every possible substring, starting from index `0` to the end, and count the frequencies of `'a'`, `'b'`, and `'c'`. If a substring contains all three characters at least once, we increment our count. 
* **Why it fails:** Generating all substrings and checking their frequencies takes $O(N^2)$ or $O(N^3)$ time, which will result in a **Time Limit Exceeded (TLE)** given the constraint of $N \le 5 \times 10^4$.

### The Optimized Sliding Window Approach
This problem is a classic application of the **Sliding Window (Two Pointers)** technique. We want to find a valid window and then mathematically deduce how many substrings can be formed using that window.

1. **Window Expansion:** Use two pointers, `i` and `j` (both initially at `0`). We expand our window by moving `j` to the right, adding the current character `s[j]` to our frequency map (or an array of size 3).
2. **Valid Window Detection:** As soon as our window `[i...j]` contains at least one `'a'`, one `'b'`, and one `'c'`, we have found a valid substring.
3. **The Core Logic (The Magic Step):** If the substring from `i` to `j` is valid, then adding any characters to the right of `j` will *also* result in a valid substring. How many characters are left? There are `n - j` characters (including the `j`-th character itself if we consider the remaining suffixes). Thus, for the current start pointer `i`, there are `(n - j)` valid substrings!
4. **Window Contraction:** After adding `(n - j)` to our result, we can try to find more valid substrings by shrinking the window from the left. We decrease the frequency of the character at `s[i]`, increment `i`, and check if the window is still valid. We repeat this contraction step as long as the window remains valid.
5. **Continue:** Once the window becomes invalid, we go back to expanding `j`.

## 💻 Solution Code

### Brute-Force Solution (C++)
```cpp
// Intuition: Check every possible substring to see if it contains 'a', 'b', and 'c'.
// Approach: Nested loops to generate all substrings and track character frequencies.
// Time Complexity: O(N^2)
// Space Complexity: O(1)

class Solution {
public:
    int numberOfSubstrings(string s) {
        int n = s.length();
        int count = 0;
        
        for (int i = 0; i < n; i++) {
            vector<int> freq(3, 0);
            for (int j = i; j < n; j++) {
                freq[s[j] - 'a']++;
                // If we have at least one of each, it's valid
                if (freq[0] > 0 && freq[1] > 0 && freq[2] > 0) {
                    count++;
                }
            }
        }
        return count;
    }
};
```

### Optimized Solution (C++)

```cpp
// Intuition: Instead of checking all substrings, use a sliding window. Once a valid window is found, all extensions of this window to the right are also valid.
// Approach: Two-pointer sliding window technique. Expand the window using pointer j until it contains all three characters, then contract the window from pointer i while counting valid substrings by adding (n - j).
// Time Complexity: O(N) as each character is processed at most twice (once by j and once by i).
// Space Complexity: O(1) because only a fixed-size vector (size 3) is used to store frequencies.

class Solution {
public:
    int numberOfSubstrings(string s) {
        int n = s.length();
        int result = 0;
        
        // Vector to count frequency of characters 'a', 'b', and 'c'
        vector<int> mp(3, 0);
        
        int i = 0;
        int j = 0;
        
        // Expand the window with pointer j
        while(j < n) {
            // Include s[j] in the window
            char ch = s[j];
            mp[ch - 'a']++;
            
            // When the window contains at least one of each character, contract from left (pointer i)
            while(mp[0] > 0 && mp[1] > 0 && mp[2] > 0) {
                // All substrings starting from index i to current j and extending to the end are valid
                result += (n - j);
                
                // Remove the character at index i from the window and move i forward
                mp[s[i] - 'a']--;
                i++;
            }
            // Expand the window by moving j to the next character
            j++;
        }
        return result;
    }
};

/*
*
* Dry Run
*
* Input: s = "abcabc"
* Initial State: i = 0, j = 0, result = 0, mp = [0, 0, 0], n = 6
* 
* j = 0: s[j] = 'a' -> mp becomes [1, 0, 0]
* j = 1: s[j] = 'b' -> mp becomes [1, 1, 0]
* j = 2: s[j] = 'c' -> mp becomes [1, 1, 1]
*        -> Valid window "abc". 
*        -> Inner loop: result += (6 - 2) = 4; decrement mp for s[i] ('a'): mp becomes [0, 1, 1]; increment i to 1.
*        -> Exit inner loop as window no longer has 'a'.
* j = 3: s[j] = 'a' -> mp becomes [1, 1, 1] ("bca" inside window)
*        -> Valid window.
*        -> Inner loop: result += (6 - 3) = 3; decrement mp for s[i] ('b'): mp becomes [1, 0, 1]; increment i to 2.
*        -> Exit inner loop as window no longer has 'b'.
* j = 4: s[j] = 'b' -> mp becomes [1, 1, 1] ("cab" inside window)
*        -> Valid window.
*        -> Inner loop: result += (6 - 4) = 2; decrement mp for s[i] ('c'): mp becomes [1, 1, 0]; increment i to 3.
*        -> Exit inner loop as window no longer has 'c'.
* j = 5: s[j] = 'c' -> mp becomes [1, 1, 1] ("abc" inside window)
*        -> Valid window.
*        -> Inner loop: result += (6 - 5) = 1; decrement mp for s[i] ('a'): mp becomes [0, 1, 1]; increment i to 4.
*        -> Exit inner loop as window no longer has 'a'.
*
* Final result = 4 + 3 + 2 + 1 = 10.
*
*/

```

## 🔑 Key Insights

* **The Power of Suffixes:** The mathematical trick `result += (n - j)` is the cornerstone of this solution. By identifying the *shortest* valid window starting at index `i` and ending at `j`, we inherently identify that any substring starting at `i` and ending *anywhere after* `j` is also valid.
* **State Management:** Tracking states with a frequency map array `mp` of size 3 keeps memory usage tightly bound to $O(1)$.
