# 3751. Total Waviness of Numbers in Range I

**Problem Link:** [LeetCode - Total Waviness of Numbers in Range I](https://leetcode.com/problems/total-waviness-of-numbers-in-range-i/description)
**Difficulty:** Medium
**Tags:** Math, Dynamic Programming, Enumeration

---

## 📖 Problem Statement

You are given two integers `num1` and `num2` representing an inclusive range `[num1, num2]`.

The **waviness** of a number is defined as the total count of its peaks and valleys:
*   A digit is a **peak** if it is strictly greater than both of its immediate neighbors.
*   A digit is a **valley** if it is strictly less than both of its immediate neighbors.

**Rules:**
*   The first and last digits of a number cannot be peaks or valleys.
*   Any number with fewer than 3 digits has a waviness of `0`.

Return the total sum of waviness for all numbers in the range `[num1, num2]`.

---

## 🚦 Constraints

*   `1 <= num1 <= num2 <= 10^5`

---

## 📋 Examples

### Example 1
> **Input:** `num1 = 120`, `num2 = 130`
> **Output:** `3`
> **Explanation:** 
> In the range `[120, 130]`:
> * `120`: middle digit 2 is a peak, waviness = 1.
> * `121`: middle digit 2 is a peak, waviness = 1.
> * `130`: middle digit 3 is a peak, waviness = 1.
> All other numbers in the range have a waviness of 0. Thus, total waviness is 1 + 1 + 1 = 3.

### Example 2
> **Input:** `num1 = 198`, `num2 = 202`
> **Output:** `3`
> **Explanation:** 
> In the range `[198, 202]`:
> * `198`: middle digit 9 is a peak, waviness = 1.
> * `201`: middle digit 0 is a valley, waviness = 1.
> * `202`: middle digit 0 is a valley, waviness = 1.
> All other numbers in the range have a waviness of 0. Thus, total waviness is 1 + 1 + 1 = 3.

### Example 3
> **Input:** `num1 = 4848`, `num2 = 4848`
> **Output:** `2`
> **Explanation:** 
> Number `4848`: the second digit 8 is a peak, and the third digit 4 is a valley, giving a waviness of 2.

## 💡 Approach

Since the constraints are relatively small ($10^5$), a direct simulation approach (linear scan) is highly effective and serves as the perfect starting point. 

1. **Iterate Through the Range:** We loop through every number from `num1` to `num2`.
2. **String Conversion:** For each number, we convert it to a string. This makes it incredibly easy to access digits by their index and compare them with their left and right neighbors.
3. **Boundary Check:** If the length of the string is less than 3, it cannot contain any peaks or valleys, so we immediately return a score of `0`.
4. **Count Peaks and Valleys:** We loop from the second digit (index 1) to the second-to-last digit (index `length - 2`).
    *   **Peak Condition:** `s[i] > s[i-1]` and `s[i] > s[i+1]`
    *   **Valley Condition:** `s[i] < s[i-1]` and `s[i] < s[i+1]`
5. **Accumulate:** We add up the score for every number and return the total sum.

## 💻 Solution Code

```cpp
// Intuition: Since the maximum range is only up to 10^5, a straightforward simulation is efficient enough. Converting each number to a string allows for easy indexing to compare adjacent digits.
// Approach: Iterate from num1 to num2. Convert each number to a string and skip if its length is less than 3. Loop through the internal digits (ignoring the first and last). Increment the score if the current digit is strictly greater than (peak) or strictly less than (valley) its immediate neighbors. Sum the waviness scores of all numbers.
// Time Complexity: O((num2 - num1) * log10(num2)) — for each number in the range, we convert it to a string and scan its digits. The number of digits is proportional to log10 of the number.
// Space Complexity: O(log10(num2)) — space required to store the string representation of each number.

class Solution {
public:
    int findWaiveScore(int num) {
        // Convert number to string to easily access adjacent digits
        string s = to_string(num);

        int l = s.length();
        // A number needs at least 3 digits to have a peak or valley
        if (l < 3)
            return 0;

        int score = 0;
        // Iterate through the string, ignoring the first and last digits
        for (int i = 1; i <= l - 2; i++) {
            // Check if the current digit is a peak
            if (s[i] > s[i - 1] && s[i] > s[i + 1]) score++;

            // Check if the current digit is a valley
            if (s[i] < s[i - 1] && s[i] < s[i + 1]) score++;
        }

        return score;
    }

    int totalWaviness(int num1, int num2) {
        int score = 0;

        // Iterate through the inclusive range and accumulate the waviness score
        for (int num = num1; num <= num2; num++) {
            score += findWaiveScore(num);
        }

        return score;
    }
};

/*
*
* Dry Run
*
* Input: num1 = 120, num2 = 122
* 
* num = 120:
* s = "120", length = 3
* i = 1, s[1] = '2'. Left = '1', Right = '0'.
* '2' > '1' && '2' > '0' -> Peak found. score = 1.
* 
* num = 121:
* s = "121", length = 3
* i = 1, s[1] = '2'. Left = '1', Right = '1'.
* '2' > '1' && '2' > '1' -> Peak found. score = 1.
* 
* num = 122:
* s = "122", length = 3
* i = 1, s[1] = '2'. Left = '1', Right = '2'.
* Not strictly greater than or less than right neighbor. score = 0.
* 
* Total Score: 1 + 1 + 0 = 2.
*
*/
```

## 📝 Notes & Key Insights

* **Simulation vs. Digit DP:** The current constraints ($10^5$) allow a simple linear scan to pass smoothly. However, if the constraints were raised to $10^{18}$ or higher, this linear scan would result in a Time Limit Exceeded (TLE). In that scenario, Digit Dynamic Programming (Digit DP) would be required. 
* **String Conversion:** Using `to_string()` adds a slight overhead but results in exceptionally clean and readable code. For micro-optimizations, one could extract digits mathematically using modulo `% 10` and division `/ 10` operations to avoid string allocations.
* **Building Blocks:** As mentioned in the inspiration: *"Great structures are never built in a single day but through the careful placement of one brick at a time."* Simple simulation approaches are often the foundation upon which more complex optimizations (like DP) are built.
