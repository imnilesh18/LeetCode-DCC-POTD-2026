## 📝 Problem Statement

You are given an `m x n` matrix `grid` of positive integers. Your task is to determine if it is possible to make **either one horizontal or one vertical cut** on the grid such that:

1. Each of the two resulting sections formed by the cut is **non-empty**.
2. The sum of the elements in both sections is **equal**.

Return `true` if such a partition exists; otherwise return `false`.

## 🚧 Constraints

- `1 <= m == grid.length <= 10^5`
- `1 <= n == grid[i].length <= 10^5`
- `2 <= m * n <= 10^5`
- `1 <= grid[i][j] <= 10^5`

---

## 💡 Approach

The problem asks whether we can slice a grid exactly in half (either horizontally or vertically) so that the sums of the two pieces are completely equal. 

Instead of iterating over every possible split and recalculating the elements recursively (which would cause a Time Limit Exceeded error), we can use the **Prefix Sum** technique optimized for rows and columns.

1. **Calculate Row and Column Sums:** Make a single pass over the grid. Calculate the total sum of all elements. Simultaneously, aggregate the sum of each individual row into a `rowSum` array, and each column into a `colSum` array.
2. **The Parity Check:** If the grid is going to be divided into two equal halves, the total sum of the grid **must be an even number**. If `total % 2 != 0`, we can immediately return `false`.
3. **Check Horizontal Splits:** Iterate from row `0` to `m-2` (we cannot cut below the last row since the bottom section would be empty). Keep an `upper` running sum by adding the current `rowSum`. If at any point `upper == total - upper`, we have found a valid horizontal cut.
4. **Check Vertical Splits:** Iterate from column `0` to `n-2` (we cannot cut past the last column). Keep a `left` running sum by adding the current `colSum`. If at any point `left == total - left`, we have found a valid vertical cut.

## 💻 Solution Code

### Brute-force Solution (Conceptual C++)
```cpp
// A naive brute force would iterate through every possible cut
// and recalculate the sum of the entire upper/lower or left/right halves using nested loops.
// Time Complexity: O((m + n) * (m * n)) -> Will give TLE!
class Solution {
public:
    bool canPartitionGrid(vector<vector<int>>& grid) {
        int m = grid.size(), n = grid[0].size();
        // Horizontal cuts
        for(int i = 0; i < m - 1; i++) {
            long long topSum = 0, bottomSum = 0;
            for(int r = 0; r <= i; r++) for(int c = 0; c < n; c++) topSum += grid[r][c];
            for(int r = i + 1; r < m; r++) for(int c = 0; c < n; c++) bottomSum += grid[r][c];
            if(topSum == bottomSum) return true;
        }
        // Vertical cuts
        for(int j = 0; j < n - 1; j++) {
            long long leftSum = 0, rightSum = 0;
            for(int r = 0; r < m; r++) for(int c = 0; c <= j; c++) leftSum += grid[r][c];
            for(int r = 0; r < m; r++) for(int c = j + 1; c < n; c++) rightSum += grid[r][c];
            if(leftSum == rightSum) return true;
        }
        return false;
    }
};
```

### Optimized Solution (C++)
```cpp
// Intuition: The problem requires splitting the grid into two equal sum parts either horizontally or vertically. A valid split is only possible if the total sum is even. By precomputing the sum of each row and each column, we can evaluate any horizontal or vertical cut in constant time.
// Approach: 
// 1. Traverse the grid to calculate the sum of each row, each column, and the total sum.
// 2. If the total sum is odd, return false immediately.
// 3. Iterate through possible horizontal cuts (rows 0 to m-2), keeping a cumulative 'upper' sum. Check if 'upper' equals 'total - upper'.
// 4. Iterate through possible vertical cuts (cols 0 to n-2), keeping a cumulative 'left' sum. Check if 'left' equals 'total - left'.
// 5. If any cut satisfies the condition, return true. Otherwise, return false.
// Time Complexity: O(m * n) where m is the number of rows and n is the number of columns, since we iterate through all elements of the grid once.
// Space Complexity: O(m + n) to store the sums of each row and each column.

class Solution {
public:
    typedef long long ll;
    bool canPartitionGrid(vector<vector<int>>& grid) {
       int m = grid.size();
       int n = grid[0].size();

        vector<ll> rowSum(m, 0);
        vector<ll> colSum(n, 0);

        ll total = 0;
        for(int i = 0; i < m; i++) {
            for(int j = 0; j < n; j++) {
                total += grid[i][j];

                // row = i
                rowSum[i] += grid[i][j];

                // col  = j
                colSum[j] += grid[i][j];
            }
        }

        if(total % 2 != 0) {
            return false;
        }

        // Horizontal Split
        ll upper = 0;
        for(int i = 0; i < m - 1; i++) {
            upper += rowSum[i];
            if(upper == total - upper) {
                return true;
            }
        }

        // Vertical Split
        ll left = 0;
        for(int j = 0; j < n - 1; j++) {
            left += colSum[j];
            if(left == total - left) {
                return true;
            }
        }

        return false;
    }
};

/*
*
* Dry Run
*
* Input: grid = [[1,4],[2,3]]
* m = 2, n = 2
*
* Calculating rowSum, colSum, and total:
* i = 0, j = 0: val = 1, total = 1, rowSum[0] = 1, colSum[0] = 1
* i = 0, j = 1: val = 4, total = 5, rowSum[0] = 5, colSum[1] = 4
* i = 1, j = 0: val = 2, total = 7, rowSum[1] = 2, colSum[0] = 3
* i = 1, j = 1: val = 3, total = 10, rowSum[1] = 5, colSum[1] = 7
* * End of loops:
* rowSum = [5, 5]
* colSum = [3, 7]
* total = 10
*
* Parity Check: 10 % 2 == 0 -> We can proceed.
*
* Horizontal Split Check:
* i = 0 (we only go up to m - 2, which is 0)
* upper += rowSum[0] -> upper = 5
* Check: upper == total - upper -> 5 == 10 - 5 -> 5 == 5
* Condition met. Returns true.
*
* Final Output: true
*
*/
```

## 📌 Notes & Key Insights
<details>
<summary>Click to view key insights</summary>

- **Early Exit (Parity Check):** One of the most critical optimizations here is checking if the total grid sum is odd. If `total % 2 != 0`, we immediately know that it's impossible to divide the grid into two equal integer halves.
- **Prefix Sum Utility:** Instead of making 2D prefix sums (which are standard for answering range sum queries in sub-matrices), decomposing the problem purely into 1D row sums and 1D column sums makes the code exceptionally clean and memory efficient.
- **Handling Constraints:** The total matrix element sum easily surpasses the 32-bit integer limit (since `10^5 * 10^5` is `10^10`). Using `long long` for sum trackers avoids integer overflow.

</details>
