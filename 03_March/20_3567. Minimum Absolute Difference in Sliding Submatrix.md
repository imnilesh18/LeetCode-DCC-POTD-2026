# 3567. Minimum Absolute Difference in Sliding Submatrix

**Difficulty:** Medium 🟡  
**Tags:** `Array`, `Matrix`, `Sliding Window`, `Ordered Set`, `Sorting`  

[![LeetCode Problem](https://img.shields.io/badge/LeetCode-Solve_Problem-FFA116?style=for-the-badge&logo=leetcode&logoColor=black)](https://leetcode.com/problems/minimum-absolute-difference-in-sliding-submatrix/)

## 📖 Problem Statement

You are given an `m x n` integer matrix `grid` and an integer `k`.

For every contiguous `k x k` submatrix of `grid`, compute the **minimum absolute difference** between any two **distinct** values within that submatrix.

Return a 2D array `ans` of size `(m - k + 1) x (n - k + 1)`, where `ans[i][j]` is the minimum absolute difference in the submatrix whose top-left corner is `(i, j)` in `grid`.

> **Note:** If all elements in the submatrix have the same value, the answer will be `0`. A submatrix `(x1, y1, x2, y2)` is a matrix that is formed by choosing all cells `matrix[x][y]` where `x1 <= x <= x2` and `y1 <= y <= y2`.

---

## 🎯 Constraints

- `1 <= m == grid.length <= 30`
- `1 <= n == grid[i].length <= 30`
- `-10^5 <= grid[i][j] <= 10^5`
- `1 <= k <= min(m, n)`

---

## 💡 Examples

### Example 1
| Input | Output |
| :--- | :--- |
| `grid = [[1,8],[3,-2]]`, `k = 2` | `[[2]]` |

**Explanation:**
There is only one possible `k x k` submatrix: `[[1, 8], [3, -2]]`.
Distinct values in the submatrix are `[1, 8, 3, -2]`.
The minimum absolute difference in the submatrix is `|1 - 3| = 2`. Thus, the answer is `[[2]]`.

### Example 2
| Input | Output |
| :--- | :--- |
| `grid = [[3,-1]]`, `k = 1` | `[[0,0]]` |

**Explanation:**
Both `k x k` submatrices have only one distinct element.
Thus, the answer is `[[0, 0]]`.

### Example 3
| Input | Output |
| :--- | :--- |
| `grid = [[1,-2,3],[2,3,5]]`, `k = 2` | `[[1,2]]` |

**Explanation:**
There are two possible `k × k` submatrices:
- **Starting at `(0, 0)`:** `[[1, -2], [2, 3]]`. Distinct values in the submatrix are `[1, -2, 2, 3]`. The minimum absolute difference in the submatrix is `|1 - 2| = 1`.
- **Starting at `(0, 1)`:** `[[-2, 3], [3, 5]]`. Distinct values in the submatrix are `[-2, 3, 5]`. The minimum absolute difference in the submatrix is `|3 - 5| = 2`.

Thus, the answer is `[[1, 2]]`.

---

## 🧠 Explanation & Approach

To solve this problem, our goal is to iterate over every possible `k x k` submatrix inside the `m x n` grid. 

1. **Grid Traversal:** The top-left corner of any `k x k` submatrix can range from row `0` to `m - k` and column `0` to `n - k`. We can use a nested loop for `(i, j)` to anchor the top-left corner.
2. **Submatrix Extraction & Sorting:** Once a top-left corner `(i, j)` is fixed, we need to extract all the elements within that `k x k` window. To find the minimum absolute difference between any two *distinct* values, we need them sorted.
3. **Using an Ordered Set:** Instead of pushing elements to a list and sorting them manually, we can insert the submatrix elements directly into a `std::set`. An ordered set automatically removes duplicate elements (giving us only distinct values) and keeps the elements strictly sorted.
4. **Calculating Minimum Difference:** - If the set has only `1` element, it means all values in the submatrix are identical. The difference is `0`.
   - If the set has more than `1` element, we iterate through it using two pointers (`prev` and `curr`). Because the set is sorted, the minimum absolute difference between any two elements will always exist between *consecutive* elements. We compute the difference and track the minimum.
5. **Storing Results:** Finally, we place this computed minimum difference into our result matrix at position `ans[i][j]`.

## 🚀 Solution Code

```cpp
// Intuition: To find the minimum absolute difference between distinct values efficiently, sorting the elements is key because the closest values will be adjacent. Using a `std::set` helps us automatically maintain a sorted collection while natively discarding any duplicate values.
// Approach: Anchor the top-left cell (i,j) for every possible k x k submatrix. Traverse the elements inside this submatrix and insert them into a `std::set`. Check if all elements are identical (size == 1) and return 0 if so. Otherwise, iterate through the sorted set with two adjacent pointers (prev and curr) to find the minimum absolute difference between consecutive distinct elements.
// Time Complexity: O((m - k + 1) * (n - k + 1) * k^2 * log(k)). The outer loops run for every valid submatrix. Inside, we insert k^2 elements into a set, taking O(log(k^2)) = O(2 * log k) -> O(log k) time per insertion.
// Space Complexity: O(k^2) to store the elements of the k x k submatrix inside the set. The result vector takes O((m-k+1)*(n-k+1)) space, which is required for the output.

class Solution {
public:
    vector<vector<int>> minAbsDiff(vector<vector<int>>& grid, int k) {
        int m = grid.size();
        int n = grid[0].size();

        // Initialize the result 2D array with dimensions (m - k + 1) x (n - k + 1)
        vector<vector<int>> result(m - k + 1, vector<int>(n - k + 1, 0));

        // Iterate over all valid top-left corners of k x k submatrices
        for(int i = 0; i <= m - k; i++) {
            for(int j = 0; j <= n - k; j++) {
                
                // Ordered set to keep elements distinct and sorted
                set<int> vals;

                // Traverse the k x k submatrix starting at (i, j)
                for(int r = i; r <= i + k - 1; r++) {
                    for(int c = j; c <= j + k - 1; c++) {
                        vals.insert(grid[r][c]); // Insert element into the set
                    }
                }

                // If all elements are the same, the difference is 0
                if(vals.size() == 1) {
                    continue; // result[i][j] is already initialized to 0
                }

                int minAbsDiff = INT_MAX;
                
                // Use iterators to check consecutive elements in the sorted set
                auto prev = vals.begin();
                auto curr = next(prev);

                // Find the minimum difference between adjacent elements
                while(curr != vals.end()) {
                    minAbsDiff = min(minAbsDiff, *curr - *prev);
                    prev = curr;
                    curr++;
                }

                // Store the computed minimum difference
                result[i][j] = minAbsDiff;
            }
        }
        return result;
    }
};

/*
*
* Dry Run
*
* Input: grid = [[1, 8], [3, -2]], k = 2
* Dimensions: m = 2, n = 2
* Result size: (2 - 2 + 1) x (2 - 2 + 1) = 1 x 1
* * Loop i = 0, j = 0:
* Submatrix limits: row 0 to 1, col 0 to 1
* Elements: 1, 8, 3, -2
* Insert into set `vals`:
* vals = {-2, 1, 3, 8} (Automatically sorted and distinct)
* * Check size: vals.size() = 4 (which is != 1)
* * Iterate through `vals`:
* 1st pass: prev = -2, curr = 1  => diff = 1 - (-2) = 3. minAbsDiff = 3
* 2nd pass: prev = 1, curr = 3   => diff = 3 - 1 = 2.    minAbsDiff = min(3, 2) = 2
* 3rd pass: prev = 3, curr = 8   => diff = 8 - 3 = 5.    minAbsDiff = min(2, 5) = 2
* * End of set. result[0][0] = 2
* * Output: [[2]]
*
*/
````

## 🔍 Key Insights

  - **Using a Set for Dual Operations:** A `std::set` in C++ is highly efficient for this specific problem because it implicitly handles sorting and duplicate removal natively as you insert the data.
  - **Adjacent Difference Optimization:** The absolute difference between any two numbers in an array is always minimized between adjacent pairs once the array is strictly sorted. This eliminates the need for an $O(N^2)$ nested comparison of all elements inside the submatrix.
