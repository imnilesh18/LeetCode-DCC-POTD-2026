## 📝 Problem Statement

Given an array of strings `nums` containing `n` unique binary strings each of length `n`, return a binary string of length `n` that **does not appear** in `nums`. If there are multiple answers, you may return **any** of them.

### 🔗 Problem Link
[Find Unique Binary String - LeetCode](https://leetcode.com/problems/find-unique-binary-string/description/)

---

## 🎯 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `nums = ["01","10"]` | `"11"` | `"11"` does not appear in `nums`. `"00"` would also be correct. |
| **2** | `nums = ["00","01"]` | `"11"` | `"11"` does not appear in `nums`. `"10"` would also be correct. |
| **3** | `nums = ["111","011","001"]` | `"101"` | `"101"` does not appear in `nums`. `"000"`, `"010"`, `"100"`, and `"110"` would also be correct. |

---

## 🧩 Constraints

- `n == nums.length`
- `1 <= n <= 16`
- `nums[i].length == n`
- `nums[i]` is either `'0'` or `'1'`.
- All the strings of `nums` are **unique**.

---

## 💡 Approach and Explanation

We are given an array `nums` of length `n`, consisting of strictly unique binary strings, and each string is also exactly of length `n`. Our task is to formulate a binary string of length `n` that is completely missing from this array.

### Approach 1: Decimal Conversion & Complete Range Check (Brute-Force)
Because the maximum string length is 16, the maximum possible decimal value represented by 16 bits is $2^{16} - 1 = 65535$. 
1. We can evaluate every binary string in the `nums` array and convert it to its decimal equivalent.
2. Store these decimal numbers in a Hash Set.
3. Start iterating from `0` to `65536`. The first integer not found inside the Hash Set is our answer!
4. Convert this integer back to a 16-bit binary string (using `std::bitset`) and extract the last `n` characters to match the required length.

### Approach 2: Optimized Range Check
We do not really need to loop all the way to `65536`. We are selecting exactly `n` strings. Therefore, the total number of items stored in our Hash Set is at most `n`.
By the **Pigeonhole Principle**, if we check numbers sequentially from `0` up to `n` (which is `n + 1` total numbers), there will absolutely be *at least one* number missing from the set. Checking from `0` to `n` drastically curtails the search space.

### Approach 3: Cantor's Diagonalization (Most Optimal)
This is the most elegant approach using **Cantor's Diagonalization Argument**. 
Instead of processing values in their entirety, we dynamically construct a binary string character by character. 
For every index `i` from `0` to `n-1`, we peek at the `i`-th character of the `i`-th string (`nums[i][i]`). If it's a `'0'`, we append `'1'` to our result; if it's a `'1'`, we append `'0'`. 
Why does this work? Because our generated string differs from the $1^{st}$ string at the $1^{st}$ position, from the $2^{nd}$ string at the $2^{nd}$ position, and so forth. It is strictly mathematically impossible for this newly generated string to match any string currently in `nums`.

## 💻 Solution Code

### Brute-Force Solution (Using Complete Range Hash Check)

```cpp
// Intuition: The maximum length of 'n' is 16. The maximum decimal value for a 16-bit binary string is 65535. We can convert all strings to decimals, store them in a set, and check for the first missing number from 0 upwards.
// Approach: Convert each binary string to an integer and insert into an unordered_set. Iterate from 0 to 65536, checking if the number exists in the set. The first missing number is our answer. Convert it back to a 16-bit binary string using bitset, and return the substring of length 'n'.
// Time Complexity: O(n^2) - Iterating through 'n' strings, and 'stoi' takes O(n) time.
// Space Complexity: O(n) - We store 'n' elements in the unordered_set.
class Solution {
public:
    string findDifferentBinaryString(vector<string>& nums) {
        unordered_set<int> st; // Set to store decimal equivalents
        
        for(string &num : nums) {
            st.insert(stoi(num, 0, 2)); // Convert binary string to decimal and insert
        }
        
        int n = nums.size();
        
        string result= "";
        
        for(int number = 0; number <= 65536; number++) {
            if(st.find(number) == st.end()) { // Find the missing number
                result = bitset<16>(number).to_string(); // Convert missing number to 16-bit binary string
                break;
            }
        }
        
        return result.substr(16-n); // Extract the last 'n' characters
        
    }
};

/*
*
* Dry Run
* nums = ["01", "10"], n = 2
* Convert "01" -> 1, "10" -> 2. Set = {1, 2}
* Loop number = 0: not in set. Found missing!
* result = bitset<16>(0) = "0000000000000000"
* return result.substr(16 - 2) -> substr(14) -> "00"
* Output: "00"
*
*/
```

### Better Solution (Optimized Pigeonhole Check)

```cpp
// Intuition: Since there are 'n' numbers in the array, by the Pigeonhole Principle, at least one number in the range [0, n] must be missing. We don't need to check up to 65536.
// Approach: Store the decimal values of all binary strings in an unordered_set. Iterate from 0 to 'n'. The first number not present in the set is a valid answer. Convert it to a binary string and return the last 'n' characters.
// Time Complexity: O(n^2) - Iterating on each string and converting each character to an integer.
// Space Complexity: O(n) - Using an unordered_set to store 'n' elements.
class Solution {
public:
    string findDifferentBinaryString(vector<string>& nums) {
        unordered_set<int> st; // Set to store decimals
        
        for(string &num : nums) {
            st.insert(stoi(num, 0, 2)); // Convert to decimal and insert
        }
        
        int n = nums.size();
        
        string result= "";
        
        for(int number = 0; number <= n; number++) { // Only check up to 'n'
            if(st.find(number) == st.end()) { // Missing number found
                result = bitset<16>(number).to_string(); // Convert to binary
                break;
            }
        }
        
        return result.substr(16-n); // Return 'n' length binary string
        
    }
};

/*
*
* Dry Run
* nums = ["00", "01"], n = 2
* "00" -> 0, "01" -> 1. Set = {0, 1}
* Range to check: 0 to 2
* number = 0: in set
* number = 1: in set
* number = 2: not in set!
* bitset<16>(2) -> "0000000000000010"
* substr(14) -> "10"
* Output: "10"
*
*/
```

### Optimized Solution (Cantor's Diagonalization)

```cpp
// Intuition: We can construct a unique string character by character. If we make sure our result string differs from the i-th string at the i-th position, it will be guaranteed to be unique and different from all strings in the array. This is known as Cantor's Diagonal Argument.
// Approach: Iterate through the array. For each index 'i', look at the i-th string's i-th character. If it is '0', append '1' to the result. If it is '1', append '0'. The final constructed string will differ from every string by at least one character.
// Time Complexity: O(n) - We only iterate through the array once and check one character per string.
// Space Complexity: O(1) - No extra space is used apart from the result string.
class Solution {
public:
    string findDifferentBinaryString(vector<string>& nums) {
        int n = nums.size();
        
        string result;
        
        for(int i = 0; i<n; i++) {
            char ch = nums[i][i]; // Get the diagonal character
            
            result += (ch == '0') ? "1" : "0"; // Flip the character and append
        }
        
        return result; // Result is guaranteed to be unique
    }
};

/*
*
* Dry Run
* nums = ["111", "011", "001"], n = 3
* i = 0: nums[0][0] is '1'. Flip to '0'. result = "0"
* i = 1: nums[1][1] is '1'. Flip to '0'. result = "00"
* i = 2: nums[2][2] is '1'. Flip to '0'. result = "000"
* Output: "000" (which is unique and not in nums)
*
*/
```

## 🔍 Key Insights

- **Mathematical Assurance:** Cantor's Diagonalization demonstrates profound mathematical beauty, allowing an $O(N)$ execution bypassing comprehensive string generations.
- **Pigeonhole Principle:** Realizing that in an array of $N$ variables, tracking indices exclusively up to $N$ completely invalidates checking all bit values drastically cuts iteration loops.
- **Language Utility Hacks:** Using standard string-manipulation utility libraries (`stoi` and `std::bitset`) allows cleaner operations switching bases implicitly compared to building base conversions mathematically from scratch.
