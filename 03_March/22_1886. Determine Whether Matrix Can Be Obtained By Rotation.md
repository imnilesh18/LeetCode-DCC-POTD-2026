## 📝 Problem Statement

Given two `n x n` binary matrices `mat` and `target`, return `true` if it is possible to make `mat` equal to `target` by **rotating** `mat` in **90-degree increments**, or `false` otherwise.

## 🚫 Constraints

- `n == mat.length == target.length`
- `n == mat[i].length == target[i].length`
- `1 <= n <= 10`
- `mat[i][j]` and `target[i][j]` are either `0` or `1`.

---

## 💡 Approach

A matrix will return to its original form after exactly 4 rotations of 90 degrees (which equals 360 degrees). Thus, the matrix can only exist in 4 distinct orientations.
We can check all 4 possibilities:
1.  Check if `mat` currently equals `target`. If it does, return `true`.
2.  If not, rotate `mat` by 90 degrees clockwise in-place.
3.  Repeat this process up to 4 times.
4.  If after 4 iterations there is still no match, it is impossible, so we return `false`.

**How to rotate an `n x n` matrix by 90 degrees clockwise in-place?**
Instead of using extra space, we can achieve this in two mathematical steps:
1.  **Transpose the Matrix:** Convert rows into columns by swapping `mat[i][j]` with `mat[j][i]`. 
2.  **Reverse each Row:** Reversing the elements of each row horizontally completes the 90-degree clockwise rotation.

---

## 💻 Solution Code

```cpp
// Intuition: A matrix has only 4 unique orientations when rotated by 90 degrees. We can check all 4 orientations by rotating the matrix up to 4 times and comparing it with the target.
// Approach: Loop 4 times. In each iteration, check if the current matrix equals the target. If it does, return true. Otherwise, rotate the matrix by 90 degrees clockwise. To rotate in-place, first transpose the matrix (swap mat[i][j] with mat[j][i]), then reverse each row. If no match is found after 4 checks, return false.
// Time Complexity: O(n^2), where n is the number of rows/cols. Checking equality and rotating the matrix both take O(n^2) time, and we do this at most 4 times.
// Space Complexity: O(1), as the rotation is done in-place without using any extra auxiliary matrices.

class Solution {
public:
    int n;

    void rotate(vector<vector<int>>& mat) {
        // Transpose the matrix: swap mat[i][j] with mat[j][i]
        for(int i = 0; i < n; i++) {
            // Start j from i to swap only the upper triangle with the lower triangle
            for(int j = i; j < n; j++) {
                swap(mat[i][j], mat[j][i]);
            }
        }

        // Reverse each row to complete the 90-degree clockwise rotation
        for(int i = 0; i < n; i++) {
            reverse(mat[i].begin(), mat[i].end());
        }
    }

    bool findRotation(vector<vector<int>>& mat, vector<vector<int>>& target) {
        n = mat.size();

        // A matrix can be rotated at most 4 times (360 degrees) to return to its original state
        for(int c = 1; c <= 4; c++) {
            
            bool equal = true;
            
            // Check if current orientation of mat is equal to target
            for(int i = 0; i < n; i++) {
                for(int j = 0; j < n; j++) {
                    if(mat[i][j] != target[i][j]) {
                        equal = false;
                        break;
                    }
                }
                // If a mismatch is found, no need to check further for this orientation
                if(!equal)
                    break;
            }

            // If mat matches target in this orientation, return true
            if(equal)
                return true;
                
            // Rotate the matrix by 90 degrees for the next iteration check
            rotate(mat);
        }

        // If no match is found after 4 rotations, return false
        return false;
    }
};

/*
 *
 * Dry Run
 * Input: mat = [[0,1],[1,0]], target = [[1,0],[0,1]]
 * n = 2
 * * Iteration 1 (c = 1):
 * - Check equality: mat[0][0] (0) != target[0][0] (1). Mismatch found. equal = false.
 * - Rotate mat:
 * - Transpose: swap(mat[0][1], mat[1][0]) -> mat becomes [[0,1],[1,0]]
 * - Reverse rows: row 0 becomes [1,0], row 1 becomes [0,1] -> mat becomes [[1,0],[0,1]]
 * * Iteration 2 (c = 2):
 * - Check equality: mat == target. Every element matches. equal = true.
 * - Returns true.
 * * Result: true
 *
 */
````

## 🔍 Key Insights

  - **State Finiteness:** A 2D matrix can only have a maximum of 4 distinct visual states when subjected to 90-degree rotations. There is never a need to rotate endlessly.
  - **In-Place Rotation Logic:** Remembering that `90° Clockwise Rotation = Transpose + Reverse Rows` is a standard and highly reusable mathematical trick for 2D matrix array challenges. (Conversely, `90° Anti-Clockwise = Transpose + Reverse Columns`).

## 📌 Notes

  - This is a fantastic foundational problem for practicing matrix manipulation techniques without allocating separate memory (extra arrays).
  - Always ensure you start `j` from `i` (`j = i`) during the transpose step to prevent double-swapping elements back to their original spots.
