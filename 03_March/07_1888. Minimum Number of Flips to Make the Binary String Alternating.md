## 📝 Problem Statement

You are given a binary string `s`. You are allowed to perform two types of operations on the string in any sequence:

* **Type-1**: Remove the character at the start of the string `s` and append it to the end of the string.
* **Type-2**: Pick any character in `s` and flip its value, i.e., if its value is `'0'` it becomes `'1'` and vice-versa.

Return the **minimum** number of **type-2** operations you need to perform such that `s` becomes alternating.

> The string is called alternating if no two adjacent characters are equal. 
> For example, the strings `"010"` and `"1010"` are alternating, while the string `"0100"` is not.

---

## 🚀 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `s = "111000"` | `2` | Use the 1st operation two times to make `s = "100011"`. Then, use the 2nd operation on the 3rd and 6th elements to make `s = "101010"`. |
| **2** | `s = "010"` | `0` | The string is already alternating. |
| **3** | `s = "1110"` | `1` | Use the 2nd operation on the 2nd element to make `s = "1010"`. |

---

## ⚠️ Constraints

* `1 <= s.length <= 10^5`
* `s[i]` is either `'0'` or `'1'`.

---

## 💡 Approach & Explanation

The problem asks for the minimum type-2 operations (flips) to make the string alternating, while allowing infinite type-1 operations (shifting the first character to the end). 

Since the target string must be alternating, there are only **two possible valid formats**:
1.  Starts with `'0'`: `010101...` (Let's call this `S1`)
2.  Starts with `'1'`: `101010...` (Let's call this `S2`)

The cyclic nature of the type-1 operation means we can treat the string as circular. A widely used trick for circular arrays/strings is to append the string to itself (`s = s + s`). Any window of size `n` within this doubled string represents a valid state obtained after a certain number of type-1 operations.

We can progressively optimize this:

### 1️⃣ Brute Force with Physical Concatenation (Approach 1)
* Concatenate the string `s` with itself to simulate circular shifts.
* Generate the two target alternating strings (`S1` and `S2`) up to length `2n`.
* Maintain a sliding window of size `n`. As the window slides, check how many characters differ from `S1` and `S2`. Add mismatches for incoming characters and remove mismatches for outgoing characters.
* Update the minimum flips found when the window is exactly size `n`.

### 2️⃣ Logical Concatenation using Modulo (Approach 2)
* Instead of modifying the original string (`s = s + s`) which takes `O(n)` auxiliary space, we can map indices cyclically using the modulo operator `(index % n)`.
* We still pre-generate the target strings `S1` and `S2` to compare against.

### 3️⃣ Complete Constant Space Optimization (Approach 3)
* We don't actually need to store `S1` and `S2` in memory! 
* We can mathematically deduce the expected character for any target string based purely on the current index's parity (whether the index is even or odd).
* For a target starting with `'1'`, even indices should be `'1'` and odd indices should be `'0'`. The reverse is true for a target starting with `'0'`. 
* By dynamically generating expected characters using `index % 2`, we reduce our Space Complexity to **O(1)**.

## 💻 Solution Code

### Approach 1: Sliding Window (Modifying Input String)

```cpp
// Intuition: The problem allows cyclic shifts (Type-1) and character flips (Type-2). A cyclic shift can be simulated by concatenating the string to itself. Since the target is an alternating string, there are only two possible targets.
// Approach: Append the string to itself to handle cyclic shifts. Create two target strings of length 2n ("0101..." and "1010..."). Use a sliding window of size n to track the number of mismatches (flips needed) for both targets. Update the minimum flips when the window size reaches n.
// Time Complexity: O(n) because we iterate through the concatenated string of length 2n once.
// Space Complexity: O(n) to store the concatenated string and target strings of length 2n.
class Solution {
public:
    int minFlips(string s) {
        int n = s.size();
        s += s; // append string to simulate circular shifts

        string s1, s2;

        // generate the two alternating patterns
        for(int i = 0; i < 2*n; i++) {
            s1 += (i % 2 ? '0' : '1');
            s2 += (i % 2 ? '1' : '0');
        }

        int i = 0, j = 0;
        int result1 = 0, result2 = 0;
        int result = INT_MAX;

        while(j < 2*n) {
            // expand window: check mismatches for incoming char
            if(s[j] != s1[j]) result1++;
            if(s[j] != s2[j]) result2++;

            // shrink if window exceeds original length n
            if(j - i + 1 > n) {
                // remove mismatch contribution of the outgoing char
                if(s[i] != s1[i]) result1--;
                if(s[i] != s2[i]) result2--;
                i++;
            }

            // window size exactly n: capture the minimum flips
            if(j - i + 1 == n)
                result = min({result, result1, result2});

            j++;
        }

        return result;
    }
};
/*
*
* Dry Run
* Input: s = "111000", n = 6
* Concatenated s = "111000111000"
* s1 = "101010101010"
* s2 = "010101010101"
* Window expands to size 6.
* j = 5: s[0..5] = "111000", mismatches against s1 = 3, s2 = 3. result = 3.
* j = 6: sliding window moves to s[1..6] = "110001".
* Outgoing char s[0] mismatch removed. Incoming char s[6] mismatch added.
* Continues sliding. Best match is window "100011" vs "101010", taking 2 flips.
* Return 2.
*
*/
```

### Approach 2: Sliding Window (Without Modifying Input String)

```cpp
// Intuition: Concatenating the string physically requires extra memory. We can logically simulate the concatenated string by using the modulo operator on the indices.
// Approach: Pre-generate the target strings of length 2n. Use a sliding window of length n over the logical range [0, 2n-1], mapping indices to the original string using j % n and i % n.
// Time Complexity: O(n) as we traverse 2n elements.
// Space Complexity: O(n) due to the pre-generated target strings s1 and s2.
class Solution {
public:
    int minFlips(string s) {
        int n = s.size();

        string s1, s2;

        // generate the two alternating patterns
        for(int i = 0; i < 2*n; i++) {
            s1 += (i % 2 ? '0' : '1');
            s2 += (i % 2 ? '1' : '0');
        }

        int i = 0, j = 0;
        int result1 = 0, result2 = 0;
        int result = INT_MAX;

        while(j < 2*n) {
            
            // expand window: map cyclical index using % n
            if(s[j%n] != s1[j]) 
                result1++;
            if(s[j%n] != s2[j]) 
                result2++;

            // shrink if window exceeds n
            if(j - i + 1 > n) {
                // remove outgoing char using % n
                if(s[i%n] != s1[i]) 
                    result1--;
                if(s[i%n] != s2[i]) 
                    result2--;
                i++;
            }

            // window size exactly n: capture the minimum flips
            if(j - i + 1 == n)
                result = min({result, result1, result2});

            j++;
        }

        return result;
    }
};
/*
*
* Dry Run
* Input: s = "111000", n = 6
* No physical string appended. Window spans j from 0 to 11 logically.
* j = 0 to 5 behaves same as before.
* j = 6 logically looks at s[6%6] which is s[0].
* Parity matches circular shift flawlessly without memory hit for string append.
* Minimum flips achieved effectively.
* Return 2.
*
*/
```

### Approach 3: Optimized Sliding Window (O(1) Space)

```cpp
// Intuition: We don't even need to pre-build the target strings. The expected character for any index in an alternating string only depends on whether the index is even or odd.
// Approach: For target 1, even indices are '1' and odd indices are '0'. Target 2 is the opposite. Track mismatches on the fly by checking (j % 2). Slide the window, adjusting counts when characters leave and enter based on their index parity.
// Time Complexity: O(n) since the loop runs 2n times, performing O(1) mathematical operations each step.
// Space Complexity: O(1) as no extra strings or arrays are created.
class Solution {
public:
    int minFlips(string s) {
        int n = s.length();

        // variables to track window operations
        int result = INT_MAX;
        int flip1 = 0;
        int flip2 = 0;

        int i = 0;
        int j = 0;

        while(j < 2*n) {

            // determine expected characters based on index parity dynamically
            char expectedCharS1 = (j%2) ? '1' : '0';
            char expectedCharS2 = (j%2) ? '0' : '1';

            // expand window: add flips for mismatches
            if(s[j%n] != expectedCharS1) {
                flip1++;
            }
            if(s[j%n] != expectedCharS2) {
                flip2++;
            }

            // shrink the window from left if it exceeds length n
            if(j - i + 1 > n) { 
                // calculate expected character for the outgoing index
                expectedCharS1 = (i%2) ? '1' : '0';
                expectedCharS2 = (i%2) ? '0' : '1';

                // remove flip contribution of the outgoing char
                if(s[i%n] != expectedCharS1) {
                    flip1--;
                }
                if(s[i%n] != expectedCharS2) {
                    flip2--;
                }
                i++;
            }

            // record minimum flips when window is exactly size n
            if(j - i + 1 == n) {
                result = min({result, flip1, flip2});
            }

            j++;
        }
        return result;
    }
};
/*
*
* Dry Run
* Input: s = "111000", n = 6
* No extra arrays/strings created.
* j = 0: expected S1='0', S2='1'. s[0]='1'. flip1++, flip2 remains same.
* j = 1: expected S1='1', S2='0'. s[1]='1'. flip2++.
* Progress to j = 5 (window size 6). Check min(result, flip1, flip2).
* Progress to j = 6. outgoing index i=0. Removing s[0] based on (i%2). Add s[6%6] based on (j%2).
* Efficient tracking identifies lowest flips across all sliding states dynamically.
* Returns 2.
*
*/
```

## 🧠 Key Insights

* **Virtual Rotation**: Treating the array computationally as `2 * n` in size using the modulo operator `% n` avoids actual physical modification of strings/arrays, saving spatial complexity.
* **Target Inference**: Recognizing that alternating sequences strictly map to index parities (`i % 2`) is the crucial insight to eliminate mapping arrays completely.
* **Constant Time Updates**: Utilizing sliding window means updating the result takes exactly `O(1)` per character added/removed, as opposed to iterating internal segments dynamically.
