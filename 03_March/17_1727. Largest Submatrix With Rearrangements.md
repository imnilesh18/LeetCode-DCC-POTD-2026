## 📝 Problem Statement

You are given a binary matrix `matrix` of size `m x n`, and you are allowed to rearrange the **columns** of the `matrix` in any order.

Return *the area of the largest submatrix within* `matrix` *where **every** element of the submatrix is* `1` *after reordering the columns optimally.*

## ⚠️ Constraints

- `m == matrix.length`
- `n == matrix[i].length`
- `1 <= m * n <= 10^5`
- `matrix[i][j]` is either `0` or `1`.

---

## 💡 Key Insights & Explanation

The core challenge is that we can only rearrange **entire columns**, not individual cells. 

If we process the matrix row by row from top to bottom, we can treat the consecutive `1`s ending at the current row as "towers" of varying heights. For any given row, if we know the height of the `1`s tower in each column, we want to form the largest possible rectangle.

To maximize the area, we should group the tallest towers together. By **sorting the column heights in descending order**, we can easily calculate the maximum possible rectangle area ending at the current row. For a sorted array of heights, the maximum rectangle formed by the first `i + 1` columns (which serve as our base width) is bounded by the height of the `i`-th column. Area = `heights[i] * (i + 1)`.

We can optimize the sorting step out completely in our final approach by observing that if we process columns in the order of their previous row's descending heights, their relative order is maintained.

<details>
<summary><b>Approach 1: Modifying the Input Matrix (Sorting)</b></summary>

```cpp
// Intuition: We want to find the largest submatrix of 1s. If we know the consecutive 1s ending at each cell for a row, we can sort these heights descendingly to greedily find the maximum area.
// Approach: 
// 1. Iterate through each row and column. If the current cell is 1, add the value of the cell above it to accumulate the height.
// 2. For each row, take the accumulated heights, sort them in descending order.
// 3. The area with the current height as the minimum is height * (index + 1). Keep track of the max area.
// Time Complexity: O(m * n log n) where m is number of rows and n is number of columns (due to sorting each row).
// Space Complexity: O(n) for the currRow copy (if we don't count the input matrix modification).
class Solution {
public:
    int largestSubmatrix(vector<vector<int>>& matrix) {
        int m = matrix.size();
        int n = matrix[0].size();
        int result = 0;
        
        for(int row = 0; row < m; row++) {
            
            // Accumulate heights
            for(int col = 0; col < n; col++) {
                if(matrix[row][col] == 1 && row > 0) {
                    matrix[row][col] += matrix[row-1][col];
                }
            }
            
            // Make a copy to sort, keeping the original matrix state for subsequent row calculations
            vector<int> currRow = matrix[row];
            sort(begin(currRow), end(currRow), greater<int>());
            
            // Calculate max area for the sorted heights
            for(int col = 0; col < n; col++) {
                int base   = (col+1); // Columns behind this have height >= currRow[col]
                int height = currRow[col];
                
                result = max(result, base*height);
            }
        }
        
        return result;
    }
};

/*
*
* Dry Run
* Matrix: [[1,0,1], [1,1,1]]
* * Row 0: heights = [1,0,1]. 
* Sorted desc: [1,1,0].
* Area calculation: max(1*1, 1*2, 0*3) = 2.
* * Row 1: heights = [2,1,2] (added from Row 0). 
* Sorted desc: [2,2,1].
* Area calculation: max(2*1, 2*2, 1*3) = 4.
* * Final Max Area = 4.
*
*/
```

<summary><b>Approach 2: Without Modifying the Input Matrix (Sorting)</b></summary>

```cpp
// Intuition: Similar to Approach 1, but we avoid modifying the original input matrix, which is a good practice in interviews.
// Approach:
// 1. Maintain a `prevRow` array to store the heights of the previous row.
// 2. For each row, create a `currRow`. If matrix[row][col] is 1, currRow[col] = prevRow[col] + 1.
// 3. Sort a copy of `currRow` descendingly, calculate max area.
// 4. Update `prevRow = currRow` for the next iteration.
// Time Complexity: O(m * n log n) due to sorting each row.
// Space Complexity: O(n) for storing the previous and current row heights.
class Solution {
public:
    int largestSubmatrix(vector<vector<int>>& matrix) {
        int m = matrix.size();
        int n = matrix[0].size();
        vector<int> prevRow(n, 0);
        int result = 0;
        
        for(int row = 0; row < m; row++) {
            vector<int> currRow = matrix[row];
            
            // Accumulate heights using prevRow
            for(int col = 0; col < n; col++) {
                if(currRow[col] == 1) {
                    currRow[col] += prevRow[col];
                }
            }
            
            // Sort to greedily maximize area
            vector<int> sortedRow = currRow;
            sort(begin(sortedRow), end(sortedRow), greater<int>());
            
            for(int col = 0; col < n; col++) {
                int base   = (col+1); // Base width
                int height = sortedRow[col];
                
                result = max(result, base*height);
            }
            
            // Update prevRow for the next iteration
            prevRow = currRow;
        }
        
        return result;
    }
};

/*
*
* Dry Run
* Matrix: [[0,0,1], [1,1,1]]
* * Row 0: currRow = [0,0,1]. Sorted: [1,0,0]. 
* Area = 1. prevRow = [0,0,1].
* * Row 1: matrix row is [1,1,1]. currRow becomes [1,1,2]. 
* Sorted: [2,1,1]. 
* Area = max(2*1, 1*2, 1*3) = 2. 
* prevRow = [1,1,2].
*
*/
```

<summary><b>Approach 3: Optimal approach (Without Sorting)</b></summary>

```cpp
// Intuition: We can avoid the O(n log n) sorting at each row. If we process columns in the order of their previous heights (which are naturally sorted), we can maintain the sorted order in O(n) time.
// Approach: 
// 1. Maintain a vector of pairs `prevHeights` storing {height, column_index} in strictly descending sorted order.
// 2. For the current row, first iterate through `prevHeights`. If the matrix has a 1 at that column, it extends the previous tower. Push {height+1, col} to `heights`.
// 3. Then, iterate through all columns from 0 to n-1 to find newly started 1s (where matrix is 1 but it wasn't tracked in `prevHeights`). Push {1, col} to `heights`.
// 4. Calculate max area using the naturally sorted `heights`. Update `prevHeights`.
// Time Complexity: O(m * n) as we eliminated the sorting step entirely.
// Space Complexity: O(n) to store the heights and seen arrays.
class Solution {
public:
    int largestSubmatrix(vector<vector<int>>& matrix) {
        int m = matrix.size();
        int n = matrix[0].size();
        
        // Stores pairs of {height, column_index}
        vector<pair<int,int>> prevHeights;
        int result = 0;

        for (int row = 0; row < m; row++) {
            vector<pair<int,int>> heights;
            vector<bool> seen = vector<bool>(n, false);

            // 1. Process columns that already had a height > 0 in the previous row
            // Since prevHeights is already in descending order, adding 1 keeps them in descending order
            for (auto [height, col] : prevHeights) {
                if (matrix[row][col] == 1) {
                    heights.push_back({height + 1, col});
                    seen[col] = true;
                }
            }

            // 2. Process columns that newly became 1 (height is 1)
            // These naturally belong at the end of our descending list
            for (int col = 0; col < n; col++) {
                if (seen[col] == false && matrix[row][col] == 1) {
                    heights.push_back({1, col});
                }
            }

            // 3. Calculate max area using the naturally sorted heights
            for (int i = 0; i < heights.size(); i++) {
                int base   = (i+1);
                int height = heights[i].first;
                result = max(result, base*height);
            }

            // Update for the next row
            prevHeights = heights;
        }

        return result;
    }
};

/*
*
* Dry Run
* Matrix: [[1,1,0], [1,0,1]]
* * prevHeights = []
* Row 0 ([1,1,0]): 
* No prevHeights. New ones at col 0, 1.
* heights = [{1,0}, {1,1}]. 
* Area = max(1*1, 1*2) = 2. 
* prevHeights = [{1,0}, {1,1}]
* * Row 1 ([1,0,1]): 
* From prev: col 0 has 1 -> push {2,0}. col 1 has 0 -> ignore.
* From new: col 2 has 1 -> push {1,2}.
* heights = [{2,0}, {1,2}]. (Naturally sorted!)
* Area = max(2, 2*1, 1*2) = 2.
*
*/
```
