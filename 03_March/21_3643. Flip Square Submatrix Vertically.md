## 📝 Problem Statement

You are given an `m x n` integer matrix `grid`, and three integers `x`, `y`, and `k`.

The integers `x` and `y` represent the row and column indices of the **top-left corner** of a **square submatrix**, and the integer `k` represents the size (side length) of the square submatrix.

Your task is to **flip the submatrix** by reversing the order of its rows vertically. Return the updated matrix.

## 🎯 Constraints

- `m == grid.length`
- `n == grid[i].length`
- `1 <= m, n <= 50`
- `1 <= grid[i][j] <= 100`
- `0 <= x < m`
- `0 <= y < n`
- `1 <= k <= min(m - x, n - y)`

### 🧠 Approach & Translation
1. **Identify the Boundaries**: Start by pinpointing the exact coordinates of the `k x k` submatrix. The starting row is `x` and the ending row is `x + k - 1`. The starting column is `y` and the ending column is `y + k - 1`.
2. **Two-Pointer Reversal**: Instead of creating a new matrix or a temporary array, we can vertically flip the submatrix in-place. We use two pointers for the rows:
   - `startRow` tracking the top of the un-swapped submatrix.
   - `endRow` tracking the bottom of the un-swapped submatrix.
3. **Column Iteration**: For each corresponding top and bottom row, iterate through the columns of the submatrix (from `startCol` to `endCol`) and swap the elements at `grid[startRow][col]` and `grid[endRow][col]`.
4. **Pointer Adjustment**: After processing an entire row, increment `startRow` (handled by the outer `for` loop) and decrement `endRow` until they meet or cross in the middle.
</details>

---

## 🔑 Key Insights

- **In-Place Modification**: By simply swapping elements between the `i`-th row from the top and the corresponding row from the bottom of the submatrix, we achieve an `O(1)` space complexity.
- **Independent Columns**: During the vertical flip, values within the same column are completely independent of values in other columns. This allows for a straightforward nested loop logic without intermediate overlapping states.

## 💻 Solution Code

```cpp
// Intuition: The problem requires vertically flipping a k x k submatrix starting at (x, y). This can be effectively achieved by swapping the elements of the top row with the bottom row of the submatrix, moving gradually towards the center.
// Approach: We define the boundary of the submatrix using `startRow`, `endRow`, `startCol`, and `endCol`. Using a two-pointer approach for the rows, we swap the values `grid[i][j]` and `grid[endRow][j]`. The `endRow` pointer moves upwards as the outer loop iterator `i` moves downwards.
// Time Complexity: O(k^2), as we iterate through roughly half of the elements in the k x k submatrix to perform swaps.
// Space Complexity: O(1), as we perform the vertical flip in-place without utilizing any extra auxiliary memory.

class Solution {
public:
    vector<vector<int>> reverseSubmatrix(vector<vector<int>>& grid, int x, int y, int k) {
        
        // Define top and bottom row boundaries for the submatrix
        int startRow = x;
        int endRow = x + k - 1;

        // Define left and right column boundaries
        int startCol = y;
        int endCol = y + k - 1;

        // Iterate halfway through the rows of the submatrix
        for(int i = startRow; i <= endRow; i++) {
            
            // Swap elements across all columns for the current row pair
            for(int j = startCol; j <= endCol; j++) {
                swap(grid[i][j], grid[endRow][j]);
            }
            
            // Move the bottom row pointer upwards towards the center
            endRow--;
        }

        // Return the vertically flipped grid
        return grid;
    }
};

/*
*
* Dry Run
*
* grid = [[1, 2], [3, 4]], x = 0, y = 0, k = 2
* startRow = 0, endRow = 1, startCol = 0, endCol = 1
* * Loop 1 (i = 0):
* - Is i (0) <= endRow (1)? Yes.
* - j = 0: swap(grid[0][0], grid[1][0]) => grid becomes [[3, 2], [1, 4]]
* - j = 1: swap(grid[0][1], grid[1][1]) => grid becomes [[3, 4], [1, 2]]
* - endRow decrements to 0
*
* Loop 2 (i = 1):
* - Is i (1) <= endRow (0)? No. Loop terminates.
*
* Output: [[3, 4], [1, 2]]
*
*/
```

---

## 📝 Notes

- Make sure the loop limits strictly adhere to the `k x k` bounds to avoid `Index Out of Bounds` errors.
- The `endRow` decrements while the `for` loop implicitly increments `i`. Once `i > endRow`, it means we've reached or crossed the middle of the submatrix, avoiding double-swapping which would revert the matrix to its original form.
