## 📝 Problem Statement

Given an `n x n` binary grid, in one step you can choose two **adjacent rows** of the grid and swap them.

A grid is said to be **valid** if all the cells above the main diagonal are **zeros**.

Return *the minimum number of steps* needed to make the grid valid, or `-1` if the grid cannot be valid.

The main diagonal of a grid is the diagonal that starts at cell `(1, 1)` and ends at cell `(n, n)`.

## 🛑 Constraints

- `n == grid.length == grid[i].length`
- `1 <= n <= 200`
- `grid[i][j]` is either `0` or `1`

---

## 💡 Approach & Explanation

To make all cells above the main diagonal strictly zero, we need to focus on the count of trailing (continuous) zeros in each row.

### Key Observations:
1. For row `0` (the topmost row), all elements from index `1` to `n-1` must be zero. This implies the row needs **at least `n - 1`** trailing zeros.
2. For row `1`, all elements from index `2` to `n-1` must be zero. This row needs **at least `n - 2`** trailing zeros.
3. Generalizing this, row `i` needs **at least `n - i - 1`** trailing zeros for the grid to be valid.

### Step-by-Step Logic:
1. **Precomputation:** First, determine how many consecutive trailing zeros exist for each row. Store this in an array `endZeros`.
2. **Finding the Candidates:** Start iterating row by row (from `i = 0` to `n-1`). Calculate the required trailing zeros for the current row: `need = n - i - 1`.
3. **Nearest Match (Greedy Strategy):** Traverse downwards from the current row `i` to find the **nearest** row `j` that satisfies `endZeros[j] >= need`. 
   - We pick the nearest valid row to minimize the number of adjacent swaps. This logic closely mimics the behavior of **Bubble Sort**.
4. **Validation:** If no such row `j` exists that satisfies the condition for row `i`, it is impossible to make the grid valid. Return `-1`.
5. **Simulation (Swapping):** Once the nearest valid row `j` is found, calculate the required steps to move it to position `i` (which is exactly `j - i` steps). Add this to our `steps` counter.
6. **Update State:** "Bubble up" this selected row inside the `endZeros` array by repeatedly swapping adjacent elements from `j` up to `i`.
7. Continue until all rows are validated. Return the total swap `steps`.

## 💻 Solution Code

### Optimized Solution (C++)

```cpp
// Intuition:
// To make the grid valid, all cells above the main diagonal must be zero. This translates to a specific requirement for each row: row `i` needs at least `n - i - 1` consecutive zeros at its end. Since we can only swap adjacent rows, we need to find the nearest valid row and "bubble" it up to the current position, exactly like Bubble Sort.
//
// Approach:
// 1. Calculate the number of trailing zeros for each row and store it in an array `endZeros`.
// 2. Iterate through each row `i` from 0 to n-1.
// 3. For each row, calculate the required consecutive trailing zeros: `need = n - i - 1`.
// 4. Search for the closest row `j` (from `i` to `n-1`) that satisfies `endZeros[j] >= need`.
// 5. If no such row is found, the grid cannot be made valid, return -1.
// 6. Otherwise, add the number of swaps required (`j - i`) to the total steps.
// 7. Simulate the swap by moving the element at `j` up to `i` in the `endZeros` array.
// 8. Return the total steps.
//
// Time Complexity: O(n^2) - We iterate through n rows. For each row, we might search up to n remaining rows and bubble up the chosen row, taking O(n) time. Total time is O(n^2).
// Space Complexity: O(n) - We use an array of size n to store the count of trailing zeros for each row.

class Solution {
public:
    int minSwaps(vector<vector<int>>& grid) {
        int n = grid.size(); // rows == columns

        vector<int> endZeros(n, 0);
        // endZeros[i] = count of consecutive 0s from the end of ith row

        for(int i = 0; i < n; i++) {
            int j = n - 1; // start from end
            int count = 0;
            
            // Count trailing zeros for current row
            while(j >= 0 && grid[i][j] == 0) {
                count++;
                j--;
            }
            endZeros[i] = count;
        }

        int steps = 0;

        for(int i = 0; i < n; i++) {
            int need = n - i - 1; // Required zeros at the end
            int j = i;

            // Find nearest row satisfying condition
            while(j < n && endZeros[j] < need) {
                j++;
            }

            // Return -1 if no valid row found
            if(j == n) {
                return -1;
            }

            // Add number of adjacent swaps needed
            steps += j - i;

            // Simulate the swaps in endZeros array
            while(j > i) {
                swap(endZeros[j], endZeros[j-1]);
                j--;
            }
        }

        return steps;
    }
};

/*
*
* Dry Run
*
* Input: grid = [[0,0,1],[1,1,0],[1,0,0]], n = 3
* * 1. Calculate endZeros:
* Row 0: [0,0,1] -> 0 trailing zeros. endZeros[0] = 0
* Row 1: [1,1,0] -> 1 trailing zero.  endZeros[1] = 1
* Row 2: [1,0,0] -> 2 trailing zeros. endZeros[2] = 2
* endZeros = [0, 1, 2]
*
* 2. Iterate to satisfy each row:
* i = 0: need = 3 - 0 - 1 = 2. 
* Search from j = 0. endZeros[0]=0, endZeros[1]=1, endZeros[2]=2.
* Match found at j = 2. 
* Steps += (2 - 0) = 2.
* Bubble up index 2 to 0: swap(1,2), swap(0,1).
* endZeros becomes [2, 0, 1].
*
* i = 1: need = 3 - 1 - 1 = 1.
* Search from j = 1. endZeros[1]=0, endZeros[2]=1.
* Match found at j = 2.
* Steps += (2 - 1) = 1.
* Bubble up index 2 to 1: swap(1,2).
* endZeros becomes [2, 1, 0].
*
* i = 2: need = 3 - 2 - 1 = 0.
* Search from j = 2. endZeros[2]=0 >= need (0).
* Match found at j = 2.
* Steps += (2 - 2) = 0.
* endZeros remains [2, 1, 0].
*
* 3. Total steps = 2 + 1 + 0 = 3. Return 3.
*
*/

```

## 📌 Notes & Key Insights

* **Greedy + Simulation:** The problem boils down to a greedy choice. At every step, picking the nearest possible valid row inherently guarantees the minimum number of swaps. Searching further down the matrix would just yield redundant swaps.
* **State Abstraction:** The entire 2D matrix can be effectively abstracted into a 1D array representing the properties we actually care about (the trailing zero count). Modifying the 1D array instead of the 2D matrix saves massive memory and computational overhead during simulations.
* **Bubble Sort Characteristic:** The algorithm is functionally similar to a localized Bubble Sort, finding the right element and continuously "bubbling" it to its sorted target position.
