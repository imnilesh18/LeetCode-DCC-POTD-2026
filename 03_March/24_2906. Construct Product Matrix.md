## 📜 Problem Statement

Given a **0-indexed** 2D integer matrix `grid` of size `n * m`, we define a **0-indexed** 2D matrix `p` of size `n * m` as the **product matrix** of `grid` if the following condition is met:

Each element `p[i][j]` is calculated as the product of all elements in `grid` except for the element `grid[i][j]`. This product is then taken modulo `12345`.

**Return** the product matrix of `grid`.

---

## 🚦 Constraints

- `1 <= n == grid.length <= 10^5`
- `1 <= m == grid[i].length <= 10^5`
- `2 <= n * m <= 10^5`
- `1 <= grid[i][j] <= 10^9`

---

## 🧠 Approach

<details>
<summary><b>Click here to view the step-by-step logic!</b></summary>

1. **The Core Dilemma:** The initial thought might be to calculate the total product of all elements in the matrix and then divide by `grid[i][j]` to get the product excluding that cell. However, the final answer requires a modulo operation (`12345`). We cannot safely divide numbers after taking a modulo unless we use **Modular Multiplicative Inverse**. 
   *But here's the catch:* Modular Multiplicative Inverse requires the modulo number to be prime. `12345` is **not** a prime number. Hence, division is completely off the table.

2. **Prefix and Suffix Products (The 1D Analogy):**
   Treat the 2D grid conceptually as a 1D flattened array. To find the product of all elements except the one at the current index, we simply need:
   `(Product of all elements to its left) * (Product of all elements to its right)`
   
3. **Space Optimization:**
   Instead of using two extra 2D matrices to store prefix and suffix arrays, we can generate the output matrix `p` directly:
   - **Pass 1 (Right-to-Left, Bottom-to-Top):** Iterate backward over the matrix. For every cell, record the `suffix` product computed so far into `p[i][j]`, and then multiply `suffix` by `grid[i][j]`.
   - **Pass 2 (Left-to-Right, Top-to-Bottom):** Iterate forward. For every cell, multiply the existing `p[i][j]` by the `prefix` product computed so far. Then, multiply `prefix` by `grid[i][j]`.

This clever overlapping avoids excessive memory consumption while keeping our algorithm at a highly optimal $O(N \times M)$ runtime!

</details>

## 💻 Solution Code

### Brute-Force Solution (C++)
> **Note:** This solution is only for theoretical understanding and will result in a **Time Limit Exceeded (TLE)** error on large inputs.

```cpp
// Intuition: The most basic way is to visit every element and, for each element, traverse the entire matrix again to multiply every other element.
// Approach: Use four nested loops. Two for the current element, and two to iterate through the matrix for calculating the product excluding the current element.
// Time Complexity: O((n*m)^2)
// Space Complexity: O(n*m) for the result matrix
class Solution {
public:
    vector<vector<int>> constructProductMatrix(vector<vector<int>>& grid) {
        int MOD = 12345;
        int n = grid.size(), m = grid[0].size();
        vector<vector<int>> p(n, vector<int>(m, 1));
        
        for(int i = 0; i < n; i++) {
            for(int j = 0; j < m; j++) {
                long long prod = 1;
                // Calculate product of all other elements
                for(int r = 0; r < n; r++) {
                    for(int c = 0; c < m; c++) {
                        if(r == i && c == j) continue;
                        prod = (prod * grid[r][c]) % MOD;
                    }
                }
                p[i][j] = prod;
            }
        }
        return p;
    }
};
```

### Optimized Solution (C++)
> Highly efficient approach using prefix and suffix running products.

```cpp
// Intuition: Since modulo 12345 is not prime, modular division is invalid. We use prefix and suffix products to gather the multiplications before and after an element dynamically.
// Approach: 
// 1. Initialize result matrix `p`. Let `suffix = 1` and `prefix = 1`.
// 2. Iterate backward to populate `p[i][j]` with the suffix product, then update `suffix`.
// 3. Iterate forward, multiply the pre-populated `p[i][j]` by `prefix`, then update `prefix`.
// Time Complexity: O(n * m), as we make two linear passes over the 2D grid.
// Space Complexity: O(n * m) to construct the return matrix `p`. No extra auxiliary space is used.

class Solution {
public:
    vector<vector<int>> constructProductMatrix(vector<vector<int>>& grid) {
        int MOD = 12345;
        int n = grid.size();
        int m = grid[0].size();

        // result = product matrix initialized to size n*m
        vector<vector<int>> p(n, vector<int>(m)); 

        // Pass 1: Compute suffix products and store directly in the result matrix
        long long suffix = 1;
        for(int i = n - 1; i >= 0; i--) {
            for(int j = m - 1; j >= 0; j--) {
                // Assign current suffix product to the cell
                p[i][j] = suffix;
                // Multiply suffix by the current grid element for the next iteration
                suffix = (suffix * grid[i][j]) % MOD;
            }
        }

        // Pass 2: Compute prefix products on the fly and multiply with the stored suffix
        long long prefix = 1;
        for(int i = 0; i < n; i++) {
            for(int j = 0; j < m; j++) {
                // Multiply the stored suffix product by the incoming prefix product
                p[i][j] = (prefix * p[i][j]) % MOD;
                // Multiply prefix by the current grid element for the next iteration
                prefix = (prefix * grid[i][j]) % MOD;
            }
        }

        return p;
    }
};

/*
*
* Dry Run
*
* Input: grid = [[1,2],[3,4]], MOD = 12345
* Dimensions: n = 2, m = 2
* * --- Pass 1: Suffix Products (Bottom-Right to Top-Left) ---
* Initial suffix = 1
* At (1,1) val=4: p[1][1] = 1, suffix = (1 * 4) % MOD = 4
* At (1,0) val=3: p[1][0] = 4, suffix = (4 * 3) % MOD = 12
* At (0,1) val=2: p[0][1] = 12, suffix = (12 * 2) % MOD = 24
* At (0,0) val=1: p[0][0] = 24, suffix = (24 * 1) % MOD = 24
* State of p: [[24, 12], [4, 1]]
*
* --- Pass 2: Prefix Products (Top-Left to Bottom-Right) ---
* Initial prefix = 1
* At (0,0) val=1: p[0][0] = (24 * 1) = 24, prefix = (1 * 1) % MOD = 1
* At (0,1) val=2: p[0][1] = (12 * 1) = 12, prefix = (1 * 2) % MOD = 2
* At (1,0) val=3: p[1][0] = (4 * 2) = 8, prefix = (2 * 3) % MOD = 6
* At (1,1) val=4: p[1][1] = (1 * 6) = 6, prefix = (6 * 4) % MOD = 24
* Final state of p: [[24, 12], [8, 6]]
* */
```

## 📝 Notes
- Taking modulo at every step avoids integer overflow (`suffix` and `prefix` can easily cross limits if not modulated). Always use `long long` for running products to prevent overflow before the `% MOD` operation executes.
- Recognizing that modular division via inverse is invalid for non-prime numbers is crucial. Without this realization, you might face runtime logical flaws.

## 🔍 Key Insights
- Whenever you face an `except self` logic condition, the **Prefix/Suffix running products** pattern is generally the gold standard.
- Working directly on the answer array `p` to accumulate suffix and then combine it with prefix is a vital space-saving strategy, keeping $S.C. = O(1)$ excluding the answer array format.
