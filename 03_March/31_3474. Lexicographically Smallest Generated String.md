## 📜 Problem Statement

**Problem Link:** [Lexicographically Smallest Generated String](https://leetcode.com/problems/lexicographically-smallest-generated-string/description/)

You are given two strings, `str1` and `str2`, of lengths `n` and `m`, respectively.

A string `word` of length `n + m - 1` is defined to be **generated** by `str1` and `str2` if it satisfies the following conditions for **each** index `0 <= i <= n - 1`:

* If `str1[i] == 'T'`, the **substring** of `word` with size `m` starting at index `i` is **equal** to `str2`, i.e., `word[i..(i + m - 1)] == str2`.
* If `str1[i] == 'F'`, the **substring** of `word` with size `m` starting at index `i` is **not equal** to `str2`, i.e., `word[i..(i + m - 1)] != str2`.

Return the **lexicographically smallest** possible string that can be **generated** by `str1` and `str2`. If no string can be generated, return an empty string `""`.

---

## 📋 Constraints

- `1 <= n == str1.length <= 10^4`
- `1 <= m == str2.length <= 500`
- `str1` consists only of `'T'` or `'F'`.
- `str2` consists only of lowercase English characters.

---

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `str1 = "TFTF", str2 = "ab"` | `"ababa"` | At index 0 ('T'), substring is "ab". At index 1 ('F'), substring is "ba" != "ab". At index 2 ('T'), substring is "ab". At index 3 ('F'), substring is "ba" != "ab". "ababa" is the lexicographically smallest valid string. |
| **2** | `str1 = "TFTF", str2 = "abc"` | `""` | No string can satisfy the conditions because placing "abc" at index 0 and 2 causes conflicting character requirements at overlapping indices. |
| **3** | `str1 = "F", str2 = "d"` | `"a"` | The string "a" is not equal to "d", and it is the lexicographically smallest possible string. |

## 🧠 Approach

<details>
<summary><b>Click to expand the detailed English explanation</b></summary>
<br>

To solve this problem, we need to construct a string of length $N = n + m - 1$. We can achieve this using a multi-step **Greedy Strategy**:

1.  **Process all 'T' conditions first (The Absolute Constraints):**
    * Whenever we see a `'T'` at index `i` in `str1`, we have absolutely no choice: the substring starting at `i` of length `m` *must* be exactly `str2`.
    * We iterate through `str1`. If we find a `'T'`, we attempt to place `str2` in our resulting `word`.
    * If the spot is already occupied by a character that contradicts `str2` (a collision), it means the configuration is impossible, and we immediately return `""`.

2.  **Fill empty spaces greedily (Lexicographically Smallest Goal):**
    * After processing all `'T'`s, there might be empty spaces remaining in our `word`.
    * Since we want the *lexicographically smallest* string, we should greedily fill all remaining empty spaces with the smallest alphabet character: `'a'`.
    * Crucially, we must keep track of the positions where we placed these `'a'`s. Why? Because these are the *only* characters we are allowed to alter later if we need to fix any violated `'F'` conditions. We can store this in a boolean array `canChange`.

3.  **Process all 'F' conditions (Validating and Breaking Equality):**
    * Now, we iterate through `str1` again to validate the `'F'` conditions.
    * If `str1[i] == 'F'`, we check if the substring in `word` starting at `i` is currently equal to `str2`.
    * If it is equal, we have a violation! We must change at least one character in this substring to break the equality.
    * **How to change it optimally?** * We should only change a character that was marked as `canChange == true`.
        * To keep the string *lexicographically smallest*, we should change the character closest to the **end** of the substring (right-to-left traversal). Changing a character at the end of a string increases its lexicographical value less than changing a character at the beginning (e.g., changing `aaaa` to `aaab` is better than `baaa`).
        * We change that `'a'` to a `'b'`. Just one change is enough to break the equality.
        * If we scan the entire substring and find no changeable character, it means the equality cannot be broken without violating a `'T'` condition. Thus, it's impossible, and we return `""`.

## 💻 Solution Code

### Optimized Solution (Greedy + String Matching)

```cpp
// Intuition: 
// To generate the lexicographically smallest string, we must strictly satisfy all 'T' conditions first.
// Any unconstrained positions should be filled with 'a' for the smallest result.
// Finally, we validate 'F' conditions. If an 'F' condition is violated, we fix it by changing the rightmost changeable 'a' to 'b' to minimize the lexicographical increase.

// Approach: 
// 1. Initialize a string `word` of length n + m - 1 with a dummy character '$'.
// 2. Initialize a boolean array `canChange` to track which positions we manually fill and can safely alter later.
// 3. Process all 'T' in str1: forcefully place str2 in word starting at index i. If a collision occurs (conflicting characters), return "".
// 4. Process empty spaces: fill any remaining '$' with 'a' (to make it lexicographically smallest) and mark them as changeable (true).
// 5. Process all 'F' in str1: check if the substring starting at i equals str2.
//    - If it does, we must break the match to satisfy 'F'.
//    - Traverse the substring from right to left (to minimize lexicographical impact).
//    - Change the first changeable 'a' to 'b'.
//    - If no character can be changed, return "".
// 6. Return the constructed word.

// Time Complexity: O(n * m)
// We iterate through `str1` of length n. For each 'T' and 'F', we potentially iterate through a substring of length m. Thus, the worst-case time complexity is bounded by O(n * m).

// Space Complexity: O(n + m - 1)
// The space is strictly used to store the resulting string `word` of length N = n + m - 1, and the boolean tracking array `canChange` of the same length.

class Solution {
public:
    // Helper function to check if the substring matches str2
    bool isSame(string &word, string & str2, int i, int m) {
        for(int j = 0; j < m; j++) {
            if(word[i] != str2[j]) {
                return false;
            }
            i++;
        }
        return true;
    }

    string generateString(string str1, string str2) {
        int n = str1.length();
        int m = str2.length();

        int N = n + m - 1;
        // Initialize the result word with a dummy character '$'
        string word(N, '$');
        
        // Array to track indices that can be modified later
        vector<bool> canChange(N, false);

        // Step 1: Process all 'T' constraints
        for(int i = 0; i < n; i++) {
            if(str1[i] == 'T') {
                int i_ = i;
                for(int j = 0; j < m; j++) {
                   // word[i] = str2[j], word[i+1] = str2[j+1] and so on...
                   // Check for collision: if occupied by a different character, it's impossible
                   if(word[i_] != '$' && word[i_] != str2[j]) {
                        return "";
                   }
                   word[i_] = str2[j];
                   i_++;
                }
            }
        }

        // Step 2: Fill the remaining unconstrained spaces with 'a'
        for(int i = 0; i < N; i++) {
            if(word[i] == '$') {
                word[i] = 'a'; // Lexicographically smallest choice
                canChange[i] = true; // Mark as modifiable
            }
        }

        // Step 3: Process all 'F' constraints
        for(int i = 0; i < n; i++) {
            if(str1[i] == 'F') {
                // If same, then we need to break the equality
                if(isSame(word, str2, i, m)) { 
                    
                    bool changed = false;
                    // Traverse right-to-left to modify the latest possible character 
                    // This ensures the string remains lexicographically smallest
                    for(int k = i + m - 1; k >= i; k--) {
                        if(canChange[k] == true) {
                            word[k] = 'b'; // 'b' is the next smallest character after 'a'
                            changed = true;
                            break; // Equality broken, exit loop
                        }
                    }

                    // If we were not able to change any character to break equality
                    if(changed == false) { 
                        return "";
                    }
                }
            }
        }

        return word;
    }
};

/*
*
* Dry Run
* str1 = "TFTF", str2 = "ab"
* n = 4, m = 2 -> N = 5
* Initial word = "$$$$$", canChange = [F, F, F, F, F]
* * Processing 'T's:
* i = 0 ('T'): word[0..1] becomes "ab" -> word = "ab$$$"
* i = 2 ('T'): word[2..3] becomes "ab" -> word = "abab$"
* * Filling empty spaces:
* word[4] is '$', fill with 'a'. 
* word = "ababa", canChange = [F, F, F, F, T]
* * Processing 'F's:
* i = 1 ('F'): Check word[1..2] == "ba". "ba" != "ab" (str2). Valid.
* i = 3 ('F'): Check word[3..4] == "ba". "ba" != "ab" (str2). Valid.
* * Result: "ababa"
*
*/
````

## 🎯 Key Insights

  * **Strict vs Flexible Assignments:** The problem elegantly splits into "must-do" constraints (`T`) and "must-not-do" constraints (`F`). Fixing `T` first establishes the unchangeable skeleton of the string.
  * **Greedy Sub-choices:** Using `'a'` for unconstrained spots handles the "lexicographically smallest" requirement immediately.
  * **Right-to-Left Alteration:** When an `F` constraint is violated, modifying the character at the rightmost possible index ensures the overall string is kept as small as possible. For example, `(a,a,a)` altered to `(a,a,b)` is smaller than `(b,a,a)`.
