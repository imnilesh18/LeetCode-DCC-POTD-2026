## 📝 Problem Statement

You are given two strings `s1` and `s2`, both of length `4`, consisting of lowercase English letters.

You can apply the following operation on any of the two strings **any** number of times:
- Choose any two indices `i` and `j` such that `j - i = 2`, then **swap** the two characters at those indices in the string.

Return `true` if you can make the strings `s1` and `s2` equal, and `false` otherwise.

---

## 🎯 Constraints

- `s1.length == s2.length == 4`
- `s1` and `s2` consist only of lowercase English letters.

---

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `s1 = "abcd", s2 = "cdab"` | `true` | We can do the following operations on `s1`:<br>1. Choose indices `i = 0, j = 2`. The resulting string is `s1 = "cbad"`.<br>2. Choose indices `i = 1, j = 3`. The resulting string is `s1 = "cdab" = s2`. |
| **2** | `s1 = "abcd", s2 = "dacb"` | `false` | It is not possible to make the two strings equal. |

---

## 🧠 Approach

Because the length of both strings is strictly fixed at `4`, and we can only swap characters if the difference between their indices is `2` (`j - i = 2`), we can deduce the following:
1. **Even Indices:** Characters at index `0` can only be swapped with index `2`.
2. **Odd Indices:** Characters at index `1` can only be swapped with index `3`.

This means the characters at even indices form one independent group, and characters at odd indices form another. Therefore, for `s1` to be transformable into `s2`:
- The set of characters at indices `(0, 2)` in `s1` must exactly match the set of characters at indices `(0, 2)` in `s2`.
- The set of characters at indices `(1, 3)` in `s1` must exactly match the set of characters at indices `(1, 3)` in `s2`.

Since the length is so small, we can directly check these conditions. Alternatively, a frequency map can cleanly separate the odd and even indices checking, which is highly scalable for larger string lengths.

## 💻 Solution Code

<details>
<summary><b>1️⃣ Brute-force / Direct Check Solution (C++)</b></summary>

```cpp
// Intuition: Since the string length is exactly 4, we are only allowed to swap characters at indices (0, 2) and indices (1, 3).
// Approach: Directly check if the sets of characters at the even indices (0, 2) and the odd indices (1, 3) match between s1 and s2 either directly or by swapping.
// Time Complexity: O(1) - Constant time as we are only performing exactly two boolean checks regardless of input.
// Space Complexity: O(1) - No extra space is utilized.
class Solution {
public:
    bool canBeEqual(string s1, string s2) {
        // Check if even indices (0 and 2) match directly or cross-match
        bool condition1 = (s1[0] == s2[0] && s1[2] == s2[2]) || (s1[0] == s2[2] && s1[2] == s2[0]);

        // Check if odd indices (1 and 3) match directly or cross-match
        bool condition2 = (s1[1] == s2[1] && s1[3] == s2[3]) || (s1[1] == s2[3] && s1[3] == s2[1]);

        // Both even and odd pairs must satisfy the matching condition
        return condition1 && condition2;
    }
};

/*
*
* Dry Run
*
* Input: s1 = "abcd", s2 = "cdab"
* * Even Indices (0, 2):
* s1[0]='a', s1[2]='c' | s2[0]='c', s2[2]='a'
* condition1 = (a==c && c==a) || (a==a && c==c) -> false || true -> true
* * Odd Indices (1, 3):
* s1[1]='b', s1[3]='d' | s2[1]='d', s2[3]='b'
* condition2 = (b==d && d==b) || (b==b && d==d) -> false || true -> true
* * Return: condition1 && condition2 -> true && true = true
*
*/
```

<summary><b>2️⃣ Optimized / Frequency Map Solution (C++)</b></summary>

```cpp
// Intuition: By keeping independent frequency counts of characters at even and odd indices, we can verify if the required character sets are identical between the two strings.
// Approach: Utilize two frequency arrays of size 26. Iterate through the strings; increment counts for characters in s1 and decrement for s2 based on their parity (even or odd index). A completely zeroed-out array at the end means the strings can be made equal.
// Time Complexity: O(N) - Where N is the string length. Since N is fixed at 4, this effectively evaluates to O(1).
// Space Complexity: O(1) - Fixed space of two arrays of size 26.
class Solution {
public:
    bool canBeEqual(string s1, string s2) {
        int even[26] = {0}; // Frequency map for even indices
        int odd[26] = {0};  // Frequency map for odd indices

        int n = s1.length();

        for(int i = 0; i < 4; i++) {
            if(i % 2 == 0) { // even indices
                even[s1[i] - 'a']++;
                even[s2[i] - 'a']--;
            } else { // odd indices
                odd[s1[i] - 'a']++;
                odd[s2[i] - 'a']--;
            }
        }

        // Verify if all frequencies perfectly balance out to zero
        for(int i = 0; i < 26; i++) {
            if(even[i] != 0 || odd[i] != 0)
                return false;
        }

        return true;
    }
};

/*
*
* Dry Run
*
* Input: s1 = "abcd", s2 = "cdab"
* * i=0 (even): even['a'-'a']++, even['c'-'a']-- => even[0]=1, even[2]=-1
* * i=1 (odd) : odd['b'-'a']++, odd['d'-'a']--  => odd[1]=1, odd[3]=-1
* * i=2 (even): even['c'-'a']++, even['a'-'a']-- => even[2]=0, even[0]=0
* * i=3 (odd) : odd['d'-'a']++, odd['b'-'a']--  => odd[3]=0, odd[1]=0
* * After the loop, all elements in both `even` and `odd` arrays are 0.
* Return: true
*
*/
```

## 📌 Notes & Key Insights
- Since the allowed operation is bounded by `j - i = 2`, characters can never jump from an even index to an odd index, or vice versa. This concept is commonly referred to as maintaining **parity**.
- While the Direct Check `O(1)` approach is simpler to write for strings of fixed length `4`, the Frequency Map `O(N)` approach is far superior conceptually.
