## 📝 Problem Statement
Given a binary string `s` ​​​​​without leading zeros, return `true`​​​ if `s` contains **at most one contiguous segment of ones**. Otherwise, return `false`.

### ⛔ Constraints
- `1 <= s.length <= 100`
- `s[i]`​​​​ is either `'0'` or `'1'`.
- `s[0]` is `'1'`.

### 📌 Examples

| Input | Output | Explanation |
| :--- | :--- | :--- |
| `s = "1001"` | `false` | The ones do not form a contiguous segment. There is a break of zeros between them. |
| `s = "110"` | `true` | The string contains only one contiguous segment of ones. |

## 💡 Explanation & Approach

### Approach 1: Counting the Segments (Brute-Force)
A simple and straightforward approach is to keep track of how many contiguous segments of ones we have seen so far.
1. We know the string will **always** start with `'1'` (given in constraints). Therefore, the first segment always starts at the `0-th` index.
2. We can initialize a `count` variable to keep track of these segments.
3. We iterate through the string. Whenever we encounter a `'1'`, we increment our segment `count`.
4. To handle the rest of the ones in that exact same segment, we use an inner `while` loop to skip all consecutive `'1'`s.
5. If we encounter a `'0'`, we simply ignore it and move to the next character.
6. At any point during the iteration, if our `count` strictly exceeds `1` (meaning we found a second segment), we immediately return `false`.

### Approach 2: Substring Search (Optimized One-Liner)
If we observe the problem carefully, an even simpler approach emerges. 
1. The string strictly begins with `'1'` and has no leading zeros.
2. The only way a **second** contiguous segment of ones can start is if there is a `'0'` that occurs *before* it. 
3. This implies that for a string to have more than one segment of ones, it **must contain the substring `"01"`** (a zero immediately followed by a one).
4. If the substring `"01"` is found anywhere in the string, it guarantees the existence of multiple segments.
5. We can easily implement this using the built-in string `find` function to check if `"01"` is present. If it is not present, we return `true`; otherwise, we return `false`.

### 1️⃣ Brute-force Solution (C++)

```cpp
// Intuition: The string always starts with '1'. We can iterate through the string and count the number of contiguous segments of '1's. If the count exceeds 1, we return false.
// Approach:
// 1. Initialize a `count` variable to keep track of the number of '1' segments.
// 2. Iterate through the string using an index `i`.
// 3. Whenever a '1' is found, increment the `count`.
// 4. Use an inner loop to skip all consecutive '1's belonging to the current segment.
// 5. If a '0' is encountered, simply move to the next character.
// 6. If `count` becomes greater than 1 at any point, return false.
// Time Complexity: O(N) where N is the length of the string, as each character is visited at most twice.
// Space Complexity: O(1) as we are using a constant amount of extra space.
class Solution {
public:
    bool checkOnesSegment(string s) {
        int n = s.length();

        int count = 0; // count of contiguous segment of '1'

        int i = 0;
        while(i < n) {
            // Found start of a segment
            if(s[i] == '1') {
                count++;
                // Skip all consecutive '1's in this segment
                while(i < n && s[i] == '1') {
                    i++;
                }
            } else {
                // If it's '0', just move forward
                i++;
            }

            // If more than 1 segment is found, return false
            if(count > 1) {
                return false;
            }
        }
        // If loop completes, there is at most 1 segment
        return true;
    }
};

/*
*
* Dry Run
* Input: s = "1001"
* i = 0, s[0] = '1'. count becomes 1. Inner loop runs, i becomes 1.
* count is 1. Not > 1.
* i = 1, s[1] = '0'. Else block runs, i becomes 2.
* i = 2, s[2] = '0'. Else block runs, i becomes 3.
* i = 3, s[3] = '1'. count becomes 2. Inner loop runs, i becomes 4.
* count is 2. Condition count > 1 is true. Returns false.
*
*/
```

### 2️⃣ Optimized Solution (C++)

```cpp
// Intuition: Since the string is guaranteed to start with '1' and has no leading zeros, a second segment of '1's can only start if there is a '0' followed by a '1' somewhere in the string.
// Approach:
// 1. Check if the substring "01" exists in the given string `s`.
// 2. We can use the built-in `find` method for strings.
// 3. If "01" is found, it implies there's a break of zeros followed by a new segment of ones, so we return false.
// 4. If `find` returns `string::npos` (meaning "01" is not found), we return true.
// Time Complexity: O(N) where N is the length of the string, as the `find` function scans the string.
// Space Complexity: O(1) as no extra space is required.
class Solution {
public:
    bool checkOnesSegment(string s) {
        // If "01" is not found, it returns true (at most 1 segment)
        return s.find("01") == string::npos;
    }
};

/*
*
* Dry Run
* Input: s = "1001"
* The find function searches for "01" in "1001".
* It finds "01" starting at index 1 ("1[01]1").
* Since it is found, s.find("01") does not equal string::npos.
* Returns false.
*
*/

```

## 🔍 Key Insights

* The constraint `s[0] == '1'` is the biggest hint in this problem. It structurally guarantees that if there are any zeros, they must appear *after* the initial segment of ones.
* Due to this guarantee, identifying the sequence `"01"` is functionally equivalent to detecting multiple isolated components of ones.
* String manipulation techniques and language-provided built-ins (like `.find()`) can significantly reduce the boilerplate code.
