# 1758. Minimum Changes To Make Alternating Binary String

![Difficulty: Easy](https://img.shields.io/badge/Difficulty-Easy-brightgreen?style=for-the-badge)
![Tags: String, String Matching](https://img.shields.io/badge/Tags-String_|_String_Matching-blue?style=for-the-badge)

## 📜 Problem Statement

You are given a string `s` consisting only of the characters `'0'` and `'1'`. In one operation, you can change any `'0'` to `'1'` or vice versa.

The string is called **alternating** if no two adjacent characters are equal. For example, the string `"010"` is alternating, while the string `"0100"` is not.

Return *the minimum number of operations needed to make `s` alternating*.

---

## 🎯 Constraints

- `1 <= s.length <= 10^4`
- `s[i]` is either `'0'` or `'1'`.

---

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `s = "0100"` | `1` | If you change the last character to '1', s will be "0101", which is alternating. |
| **2** | `s = "10"` | `0` | `s` is already alternating. |
| **3** | `s = "1111"` | `2` | You need two operations to reach "0101" or "1010". |

---

## 🧠 Key Insights & Explanation

A naive approach to solving this problem might be to greedily iterate through the string and change a character whenever it matches the previous one. However, this **greedy approach fails**. For example, in the string `10000000`, a greedy approach would result in 5 operations, but the optimal answer is 3 operations.

**The Observation:**
Since the string can only contain `'0'`s and `'1'`s, there are strictly **only two valid patterns** for an alternating binary string of length `n`:
1.  **Starts with '0':** `01010101...` (Even indices must have '0', odd indices must have '1')
2.  **Starts with '1':** `10101010...` (Even indices must have '1', odd indices must have '0')

**The Approach:**
We simply check our given string against these two possible valid patterns. We count how many characters mismatch the first pattern, and how many mismatch the second pattern. The minimum of these two mismatch counts will be our answer.

**Mathematical Optimization:**
Notice that the two patterns are exact opposites. If a character at an index matches pattern 1, it will definitely mismatch pattern 2, and vice versa. 
This means if it takes `X` operations to convert the string to pattern 1, it will always take exactly `n - X` operations to convert it to pattern 2 (where `n` is the length of the string). We only need to calculate the operations for one pattern, derive the other by subtracting from `n`, and return the minimum.

## 💻 Solution Code

### ❌ Wrong Brute-Force / Greedy Solution

```cpp
// Intuition: Greedily flip the current character if it is the same as the previous character.
// Approach: Iterate through the string starting from index 1. Check if the current character is equal to the previous one. If yes, logically flip it and increment the result counter.
// Time Complexity: O(n) because we iterate through the string once.
// Space Complexity: O(1) as we modify the string in-place without extra auxiliary space.
class Solution {
public:
    int minOperations(string s) {
        int n = s.length();
        int result = 0;
        // Traverse string from the second character
        for(int i = 1; i < n; i++) {
            // Check for adjacent duplicates
            if(s[i-1] == s[i]) {
                // Incorrectly overwrites with the same char, but counting continues greedily
                s[i] = (s[i-1] == '1') ? '1' : '0';
                result++;
            }
        }
        return result;
    }
};

/*

Dry Run

Input: "1000"
n = 4, result = 0
i = 1: s[0]('1') != s[1]('0'), no change, result = 0
i = 2: s[1]('0') == s[2]('0'), condition met, result = 1
i = 3: s[2]('0') == s[3]('0'), condition met, result = 2
Output: 2

Note: This greedy logic produces wrong outputs for global minimums (e.g., input "10000000").

*/

```

### ✅ Approach 1: Count Both Patterns

```cpp
// Intuition: An alternating binary string can only take two forms: starting with '0' or starting with '1'. We just need to find the differences against both forms.
// Approach: Traverse the string and maintain two separate counters for mismatches against the "0101..." pattern and the "1010..." pattern. Return the minimum.
// Time Complexity: O(n) since we iterate through the string exactly once.
// Space Complexity: O(1) because we only use two integer variables to keep track of the operations.
class Solution {
public:
    int minOperations(string s) {
        int n = s.length();
        int starts_with_0 = 0; // operations needed for 01010101...
        int starts_with_1 = 0; // operations needed for 10101010...

        for(int i = 0; i < n; i++) {
            if(i % 2 != 0) { // odd index
                // For 1010... pattern, odd index must be '0'
                if(s[i] == '0') {
                    starts_with_1++;
                } else {
                    // For 0101... pattern, odd index must be '1'
                    starts_with_0++;
                }
            } else { // even index
                // For 0101... pattern, even index must be '0'
                if(s[i] == '0') {
                    starts_with_0++;
                } else {
                    // For 1010... pattern, even index must be '1'
                    starts_with_1++;
                }
            }
        }

        // Return the best path
        return min(starts_with_0, starts_with_1);
    }
};

/*

Dry Run

Input: "0100"
n = 4, starts_with_0 = 0, starts_with_1 = 0
i = 0 (even), s[0] = '0': starts_with_0 = 1, starts_with_1 = 0
i = 1 (odd),  s[1] = '1': starts_with_0 = 1, starts_with_1 = 1
i = 2 (even), s[2] = '0': starts_with_0 = 2, starts_with_1 = 1
i = 3 (odd),  s[3] = '0': starts_with_0 = 2, starts_with_1 = 2
Return min(2, 2) = 2

*/

```

### 🚀 Approach 2: Optimized Pattern Counting (Math Trick)

```cpp
// Intuition: If it takes X operations to turn the string into one pattern, it strictly takes (n - X) operations to turn it into the completely opposite pattern.
// Approach: Only calculate the mismatches for one target pattern (e.g., "1010..."). The mismatches for the "0101..." pattern will trivially be (n - operations). Return the minimum.
// Time Complexity: O(n) as we do a single pass over the string length.
// Space Complexity: O(1) as we rely on a single counter variable.
class Solution {
public:
    int minOperations(string s) {
        int n = s.length();
        int starts_with_0 = 0; // tracks operations to form 10101010...

        for(int i = 0; i < n; i++) {
            if(i % 2 != 0) { // odd index
                // If it is '1', it mismatches the 1010... pattern
                if(s[i] == '1') {
                    starts_with_0++;
                }
            } else { // even index
                // If it is '0', it mismatches the 1010... pattern
                if(s[i] == '0'){
                    starts_with_0++;
                }
            }
        }

        // Mathematical deduction for the other pattern
        int starts_with_1 = n - starts_with_0;

        // Return the optimized minimum
        return min(starts_with_0, starts_with_1);
    }
};

/*

Dry Run

Input: "1111"
n = 4, starts_with_0 = 0
i = 0 (even), s[0] = '1': matches "1010...", starts_with_0 = 0
i = 1 (odd),  s[1] = '1': mismatches "1010...", starts_with_0 = 1
i = 2 (even), s[2] = '1': matches "1010...", starts_with_0 = 1
i = 3 (odd),  s[3] = '1': mismatches "1010...", starts_with_0 = 2
Calculate starts_with_1 = 4 - 2 = 2
Return min(2, 2) = 2

*/

```

## 📌 Notes & Further Exploration

* **The Pitfall of Greedy Algorithms**: This problem perfectly illustrates that making locally optimal decisions (fixing adjacencies one by one) does not necessarily lead to a globally optimal solution. Always verify if the problem has fixed global boundaries (like two static patterns here).
* **Power of Mathematical Observations**: The substitution `n - starts_with_0` simplifies code drastically, making it faster and less error-prone. In many bit manipulation and pattern questions, recognizing that exactly two complementary states exist can save you half the work.
* **Reference**: Keep an eye on similar string transformation questions where parity checks (odd/even indices rules) are required.
