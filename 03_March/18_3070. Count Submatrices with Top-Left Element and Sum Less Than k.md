# 3070. Count Submatrices with Top-Left Element and Sum Less Than k

**Difficulty:** Medium  
**Problem Link:** [LeetCode - 3070. Count Submatrices with Top-Left Element and Sum Less Than k](https://leetcode.com/problems/count-submatrices-with-top-left-element-and-sum-less-than-k/description/)

---

## 📝 Problem Statement

You are given a **0-indexed** integer matrix `grid` and an integer `k`.

Return the **number of submatrices** that contain the top-left element of the `grid`, and have a sum less than or equal to `k`.

---

## 💡 Examples

### Example 1:
| Input | Output | Explanation |
| :--- | :---: | :--- |
| `grid = [[7,6,3],[6,6,1]], k = 18` | `4` | There are exactly 4 submatrices that contain the top-left element of `grid` and have a sum less than or equal to `18`. |

### Example 2:
| Input | Output | Explanation |
| :--- | :---: | :--- |
| `grid = [[7,2,9],[1,5,0],[2,6,6]], k = 20` | `6` | There are exactly 6 submatrices that contain the top-left element of `grid` and have a sum less than or equal to `20`. |

---

## ⚙️ Constraints

- `m == grid.length`
- `n == grid[i].length`
- `1 <= n, m <= 1000`
- `0 <= grid[i][j] <= 1000`
- `1 <= k <= 10^9`

---

## 🧠 Approach & Explanation

### The Challenge
The core problem asks us to find how many submatrices (all starting from the `(0,0)` cell) sum up to a value `<= k`. 

### Brute Force Intuition
A naive approach would be to fix the bottom-right corner of the submatrix at every possible cell `(i, j)`. Then, we can use a nested loop to calculate the sum of all elements from `(0, 0)` to `(i, j)`. However, since $m, n \le 1000$, the maximum number of operations is approximately $10^{12}$, which will result in a **Time Limit Exceeded (TLE)** error.

### Optimized Approach (2D Prefix Sum)
To avoid recalculating the overlapping regions of the submatrices repeatedly, we can compute a **2D Prefix Sum**. 
Instead of looking at the entire grid every time, we can calculate the sum of the submatrix ending at `(i, j)` on the fly:
1. The sum at `(i, j)` is its own value `grid[i][j]`.
2. Add the prefix sum of the elements strictly above it: `grid[i-1][j]`.
3. Add the prefix sum of the elements strictly to its left: `grid[i][j-1]`.
4. By adding the top and left submatrices, the top-left diagonal section `grid[i-1][j-1]` gets added **twice**. To fix this repetition, we subtract it once.
5. If the total calculated sum is `<= k`, we increment our `count`.
6. **Early Exit Optimization**: Since all numbers in the grid are non-negative (`grid[i][j] >= 0`), if at any point the prefix sum exceeds `k`, moving further to the right in the same row will only increase the sum. Thus, we can safely `break` the inner loop early.

## 💻 Solution Code

### Brute-Force Solution

```cpp
// Intuition: The most basic way to solve this is to try all possible submatrices starting at the top-left (0,0). We fix the bottom-right corner at (i,j) and sum all elements inside.
// Approach: Use nested loops to fix the bottom-right cell (i, j). For each fixed cell, use another set of nested loops to iterate from (0,0) to (i,j) and calculate the sum. If the sum is <= k, increment the count.
// Time Complexity: O(m^2 * n^2), where m is rows and n is columns. Calculating sum for each cell takes O(m*n), and there are O(m*n) cells.
// Space Complexity: O(1), as we only use a few integer variables.
class Solution {
public:
    int countSubmatrices(vector<vector<int>>& grid, int k) {
        int m = grid.size();
        int n = grid[0].size();
        int count = 0;

        // Fix the bottom-right corner at (i, j)
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {

                // Sum from (0,0) to (i,j)
                int sum = 0;
                for (int m1 = 0; m1 <= i; m1++) {
                    for (int n1 = 0; n1 <= j; n1++) {
                        sum += grid[m1][n1];
                    }
                }

                // Increment if valid
                if (sum <= k) count++;
            }
        }

        return count;
    }
};

/*
*
* Dry Run
* grid = [[7,6,3],[6,6,1]], k = 18
* i=0, j=0: sum = grid[0][0] = 7. 7 <= 18, count=1
* i=0, j=1: sum = grid[0][0]+grid[0][1] = 13. 13 <= 18, count=2
* ... and so on. Re-computes everything repeatedly.
*
*/
```

### Optimized Solution (2D Prefix Sum)

```cpp
// Intuition: The brute force approach calculates overlapping submatrices repeatedly. We can use a 2D prefix sum to compute the sum of the submatrix ending at (i,j) in O(1) time using previously computed results.
// Approach: For each cell (i, j), update it to store the sum of the submatrix from (0,0) to (i,j). The relation is: sum(i,j) = current_val + sum(i-1, j) + sum(i, j-1) - sum(i-1, j-1). If the sum <= k, increment count; otherwise, break early as subsequent elements in the row will only increase the sum.
// Time Complexity: O(m * n), as we visit each cell exactly once.
// Space Complexity: O(1), since we modify the grid in-place to store prefix sums.
class Solution {
public:
    int countSubmatrices(vector<vector<int>>& grid, int k) {
        int m = grid.size();
        int n = grid[0].size();

        int count = 0;
        
        for(int i = 0; i < m; i++) {
            for(int j = 0; j < n; j++) {
                
                // Add sum from the top submatrix
                if (i > 0) 
                  grid[i][j] += grid[i - 1][j];
                
                // Add sum from the left submatrix
                if (j > 0) 
                  grid[i][j] += grid[i][j - 1];
                
                // Subtract the doubly added top-left diagonal submatrix
                if (i > 0 && j > 0) 
                  grid[i][j] -= grid[i - 1][j - 1];
                
                // Check if current submatrix sum is within limit
                if(grid[i][j] <= k)
                    count++;
                else 
                    break; // Subsequent elements in this row will exceed k
            }
        }

        return count;
    }
};

/*
*
* Dry Run
* grid = [[7,6,3],[6,6,1]], k = 18
* i=0, j=0: grid[0][0] = 7. 7<=18 -> count=1
* i=0, j=1: grid[0][1] = 6 + grid[0][0](7) = 13. 13<=18 -> count=2
* i=0, j=2: grid[0][2] = 3 + grid[0][1](13) = 16. 16<=18 -> count=3
* i=1, j=0: grid[1][0] = 6 + grid[0][0](7) = 13. 13<=18 -> count=4
* i=1, j=1: grid[1][1] = 6 + 13 (top) + 13 (left) - 7 (diag) = 25. 25 > 18 -> break
* Return count = 4
*
*/
```

## 📌 Notes & Key Insights
- **In-place Prefix Sum Arrays:** Computing prefix sums directly in the given array is a great way to maintain an `O(1)` space complexity.
- **Inclusion-Exclusion Principle:** 2D prefix sums naturally follow the Principle of Inclusion-Exclusion. We include `Top` and `Left`, but doing so includes the `Top-Left Diagonal` twice, which means we must exclude it once (`- grid[i-1][j-1]`).
- **Pruning / Early Exit:** The assumption that all input array elements are non-negative helps us optimize further. If traversing horizontally across rows causes the sum to breach the threshold `k`, any further movement rightward will strictly yield values `> k`. Using a simple `break` saves processing time.
