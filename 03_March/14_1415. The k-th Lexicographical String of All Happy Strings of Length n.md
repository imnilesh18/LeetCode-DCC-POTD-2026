## 📝 Problem Statement

A **happy string** is a string that:
* Consists only of letters of the set `['a', 'b', 'c']`.
* `s[i] != s[i + 1]` for all values of `i` from `1` to `s.length - 1` (string is 1-indexed).

For example, strings `"abc"`, `"ac"`, `"b"`, and `"abcbabcbcb"` are all happy strings, while strings `"aa"`, `"baa"`, and `"ababbc"` are not happy strings.

Given two integers `n` and `k`, consider a list of all happy strings of length `n` sorted in lexicographical order.

Return *the k-th string* of this list or return an **empty string** if there are less than `k` happy strings of length `n`.

### The Core Idea: Backtracking
The problem asks us to generate permutations of characters (`'a'`, `'b'`, `'c'`) up to length `n` while adhering to the "no adjacent duplicates" rule. Because we need to find the $k^{th}$ valid string in **lexicographical order**, we must be thoughtful about how we generate them. 

Since 'a' < 'b' < 'c', we can guarantee lexicographical sorting directly during generation by universally trying to append `'a'` first, then `'b'`, then `'c'`.

### Approach 1: Generating All Strings (Brute Force)
1. Use an empty `curr` string and a backtracking function.
2. For each step, iterate through our choices (`'a'`, `'b'`, `'c'`).
3. If the chosen character matches the very last character of the `curr` string, skip it to maintain "happiness".
4. Append the character (Do), recurse (Explore), and then pop it back (Undo).
5. When `curr.length() == n`, add it to an array/list.
6. Once complete, return the $(k-1)^{th}$ element (handling boundaries).

### Approach 2: Space Optimized with a Counter
Generating *all* strings strictly to find the $k^{th}$ string wastes memory ($O(n \cdot 2^n)$ space). Since we are already traversing in lexicographical order naturally, we can:
1. Maintain an integer `count` initially at `0`.
2. When a valid length $n$ string is fully formed, increment `count`.
3. If `count == k`, store this exact string as the result and exit early!
4. This drops the Space Complexity dramatically from $O(n \cdot 2^n)$ down to just $O(n)$ for the recursion stack.

## 💻 Solution Code

### 1. Brute Force (Storing All Strings)

```cpp
// Intuition: The problem requires generating the k-th lexicographically smallest "happy string" of length n. By always trying 'a', then 'b', then 'c', we naturally generate them in lexicographical order. We can store them all and pick the k-th element.
// Approach: 
// 1. Use a recursive backtracking function that builds a string character by character.
// 2. Try adding 'a', 'b', or 'c' to the current string, ensuring it is not the same as the last character.
// 3. When the string reaches length n, add it to a results vector.
// 4. Finally, check if we have at least k strings. If yes, return the (k-1)-th string; otherwise, return an empty string.
// Time Complexity: O(n * 2^n) - There are 3 choices for the first char, and 2 choices for the rest. Building/storing strings takes O(n).
// Space Complexity: O(n * 2^n) - We are storing all possible valid 2^n strings of length n.

class Solution {
public:
    void solve(int n, string &curr, vector<string> &result) {
        // Base case: string reached the target length
        if (curr.length() == n) {
            result.push_back(curr);
            return;
        }
        
        // Loop through allowed characters in lexicographical order
        for (char ch = 'a'; ch <= 'c'; ch++) {
            // Prevent adjacent repeating characters
            if (!curr.empty() && curr.back() == ch)
                continue;
            
            // Do
            curr.push_back(ch);

            // Explore
            solve(n, curr, result);

            // Undo
            curr.pop_back();
        }
    }

    string getHappyString(int n, int k) {
        string curr = "";
        vector<string> result;
        
        // Populate the result array with all happy strings
        solve(n, curr, result);

        // If fewer than k valid strings exist
        if (result.size() < k) 
            return "";
        
        // Return the k-th sequence (0-indexed mapping)
        return result[k - 1];
    }
};

/*
*
* Dry Run
* Input: n = 1, k = 3
* Call solve(1, "", result).
* -> Loop 'a': curr="a". solve(). Length == 1, result=["a"]. Undo.
* -> Loop 'b': curr="b". solve(). Length == 1, result=["a", "b"]. Undo.
* -> Loop 'c': curr="c". solve(). Length == 1, result=["a", "b", "c"]. Undo.
* result.size() is 3. Since k=3, return result[3-1] => "c".
*
*/
```

### 2. Optimized Space (Counting On-The-Fly)

```cpp
// Intuition: Storing all generated strings consumes unnecessary space. Since we traverse the recursion tree in lexicographical order, we can simply keep a running counter and halt the search the moment our counter hits k.
// Approach: 
// 1. Maintain a counter to track the number of valid length-n happy strings formed.
// 2. Build the string character by character (trying 'a', then 'b', then 'c').
// 3. Increment the counter when a string hits length n. If counter == k, copy it to result.
// 4. Implement an early exit condition to prune the remainder of the search tree once the result is found.
// Time Complexity: O(n * 2^n) - Worst-case time to traverse branches, though it breaks early upon hitting k.
// Space Complexity: O(n) - Only the recursion stack overhead and the space for the tracking variables.

class Solution {
public:
    void solve(int n, string &curr, int &count, int k, string &result) {
        // Base case: target string length reached
        if (curr.length() == n) {
            count++;
            if (count == k) {
                result = curr;  // Store only the k-th string
            }
            return;
        }
        
        // Explore lexographically ('a' -> 'b' -> 'c')
        for (char ch = 'a'; ch <= 'c'; ch++) {
            // Disallow adjacent duplicate characters
            if (!curr.empty() && curr.back() == ch)
                continue;
            
            // Do
            curr.push_back(ch);

            // Explore
            solve(n, curr, count, k, result);

            // If result is found, exit early (Pruning)
            if (!result.empty()) return;

            // Undo
            curr.pop_back();
        }
    }

    string getHappyString(int n, int k) {
        string curr = "";
        string result = "";
        int count = 0;
        
        // Start backtracking
        solve(n, curr, count, k, result);

        return result;
    }
};

/*
*
* Dry Run
* Input: n = 3, k = 9
* Init curr="", count=0, result=""
* Try 'a' first -> "aba", "abc", "aca", "acb" (count becomes 4)
* Try 'b' next -> "bab", "bac", "bca", "bcb" (count becomes 8)
* Try 'c' next -> curr="c" -> "ca" -> "cab" (Length reaches 3, count becomes 9)
* count == k (9 == 9), result="cab". 
* The early return triggers, bubbling up and terminating the entire recursion tree.
* Returns "cab".
*
*/
```

## 🧠 Key Insights

- **Family Template of Backtracking:** Both approaches utilize the classic `<Do> -> <Explore> -> <Undo>` state-space traversal technique.
- **Inbuilt Sorting:** Processing the loop sequentially from `'a'` to `'c'` perfectly simulates a lexicographically ordered string generation, eliminating the need for a separate `.sort()` operation.
- **Constant Width Decision Tree:** At the root, we have $3$ branches (`a`, `b`, `c`). At every subsequent node, we strictly have $2$ branches (skipping the immediate parent's character). This limits the tree width explicitly.
