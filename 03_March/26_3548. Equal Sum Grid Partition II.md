## 📝 Problem Statement

**Problem Link:** [Equal Sum Grid Partition II](https://leetcode.com/problems/equal-sum-grid-partition-ii/description/)

You are given an `m x n` matrix `grid` of positive integers. Your task is to determine if it is possible to make **either one horizontal or one vertical cut** on the grid such that:

1. Each of the two resulting sections formed by the cut is **non-empty**.
2. The sum of elements in both sections is **equal, or can be made equal by discounting at most one single cell in total** (from either section).
3. If a cell is discounted, the rest of the section must **remain connected**.

Return `true` if such a partition exists; otherwise, return `false`.

> **Note:** A section is **connected** if every cell in it can be reached from any other cell by moving up, down, left, or right through other cells in the section.

---

## 🛑 Constraints

- `1 <= m == grid.length <= 10^5`
- `1 <= n == grid[i].length <= 10^5`
- `2 <= m * n <= 10^5`
- `1 <= grid[i][j] <= 10^5`

## 💡 Approach & Intuition

The core challenge of this problem is efficiently checking if discounting a cell leaves the remaining section **connected** without running an expensive Breadth-First Search (BFS) or Depth-First Search (DFS) for every possible removal, which would cause a Time Limit Exceeded (TLE) error.

Instead of running DFS, we can rely on mathematical deduction and corner-case analysis:
1. **Horizontal Cuts Logic:**
   - As we traverse the grid row by row (from `i = 0` to `m-2`), we maintain a running sum of the `top` section and store its elements in a hash set. 
   - The `bottom` sum is simply `total_sum - top_sum`. 
   - The difference we need to discount is `diff = top - bottom`. 
   - If `diff == 0`, it's a perfect split. 
   - If `diff > 0`, we need to find if `diff` exists in the `top` section and if it's "safe" to remove. 
2. **Safe Removals (Avoiding Disconnections):**
   - Certain boundary cells are *always* safe to remove: `grid[0][0]`, `grid[0][n-1]`, and `grid[i][0]`.
   - If the top section has more than one row (`i > 0`) and more than one column (`n > 1`), it behaves like a 2D blob. In this specific scenario, any cell we've seen so far (`st.count(diff)`) is generally safe to remove without splitting the blob into completely unreachable components.
3. **Handling Negative Differences (Bottom Section Removals):**
   - What if `diff` is negative, meaning the bottom section is larger? Instead of writing complex logic to search the bottom section, we can just **reverse the grid upside down** and run the exact same top-down horizontal check again!
4. **Handling Vertical Cuts:**
   - Rather than writing a separate complex loop for vertical cuts, we can simply **transpose the grid** (turning rows into columns and vice-versa) and reuse our highly efficient `checkHorCuts` function (both normal and reversed).

---

## 🔑 Key Insights

- **Code Reusability:** By leveraging grid reversal and transposition, we reduce the problem to writing just *one* robust logic function (`checkHorCuts`). 
- **O(1) Connectivity Check:** Replacing DFS with geometric deductions (e.g., checking if `i > 0 && n > 1`) drastically reduces the time complexity from $O((M \times N)^2)$ to $O(M \times N)$.
- **Set Tracking:** By keeping track of elements seen on the fly using a Hash Set, we can check for the presence of the required `diff` element in $O(1)$ time.

## 💻 Solution Code

### Optimized Solution (C++)

```cpp
// Intuition: The problem requires partitioning a grid with at most one safe cell removal. Instead of using expensive DFS for connectivity checks, we can mathematically deduce safe removal spots (corners/edges) and reuse horizontal cut logic for vertical cuts via grid transposition and reversal.
// Approach: 
// 1. Compute total sum. 
// 2. Define `checkHorCuts` to evaluate horizontal splits row-by-row, tracking `top` sum and seen elements.
// 3. For a cut, if `diff = top - bottom` > 0, check if `diff` is a safe corner or if the section is large enough (`i > 0 && n > 1`) to safely remove any present `diff`.
// 4. Run `checkHorCuts` on the original grid.
// 5. Reverse the grid to handle cases where the bottom section needs a removal, and run `checkHorCuts` again.
// 6. Transpose the grid to convert vertical cuts into horizontal cuts, then repeat normal and reversed checks.
// Time Complexity: O(m * n) where m is the number of rows and n is columns. The grid is traversed a constant number of times (normal, reversed, transposed, transposed-reversed).
// Space Complexity: O(m * n) to store elements in the hash set during traversal, and for the transposed grid creation.

class Solution {
public:
    typedef long long ll;
    ll total = 0;

    bool checkHorCuts(vector<vector<int>>& grid) {
        int m = grid.size();
        int n = grid[0].size();

        unordered_set<ll> st;
        ll top = 0;

        // Iterate through valid horizontal cut positions (excluding the very bottom)
        for(int i = 0; i <= m-2; i++) {

            // Process current row, adding to top sum and set
            for(int j = 0; j < n; j++) {
                st.insert(grid[i][j]);
                top += grid[i][j];
            }

            ll bottom = (total - top);
            ll diff = top - bottom; // The exact value we need to discard from the top

            // Perfect split without removals
            if (diff == 0) return true;

            // Safe corner/boundary removals that won't break connectivity
            if (diff == (ll)grid[0][0])   return true;
            if (diff == (ll)grid[0][n-1]) return true;
            if (diff == (ll)grid[i][0]) return true;

            // If the section is a 2D block (not a single row/col), any present element is safe
            if(i > 0 && n > 1 && st.count(diff)) {
                return true;
            }
        }

        return false;
    }

    bool canPartitionGrid(vector<vector<int>>& grid) {
        int m = grid.size();
        int n = grid[0].size();

        // Step 1: Calculate total sum of the grid
        for(int i = 0; i < m; i++) {
            for(int j = 0; j < n; j++) {
                total += grid[i][j];
            }
        }

        // Step 2: Check horizontal cuts
        if(checkHorCuts(grid)) {
            return true;
        }

        // Step 3: Reverse grid to check bottom-heavy horizontal cuts
        reverse(begin(grid), end(grid));
        if(checkHorCuts(grid)) {
            return true;
        }

        // Restore original grid orientation
        reverse(begin(grid), end(grid)); 

        // Step 4: Vertical cuts checking by using checkHorCuts on Transposed Grid
        vector<vector<int>> transposeGrid(n, vector<int>(m)); 

        for(int i = 0; i < m; i++) {
            for(int j = 0; j < n; j++) {
                transposeGrid[j][i] = grid[i][j];
            }
        }

        // Step 5: Check vertical cuts (now horizontal on transposed grid)
        if(checkHorCuts(transposeGrid)) {
            return true;
        }

        // Step 6: Reverse transposed grid to check right-heavy vertical cuts
        reverse(begin(transposeGrid), end(transposeGrid));
        if(checkHorCuts(transposeGrid)) {
            return true;
        }

        // No valid partition found
        return false;
    }
};

/*
*
* Dry Run
* Input: grid = [[1,4],[2,3]]
* Total Sum = 1 + 4 + 2 + 3 = 10
* * --- checkHorCuts(grid) ---
* i = 0: (Cut after first row)
* Elements added to st: {1, 4}
* top sum = 5
* bottom sum = total - top = 10 - 5 = 5
* diff = top - bottom = 5 - 5 = 0
* diff == 0 -> Returns true!
* * Result: true
*
*/
```
