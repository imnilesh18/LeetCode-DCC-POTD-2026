# 3212. Count Submatrices With Equal Frequency of X and Y

**Difficulty:** Medium

## 📝 Problem Statement

Given a 2D character matrix `grid`, where `grid[i][j]` is either `'X'`, `'Y'`, or `'.'`, return the number of submatrices that contain:
1. `grid[0][0]` (Every submatrix must start from the top-left corner).
2. An **equal frequency** of `'X'` and `'Y'`.
3. At least **one** `'X'`.

## 🚀 Constraints

- `1 <= grid.length, grid[i].length <= 1000`
- `grid[i][j]` is either `'X'`, `'Y'`, or `'.'`.

---

## 💡 Approach

A brute-force approach to count all valid submatrices starting from `(0,0)` to any `(i,j)` would require repeatedly iterating over grid elements, leading to excessive Time Complexity (like $O(M^2 \times N^2)$). 

**Optimization using 2D Prefix Sum (Cumulative Sum):**
To compute the frequencies efficiently in $O(1)$ time for any submatrix ending at `(i,j)`, we can precompute the counts using a 2D prefix sum technique.

1. Create two 2D arrays, `cumSumX` and `cumSumY`, which will store the cumulative count of `'X'` and `'Y'` up to the cell `(i,j)`.
2. For any cell `(i,j)`, the cumulative sum incorporates:
   - The value at the current cell.
   - The cumulative sum from the cell directly above `(i-1, j)`.
   - The cumulative sum from the cell directly to the left `(i, j-1)`.
   - Since adding the top and left cell values counts the top-left diagonal submatrix `(i-1, j-1)` **twice**, we must subtract it to avoid duplicate counting.
   
   **Formula:**
   `cumSum[i][j] = current_cell_val + cumSum[i-1][j] + cumSum[i][j-1] - cumSum[i-1][j-1]`
   
3. After computing the count for the submatrix bounded by `(0,0)` to `(i,j)`, we check if the count of `'X'` equals the count of `'Y'` AND if the count of `'X'` is strictly greater than 0.
4. If the conditions are met, we increment our valid submatrix counter.

---

## 💻 Solution Code

```cpp
// Intuition: To efficiently find the count of 'X's and 'Y's in any submatrix starting from (0,0) to (i,j), we can use a 2D prefix sum (cumulative sum) approach. This avoids recalculating the counts for overlapping regions and brings the time complexity down from cubic to O(m*n).
// Approach: 
// 1. Create two 2D arrays, cumSumX and cumSumY, to store the prefix sums of 'X' and 'Y' respectively.
// 2. Iterate through each cell (i, j) in the grid.
// 3. For each cell, the number of 'X's in the submatrix from (0,0) to (i,j) is the current cell's 'X' count plus the sum from the submatrix above it, plus the sum from the submatrix to its left, minus the overlapping top-left diagonal submatrix.
// 4. Repeat the same calculation for 'Y'.
// 5. If the total count of 'X' is equal to the total count of 'Y' and there is at least one 'X', increment our valid submatrix count.
// 6. Return the final count.
// Time Complexity: O(m * n), where m is the number of rows and n is the number of columns, because we visit each cell exactly once.
// Space Complexity: O(m * n) to store the cumulative sums for 'X' and 'Y' in two separate 2D arrays.

class Solution {
public:
    int numberOfSubmatrices(vector<vector<char>>& grid) {
        int m = grid.size();
        int n = grid[0].size();

        // 2D arrays to store prefix sums of 'X' and 'Y'
        vector<vector<int>> cumSumX(m, vector<int>(n, 0));
        vector<vector<int>> cumSumY(m, vector<int>(n, 0));

        int count = 0; // To store valid submatrices count

        for(int i = 0; i < m; i++) {
            for(int j = 0; j < n; j++) {
                
                // Initialize current cell count
                cumSumX[i][j] = (grid[i][j] == 'X');
                cumSumY[i][j] = (grid[i][j] == 'Y');

                // Add top submatrix sum
                if(i - 1 >= 0) {
                    cumSumX[i][j] += cumSumX[i-1][j];
                    cumSumY[i][j] += cumSumY[i-1][j];
                }

                // Add left submatrix sum
                if(j - 1 >= 0) {
                    cumSumX[i][j] += cumSumX[i][j-1];
                    cumSumY[i][j] += cumSumY[i][j-1];
                }

                // Subtract the overlapping top-left diagonal sum
                if(i - 1 >= 0 && j - 1 >= 0) {
                    cumSumX[i][j] -= cumSumX[i-1][j-1];
                    cumSumY[i][j] -= cumSumY[i-1][j-1];
                }
                
                // Check if conditions are met: equal frequencies and at least one 'X'
                if(cumSumX[i][j] == cumSumY[i][j] && cumSumX[i][j] > 0) {
                    count++;
                }
            }
        }

        return count;
    }
};

/*
 *
 * Dry Run
 *
 * Input: grid = [["X","Y","."],["Y",".","."]]
 * m = 2, n = 3
 * * i = 0, j = 0 (grid = 'X')
 * cumSumX[0][0] = 1, cumSumY[0][0] = 0
 * count = 0
 * * i = 0, j = 1 (grid = 'Y')
 * cumSumX[0][1] = 0 + cumSumX[0][0] = 1
 * cumSumY[0][1] = 1 + cumSumY[0][0] = 1
 * cumSumX == cumSumY and cumSumX > 0 -> count = 1
 * * i = 0, j = 2 (grid = '.')
 * cumSumX[0][2] = 0 + 1 = 1
 * cumSumY[0][2] = 0 + 1 = 1
 * cumSumX == cumSumY and cumSumX > 0 -> count = 2
 * * i = 1, j = 0 (grid = 'Y')
 * cumSumX[1][0] = 0 + cumSumX[0][0] = 1
 * cumSumY[1][0] = 1 + cumSumY[0][0] = 1
 * cumSumX == cumSumY and cumSumX > 0 -> count = 3
 * * i = 1, j = 1 (grid = '.')
 * cumSumX[1][1] = 0 + cumSumX[0][1] + cumSumX[1][0] - cumSumX[0][0] = 0 + 1 + 1 - 1 = 1
 * cumSumY[1][1] = 0 + cumSumY[0][1] + cumSumY[1][0] - cumSumY[0][0] = 0 + 1 + 1 - 0 = 2
 * Condition not met. count = 3
 * * i = 1, j = 2 (grid = '.')
 * cumSumX[1][2] = 0 + cumSumX[0][2] + cumSumX[1][1] - cumSumX[0][1] = 0 + 1 + 1 - 1 = 1
 * cumSumY[1][2] = 0 + cumSumY[0][2] + cumSumY[1][1] - cumSumY[0][1] = 0 + 1 + 2 - 1 = 2
 * Condition not met. count = 3
 * * Final Output: 3
 *
 */
````

## 📊 Key Insights

  - **Inclusion-Exclusion Principle:** Expanding a 2D cumulative matrix follows the basic algebraic inclusion-exclusion principle, which states that to find the union of sizes of overlapping areas, you add the sub-areas and subtract their overlapping intersection.
  - **Space Optimization:** You can reduce the Space Complexity from $O(M \times N)$ to $O(N)$ by keeping track of only the current row and the previous row since computing the prefix sums only requires elements adjacent to the current working row.
