## 📝 Problem Statement

Given an `m x n` binary matrix `mat`, return the number of **special positions** in `mat`.

A position `(i, j)` is called **special** if `mat[i][j] == 1` and all other elements in row `i` and column `j` are `0` (rows and columns are **0-indexed**).

---

## 🎯 Constraints

- `m == mat.length`
- `n == mat[i].length`
- `1 <= m, n <= 100`
- `mat[i][j]` is either `0` or `1`.

## 🧠 Approach

### 1. Brute Force Method
The simplest approach is to check every cell in the matrix. If a cell contains a `1`, we traverse its entire row and its entire column to ensure there are no other `1`s. If no other `1`s are found, we increment our count of special positions. While straightforward, this results in redundant checks because we scan rows and columns multiple times.

### 2. Optimized Method (Precomputation)
To optimize the solution, we can compute the frequencies beforehand. We create two arrays: `rowCount` and `colCount` to store the total number of `1`s in each row and column, respectively. 
By iterating through the matrix once, we populate these arrays. In a second iteration, whenever we find a `1` at `(i, j)`, we simply check if `rowCount[i] == 1` and `colCount[j] == 1`. If both are true, it means it's the unique `1` in that row and column, making it a special position. This significantly reduces our time complexity!

## 💻 Solution Code

### Approach 1: Brute Force

```cpp
// Intuition: For each 1 encountered in the matrix, we can directly verify if it is the only 1 in its respective row and column by iterating through them.
// Approach: Iterate through every cell in the m x n matrix. If a cell contains a 1, traverse its entire row and column. If no other 1s are found, it is a special position.
// Time Complexity: O((m * n) * (m + n)), because for each cell, we traverse its entire row (length n) and column (length m).
// Space Complexity: O(1), as no extra space is utilized apart from basic variables.
class Solution {
public:
    int numSpecial(vector<vector<int>>& mat) {
        int result = 0; // Tracks the number of special positions
        int m = mat.size(); // Number of rows
        int n = mat[0].size(); // Number of columns
        
        for (int row = 0; row < m; row++) {
            for (int col = 0; col < n; col++) {
                // Skip if the current cell is 0
                if (mat[row][col] == 0) {
                    continue;
                }
                
                bool good = true; // Flag to verify if the position is special
                
                // Check the entire column for other 1s
                for (int r = 0; r < m; r++) {
                    if (r != row && mat[r][col] == 1) {
                        good = false;
                        break; // Not special, exit loop
                    }
                }
                
                // Check the entire row for other 1s
                for (int c = 0; c < n; c++) {
                    if (c != col && mat[row][c] == 1) {
                        good = false;
                        break; // Not special, exit loop
                    }
                }
                
                // If no other 1s were found, increment the result
                if (good) {
                    result++;
                }
            }
        }
        
        return result;
    }
};

/*
*
* Dry Run
*
* Input: mat = [[1,0,0], [0,0,1], [1,0,0]]
* * row = 0, col = 0: mat[0][0] = 1.
* Check col 0: mat[1][0]=0, mat[2][0]=1 -> good = false.
* row = 1, col = 2: mat[1][2] = 1.
* Check col 2: mat[0][2]=0, mat[2][2]=0.
* Check row 1: mat[1][0]=0, mat[1][1]=0.
* good = true -> result = 1.
* row = 2, col = 0: mat[2][0] = 1.
* Check col 0: mat[0][0]=1 -> good = false.
* * Final Result: 1
*
*/
```

### Approach 2: Optimized (Storing count of 1s in rows and cols)

```cpp
// Intuition: We can optimize the redundant row and column checks by precomputing the total number of 1s in each row and column.
// Approach: Create two frequency arrays, rowCount and colCount. Traverse the matrix once to populate these frequencies. Traverse the matrix again, and if mat[i][j] == 1, simply check if rowCount[i] == 1 and colCount[j] == 1.
// Time Complexity: O(m * n), as we traverse the matrix a constant number of times (twice).
// Space Complexity: O(m + n), extra space used by the rowCount and colCount arrays.
class Solution {
public:
    int numSpecial(vector<vector<int>>& mat) {
        int m = mat.size(); // Number of rows
        int n = mat[0].size(); // Number of columns
        
        vector<int> rowCount(m, 0); // Stores count of 1s in each row
        vector<int> colCount(n, 0); // Stores count of 1s in each column
        
        // First pass: Precompute the counts of 1s
        for (int row = 0; row < m; row++) {
            for (int col = 0; col < n; col++) {
                if (mat[row][col] == 1) {
                    rowCount[row]++;
                    colCount[col]++;
                }
            }
        }
        
        int result = 0; // Tracks the number of special positions
        
        // Second pass: Check for special positions using precomputed counts
        for (int row = 0; row < m; row++) {
            for (int col = 0; col < n; col++) {
                // Check if current cell is 1 and it is the only 1 in its row and column
                if (mat[row][col] == 1) {
                    if (rowCount[row] == 1 && colCount[col] == 1) {
                        result++;
                    }
                }
            }
        }
        
        return result;
    }
};

/*
*
* Dry Run
*
* Input: mat = [[1,0,0], [0,1,0], [0,0,1]]
* * First Pass (Counting):
* row = 0: mat[0][0] = 1 -> rowCount[0]=1, colCount[0]=1
* row = 1: mat[1][1] = 1 -> rowCount[1]=1, colCount[1]=1
* row = 2: mat[2][2] = 1 -> rowCount[2]=1, colCount[2]=1
* * rowCount = [1, 1, 1], colCount = [1, 1, 1]
* * Second Pass (Evaluating):
* row = 0, col = 0: mat[0][0]=1. rowCount[0]=1 & colCount[0]=1 -> result = 1.
* row = 1, col = 1: mat[1][1]=1. rowCount[1]=1 & colCount[1]=1 -> result = 2.
* row = 2, col = 2: mat[2][2]=1. rowCount[2]=1 & colCount[2]=1 -> result = 3.
* * Final Result: 3
*
*/
```

## 🔑 Key Insights

- **Trade-offs:** By expending a small amount of extra space $O(m+n)$ to store row and column statistics, we drastically reduce our time complexity from cubic bounds down to a linear pass $O(m \times n)$.
- **Preprocessing Paradigm:** This is a classic pattern in grid/matrix problems. Precalculating row/col properties stops inner loops from repeatedly reading the same data.
