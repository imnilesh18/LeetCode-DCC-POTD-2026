## 📖 Problem Statement

You are given 3 positive integers `zero`, `one`, and `limit`.

A **binary array** `arr` is called **stable** if:
- The number of occurrences of `0` in `arr` is **exactly** `zero`.
- The number of occurrences of `1` in `arr` is **exactly** `one`.
- Each **subarray** of `arr` with a size greater than `limit` must contain **both** `0` and `1`.

Return the **total** number of **stable** binary arrays.  
Since the answer may be very large, return it **modulo** $10^9 + 7$.

## 🧠 Key Insights & Approach

This problem is an extension of Part I, where the limits were originally up to $200$. In Part I, an $O(\text{zero} \times \text{one} \times \text{limit})$ approach was acceptable. However, with the constraints increased to $1000$, $1000^3 = 10^9$ operations will result in a **Time Limit Exceeded (TLE)** error. 

We need to optimize our $O(N^3)$ Dynamic Programming approach to $O(N^2)$ by removing the innermost limit iteration loop. We can do this mathematically by analyzing the repetitive work done across states.

### Mathematical Derivation for $O(1)$ Transitions
If we look closely at how we calculate the number of combinations ending in `1` for state $(i, j)$:
$$dp[i][j][1] = dp[i][j-1][0] + dp[i][j-2][0] + \dots + dp[i][j-limit][0]$$ *(Equation 1)*

Now, let's write the same equation for the next state, $(i, j+1)$:
$$dp[i][j+1][1] = dp[i][j][0] + dp[i][j-1][0] + \dots + dp[i][j+1-limit][0]$$ *(Equation 2)*

By subtracting **Equation 1** from **Equation 2**, almost all terms cancel out, leaving us with:
$$dp[i][j+1][1] - dp[i][j][1] = dp[i][j][0] - dp[i][j-limit][0]$$

To standardize the formula for state $(i, j)$, substitute $j+1$ with $j$:
$$dp[i][j][1] = dp[i][j-1][1] + dp[i][j-1][0] - dp[i][j-1-limit][0]$$

By applying the exact same substitution logic for the combinations ending in `0`, we derive:
$$dp[i][j][0] = dp[i-1][j][0] + dp[i-1][j][1] - dp[i-1-limit][j][1]$$

This elegant mathematical reduction allows us to transition states in $O(1)$ time instead of $O(limit)$, turning an $O(N^3)$ problem into a highly optimized $O(N^2)$ problem!

## 💻 Code Implementations

### 1️⃣ Approach 1: Recursion + Memoization (Optimized)

```cpp
// Intuition: We want to find the number of valid binary arrays. We build the array element by element. To avoid iterating over the 'limit' continuously, we mathematically deduce a constant-time transition by observing the overlapping subproblems from Part I.
// Approach: Use 3D Memoization `t[zero][one][last_placed]`. Instead of looping `1` to `limit`, we use the deduced mathematical transition: `t[i][j][1] = t[i][j-1][0] + t[i][j-1][1] - t[i][j-1-limit][0]`. We compute recursively and memoize overlapping subproblems.
// Time Complexity: O(zero * one) - Each state (i, j, last) is computed in O(1) time. There are roughly 2 * zero * one distinct states.
// Space Complexity: O(zero * one) - For the 3D memoization table `t` and the recursion call stack depth.

class Solution {
public:
    int M = 1e9 + 7;
    vector<vector<vector<int>>> t;

    int solve(int i, int j, int last, int limit) {
        // Base case: no elements left to place
        if (i == 0 && j == 0) 
            return 0;

        // Base case: only zeros are left
        if (j == 0) {
            if (last == 1) return 0; 
            return (i <= limit) ? 1 : 0;
        }

        // Base case: only ones are left
        if (i == 0) {
            if (last == 0) return 0; 
            return (j <= limit) ? 1 : 0;
        }

        // Return already computed valid subproblem
        if (t[i][j][last] != -1)
            return t[i][j][last];

        int result = 0;

        // Transition for ending with 0
        if (last == 0) {
            result = (solve(i-1, j, 0, limit) + solve(i-1, j, 1, limit)) % M;

            // Subtract out-of-bounds invalid combinations
            if (i-1 >= limit)
                result = (result - solve(i-1-limit, j, 1, limit) + M) % M;

        // Transition for ending with 1
        } else {
            result = (solve(i, j-1, 0, limit) + solve(i, j-1, 1, limit)) % M;

            // Subtract out-of-bounds invalid combinations
            if (j-1 >= limit)
                result = (result - solve(i, j-1-limit, 0, limit) + M) % M;
        }

        // Store and return the result
        return t[i][j][last] = result;
    }

    int numberOfStableArrays(int zero, int one, int limit) {
        // Initialize memoization 3D array with -1
        t.assign(zero+1, vector<vector<int>>(one+1, vector<int>(2, -1)));

        // Answer is sum of valid combinations ending in 0 and ending in 1
        return (solve(zero, one, 0, limit) + solve(zero, one, 1, limit)) % M;
    }
};

/*
* Dry Run
* Input: zero = 1, one = 1, limit = 2
* * solve(1, 1, 0, 2):
* - i=1, j=1, last=0
* - Computes: solve(0, 1, 0, 2) + solve(0, 1, 1, 2)
* - solve(0, 1, 0, 2) -> returns 0 (Base case: only 1s left, but last is 0)
* - solve(0, 1, 1, 2) -> returns 1 (Base case: j <= limit, returns 1)
* - result = (0 + 1) % M = 1
*
* solve(1, 1, 1, 2):
* - i=1, j=1, last=1
* - Computes: solve(1, 0, 0, 2) + solve(1, 0, 1, 2)
* - solve(1, 0, 0, 2) -> returns 1 (Base case: i <= limit, returns 1)
* - solve(1, 0, 1, 2) -> returns 0 (Base case: only 0s left, but last is 1)
* - result = (1 + 0) % M = 1
*
* Final Answer: (1 + 1) % M = 2.
*/
```

### 2️⃣ Approach 2: Bottom-Up Dynamic Programming (Optimal)

```cpp
// Intuition: The recursive formulation can run into stack depth overhead given bounds up to 1000. Converting the mathematically deduced constant-time transitions into a bottom-up DP table beautifully mitigates recursion limits and cleanly calculates the total counts.
// Approach: Initialize a 3D DP array `t[zero+1][one+1][2]`. Base cases are pre-filled for strictly sequences of 0s or strictly 1s bounded by `limit`. Traverse over all `i` (zeros) and `j` (ones), applying the mathematically simplified $O(1)$ transitions to populate the DP table. Sum valid arrays ending in 0 and 1 for the final answer.
// Time Complexity: O(zero * one) - Nested loops processing `zero` and `one` states in O(1) time each.
// Space Complexity: O(zero * one) - A 3D DP array `t` of size (zero+1) x (one+1) x 2 is required.

class Solution {
public:
    int M = 1e9+7;
    int numberOfStableArrays(int zero, int one, int limit) {
        // Initialize DP table t[zero+1][one+1][2] to 0
        vector<vector<vector<int>>> t(zero+1, vector<vector<int>>(one+1, vector<int>(2, 0)));

        // Base cases: Place purely 0s (up to limit)
        for(int i = 0; i <= min(zero, limit); i++)
            t[i][0][0] = 1;
        
        // Base cases: Place purely 1s (up to limit)
        for(int j = 0; j <= min(one, limit); j++) {
            t[0][j][1] = 1;
        }

        // Iterate over all possible counts of 0s and 1s
        for(int i = 0; i <= zero; i++) { // i = # 0s
            for(int j = 0; j <= one; j++) { // j = # 1s

                // Skip purely 0s or purely 1s (already handled in base cases)
                if(i == 0 || j == 0)
                    continue;

                // Calculate states ending in 1
                t[i][j][1] = (t[i][j-1][0] + t[i][j-1][1]) % M;
                
                if(j-1 >= limit) {
                    // Subtract invalid groupings that surpass limit
                    t[i][j][1] = (t[i][j][1] - t[i][j-1-limit][0] + M) % M;
                }

                // Calculate states ending in 0
                t[i][j][0] = (t[i-1][j][0] + t[i-1][j][1]) % M;
                
                if(i-1 >= limit) {
                    // Subtract invalid groupings that surpass limit
                    t[i][j][0] = (t[i][j][0] - t[i-1-limit][j][1] + M) % M;
                }
            }
        }

        // Combine the combinations ending in 0 and 1 using all available zeros and ones
        return (t[zero][one][0] + t[zero][one][1]) % M;
    }
};

/*
* Dry Run
* Input: zero = 1, one = 1, limit = 2
* * Initialization:
* t[0][0][0] = 1, t[1][0][0] = 1
* t[0][0][1] = 1, t[0][1][1] = 1
* * Loop Processing for i=1, j=1:
* - Calculate t[1][1][1]:
* t[1][1][1] = (t[1][0][0] + t[1][0][1]) % M = (1 + 0) = 1
* Condition (j-1 >= limit) -> (0 >= 2) False
* * - Calculate t[1][1][0]:
* t[1][1][0] = (t[0][1][0] + t[0][1][1]) % M = (0 + 1) = 1
* Condition (i-1 >= limit) -> (0 >= 2) False
* * Result Calculation:
* return (t[1][1][0] + t[1][1][1]) % M = (1 + 1) % M = 2
*/
```
