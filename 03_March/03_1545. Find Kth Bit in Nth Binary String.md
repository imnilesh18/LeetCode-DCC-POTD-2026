## 📝 Problem Statement

Given two positive integers `n` and `k`, the binary string `Sn` is formed as follows:

- `S1 = "0"`
- `Si = Si - 1 + "1" + reverse(invert(Si - 1))` for `i > 1`

Where `+` denotes the concatenation operation, `reverse(x)` returns the reversed string `x`, and `invert(x)` inverts all the bits in `x` (`0` changes to `1` and `1` changes to `0`).

For example, the first four strings in the above sequence are:
- `S1 = "0"`
- `S2 = "011"`
- `S3 = "0111001"`
- `S4 = "011100110110001"`

Return the `kth` bit in `Sn`. It is guaranteed that `k` is valid for the given `n`.

| Example | Input | Output | Explanation |

| **1** | `n = 3, k = 1` | `"0"` | S3 is "0111001". The 1st bit is "0". |
| **2** | `n = 4, k = 11` | `"1"` | S4 is "011100110110001". The 11th bit is "1". |

## 🛑 Constraints

- `1 <= n <= 20`
- `1 <= k <= 2^n - 1`

---

## 🧠 Approach & Explanation

The problem defines a recursive string generation. Let's analyze the string structure:
For any given $n$, the length of the string $S_n$ will always be $2^n - 1$.
The string is symmetrically constructed around a middle element:
$$S_n = S_{n-1} + \text{"1"} + \text{reverse}(\text{invert}(S_{n-1}))$$

Instead of fully constructing the string (which will take $O(2^n)$ memory and time), we can leverage recursion to find the answer efficiently by determining which half the $k$-th bit lies in:

1. **Calculate the Middle Index:** The total length of $S_n$ is `(1 << n) - 1`. The middle bit is located at `ceil(length / 2.0)`.

2. **Check the Location of $k$:**
   - **Left Half ($k < \text{mid}$):** The left half is identical to the string $S_{n-1}$. We can directly return the answer of the recursive call `findKthBit(n-1, k)`.
   - **Exactly at the Middle ($k == \text{mid}$):** The middle bit added to concatenate both halves is always `"1"`. We can immediately return `'1'`.
   - **Right Half ($k > \text{mid}$):** The right half is the reverse and inverted version of $S_{n-1}$. We need to find the mirrored position in $S_{n-1}$. The corresponding index in $S_{n-1}$ is `length - (k - 1)`. We recursively find the bit at this position in $S_{n-1}$, and then return the flipped value (if it's `'0'`, return `'1'`, and vice versa).

3. **Base Case:** When $n = 1$, the string is just `"0"`, so we return `'0'`.

## 💻 Solution Code

```cpp
// Intuition: The problem follows a clear recursive pattern where each string Sn is formed by taking Sn-1, adding a '1' in the middle, and then appending the inverted and reversed Sn-1. Thus, we can find the kth bit by recursively narrowing down the search space to the corresponding half or the middle bit.
// Approach:
// 1. Calculate the length of Sn, which is (2^n) - 1.
// 2. Find the middle position.
// 3. If k is in the first half, it matches the kth bit of Sn-1.
// 4. If k is exactly the middle element, it's always '1'.
// 5. If k is in the second half, it corresponds to the (length - k + 1)-th bit of Sn-1, but inverted. Recursively fetch this bit and flip it.
// Time Complexity: O(n) because we reduce n by 1 in each recursive step, leading to at most n calls.
// Space Complexity: O(n) due to the system call stack used for the recursion depth of n.
class Solution {
public:
    char findKthBit(int n, int k) {
        
        // Base case: for S1, the only character is '0'
        if(n == 1) {
            return '0';
        }

        // Calculate the total length of the current string Sn
        int length = (1 << n) - 1; // pow(2, n) - 1

        // If k is in the left half, it's identical to Sn-1
        if(k < ceil(length/2.0)) {
            return findKthBit(n-1, k);
        } 
        // If k is exactly at the middle, the appended bit is always '1'
        else if(k == ceil(length/2.0)) {
            return '1';
        } 
        // If k is in the right half, find its mirrored position in Sn-1
        else {
            char ch = findKthBit(n-1, length-(k-1)); // handled reversed
            return (ch == '0') ? '1' : '0'; // handled flipped bit
        }

    }
};

/*
*
* Dry Run
* n = 3, k = 1
* findKthBit(3, 1)
* length = (1 << 3) - 1 = 7, mid = ceil(7/2.0) = 4
* k (1) < mid (4), so call findKthBit(2, 1)
* * findKthBit(2, 1)
* length = (1 << 2) - 1 = 3, mid = ceil(3/2.0) = 2
* k (1) < mid (2), so call findKthBit(1, 1)
* * findKthBit(1, 1)
* n == 1, base case hit, returns '0'
* * findKthBit(2, 1) returns '0'
* findKthBit(3, 1) returns '0'
* Final Output: '0'
*
*/
```

## 🔍 Key Insights

- **Avoid String Construction:** Building the strings literally would result in Memory Limit Exceeded (MLE) and Time Limit Exceeded (TLE) because the maximum string size would be $2^{20} - 1$, which is about 1 million characters.
- **Recursive Leap of Faith:** Recognizing that parts of a complex recursive string strictly map back to previous, smaller string states allows us to shrink the problem space from $O(2^n)$ down to $O(n)$.
- **Mirror Indexing Formula:** The formula `length - (k - 1)` elegantly pinpoints the original position of a character before the string was reversed.
