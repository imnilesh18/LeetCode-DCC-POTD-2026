## 📝 Problem Statement

You are given an `m x n` integer matrix `mat` and an integer `k`. The matrix rows are `0`-indexed.

The following process happens `k` times:
- **Even-indexed rows** (`0, 2, 4, ...`) are cyclically shifted to the **left**.
<div align="center">
  <img src="https://assets.leetcode.com/uploads/2024/05/19/lshift.jpg" alt="Left Shift">
</div>

- **Odd-indexed rows** (`1, 3, 5, ...`) are cyclically shifted to the **right**.
<div align="center">
  <img src="https://assets.leetcode.com/uploads/2024/05/19/rshift-stlone.jpg" alt="Right Shift">
</div>

Return `true` if the final modified matrix after `k` steps is **identical** to the original matrix, and `false` otherwise.

## 🧠 Explanation & Approach

### Key Insight: Cyclic Shifts Repeat
A crucial mathematical property of shifting an array (or row) of length `n` is that shifting it `n` times brings it back to its exact original state. Therefore, shifting a row `k` times is mathematically equivalent to shifting it `k % n` times. This normalization (`k = k % n`) prevents redundant operations.

### Approach 1: Simulation (Brute Force)
1. Create a copy of the original matrix (`temp = mat`).
2. Simplify the number of shifts using modulo arithmetic: `k = k % n`. If `k == 0`, no shifts are needed, return `true`.
3. Iterate through each row of the matrix.
4. For **even** rows (`i % 2 == 0`), perform a **left shift** by `k` places. In C++, this can be elegantly handled using `std::rotate` with forward iterators.
5. For **odd** rows (`i % 2 != 0`), perform a **right shift** by `k` places. This can be handled using `std::rotate` with reverse iterators.
6. After processing all rows, compare the modified matrix `mat` with the stored original `temp`. If they match, return `true`.

### Approach 2: Index Mapping (Optimized, O(1) Space)
Instead of allocating memory to clone the entire matrix and spending processing power to physically move elements, we can predict the outcome logically.
1. Simplify the number of shifts using `k = k % n`.
2. For any element currently sitting at index `j` in an **even** row, a left shift means the element that will eventually land at index `j` is currently at `(j + k) % n`.
3. For any element at index `j` in an **odd** row, a right shift means the element that will replace it comes from the left side, specifically at `(j - k + n) % n`. (Adding `n` avoids negative mod issues).
4. Traverse the matrix. For each cell `(i, j)`, calculate the `finalIdx` from where its replacement will come. 
5. If `mat[i][j]` is not equal to `mat[i][finalIdx]`, the matrix is altered. Return `false` immediately. If the loop completes, return `true`.

## 💻 Solution Code

### 1️⃣ Brute-force Solution (Simulation)

```cpp
// Intuition: We can directly simulate the shift operations as described. Since shifts are cyclic and repeat every 'n' times, we can optimize the number of shifts using modulo arithmetic (k = k % n).
// Approach: Create a copy of the original matrix. Iterate through each row. If the row index is even, perform a left shift by 'k' using std::rotate. If it's odd, perform a right shift using std::rotate with reverse iterators. Finally, compare the modified matrix with the original copy.
// Time Complexity: O(m * n), where m is the number of rows and n is the number of columns. We visit and rotate elements in each row.
// Space Complexity: O(m * n) to store the copy of the original matrix for final comparison.
class Solution {
public:
    bool areSimilar(vector<vector<int>>& mat, int k) {
        vector<vector<int>> temp = mat; // Store original matrix for comparison
        int m = mat.size();
        int n = mat[0].size();
        k = k % n; // Simplify k to avoid redundant full cyclic rotations
        
        if(k == 0) { // No shifting needed
            return true;
        }
        
        for(int i = 0; i < m; i++) {
            if(i % 2) { // Odd row: right shift
                rotate(rbegin(mat[i]), rbegin(mat[i]) + k, rend(mat[i]));
            } else { // Even row: left shift
                rotate(begin(mat[i]), begin(mat[i]) + k, end(mat[i]));
            }
        }

        return temp == mat; // Compare modified matrix with the original
    }
};

/*
*
* Dry Run
* Input: mat = [[1,2,1,2],[5,5,5,5],[6,3,6,3]], k = 2
* m = 3, n = 4. k = 2 % 4 = 2.
* Row 0 (Even): Left shift [1,2,1,2] by 2 -> [1,2,1,2]
* Row 1 (Odd): Right shift [5,5,5,5] by 2 -> [5,5,5,5]
* Row 2 (Even): Left shift [6,3,6,3] by 2 -> [6,3,6,3]
* Modified mat == temp. Return true.
*
*/
````

### 2️⃣ Optimized Solution (Index Mapping)

```cpp
// Intuition: Instead of physically rotating the matrix which takes O(m*n) space, we can mathematically determine which element will replace the current element after 'k' shifts and check if they are identical.
// Approach: For each cell at (i, j), determine its future replacing element's index based on the row type. For even rows (left shift), the element comes from index (j + k) % n. For odd rows (right shift), it comes from (j - k + n) % n. If any cell doesn't match its future replacing element, return false.
// Time Complexity: O(m * n) because we traverse every cell in the m x n matrix exactly once.
// Space Complexity: O(1) as we only use a few integer variables for index calculations without extra array copies.
class Solution {
public:
    bool areSimilar(vector<vector<int>>& mat, int k) {
        int m = mat.size();
        int n = mat[0].size();

        k = (k % n); // Simplify k rotations

        if(k == 0) { // No shifting needed
            return true;
        }

        for(int i = 0; i < m; i++) {
            for(int j = 0; j < n; j++) {

                int currIdx = j;
                int finalIdx;

                if(i % 2 == 0) { // Even row : left shift element source
                    finalIdx = (j + k) % n;
                } else { // Odd row : right shift element source (add n to avoid negative mod)
                    finalIdx = (j - k + n) % n;
                }

                // Mismatch found, matrix won't be similar
                if(mat[i][currIdx] != mat[i][finalIdx]) {
                    return false;
                }

            }
        }

        return true; // All elements matched their replacements
    }
};

/*
*
* Dry Run
* Input: mat = [[1,2,3],[4,5,6],[7,8,9]], k = 4
* m = 3, n = 3. k = 4 % 3 = 1.
* i=0, j=0 (Even): curr=0, finalIdx=(0+1)%3=1. mat[0][0](1) != mat[0][1](2). 
* Mismatch found right away since 1 != 2. Return false.
*
*/
```
