## 📝 Problem Statement

The **complement** of an integer is the integer you get when you flip all the `0`'s to `1`'s and all the `1`'s to `0`'s in its binary representation.
    
For example, The integer `5` is `"101"` in binary and its complement is `"010"` which is the integer `2`.
    
Given an integer `n`, return its complement.

*(Note: This question is the same as LeetCode 476: Number Complement.)*

## ⚠️ Constraints

- `0 <= n < 10^9`

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `n = 5` | `2` | 5 is "101" in binary, with complement "010" in binary, which is 2 in base-10. |
| **2** | `n = 7` | `0` | 7 is "111" in binary, with complement "000" in binary, which is 0 in base-10. |
| **3** | `n = 10` | `5` | 10 is "1010" in binary, with complement "0101" in binary, which is 5 in base-10. |

## 🧠 Approach

To solve this problem, we can consider three main approaches:

### Approach 1: Bit-by-Bit Extraction & Flip (Brute Force)
We can extract the binary bits of `n` one by one using the modulo operator (`n % 2`). Once a bit is extracted, we flip it (turn 1 to 0, or 0 to 1) and multiply it by its corresponding place value (`2^counter`), adding it to our final result. We then right-shift `n` (`n >> 1` or `n / 2`) and repeat until `n` becomes 0. 

### Approach 2: Bitwise XOR with Mask
We know that applying the XOR operation on any bit with `1` flips that bit (`1 ^ 1 = 0`, `0 ^ 1 = 1`). Therefore, we can generate a "mask" that consists of entirely `1`s and has the exact same bit length as the binary representation of `n` (e.g., for `5` which is `101`, our mask should be `111`). Finally, returning `n ^ mask` gives the flipped complement.

### Approach 3: Direct Mask Calculation (Optimized)
Instead of building the mask using a loop, we can directly calculate the number of bits required to represent `n` using the logarithmic formula `floor(log2(n)) + 1`. We can then instantly create the mask using `(1 << bits) - 1` and return `n ^ mask`. This is the most optimized method running in `O(1)` time.

## 💻 Solution Code

### Approach 1: Bit-by-Bit Extraction
```cpp
// Intuition: Extract bits one by one from LSB to MSB. Flip each bit and place it at its corresponding power of 2 to build the complement number.
// Approach: Loop until n becomes 0. Extract the remainder (r = n % 2). Flip the remainder (!r) and multiply it by 2^counter. Add it to the result and right shift n.
// Time Complexity: O(log(n)) - We traverse each bit of the number (dividing by 2 continuously).
// Space Complexity: O(1) - Only a few integer variables are used for calculation.
class Solution {
public:
    int bitwiseComplement(int n) {
        // Handle edge case for 0
        if(n == 0)
            return 1;
        
        int result = 0;
        int counter = 0;
        
        // Loop until all bits are processed
        while(n) {
            int r = n % 2; // Extract the least significant bit
            
            // Flip the bit (!r), multiply by appropriate power of 2, and add to result
            result += (pow(2, counter) * !r);
            
            counter++; // Increment the bit position counter
            n = n >> 1; // Right shift n to process the next bit
        }
        
        return result;
    }
};

/*
*
* Dry Run
* Input: n = 5 (binary: 101)
*
* Iteration 1:
* r = 5 % 2 = 1
* !r = 0
* result = 0 + (2^0 * 0) = 0
* counter = 1
* n = 5 >> 1 = 2
*
* Iteration 2:
* r = 2 % 2 = 0
* !r = 1
* result = 0 + (2^1 * 1) = 2
* counter = 2
* n = 2 >> 1 = 1
*
* Iteration 3:
* r = 1 % 2 = 1
* !r = 0
* result = 2 + (2^2 * 0) = 2
* counter = 3
* n = 1 >> 1 = 0
*
* Loop terminates. Returns result = 2.
*
*/
```

### Approach 2: Building XOR Mask
```cpp
// Intuition: XORing any bit with 1 flips it. If we build a mask of all 1s having the same bit-length as n, then n ^ mask gives the complement.
// Approach: Start with a mask of 1. While mask is less than n, left shift it by 1 and bitwise OR with 1 to append more 1s. Finally, return n ^ mask.
// Time Complexity: O(log(n)) - We build the mask bit by bit, matching the length of n's binary representation.
// Space Complexity: O(1) - We only use a single variable for the mask.
class Solution {
public:
    int bitwiseComplement(int n) {
        // Edge case: complement of 0 is 1
        if (n == 0)
            return 1;

        int mask = 1;

        // Keep appending 1s to the mask until it fully covers all bits of n
        while (mask < n) {
            mask = (mask << 1) | 1;  // builds 111...1
        }
        
        // XOR n with the mask of 1s to flip all bits
        return n ^ mask;
    }
};

/*
*
* Dry Run
* Input: n = 5 (binary: 101)
*
* Initial: mask = 1 (binary: 1)
* Iteration 1: mask (1) < n (5) -> True
* mask = (1 << 1) | 1 = 2 | 1 = 3 (binary: 11)
* * Iteration 2: mask (3) < n (5) -> True
* mask = (3 << 1) | 1 = 6 | 1 = 7 (binary: 111)
*
* Iteration 3: mask (7) < n (5) -> False
* Loop terminates.
* Return n ^ mask = 5 ^ 7 = (101 ^ 111) = 010 (which is 2)
*
*/
```

### Approach 3: Optimized Direct Mask
```cpp
// Intuition: The number of bits needed to represent n can be found directly using logarithms. We can then instantly create the required mask of 1s.
// Approach: Calculate bit length using floor(log2(n)) + 1. Create a mask by left-shifting 1 by the number of bits and subtracting 1. Return n ^ mask.
// Time Complexity: O(1) - Calculating log2 and basic bitwise operations take constant time.
// Space Complexity: O(1) - Only a few integer variables are used.
class Solution {
public:
    int bitwiseComplement(int n) {
        // Edge case: complement of 0 is 1
        if (n == 0)
            return 1;

        // Find the total number of bits required to represent n
        int bits = floor(log2(n)) + 1;  // number of bits
        
        // Create a mask with all 1s of length 'bits'
        int mask = (1 << bits) - 1;     // 2^bits - 1 = 111...1

        // XOR n with the mask to get the complement
        return n ^ mask;
    }
};

/*
*
* Dry Run
* Input: n = 5 (binary: 101)
*
* bits = floor(log2(5)) + 1 = 2 + 1 = 3
* mask = (1 << 3) - 1 = 8 - 1 = 7 (binary: 111)
* Return n ^ mask = 5 ^ 7 = 2
*
*/
```

## 📝 Notes
- Handling the edge case `n == 0` separately is critical because mathematically `log2(0)` is undefined, and bitwise shifts on 0 can produce unexpected logic without explicit handling.

## 🔑 Key Insights
- The XOR (`^`) operator is a very powerful tool for toggling/flipping bits. XORing any bit with `1` results in the exact opposite bit.
- We can dynamically build a bitmask (composed entirely of `1`s) in a loop by continuously left-shifting and OR-ing with 1: `(mask << 1) | 1`.
- An even faster mathematical shortcut is using `(1 << bits) - 1`, which creates a solid block of `1`s of any desired length in perfect `O(1)` time.
