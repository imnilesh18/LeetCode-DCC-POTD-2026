## 📝 Problem Statement

You are given two strings `s1` and `s2`, both of length `n`, consisting of lowercase English letters.

You can apply the following operation on **any** of the two strings **any** number of times:
- Choose any two indices `i` and `j` such that `i < j` and the difference `j - i` is **even**, then **swap** the two characters at those indices in the string.

Return `true` if you can make the strings `s1` and `s2` equal, and `false` otherwise.

---

## 🎯 Constraints

- `n == s1.length == s2.length`
- `1 <= n <= 10^5`
- `s1` and `s2` consist only of lowercase English letters.

## 🧠 Approach

This problem is a direct extension of Part I. The core hint lies in the condition that you can only swap characters at indices `i` and `j` if their difference `j - i` is even. 
- The difference between two even numbers is always even.
- The difference between two odd numbers is always even.
- The difference between an even and an odd number is always odd.

This means characters currently sitting at **even indices** can only be swapped with other characters at **even indices**. Similarly, characters at **odd indices** can only be swapped with other characters at **odd indices**. 

Therefore, for `s1` to be transformable into `s2`:
1. The frequency of characters at all even indices in `s1` must exactly match the frequency of characters at all even indices in `s2`.
2. The frequency of characters at all odd indices in `s1` must exactly match the frequency of characters at all odd indices in `s2`.

We can easily track these frequencies using two size-26 integer arrays, resulting in a highly efficient `O(n)` time complexity solution.

## 💻 Solution Code

```cpp
// Intuition: Since we can only swap characters where the index difference `j - i` is even, characters can never change their parity. Even-indexed characters stay at even indices, and odd-indexed characters stay at odd indices.
// Approach: We use two frequency arrays of size 26 (one for even indices, one for odd indices). We increment counts for characters in s1 and decrement for s2. Finally, if all counts balance out to zero, the strings can be made equal.
// Time Complexity: O(n) where n is the length of the string, as we iterate through the strings exactly once.
// Space Complexity: O(1) because we only use two arrays of fixed size 26, independent of input size.
class Solution {
public:
    bool checkStrings(string s1, string s2) {
        int even[26] = {0}; // Track characters at even indices
        int odd[26] = {0};  // Track characters at odd indices

        int n = s1.length();

        for(int i = 0; i < n; i++) {
            if(i % 2 == 0) { // Even indices
                even[s1[i] - 'a']++;
                even[s2[i] - 'a']--;
            } else { // Odd indices
                odd[s1[i] - 'a']++;
                odd[s2[i] - 'a']--;
            }
        }

        // Check if all character frequencies perfectly balance out
        for(int i = 0; i < 26; i++) {
            if(even[i] != 0 || odd[i] != 0)
                return false;
        }

        return true;
    }
};

/*
*
* Dry Run
*
* s1 = "abe", s2 = "bea"
* n = 3
*
* i = 0 (even): 
* s1[0]='a', s2[0]='b' -> even['a'-'a']++, even['b'-'a']-- -> even[0]=1, even[1]=-1
* i = 1 (odd): 
* s1[1]='b', s2[1]='e' -> odd['b'-'a']++, odd['e'-'a']-- -> odd[1]=1, odd[4]=-1
* i = 2 (even): 
* s1[2]='e', s2[2]='a' -> even['e'-'a']++, even['a'-'a']-- -> even[4]=1, even[0]=0
* Loop ends. Check arrays:
* even[1] = -1 (which is != 0) -> returns false immediately.
* Output: false
*
*/
```

## 📌 Notes & Key Insights

- **Parity Property:** The foundational logic of this problem is understanding that `j - i = even` restricts character movement exclusively within their starting parity (even stays even, odd stays odd).
- **Frequency Map Optimization:** Sorting the even/odd grouped characters from both strings and comparing them would take `O(n log n)` time. Using hash tables (or arrays of size 26) optimizes this validation check down to `O(n)` time.
