## 📝 Problem Statement

You are given 3 positive integers `zero`, `one`, and `limit`.

A **binary array** `arr` is called **stable** if:
1. The number of occurrences of `0` in `arr` is **exactly** `zero`.
2. The number of occurrences of `1` in `arr` is **exactly** `one`.
3. Each **subarray** of `arr` with a size greater than `limit` must contain **both** `0` and `1`.

Return the **total** number of **stable** binary arrays. 
Since the answer may be very large, return it **modulo** $10^9 + 7$.

## ⚠️ Constraints

- `1 <= zero, one, limit <= 200`

## 💡 Explanation and Approach

### Decoding the Problem
The most confusing part of this problem statement is the third rule: *"Each subarray of `arr` with a size greater than `limit` must contain both `0` and `1`"*. 

What this indirectly means is: **You cannot have more than `limit` consecutive `0`s or `limit` consecutive `1`s anywhere in the array.** If you have a streak of identical elements longer than `limit`, it immediately forms a subarray of size `> limit` containing only one type of digit, thereby violating the rule.

### The Strategy
We can break the array creation process into smaller subproblems. Instead of placing numbers one by one, we place them in **blocks** (or streaks) of consecutive identical numbers. 
- If we just placed a block of `1`s, our next block must be `0`s. 
- The length of this block of `0`s can be anything from `1` up to `min(zerosLeft, limit)`.
- If we just placed a block of `0`s, our next block must be `1`s.
- The length of this block of `1`s can be anything from `1` up to `min(onesLeft, limit)`.

### State Tracking
We can use a recursive function with three states:
1. `onesLeft`: How many `1`s are left to place.
2. `zerosLeft`: How many `0`s are left to place.
3. `lastWasOne`: A boolean indicating if the previous block was `1`s (true) or `0`s (false).

**Base Case:** If `onesLeft == 0` and `zerosLeft == 0`, we've successfully used all digits without breaking the limit rules. Return `1`.

We memoize this using a 3D array `t[onesLeft][zerosLeft][lastWasOne]` to optimize overlapping subproblems.

## 💻 Solution Codes

### 1️⃣ Approach 1: Recursion + Memoization

```cpp
// Intuition:
// The problem asks us to form binary arrays with exactly `zero` 0s and `one` 1s without exceeding `limit` consecutive identical elements. 
// We can build the array by placing chunks of 0s or 1s. At each step, if we placed 1s previously, we must now place 0s, and vice versa. 
// The number of identical elements we can place at once ranges from 1 to min(available elements, limit).

// Approach:
// 1. Use recursion to find all valid arrangements. State variables are `onesLeft`, `zerosLeft`, and a boolean `lastWasOne`.
// 2. Base Case: If `onesLeft` and `zerosLeft` are both 0, a valid array is formed (return 1).
// 3. If `lastWasOne` is true, iterate `len` from 1 to min(zerosLeft, limit) and recursively call for `zerosLeft - len` and `lastWasOne = false`.
// 4. If `lastWasOne` is false, iterate `len` from 1 to min(onesLeft, limit) and recursively call for `onesLeft - len` and `lastWasOne = true`.
// 5. Use a 3D memoization table `t` to store results of overlapping subproblems.

// Time Complexity: O(one * zero * limit)
// There are O(one * zero * 2) states. For each state, we run a loop of maximum size `limit`.
// Space Complexity: O(one * zero)
// The 3D memoization array takes O(201 * 201 * 2) space. The recursion stack depth is at most O(one + zero).

class Solution {
public:
    int M = 1e9 + 7;
    int t[201][201][2];

    int solve(int onesLeft, int zerosLeft, bool lastWasOne, int limit) {
        // Base case: successfully used all required digits
        if(onesLeft == 0 && zerosLeft == 0) {
            return 1;
        }

        // Return memoized result if available
        if(t[onesLeft][zerosLeft][lastWasOne] != -1) {
            return t[onesLeft][zerosLeft][lastWasOne];
        }

        int result = 0;

        if(lastWasOne == true) { 
            // Last placed chunk was 1s, now explore placing 0s
            for(int len = 1; len <= min(zerosLeft, limit); len++) {
                result = (result + solve(onesLeft, zerosLeft - len, false, limit)) % M;
            }
        } else { 
            // Last placed chunk was 0s, now explore placing 1s
            for(int len = 1; len <= min(onesLeft, limit); len++) {
                result = (result + solve(onesLeft - len, zerosLeft, true, limit)) % M;
            }
        }

        return t[onesLeft][zerosLeft][lastWasOne] = result;
    }

    int numberOfStableArrays(int zero, int one, int limit) {
        memset(t, -1, sizeof(t));
        
        // Start exploring by assuming the sequence begins with a chunk of 1s (last was 0)
        int startWithOne = solve(one, zero, false, limit);
        
        // Start exploring by assuming the sequence begins with a chunk of 0s (last was 1)
        int startWithZero = solve(one, zero, true, limit);

        return (startWithOne + startWithZero) % M;
    }
};

/*
*
* Dry Run
* zero = 1, one = 1, limit = 2
*
* startWithOne = solve(1, 1, false, 2)
* -> lastWasOne is false, explore placing 1s:
* len = 1: solve(0, 1, true, 2)
* -> lastWasOne is true, explore placing 0s:
* len = 1: solve(0, 0, false, 2) -> returns 1 (Base Case reached)
* -> returns 1
* -> startWithOne = 1
*
* startWithZero = solve(1, 1, true, 2)
* -> lastWasOne is true, explore placing 0s:
* len = 1: solve(1, 0, false, 2)
* -> lastWasOne is false, explore placing 1s:
* len = 1: solve(0, 0, true, 2) -> returns 1 (Base Case reached)
* -> returns 1
* -> startWithZero = 1
*
* Total Result = (startWithOne + startWithZero) % M 
* Total Result = (1 + 1) % 1000000007 = 2
*
*/
```

### 2️⃣ Approach 2: Bottom-Up DP

```cpp
// Intuition:
// To avoid the recursion stack overhead, we can build the solution iteratively. 
// The transitions map exactly from the top-down logic: we build the counts iteratively from 0 up to `one` and `zero`.

// Approach:
// 1. Initialize a 3D DP array `t` with 0s. 
// 2. Set the base cases: `t[0][0][0] = 1` and `t[0][0][1] = 1` (0 ones and 0 zeros left means 1 valid configuration).
// 3. Iterate over all combinations of `onesLeft` (0 to `one`) and `zerosLeft` (0 to `zero`).
// 4. For each state, compute the ways to place blocks of 0s (looping `len` up to `limit`) and store it in `t[onesLeft][zerosLeft][1]`.
// 5. Compute the ways to place blocks of 1s (looping `len` up to `limit`) and store it in `t[onesLeft][zerosLeft][0]`.
// 6. The final answer is the sum of starting with 1s (`t[one][zero][0]`) and starting with 0s (`t[one][zero][1]`).

// Time Complexity: O(one * zero * limit)
// The nested loops process `one * zero` states, and inner loops iterate up to `limit` times.
// Space Complexity: O(one * zero)
// The 3D DP table requires O(201 * 201 * 2) contiguous memory.

class Solution {
public:
    int M = 1e9 + 7;
    int t[201][201][2];

    int numberOfStableArrays(int zero, int one, int limit) {
        memset(t, 0, sizeof(t));

        // Base case: solve(0, 0, lastWasOne) = 1
        t[0][0][0] = 1;
        t[0][0][1] = 1;

        for(int onesLeft = 0; onesLeft <= one; onesLeft++) {
            for(int zerosLeft = 0; zerosLeft <= zero; zerosLeft++) {

                // Skip the base case processing as it's already seeded
                if(onesLeft == 0 && zerosLeft == 0) continue;

                int result = 0;

                // Explore placing 0s (equivalent to lastWasOne == true)
                result = 0;
                for(int len = 1; len <= min(zerosLeft, limit); len++) {
                    result = (result + t[onesLeft][zerosLeft - len][0]) % M;
                }
                t[onesLeft][zerosLeft][1] = result;

                // Explore placing 1s (equivalent to lastWasOne == false)
                result = 0;
                for(int len = 1; len <= min(onesLeft, limit); len++) {
                    result = (result + t[onesLeft - len][zerosLeft][1]) % M;
                }
                t[onesLeft][zerosLeft][0] = result;
            }
        }

        // Fetch total ways starting with 1s (last was 0)
        int startWithOne  = t[one][zero][0]; 
        // Fetch total ways starting with 0s (last was 1)
        int startWithZero = t[one][zero][1]; 

        return (startWithOne + startWithZero) % M;
    }
};

/*
*
* Dry Run
* zero = 1, one = 1, limit = 1
* * Initial Base Cases: t[0][0][0] = 1, t[0][0][1] = 1
* * State: onesLeft = 0, zerosLeft = 1
* -> Place 0s: t[0][1][1] = t[0][1-1][0] = t[0][0][0] = 1
* -> Place 1s: loop bounds fail (len <= min(0, 1) is 1 <= 0), t[0][1][0] = 0
* * State: onesLeft = 1, zerosLeft = 0
* -> Place 0s: loop bounds fail (len <= min(0, 1)), t[1][0][1] = 0
* -> Place 1s: t[1][0][0] = t[1-1][0][1] = t[0][0][1] = 1
* * State: onesLeft = 1, zerosLeft = 1
* -> Place 0s: t[1][1][1] = t[1][1-1][0] = t[1][0][0] = 1
* -> Place 1s: t[1][1][0] = t[1-1][1][1] = t[0][1][1] = 1
* * Results extracted at the end:
* startWithOne = t[1][1][0] = 1
* startWithZero = t[1][1][1] = 1
* Total = 1 + 1 = 2.
*
*/
```
