# 3666. Minimum Operations to Equalize Binary String

**Difficulty:** Hard 🔴  
**Tags:** `String`, `Breadth-First Search (BFS)`, `Math`, `Greedy`

---

## 📜 Problem Statement

You are given a binary string `s`, and an integer `k`.

In one operation, you must choose **exactly** `k` **different** indices and **flip** each `'0'` to `'1'` and each `'1'` to `'0'`.

Return the **minimum** number of operations required to make all characters in the string equal to `'1'`. If it is not possible, return `-1`.

### 📋 Constraints

- `1 <= s.length <= 10^5`
- `s[i]` is either `'0'` or `'1'`.
- `1 <= k <= s.length`

---

## 📊 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `s = "110", k = 1` | `1` | There is one `'0'` in `s`. Since `k = 1`, we can flip it directly in one operation. |
| **2** | `s = "0101", k = 3` | `2` | Operation 1: Flip indices `[0, 1, 3]`. `s` changes from `"0101"` to `"1000"`. <br> Operation 2: Flip indices `[1, 2, 3]`. `s` changes from `"1000"` to `"1111"`. <br> Minimum operations = 2. |
| **3** | `s = "101", k = 2` | `-1` | Since `k = 2` and `s` has only one `'0'`, it is impossible to flip exactly `k` indices to make all `'1'`. Hence, the answer is `-1`. |

---

## 🧠 Approach & Explanation

Instead of storing and tracking the actual configurations of the string (which has up to $10^5$ characters and leads to too many string states), we only need to keep track of the **number of zeros**. Our ultimate goal is to reach a state where the count of zeros is `0`.

### State Transition (Math Formulation)
Let the initial number of zeros in the string be `z`, and the total length of the string be `n`.
In a single operation, we must pick exactly `k` indices. Suppose we choose to flip `f` zeros out of these `k` indices. 
- Since we picked `f` zeros, the remaining `k - f` indices we picked must be ones.
- Flipping `f` zeros turns them into ones (decreasing zero count by `f`).
- Flipping `k - f` ones turns them into zeros (increasing zero count by `k - f`).

The new number of zeros $Z'$ will be:
$Z' = z - f + (k - f) = z + k - 2f$

### Finding the Range for `f`
We cannot simply pick any value for `f`. It must be logically valid:
1. We cannot flip more zeros than we currently have: $f \le z$.
2. We cannot pick more than $k$ indices: $f \le k$.
   - **Maximum zeros we can flip:** $f_{max} = \min(k, z)$
3. The number of ones we choose to flip ($k - f$) cannot exceed the total number of ones currently in the string ($n - z$). 
   $k - f \le n - z \implies f \ge k - n + z$.
4. We cannot flip a negative number of zeros: $f \ge 0$.
   - **Minimum zeros we can flip:** $f_{min} = \max(0, k - n + z)$

### The Shortest Path (BFS)
Because we want the *minimum* number of operations, this elegantly maps to a **Shortest Path Problem on an Unweighted Graph**.
- **Nodes/States:** Number of zeros (ranging from `0` to `n`).
- **Edges:** A valid transition from $z$ to $z'$ takes 1 operation.
- **Target:** Reach state `0`.

### Optimization for Hard Constraint
A simple BFS would loop over all possible values of `f` (from $f_{min}$ to $f_{max}$) for every state, taking $O(n^2)$ time, resulting in TLE.
Notice that the equation $Z' = z + k - 2f$ steps by `2`. This means all reachable next states share the **same parity** (they are all even or all odd). 
Instead of checking every state and running into redundant states we already visited, we can keep two `std::set`s of unvisited nodes: `evenSet` and `oddSet`. Using `lower_bound`, we can directly find valid unvisited states, add them to the queue, and instantly remove them. Since each state (0 to n) is erased at most once, this reduces the time complexity to $O(n \log n)$.

## 💻 Brute-force Solution

```cpp
// Intuition: We can model the problem as finding the shortest path in an unweighted graph where nodes represent the count of zeros. BFS is ideal for this.
// Approach: Track the count of '0's instead of the string. From a state with `z` zeros, try flipping `f` zeros where `f` is valid. Queue the new zero counts and return steps when zero count hits 0.
// Time Complexity: O(n^2) because for each state (up to n), we might iterate through O(n) possible values of `f`.
// Space Complexity: O(n) for the BFS queue and the operations array tracking visited states.
class Solution {
public:
    int minOperations(string s, int k) {
        int n = s.length();

        int startZeros = 0;
        // Count initial number of zeros in the string
        for(char &ch : s) {
            if(ch == '0')
                startZeros++;
        }

        // If string already has all 1s, 0 steps needed
        if(startZeros == 0) {
            return 0; //no operations required
        }

        vector<int> operations(n+1, -1);
        //operations[z] = min operations required to reach state in which we have z 0s

        queue<int> que;
        que.push(startZeros);
        operations[startZeros] = 0;

        while(!que.empty()) {
            int z = que.front();
            que.pop();

            // Calculate minimum and maximum zeros we can flip in this operation
            int minF = max(0, k-n+z);
            int maxF = min(k, z);

            // Iterate over all possible counts of zeros we can flip
            for(int f = minF; f <= maxF; f++) {
                int newZ = z + k - 2*f;

                // Process unvisited states
                if(operations[newZ] == -1) {
                    operations[newZ] = operations[z] + 1;

                    // Reached target state where zero count is 0
                    if(newZ == 0) {
                        return operations[newZ];
                    }

                    que.push(newZ);
                }
            }
        }

        // Target state unreachable
        return -1;
    }
};

/*
*
* Dry Run
* s = "0101", k = 3, n = 4
* Start zeros = 2
* Push 2 to queue, ops[2] = 0
* Pop 2. minF = max(0, 3-4+2) = 1. maxF = min(3, 2) = 2
* Iterate f = 1: newZ = 2 + 3 - 2(1) = 3. ops[3] = 1. Q pushes 3
* Iterate f = 2: newZ = 2 + 3 - 2(2) = 1. ops[1] = 1. Q pushes 1
* Pop 1. minF = max(0, 3-4+1) = 0. maxF = min(3, 1) = 1
* Iterate f = 0: newZ = 1 + 3 - 2(0) = 4. ops[4] = 2. Q pushes 4
* Iterate f = 1: newZ = 1 + 3 - 2(1) = 2. ops[2] != -1 (already visited)
* Pop 3. minF = max(0, 3-4+3) = 2. maxF = min(3, 3) = 3
* Iterate f = 2: newZ = 3 + 3 - 2(2) = 2. Visited.
* Iterate f = 3: newZ = 3 + 3 - 2(3) = 0. ops[0] = 2. Target 0 reached! Returns 2.
*
*/
```

## 🚀 Optimized Solution

```cpp
// Intuition: The next states generated from `z` form a contiguous sequence with a step of 2, meaning they all share the same parity. By keeping track of unvisited states in separate Even and Odd sets, we can skip redundant visits.
// Approach: Calculate the min and max possible new zero counts. Based on parity, use `lower_bound` on the corresponding unvisited set to find valid states, process them, queue them, and immediately erase them from the set.
// Time Complexity: O(n log n) because each state (from 0 to n) is inserted and erased from the set at most once, and lower_bound takes O(log n).
// Space Complexity: O(n) for the sets, queue, and operations array.
class Solution {
public:
    int minOperations(string s, int k) {
        int n = s.length();

        int startZeros = 0;
        // Count initial zeros
        for(char &ch : s) {
            if(ch == '0')
                startZeros++;
        }

        if(startZeros == 0) {
            return 0; //no operations required
        }

        vector<int> operations(n+1, -1);
        //operations[z] = min operations required to reach state in which we have z 0s

        // Track unvisited even and odd zero-count states
        set<int> evenSet;
        set<int> oddSet;

        for(int count = 0; count <= n; count++) {
            if(count%2 == 0) {
                evenSet.insert(count);
            } else {
                oddSet.insert(count);
            }
        }

        queue<int> que;
        que.push(startZeros);
        operations[startZeros] = 0;
        
        // Remove the initial state from unvisited sets
        if(startZeros % 2 == 0) {
            evenSet.erase(startZeros);
        } else {
            oddSet.erase(startZeros);
        }

        while(!que.empty()) {
            int z = que.front();
            que.pop();
            
            // max(f) gives the minimum possible next state, min(f) gives the maximum possible next state
            int min_new_z = z + k - 2*min(k, z);
            int max_new_z = z + k - 2*max(0, k-n+z);

            // Select the relevant set based on the parity of the new zero count
            set<int> &currSet = (min_new_z%2 == 0) ? evenSet : oddSet;

            // Find the first unvisited state within the valid range
            auto it = currSet.lower_bound(min_new_z);

            while(it != currSet.end() && *it <= max_new_z) {
                int newZ = *it;

                if(operations[newZ] == -1) {
                    operations[newZ] = operations[z] + 1;

                    // If zero zeros are left, we found the shortest path
                    if(newZ == 0) {
                        return operations[newZ];
                    }

                    que.push(newZ);
                }

                // Erase visited state to prevent future redundant processing
                it = currSet.erase(it);
            }
        }

        // Return -1 if it's impossible to reach 0
        return -1;
    }
};

/*
*
* Dry Run
* s = "0101", k = 3, n = 4. Start zeros = 2
* Sets created: evenSet={0,2,4}, oddSet={1,3}
* 2 is erased from evenSet. Q = [2]. ops[2] = 0
* Pop 2. min_new_z = 2+3-2(2) = 1. max_new_z = 2+3-2(1) = 3
* Both are odd, so currSet = oddSet={1,3}
* Iterator starts at 1. Loops over oddSet from 1 to 3:
* - Process 1: ops[1] = 1. Q pushes 1. Erase 1.
* - Process 3: ops[3] = 1. Q pushes 3. Erase 3.
* Q = [1, 3], oddSet = {}
* Pop 1. min_new_z = 1+3-2(1) = 2. max_new_z = 1+3-2(0) = 4
* Both are even, currSet = evenSet={0,4}
* Iterator starts at lower_bound(2) which is 4 (since 2 was deleted!). Wait, formula says max(0, 3-4+1)=0, min(3,1)=1. min_new=1+3-2(1)=2, max_new=1+3-2(0)=4. lower_bound(2) points to 4 in evenSet.
* However, what about 0? Ah! If we pop 3: min_new=3+3-2(3)=0, max_new=3+3-2(2)=2. lower_bound(0) points to 0. 
* Process 0: ops[0] = ops[3] + 1 = 1 + 1 = 2. Returns 2!
*
*/
```

## 🔍 Key Insights

1. **State Abstraction**: The sequence of 1s and 0s doesn't matter; only the count of zeros matters. This reduces an unmanageable string state problem into a simple numeric state problem.
2. **Deterministic Parity Checks**: A sequence of swaps involving a specific $k$ shifts the total number of zeros linearly in bounds, with uniform increments of 2. Tracking `minZ` and `maxZ` safely spans all possible valid states.
3. **Graph Optimization via Sets**: Using `std::set` allows deletion of previously visited states. This acts as a robust pruning mechanism against overlapping subproblems, reducing overlapping edge computations entirely.
