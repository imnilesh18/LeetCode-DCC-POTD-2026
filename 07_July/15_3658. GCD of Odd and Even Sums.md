## 📝 Problem Statement

You are given an integer `n`. Your task is to compute the GCD (Greatest Common Divisor) of two values:
- `sumOdd`: the sum of the smallest `n` positive odd numbers.
- `sumEven`: the sum of the smallest `n` positive even numbers.

Return the GCD of `sumOdd` and `sumEven`.

## 💡 Explanation & Derivation

To solve this problem efficiently, we can use simple mathematical derivations rather than iterating through loops to find the sums. Let's break down the math for finding the sum of the first `n` odd and even numbers.

### Deriving `sumOdd`:
Any odd number can be represented in the format `2k - 1`, where `k` starts from `1`.
For example:
- If `k = 1`, `2(1) - 1 = 1`
- If `k = 2`, `2(2) - 1 = 3`
- If `k = 3`, `2(3) - 1 = 5`

To find the sum of the first `n` odd numbers, we can use the summation:
∑ (from `k = 1` to `n`) of `(2k - 1)`
= 2 * [∑ `k`] - [∑ `1`]
= 2 * [ `n(n + 1) / 2` ] - `n`
= `n(n + 1) - n`
= `n² + n - n`
= **`n²`**

So, **`sumOdd = n²`**.

### Deriving `sumEven`:
Any even number can be represented as `2k`, where `k` starts from `1`.
For example:
- If `k = 1`, `2(1) = 2`
- If `k = 2`, `2(2) = 4`

To find the sum of the first `n` even numbers:
∑ (from `k = 1` to `n`) of `2k`
= 2 * [∑ `k`]
= 2 * [ `n(n + 1) / 2` ]
= **`n(n + 1)`**

So, **`sumEven = n(n + 1)`**.

---

## 🚀 Approaches

### Approach 1: Simple Maths using GCD Function
Using our derived formulas, we know `sumOdd = n * n` and `sumEven = n * (n + 1)`. We can directly compute these two values and return their GCD using built-in math functions.

### Approach 2: Constant Time Optimization (Key Insight)
Let's analyze what we are taking the GCD of:
`GCD(n², n(n + 1))`

We can factor out the common term `n` from both expressions:
`n * GCD(n, n + 1)`

**Mathematical Insight:** The Greatest Common Divisor of any two consecutive integers `n` and `n + 1` is **always 1** (they are co-prime).
Therefore:
`n * 1 = n`

The answer is simply `n` itself! We can return it directly in `O(1)` time.

## 💻 Solution Code

### Brute-force / Basic Math Solution (C++)

```cpp
// Intuition: The sum of the first n odd numbers is always n^2, and the sum of the first n even numbers is n*(n+1). We can compute these using basic arithmetic and find their GCD.
// Approach: Apply the derived mathematical formulas to calculate sumOdd and sumEven in constant time, then pass them to the built-in __gcd() function.
// Time Complexity: O(log(n)) because computing the GCD using the Euclidean algorithm takes logarithmic time relative to the value of the numbers.
// Space Complexity: O(1) as no extra memory or data structures are utilized.

// Approach-1 (Simple Maths)
// T.C : O(log(n))
// S.C : O(1)
class Solution {
public:
    int gcdOfOddEvenSums(int n) {
        // Sum of first n odd numbers  = n * n
        // Sum of first n even numbers = n * (n + 1)
        return __gcd(n * n, n * (n + 1)); 
    }
};

/*
*
* Dry Run
*
* Input: n = 4
* sumOdd calculation: 4 * 4 = 16
* sumEven calculation: 4 * (4 + 1) = 20
* Call __gcd(16, 20)
* GCD of 16 and 20 is 4
* Return 4
*
*/
```

### Optimized Solution (C++)

```cpp
// Intuition: Factoring out 'n' from GCD(n^2, n*(n+1)) yields n * GCD(n, n+1). Because consecutive numbers (n and n+1) are strictly co-prime, their GCD is always 1, making the final result always equal to 'n'.
// Approach: Bypass all calculations and built-in GCD function calls by simply returning 'n' based on the simplified mathematical derivation.
// Time Complexity: O(1) as we perform a direct return with no calculations.
// Space Complexity: O(1) since we use no extra variables or memory.

// Approach-2 (Simple Maths and Constant Time)
// T.C : O(1)
// S.C : O(1)
class Solution {
public:
    int gcdOfOddEvenSums(int n) {
        // gcd(n*n, n*(n+1)) = n * gcd(n, n+1) = n * 1 = n
        // Consecutive integers share no common factors other than 1
        return n;
    }
};

/*
*
* Dry Run
*
* Input: n = 5
* Function execution starts.
* Formula theoretically evaluates to: 5 * GCD(5, 6) -> 5 * 1
* Code directly returns the parameter 'n' (which is 5).

*
*/
```
