## 📝 Problem Statement

You are given a positive integer `n` representing `n` cities numbered from `1` to `n`. You are also given a 2D array `roads` where `roads[i] = [ai, bi, distancei]` indicates that there is a bidirectional road between cities `ai` and `bi` with a distance equal to `distancei`. The cities graph is not necessarily connected.

The **score** of a path between two cities is defined as the **minimum distance** of a road in this path.

Return the **minimum possible score** of a path between cities `1` and `n`.

> **Note:**
> - A path is a sequence of roads between two cities.
> - It is allowed for a path to contain the same road multiple times, and you can visit cities `1` and `n` multiple times along the path.
> - The test cases are generated such that there is **at least one path** between `1` and `n`.

---

## ⛔ Constraints

- `2 <= n <= 10^5`
- `1 <= roads.length <= 10^5`
- `roads[i].length == 3`
- `1 <= ai, bi <= n`
- `ai != bi`
- `1 <= distancei <= 10^4`
- There are no repeated edges.
- There is at least one path between `1` and `n`.

## 🧠 Approach

Because the graph is undirected and we can visit edges/nodes multiple times, we do not need to restrict ourselves to a simple path. We are guaranteed that city `1` and city `n` are connected. 

Since we can cycle back and forth on any road to artificially include it in our path without penalty, the **minimum score** is effectively the **minimum weight of any edge** in the entire connected component that contains city `1`.

1. **Graph Representation**: Build an adjacency list `adj` storing pairs of `{neighbor, distance}` for each city.
2. **Traversal (DFS)**: Start a Depth First Search from city `1`. 
3. **Minimize Edge Track**: As you traverse the graph, compare every edge you cross against a running `result` variable that keeps track of the global minimum.
4. **Avoid Cycles**: Maintain a `visited` array so that you don't infinitely recurse on the same nodes.
5. **Return Result**: Once the traversal finishes exploring the component containing city `1`, `result` will hold the absolute minimum distance of all reachable edges.

---

## 💻 Solution Code

```cpp
// Intuition: Since paths can re-visit edges and nodes, the minimum score of a path from 1 to n is simply the minimum edge weight in the entire connected component containing city 1.
// Approach: Build an adjacency list for the undirected graph. Use DFS starting from node 1 to visit all reachable nodes, updating a global minimum with every edge weight encountered during the traversal.
// Time Complexity: O(V + E), where V is the number of cities and E is the number of roads. We visit each city and explore each road at most once.
// Space Complexity: O(V + E) to store the graph in an adjacency list, plus O(V) for the recursion stack and visited array.

class Solution {
public:

    void dfs(unordered_map<int, vector<pair<int, int>>> &adj, int u, vector<bool>& visited, int &result) {
        
        // Mark current city as visited
        visited[u] = true;
        
        // Traverse all connected roads from current city
        for(auto &vec : adj[u]) {
            int v = vec.first;
            int c = vec.second;
            
            // Update the global minimum score with current road's distance
            result = min(result, c);
            
            // If the neighboring city is unvisited, recursively explore it
            if(!visited[v]) {
                dfs(adj, v, visited, result);
            }
        }
    }
    
    int minScore(int n, vector<vector<int>>& roads) {
        unordered_map<int, vector<pair<int, int>>> adj;
        
        // Build the undirected graph representation
        for(auto &vec : roads) {
            
            int u = vec[0];
            int v = vec[1];
            int c = vec[2];
            
            adj[u].push_back({v, c});
            adj[v].push_back({u, c});
        }
        
        // Array to track visited cities to prevent infinite loops
        vector<bool> visited(n, false);
        int result = INT_MAX;
        
        // Start DFS traversal from city 1
        dfs(adj, 1, visited, result);
        
        return result;
    }
};

/*
*
* Dry Run
*
* Input: n = 4, roads = [[1,2,9],[2,3,6],[2,4,5],[1,4,7]]
* Graph: 
* 1 -> {2, 9}, {4, 7}
* 2 -> {1, 9}, {3, 6}, {4, 5}
* 3 -> {2, 6}
* 4 -> {2, 5}, {1, 7}
* * Setup: visited = [false, false, false, false], result = INT_MAX
* * dfs(u=1):
* visited[1] = true
* loop over adj[1]:
* edge(1->2, c=9): result = min(INT_MAX, 9) = 9
* dfs(u=2):
* visited[2] = true
* loop over adj[2]:
* edge(2->1, c=9): result = min(9, 9) = 9 (visited[1] is true)
* edge(2->3, c=6): result = min(9, 6) = 6
* dfs(u=3):
* visited[3] = true
* loop over adj[3]:
* edge(3->2, c=6): result = min(6, 6) = 6 (visited[2] is true)
* edge(2->4, c=5): result = min(6, 5) = 5
* dfs(u=4):
* visited[4] = true
* loop over adj[4]:
* edge(4->2, c=5): result = min(5, 5) = 5 (visited[2] is true)
* edge(4->1, c=7): result = min(5, 7) = 5 (visited[1] is true)
* edge(1->4, c=7): result = min(5, 7) = 5 (visited[4] is true)
* * Final Result: 5
*/
```
