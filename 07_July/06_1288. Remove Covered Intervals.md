# 1288. Remove Covered Intervals

**Problem Link:** [https://leetcode.com/problems/remove-covered-intervals/](https://leetcode.com/problems/remove-covered-intervals/)  
**Difficulty:** Medium  

---

## 📝 Problem Statement

Given an array `intervals` where `intervals[i] = [li, ri]` represent the interval `[li, ri)`, remove all intervals that are covered by another interval in the list.

The interval `[a, b)` is covered by the interval `[c, d)` if and only if `c <= a` and `b <= d`.

Return the number of remaining intervals.

---

## ⚙️ Constraints

- `1 <= intervals.length <= 1000`
- `intervals[i].length == 2`
- `0 <= li < ri <= 10^5`
- All the given intervals are **unique**.

---

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `intervals = [[1,4],[3,6],[2,8]]` | `2` | Interval `[3,6]` is covered by `[2,8]`, therefore it is removed. The remaining intervals are `[1,4]` and `[2,8]`. |
| **2** | `intervals = [[1,4],[2,3]]` | `1` | Interval `[2,3]` is entirely covered by `[1,4]`. |

---

## 🧠 Approach & Explanation

To efficiently find and remove covered intervals, we can visualize them on a number line. A larger interval will naturally engulf a smaller one if it starts before or at the same time and ends after or at the same time.

### Step-by-Step Logic:
1. **Custom Sorting:**
   We need to sort the `intervals` array. 
   - **Primary condition:** Sort the intervals based on their **start points** in **ascending** order.
   - **Secondary condition:** If two intervals have the same start point, sort them by their **end points** in **descending** order. This guarantees that the larger interval comes first and can act as an "umbrella" to cover subsequent smaller intervals sharing the same start.
2. **Checking for Coverage:**
   - **Approach 1 (O(N) Space):** Iterate through the sorted intervals and maintain a `result` array of valid intervals. For each interval, check if its start and end fall completely within the bounds of the last interval in the `result` array. If yes, skip it. If no, push it to the `result` array.
   - **Approach 2 (Simplified Check):** Since the array is already sorted by start points, the start point of any incoming interval is guaranteed to be `>=` the start point of the last added interval. Thus, we only need to check if the `end point` of the last added interval is `>=` the `end point` of the current interval.
   - **Approach 3 (Optimized Space - O(1)):** We don't actually need to store the non-covered intervals in a separate array. We just need to track the `end point` of the last valid interval (let's call it `lastIntervalKaEnd`) and a `count` variable. This optimizes our space complexity from O(N) to O(1).

## 💻 Solution Code

### Approach 1: Using O(N) Space and Sorting

```cpp
// Intuition: Sorting the intervals allows us to easily track the largest active interval. A custom sort ensures that for identical start points, the larger interval comes first.
// Approach: Sort the array (start ascending, end descending). Use a result vector to track non-covered intervals and compare the current interval with the last interval in the result.
// Time Complexity: O(n log n) due to the sorting step.
// Space Complexity: O(n) for the result array in the worst-case scenario.
//Approach-1 (Using O(n) space and sorting)
//T.C : O(nlogn)
//S.C : O(n)
class Solution {
public:
    int removeCoveredIntervals(vector<vector<int>>& intervals) {
        vector<vector<int>> result;

        int n = intervals.size();

        // Custom sort: ascending by start point, descending by end point if starts are equal
        auto lambda = [](vector<int>& vec1, vector<int>& vec2) {
            if(vec1[0] == vec2[0]) {
                return vec1[1] > vec2[1];
            }

            return vec1[0] < vec2[0];
        };

        sort(begin(intervals), end(intervals), lambda);

        // Always add the first interval as there's nothing before to cover it
        result.push_back(intervals[0]);

        for(int i = 1; i < n; i++) {
            // Check if current interval is fully covered by the last interval in result
            if(result.back()[0] <= intervals[i][0] && result.back()[1] >= intervals[i][1]) {
                continue;
            }

            // If not covered, add to result
            result.push_back(intervals[i]);
        }

        return result.size();
    }
};

/*
*
* Dry Run
* Input: intervals = [[1,4],[3,6],[2,8]]
* Sorted: [[1,4], [2,8], [3,6]]
* Initialize result = [[1,4]]
* i = 1, interval = [2,8]: result.back() is [1,4]. 1 <= 2 is true, but 4 >= 8 is false. Not covered. result = [[1,4], [2,8]]
* i = 2, interval = [3,6]: result.back() is [2,8]. 2 <= 3 is true, and 8 >= 6 is true. Covered! (Skip)
* Return result.size() -> 2
*
*/
```

### Approach 2: Using O(N) Space with Simplified Condition

```cpp
// Intuition: Since start points are already sorted in ascending order, the current interval's start will always be >= the last added interval's start. Thus, comparing start points is redundant.
// Approach: Sort exactly like Approach 1. For coverage, simply check if the last interval's end point is greater than or equal to the current interval's end point.
// Time Complexity: O(n log n) due to the sorting step.
// Space Complexity: O(n) for the result array.
//Approach-2 (Using O(n) space and sorting + Simplified condition check)
//T.C : O(nlogn)
//S.C : O(n)
class Solution {
public:
    int removeCoveredIntervals(vector<vector<int>>& intervals) {
        vector<vector<int>> result;

        int n = intervals.size();

        // Custom sort: ascending by start point, descending by end point if starts are equal
        auto lambda = [](vector<int>& vec1, vector<int>& vec2) {
            if(vec1[0] == vec2[0]) {
                return vec1[1] > vec2[1];
            }

            return vec1[0] < vec2[0];
        };

        sort(begin(intervals), end(intervals), lambda);

        result.push_back(intervals[0]);

        for(int i = 1; i < n; i++) {
            // Simplified check: only check the end boundary
            if(result.back()[1] >= intervals[i][1]) { //simplified
                continue;
            }

            result.push_back(intervals[i]);
        }

        return result.size();
    }
};

/*
*
* Dry Run
* Input: intervals = [[1,4],[2,3]]
* Sorted: [[1,4], [2,3]]
* Initialize result = [[1,4]]
* i = 1, interval = [2,3]: result.back()[1] is 4. 4 >= 3 is true. Covered! (Skip)
* Return result.size() -> 1
*
*/
```

### Approach 3: Optimized Space O(1)

```cpp
// Intuition: We don't need to store the entire sequence of non-covered intervals. To determine if an incoming interval is covered, we only need the end point of the last valid interval.
// Approach: Sort the array. Use a variable to track the last valid interval's end point and another to count the valid intervals. Skip covered ones, or update the end point and count if it's not covered.
// Time Complexity: O(n log n) due to the sorting step.
// Space Complexity: O(1) as we only use primitive tracking variables instead of an array.
//Approach-3 (Using constant space and sorting)
//T.C : O(nlogn)
//S.C : O(1)
class Solution {
public:
    int removeCoveredIntervals(vector<vector<int>>& intervals) {

        int n = intervals.size();

        // Custom sort: ascending by start point, descending by end point if starts are equal
        auto lambda = [](vector<int>& vec1, vector<int>& vec2) {
            if(vec1[0] == vec2[0]) {
                return vec1[1] > vec2[1];
            }

            return vec1[0] < vec2[0];
        };

        sort(begin(intervals), end(intervals), lambda);

        // Track the end point of the last valid interval
        int lastIntervalKaEnd = intervals[0][1];
        int count = 1; // At least the first interval will be kept

        for(int i = 1; i < n; i++) {
            // If current interval's end is within the previous valid interval's end, it's covered
            if(lastIntervalKaEnd >= intervals[i][1]) {
                continue;
            }

            // Update the end boundary tracker and increment the valid count
            lastIntervalKaEnd = intervals[i][1];
            count++;
        }

        return count;
    }
};

/*
*
* Dry Run
* Input: intervals = [[1,4],[3,6],[2,8]]
* Sorted: [[1,4], [2,8], [3,6]]
* lastIntervalKaEnd = 4, count = 1
* i = 1, interval = [2,8]: 4 >= 8 is False. Update lastIntervalKaEnd = 8, count = 2
* i = 2, interval = [3,6]: 8 >= 6 is True. Covered! (Skip)
* Return count -> 2
*
*/
```

## 🔍 Key Insights
- Sorting transforms this from an O(N^2) comparison problem into an O(N) sequential scan problem.
- A crucial edge case is when start times are identical. Sorting the end times in descending order ensures the longest interval comes first, naturally covering any shorter intervals that share the same start time.
- The condition to check for coverage simplifies to just checking the `end` times once the arrays are appropriately sorted.
