# 3336. Find the Number of Subsequences With Equal GCD

**Problem Link**: [LeetCode 3336. Find the Number of Subsequences With Equal GCD](https://leetcode.com/problems/find-the-number-of-subsequences-with-equal-gcd)  
**Difficulty**: Hard  
**Tags**: Array, Math, Dynamic Programming, Number Theory  

---

## 📝 Problem Statement

You are given an integer array `nums`.

Your task is to find the number of pairs of non-empty subsequences `(seq1, seq2)` of `nums` that satisfy the following conditions:
1. The subsequences `seq1` and `seq2` are **disjoint**, meaning no index of `nums` is common between them.
2. The **GCD** of the elements of `seq1` is equal to the **GCD** of the elements of `seq2`.

Return the total number of such pairs. Since the answer may be very large, return it modulo $10^9 + 7$.

---

## ⚙️ Constraints

- $1 \le \text{nums.length} \le 200$
- $1 \le \text{nums}[i] \le 200$

---

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `nums = [1,2,3,4]` | `10` | There are 10 subsequence pairs which have a GCD of 1 for both elements. For instance: `([1, 2, 3, 4], [1, 2, 3, 4])` (Note: Subsequences correspond to disjoint indices). |
| **2** | `nums = [10,20,30]` | `2` | Two pairs have GCD of 10. |

## 🧠 Approach & Explanation

The problem asks us to find the number of pairs of disjoint subsequences with identical GCDs. We can map this to a choice-making framework.

For every element `nums[i]` at index `i`, we have exactly **3 choices**:
1. **Skip** the element: It belongs to neither subsequence.
2. **Include in seq1**: Update the running GCD of `seq1`.
3. **Include in seq2**: Update the running GCD of `seq2`.

**State Representation:**
We need to track the current index `i`, the running GCD of the first subsequence (`first`), and the running GCD of the second subsequence (`second`). 
When an element `nums[i]` is added to a sequence with current GCD `x`, the new GCD becomes `gcd(x, nums[i])`. Note that `gcd(0, x) = x`.

**Base Case:**
When we reach the end of the array (`i == nums.size()`), we check if both subsequences are non-empty (i.e., `first != 0` and `second != 0`). If they are non-empty and `first == second`, we return `1` (a valid pair found), else `0`.

**Optimization:**
- **Recursion + Memoization**: The maximum value in `nums` is 200, so the maximum possible GCD is 200. The states are `[200][201][201]`, which easily fits into a 3D array memory structure.
- **Bottom-Up DP**: We can iteratively compute the states starting from the end of the array `n` down to `0`.
- **Space Optimized DP**: Since the states for index `i` only depend on the states of index `i+1`, we can optimize the space from a 3D array to a 2D array by merely keeping track of the `previous` layer.

## 💻 Solution 1: Recursion + Memoization

```cpp
// Intuition: We make a decision at each index to either skip the element, add it to subsequence 1, or add it to subsequence 2. We track the running GCDs for both subsequences.
// Approach: Recursively explore the 3 choices. Use a 3D table t[i][first][second] to memoize overlapping subproblems, reducing exponential complexity to polynomial.
// Time Complexity: O(n * M * M) where M is the maximum element in nums (up to 200). We visit each state at most once.
// Space Complexity: O(n * M * M) for the 3D memoization table and recursion stack.

//Approach-1 (Recursion + Memoization)
//T.C : O(n * M * M), M = max element 
//S.C : O(n * M * M), M = max element
class Solution {
    int MOD = 1e9 + 7;
    int t[201][201][201];
public:
    int solve(vector<int> &nums,int i, int first, int second){
        // Base Case: reached end of the array
        if (i == nums.size()) {
            // Check if both subsequences are non-empty
            bool bothNonEmpty = (first != 0 && second != 0);
            // Check if their GCDs match
            bool gcdsMatch    = (first == second);
            // Return 1 if valid, else 0
            return (bothNonEmpty && gcdsMatch) ? 1 : 0;
        }
        
        // Return precomputed answer if available
        if(t[i][first][second] != -1) 
            return t[i][first][second];
        
        // Skip this index entirely
        int skip = solve(nums, i + 1, first, second);

        // Include this index in seq1
        int take1 = solve(nums, i + 1, __gcd(first, nums[i]), second);

        // Include this index in seq2
        int take2 = solve(nums, i + 1, first, __gcd(second, nums[i]));
        
        // Summing up all the possibilites and caching
        return t[i][first][second] = (0LL + skip + take1 + take2) % MOD;
    }
    
    int subsequencePairCount(vector<int>& nums) {
        // Initialize memo table with -1
        memset(t, -1, sizeof(t));
        // Start from index 0 with both GCDs as 0
        return solve(nums, 0, 0, 0);
    }
};

/*
* Dry Run
* Let nums = [2, 2]
* solve(0, 0, 0)
* -> skip: solve(1, 0, 0)
* -> take1: solve(1, 2, 0)
* -> take2: solve(1, 0, 2)
* From solve(1, 2, 0):
* -> skip: solve(2, 2, 0) -> Returns 0 (seq2 empty)
* -> take1: solve(2, 2, 0) -> Returns 0
* -> take2: solve(2, 2, 2) -> Returns 1 (both non-empty, gcds match)
* Summing choices gracefully branches out keeping track of actual GCD pairs.
*/

```

## 💻 Solution 2: Bottom-Up DP

```cpp
// Intuition: The recursive formulation can be flattened to a bottom-up DP table, avoiding maximum recursion depth and function call overheads.
// Approach: Build a 3D array dp[i][first][second] iterating from i = n-1 down to 0. Populate dp[n] with the base condition first.
// Time Complexity: O(n * M * M) where M is the maximum element in nums. Nested loops guarantee bounded iterations.
// Space Complexity: O(n * M * M) to store the full 3D DP table.

//Approach-2 (Bottom Up)
//T.C : O(n * M * M), M = max element 
//S.C : O(n * M * M), M = max element
class Solution {
    int MOD = 1e9 + 7;
public:
    int subsequencePairCount(vector<int>& nums) {
        int n = nums.size();

        int maxEl = -1;
        // Find max element to bound the GCD values
        for (int x : nums) 
            maxEl = max(maxEl, x);
        
        int dp[n+1][maxEl+1][maxEl+1];

        // Base case: at index n, evaluate all possible gcd pairs
        for (int first = 0; first <= maxEl; first++) {
            for (int second = 0; second <= maxEl; second++) {
                bool bothNonEmpty = (first != 0 && second != 0);
                bool gcdsMatch    = (first == second);
                dp[n][first][second] = (bothNonEmpty && gcdsMatch) ? 1 : 0;
            }
        }

        // Iterate backwards from the end of the array
        for (int i = n - 1; i >= 0; i--) {
            for (int first = maxEl; first >= 0; first--) {
                for (int second = maxEl; second >= 0; second--) {

                    // Skip this index entirely
                    int skip  = dp[i + 1][first][second]; //solve(i+1, first, second);

                    // Include this index in seq1
                    int take1 = dp[i + 1][__gcd(first, nums[i])][second]; 

                    // Include this index in seq2
                    int take2 = dp[i + 1][first][__gcd(second, nums[i])];

                    // Assign accumulated possibilities to current state
                    dp[i][first][second] = (0LL + skip + take1 + take2) % MOD;
                }
            }
        }
 
        // Result is at initial state index 0 and GCDs 0
        return dp[0][0][0]; //return solve(nums, 0, 0, 0);
    }
};

/*
* Dry Run
* Size n, array padded with bounds MaxEl
* Table is initialized backwards
* i = n evaluates logic for actual combinations.
* Then i = n-1 calculates using established n layer combinations matching current elements.
* Reaches i=0 retaining aggregate answers spanning paths mathematically.
*/

```

## 💻 Solution 3: Space Optimized Bottom-Up DP

```cpp
// Intuition: In the 3D table, layer dp[i] only requires data from dp[i+1]. We do not need the rest of the array history, allowing us to drop the i-dimension completely.
// Approach: Utilize two 2D vectors: `prev` representing i+1 and `curr` representing i. Fill `curr` based on `prev`, then swap `prev = move(curr)` avoiding deep copies.
// Time Complexity: O(n * M * M) where M is the maximum element in nums.
// Space Complexity: O(M * M) because we only store two 2D layers of size MxM, drastically reducing memory footprint.

//Approach-3 (Bottom Up - 2D DP, space optimized)
//T.C : O(n * M * M), M = max element
//S.C : O(M * M), M = max element
class Solution {
    int MOD = 1e9 + 7;
public:
    int subsequencePairCount(vector<int>& nums) {
        int n = nums.size();

        int maxEl = -1;
        // Determine the maximum element for defining bounds
        for (int x : nums) 
            maxEl = max(maxEl, x);

        // prev = layer i+1, curr = layer i   (2D instead of 3D)
        vector<vector<int>> prev(maxEl+1, vector<int>(maxEl+1, 0));

        // Base case: initialize validation criteria in the prev state
        for (int first = maxEl; first >= 0; first--) {
            for (int second = maxEl; second >= 0; second--) {
                bool bothNonEmpty = (first != 0 && second != 0);
                bool gcdsMatch    = (first == second);
                prev[first][second] = (bothNonEmpty && gcdsMatch) ? 1 : 0;
            }
        }

        // Compute current states based on previous row dependencies
        for (int i = n - 1; i >= 0; i--) {
            vector<vector<int>> curr(maxEl+1, vector<int>(maxEl+1, 0));
            for (int first = maxEl; first >= 0; first--) {
                for (int second = maxEl; second >= 0; second--) {

                    // Skip this index entirely
                    int skip  = prev[first][second];

                    // Include this index in seq1
                    int take1 = prev[__gcd(first, nums[i])][second];

                    // Include this index in seq2
                    int take2 = prev[first][__gcd(second, nums[i])];

                    // Combine sums and map to current row
                    curr[first][second] = (0LL + skip + take1 + take2) % MOD;
                }
            }
            // Move current into previous efficiently, destroying curr object reference
            prev = move(curr);
        }

        // Return answer accumulated at the initial invocation spot
        return prev[0][0];
    }
};

/*
* Dry Run
* Prev tracks exactly what row i+1 had.
* When i = n-1, it fills Curr using combinations calculated in Prev.
* Move function efficiently transfers memory references (Curr to Prev) rather than cloning values.
* Loops dynamically drop downwards leaving memory complexity incredibly bounded.
*/

```
