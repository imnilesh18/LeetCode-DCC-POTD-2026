## 📝 Problem Statement

Given a 2D `grid` of size `m x n` and an integer `k`. You need to shift the grid `k` times.

In one shift operation:
* Element at `grid[i][j]` moves to `grid[i][j + 1]`.
* Element at `grid[i][n - 1]` moves to `grid[i + 1][0]`.
* Element at `grid[m - 1][n - 1]` moves to `grid[0][0]`.

Return the 2D grid after applying shift operation `k` times.

---

## 🚦 Constraints

* `m == grid.length`
* `n == grid[i].length`
* `1 <= m <= 50`
* `1 <= n <= 50`
* `-1000 <= grid[i][j] <= 1000`
* `0 <= k <= 100`

## 🧠 Approach

<details>
<summary><b>Click to expand the detailed walkthrough</b></summary>

This problem is a 2D variation of the classic "Rotate Array" (LeetCode 189) problem. 

**1. The 1D Rotation Trick:**
If we flatten the 2D grid into a 1D array, shifting elements to the right by `k` places can be done efficiently in $O(N)$ time and $O(1)$ space using the 3-step reversal trick:
* Reverse the entire array (from `0` to `n-1`).
* Reverse the first `k` elements (from `0` to `k-1`).
* Reverse the remaining elements (from `k` to `n-1`).

**2. Applying to a 2D Grid In-Place:**
Instead of actually allocating extra space to create a 1D array, we can *visualize* the 2D grid as a 1D array. 
* Total elements $N = \text{rows} \times \text{cols}$.
* A logical 1D index `i` maps to a 2D grid coordinate using:
  * `row = i / cols`
  * `col = i % cols`

**3. Execution:**
* First, normalize `k` by doing `k = k % N`. If `k == 0`, no shifts are needed, so we simply return the grid.
* Implement a helper function `reverse(i, j)` that takes two 1D indices, maps them to their 2D coordinates, swaps the values, and moves the pointers inward (`i++`, `j--`).
* Apply the 3-step reversal algorithm using the mapped indices.

</details>

## 💻 Solution Code

### Brute Force Solution (Extra Space)

```cpp
// Intuition: Flatten the 2D grid into a 1D array, shift elements, and put them back.
// Approach:
// 1. Create a 1D array of size m*n and copy all grid elements into it.
// 2. Calculate the new position for each element as (i + k) % (m * n).
// 3. Map the elements back into the 2D grid.
// Time Complexity: O(m * n) to traverse and copy the grid.
// Space Complexity: O(m * n) for the intermediate 1D array.

class Solution {
public:
    vector<vector<int>> shiftGrid(vector<vector<int>>& grid, int k) {
        int m = grid.size();
        int n = grid[0].size();
        int total = m * n;
        k = k % total;
        
        if (k == 0) return grid;
        
        vector<int> temp(total);
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                int flatIndex = i * n + j;
                int newIndex = (flatIndex + k) % total;
                temp[newIndex] = grid[i][j];
            }
        }
        
        vector<vector<int>> res(m, vector<int>(n));
        for (int i = 0; i < total; i++) {
            res[i / n][i % n] = temp[i];
        }
        
        return res;
    }
};

/*
* Dry Run
* grid = [[1,2],[3,4]], k = 1
* temp = [1,2,3,4], shifted temp = [4,1,2,3]
* res = [[4,1],[2,3]]
*/
```

### Optimized Solution (In-Place)

```cpp
// Intuition: Shifting a 2D grid is structurally identical to rotating a 1D array. We can perform an in-place rotation using a 3-step reversal by mapping 1D logical indices to 2D coordinates.
// Approach: 
// 1. Calculate total elements n = row * col. Normalize k = k % n.
// 2. Define a lambda function `reverse` that treats the grid like a 1D array using `i / col` for the row and `i % col` for the column.
// 3. Reverse the whole grid (0 to n - 1).
// 4. Reverse the first k elements (0 to k - 1).
// 5. Reverse the remaining n - k elements (k to n - 1).
// Time Complexity: O(m * n) where m is rows and n is cols. We perform at most 2 full passes over the matrix during the reverse operations.
// Space Complexity: O(1) as the swaps modify the grid completely in-place.

class Solution {
public:
    vector<vector<int>> shiftGrid(vector<vector<int>>& grid, int k) {
        int row = grid.size();
        int col = grid[0].size();
        
        // Total number of elements in the grid
        int n = row * col;

        // Normalize k to avoid redundant full-cycle shifts
        k = k % n;
        
        // No shift needed if k is a multiple of total elements
        if(k == 0)
            return grid;
        
        // Lambda to reverse a segment of the grid using 1D to 2D index mapping
        auto reverse = [&](int i, int j) {
            while(i < j) {
                // Map 1D indices 'i' and 'j' to 2D grid coordinates and swap
                swap(grid[i/col][i % col], grid[j / col][j % col]);
                i++;
                j--;
            }
        };

        // Step 1: Reverse the entire grid
        reverse(0, n - 1);
        // Step 2: Reverse the first k elements
        reverse(0, k - 1);
        // Step 3: Reverse the remaining n - k elements
        reverse(k, n - 1);

        return grid;
    }
};

/*
*
* Dry Run
* grid = [[1,2,3],[4,5,6],[7,8,9]], k = 1
* row = 3, col = 3, n = 9
* k = 1 % 9 = 1
* 
* Step 1: reverse(0, 8) 
* Flips entire grid -> [[9,8,7],[6,5,4],[3,2,1]]
* 
* Step 2: reverse(0, 0)
* Flips first k elements (just index 0) -> grid remains [[9,8,7],[6,5,4],[3,2,1]]
* 
* Step 3: reverse(1, 8)
* Flips remaining elements -> [[9,1,2],[3,4,5],[6,7,8]]
* 
* Final Output: [[9,1,2],[3,4,5],[6,7,8]]
*
*/
```

## 📌 Notes
* Always normalize `k` using the modulo operator (`k % total_elements`) because shifting `N` times results in the exact same array. This prevents unnecessary bounds and TLE (Time Limit Exceeded) errors.
* Passing the lambda function by reference `[&]` allows it to easily capture the `grid` and `col` variables from the enclosing scope without global declarations.

## 🚀 Key Insights
* **1D to 2D Mapping:** An element at a hypothetical 1D index `idx` exists in a 2D matrix (with `C` columns) at `row = idx / C` and `col = idx % C`. This eliminates the need for extra $O(N)$ space.
* **The Reverse Trick:** The equation `Reverse(All) -> Reverse(0 to k-1) -> Reverse(k to n-1)` is a mathematically proven optimal strategy to rotate arrays/grids in $O(1)$ space.
