## 📝 Problem Statement

You are given an `m x n` binary matrix `grid` and an integer `health`. You start on the upper-left corner `(0, 0)` and would like to get to the lower-right corner `(m - 1, n - 1)`.

You can move up, down, left, or right from one cell to another adjacent cell as long as your health remains positive.

Cells `(i, j)` with `grid[i][j] == 1` are considered **unsafe** and reduce your health by `1`.

Return `true` *if you can reach the final cell with a health value of 1 or more, and* `false` *otherwise*.

## 🛑 Constraints

- `m == grid.length`
- `n == grid[i].length`
- `1 <= m, n <= 50`
- `2 <= m * n`
- `1 <= health <= m + n`
- `grid[i][j]` is either `0` or `1`.

## 💡 Approach and Explanation

### Core Idea
Instead of finding *a* path that keeps health $\ge 1$ using plain DFS (which would explore all possible paths non-optimally), we can reframe the problem: **Find the path from the source `(0,0)` to the destination `(m-1, n-1)` that consumes the minimum amount of health.** If we can find this *minimum health consumed* (let's call it $x$), we just need to verify if `health - x >= 1`. If it is, we return `true`; otherwise, `false`.

### Why not normal BFS?
In a normal BFS, every move is considered equal (edge weight = 1). Here, moving to a cell with `0` costs `0` health, while moving to a cell with `1` costs `1` health. Because the weights are non-uniform, standard BFS cannot guarantee the shortest path to a node upon first visit.

### 1. Dijkstra's Algorithm (Valid but Slower)
Since we want the minimum cost to reach the destination, Dijkstra's Algorithm is a natural fit. We use a Min-Heap (Priority Queue) to always extend the path with the current minimum accumulated cost.
- **Time Complexity:** $O(M \times N \log(M \times N))$ due to the priority queue.

### 2. 0-1 BFS Algorithm (Optimal)
Because the edge weights are strictly `0` or `1`, we can optimize Dijkstra's using a **Double-Ended Queue (Deque)** instead of a Priority Queue. This is called **0-1 BFS**.
- When transitioning to a neighbor with cost `0`, we add it to the **front** of the deque. (It should be processed immediately as it doesn't increase the cost).
- When transitioning to a neighbor with cost `1`, we add it to the **back** of the deque. (It should be processed later).
- This simple rule organically keeps the deque sorted by distance, mimicking Dijkstra's behavior but eliminating the $O(\log V)$ factor!
- **Time Complexity:** $O(M \times N)$ since each cell is pushed and popped from the deque at most once.

## 💻 Solution Codes

### Solution 1: Dijkstra's Algorithm (C++)

```cpp
// Intuition: The problem requires finding the minimum health deduction to reach the destination. Since grid values (0 or 1) act as edge weights, Dijkstra's algorithm safely finds the shortest path (minimum cost).
// Approach: 
// 1. Initialize a 2D result array with INT_MAX to track the minimum cost to each cell.
// 2. Use a Min-Heap (priority queue) to always process the cell with the smallest accumulated cost.
// 3. From the current cell, explore all 4 directions. If the new cost is smaller than the recorded cost, update it and push to the priority queue.
// 4. Finally, check if the initial health minus the minimum cost to reach the end is at least 1.
// Time Complexity: O(m * n * log(m * n)) - In the worst case, every cell is pushed into the priority queue.
// Space Complexity: O(m * n) - For the result matrix and the priority queue.

class Solution {
    vector<vector<int>> directions = {{0, 1}, {0, -1}, {1, 0}, {-1, 0}};
    typedef pair<int, pair<int,int>> P;

public:
    bool findSafeWalk(vector<vector<int>>& grid, int health) {
        int m = grid.size(), n = grid[0].size();

        // Min-heap to fetch the path with the least cost
        priority_queue<P, vector<P>, greater<P>> pq;

        // Tracks minimum cost to reach each cell
        vector<vector<int>> result(m, vector<int>(n, INT_MAX));

        // Initialize source
        result[0][0] = grid[0][0]; 
        pq.push({result[0][0], {0, 0}});

        while (!pq.empty()) {
            auto [d, cell] = pq.top();
            auto [r, c] = cell;
            pq.pop();

            // Skip if we already found a strictly better path to this cell
            if (d > result[r][c]) continue;

            // Explore 4-directional neighbors
            for (auto &dir : directions) {
                int nr = r + dir[0];
                int nc = c + dir[1];

                // Check bounds
                if (nr < 0 || nr >= m || nc < 0 || nc >= n) continue;

                // Relaxation step
                if (d + grid[nr][nc] < result[nr][nc]) {
                    result[nr][nc] = d + grid[nr][nc];
                    pq.push({d + grid[nr][nc], {nr, nc}});
                }
            }
        }

        // Check if remaining health is strictly positive
        return health - result[m-1][n-1] >= 1;
    }
};

/*
*
* Dry Run
* grid = [[0,1,0], [0,0,0]], health = 2
* 1. start at (0,0), cost = 0. pq = {(0, (0,0))}
* 2. pop (0,0). Neighbors: (0,1) cost 1, (1,0) cost 0
* 3. push (0, (1,0)) and (1, (0,1))
* 4. pop (0, (1,0)). Neighbors: (1,1) cost 0
* 5. push (0, (1,1))
* 6. pop (0, (1,1)). Neighbors: (1,2) cost 0
* 7. push (0, (1,2))
* 8. pop (0, (1,2)). Reached end! Min cost to reach end is 0.
* 9. Check 2 - 0 >= 1 => True.
*
*/
```

### Solution 2: Optimized 0-1 BFS (C++)

```cpp
// Intuition: Since the edge weights (cell values) are strictly 0 or 1, we can optimize Dijkstra's algorithm. Instead of a priority queue, we use a Deque.
// Approach: 
// 1. Maintain a 2D result array initialized to INT_MAX.
// 2. Use a Deque to store coordinates.
// 3. If transitioning to a neighbor costs 0 (grid[nr][nc] == 0), push it to the FRONT of the deque.
// 4. If transitioning costs 1 (grid[nr][nc] == 1), push it to the BACK of the deque.
// 5. This automatically maintains a sorted order of costs without O(log V) overhead.
// Time Complexity: O(m * n) - Each cell is pushed to and popped from the deque at most once.
// Space Complexity: O(m * n) - For the result matrix and the deque.

class Solution {
    vector<vector<int>> directions = {{0, 1}, {0, -1}, {1, 0}, {-1, 0}};

public:
    bool findSafeWalk(vector<vector<int>>& grid, int health) {
        int m = grid.size(), n = grid[0].size();

        // Tracks minimum cost to reach each cell
        vector<vector<int>> result(m, vector<int>(n, INT_MAX));
        deque<pair<int,int>> dq;

        // Initialize source
        result[0][0] = grid[0][0]; 
        dq.push_front({0, 0});

        while (!dq.empty()) {
            auto [r, c] = dq.front();
            dq.pop_front();

            // Explore 4-directional neighbors
            for (auto &dir : directions) {
                int nr = r + dir[0];
                int nc = c + dir[1];

                // Check bounds
                if (nr < 0 || nr >= m || nc < 0 || nc >= n) continue;

                // Relaxation step
                if (result[r][c] + grid[nr][nc] < result[nr][nc]) {
                    result[nr][nc] = result[r][c] + grid[nr][nc];

                    // 0-1 BFS logic: 0-weight edges go to the front, 1-weight edges to the back
                    if (grid[nr][nc] == 0) 
                        dq.push_front({nr, nc});
                    else
                        dq.push_back({nr, nc});
                }
            }
        }

        // Check if remaining health is strictly positive
        return health - result[m-1][n-1] >= 1;
    }
};

/*
*
* Dry Run
* grid = [[1,0], [1,0]], health = 2
* 1. start at (0,0), cost = 1. dq = {(0,0)} (front)
* 2. pop (0,0). Neighbors: (0,1) cost 1+0=1. push_front((0,1)). (1,0) cost 1+1=2. push_back((1,0)).
* dq state: Front -> (0,1), (1,0) <- Back
* 3. pop (0,1). Neighbors: (1,1) cost 1+0=1. push_front((1,1)).
* dq state: Front -> (1,1), (1,0) <- Back
* 4. pop (1,1). End reached with cost 1.
* 5. health (2) - result[1][1] (1) = 1 >= 1 => True.
*
*/
```

## 🧠 Key Insights

- **Framing the logic:** The condition "arrive with at least 1 health" is identical to checking whether `initial_health - minimum_damage_taken >= 1`.
- **0-1 BFS Priority:** 0-1 BFS is a specialized version of Dijkstra's algorithm. Because edge weights only ever increase the current path cost by `0` or `1`, adding `0`-weight edges to the front and `1`-weight edges to the back mathematically guarantees that the queue always expands in non-decreasing order of cost.
- **Performance:** In grid-based shortest path problems where costs are strictly binary (`0` or `1`), 0-1 BFS is practically superior because it circumvents the tree-balancing overhead of Min-Heaps.
