## 📝 Problem Statement

You are given a binary string `s` of length `n`, where:
* `'1'` represents an **active section**.
* `'0'` represents an **inactive section**.

You can perform **at most one trade** to maximize the number of active sections in `s`. In a trade, you:
1. Convert a contiguous block of `'1'`s that is surrounded by `'0'`s to all `'0'`s.
2. Afterward, convert a contiguous block of `'0'`s that is surrounded by `'1'`s to all `'1'`s.

Return the **maximum number of active sections** in `s` after making the optimal trade.

> **Note:** Treat `s` as if it is augmented with a `'1'` at both ends, forming $t = '1' + s + '1'$. The augmented `'1'`s do not contribute to the final count.

---

## 📐 Constraints

* $1 \le n == s.\text{length} \le 10^5$
* `s[i]` is either `'0'` or `'1'`

| **3** | `"1000100"` | `7` | Augmented to `"110001001"`. Convert middle `'1'` to `'0'` $\rightarrow$ `"110000001"` $\rightarrow$ `"111111111"`. Final active count is `7`. |
| **4** | `"01010"` | `4` | Augmented to `"1010101"`. Target middle `'1'`, convert $\rightarrow$ `"1111101"`. Final string without padding is `"11110"`. Active count is `4`. |

## 🧠 Approach & Explanation

### Key Intuition

When we perform a trade on a block of `'1'`s surrounded by `'0'`s:
1. Turning the middle block of `'1'`s into `'0'`s connects the block of `'0'`s on its left with the block of `'0'`s on its right.
2. Next, converting this newly merged, contiguous block of `'0'`s into `'1'`s turns **both** adjacent zero-blocks into `'1'`s.
3. The original block of `'1'`s that was temporarily converted to `'0'`s becomes `'1'`s again—meaning its count does not change.
4. Therefore, the net gain in `'1'`s from performing a trade on a block of `'1'`s is equal to:
   $$\text{Gain} = (\text{Length of Left Zero Block}) + (\text{Length of Right Zero Block})$$

### Step-by-Step Algorithm

1. **Count Existing Active Sections:** Count the total number of `'1'`s already present in string `s` (`activeCount`).
2. **Collect Inactive Block Sizes:** Group all continuous segments of `'0'`s and record their lengths into an array `inactiveBlocks`.
3. **Find Maximum Pair Sum:** Since targeting a block of `'1'`s allows us to merge two adjacent zero-blocks, find the maximum sum of any two adjacent elements in `inactiveBlocks`:
   $$\text{maxPairSum} = \max(\text{inactiveBlocks}[i] + \text{inactiveBlocks}[i-1])$$
4. **Calculate Final Result:** The maximum achievable active sections will be:
   $$\text{Result} = \text{activeCount} + \text{maxPairSum}$$

## 💻 Solution Code

### Brute-Force Solution (Simulation)

```cpp
// Intuition: Simulate all possible single trade operations on every valid block of 1s and check the resulting active count.
// Approach: Iterate through the string, find valid '1' blocks surrounded by '0's, simulate both steps of trade, and track max active '1's.
// Time Complexity: O(N^2) - Simulation takes linear time for each candidate block of '1's.
// Space Complexity: O(N) - Storing modified strings during simulation.

class SolutionBruteForce {
public:
    int maxActiveSectionsAfterTrade(string s) {
        int n = s.length();
        int maxActive = count(s.begin(), s.end(), '1');
        
        // Try selecting each block of 1s surrounded by 0s
        for (int i = 0; i < n; i++) {
            if (s[i] == '1') {
                int start = i;
                while (i < n && s[i] == '1') i++;
                int end = i - 1;
                
                // Check if surrounded by 0s (considering virtual padding at ends)
                bool leftZero = (start == 0) || (s[start - 1] == '0');
                bool rightZero = (end == n - 1) || (s[end + 1] == '0');
                
                if (leftZero && rightZero) {
                    string temp = s;
                    // Step 1: Convert 1s to 0s
                    for (int j = start; j <= end; j++) temp[j] = '0';
                    
                    // Step 2: Convert combined 0s block to 1s
                    int l = start - 1;
                    while (l >= 0 && temp[l] == '0') l--;
                    int r = end + 1;
                    while (r < n && temp[r] == '0') r++;
                    
                    for (int j = l + 1; j < r; j++) temp[j] = '1';
                    
                    int currentActive = count(temp.begin(), temp.end(), '1');
                    maxActive = max(maxActive, currentActive);
                }
            }
        }
        return maxActive;
    }
};

/*
*
* Dry Run
*
* Input: s = "0100"
* Initial active count = 1
* Candidate 1s block: at index 1 ("1")
* Surrounded by '0' at index 0 and '0' at index 2 -> Valid trade!
* Step 1: "0100" -> "0000"
* Step 2: "0000" surrounded by padded 1s -> "1111"
* Resulting active count = 4
*
* Output: 4
*/
```

### Optimized Solution (Zero Block Aggregation & Pair Sum)

```cpp
// Intuition: Flipping a block of 1s between two zero-blocks merges them into a single zero-block, converting both zero-blocks into 1s.
// Approach: Count existing 1s, store continuous lengths of 0s in an array, and find the maximum sum of two adjacent zero-block lengths.
// Time Complexity: O(N) - Linear scan to count 1s, collect zero blocks, and compute max adjacent pair sum.
// Space Complexity: O(N) - Vector to store lengths of inactive (0s) blocks.

class Solution {
public:
    int maxActiveSectionsAfterTrade(string s) {
        int n = s.length();

        // existing count of 1s
        int activeCount = count(begin(s), end(s), '1');

        vector<int> inactiveBlocks;
        int i = 0;
        while(i < n) {
            if(s[i] == '0') {
                int start = i;
                while(i < n && s[i] == '0') i++;

                inactiveBlocks.push_back(i-start); // Store continuous length of 0s
            } else {
                i++; // Skip '1's
            }
        }

        int maxPairSum = 0;
        // max(inactiveBlocks[i] + inactiveBlocks[i-1])
        for(int i = 1; i < inactiveBlocks.size(); i++) {
            maxPairSum = max(maxPairSum, inactiveBlocks[i] + inactiveBlocks[i-1]); // Find max sum of two adjacent zero blocks
        }

        return maxPairSum + activeCount; // Max active sections = original 1s + max 0s converted
    }
};

/*
*
* Dry Run
*
* Input: s = "1000100"
* 
* Initial Setup:
* - activeCount = count of '1's in "1000100" = 2
* 
* Iteration 1: Collecting zero block lengths
* - Index 0: '1' -> skip
* - Indices 1..3: "000" -> length = 3, inactiveBlocks = [3]
* - Index 4: '1' -> skip
* - Indices 5..6: "00"  -> length = 2, inactiveBlocks = [3, 2]
* 
* Iteration 2: Finding max adjacent pair sum
* - i = 1: inactiveBlocks[1] + inactiveBlocks[0] = 2 + 3 = 5
* - maxPairSum = 5
* 
* Final Calculation:
* - Return maxPairSum + activeCount = 5 + 2 = 7
* 
* Output: 7
*/
```

## 🔑 Key Insights

1. **Virtual Padding Simplification:** Treat string boundaries as padded `'1'`s. This guarantees that any boundary block of `'0'`s is automatically surrounded by a `'1'` at the end.
2. **Trade Invariant:** The trade operation does not alter the net number of `'1'`s in the target `'1'` block itself; it purely acts as a converter for the two adjacent `'0'` blocks on its sides.
3. **Problem Reduction:** A two-step string transformation simplifies directly into finding the maximum sum of adjacent elements in an array of zero-block lengths!

---

## 📝 Notes

* If there are fewer than 2 zero-blocks (`inactiveBlocks.size() < 2`), `maxPairSum` remains `0`, and the answer is simply the count of existing `'1'`s.
* The algorithm runs in a single pass $O(N)$ with optimal efficiency, well within the $10^5$ constraint limit.
