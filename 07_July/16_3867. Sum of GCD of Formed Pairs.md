## 📝 Problem Statement

You are given an integer array `nums` of length `n`.

First, construct an array `prefixGcd` where for each index `i`:
- Let `mxi` be the maximum element in the subarray `nums[0...i]`.
- `prefixGcd[i] = gcd(nums[i], mxi)`.

After constructing `prefixGcd`:
1. Sort `prefixGcd` in non-decreasing order.
2. Form pairs by taking the smallest unpaired element and the largest unpaired element.
3. Repeat this process until no more pairs can be formed.
4. For each formed pair, compute the greatest common divisor (GCD) of the two elements.
5. If `n` is odd, the middle element in the `prefixGcd` array remains unpaired and should be ignored.

**Return an integer** denoting the sum of the GCD values of all formed pairs.

**Input:** `nums = [2, 6, 4]`
**Output:** `2`

- `prefixGcd = [2, 6, 2]`. After sorting, it forms `[2, 2, 6]`.
- Pair the smallest and largest elements: `gcd(2, 6) = 2`. 
- The remaining middle element `2` is ignored. Thus, the sum is `2`.

## 🚫 Constraints

- `1 <= n == nums.length <= 10^5`
- `1 <= nums[i] <= 10^9`

---

## 💡 Approach & Intuition

The problem strictly lays out the steps we need to follow, making this an algorithmic simulation. 

1. **Track the Maximum:** As we iterate through the array, keep a running maximum `maxEl` which corresponds to `mxi` at any index `i`.
2. **Compute Prefix GCDs:** For each element, calculate the greatest common divisor between the current element and the running maximum. Push this to a new array `prefixGcd`.
3. **Sort:** Sort the newly created `prefixGcd` array as required by the problem.
4. **Two Pointers Pairing:** Use a standard two-pointer approach (`i` starting at 0, `j` starting at the end) to pick the smallest and largest elements. Add their GCD to the cumulative total.
5. **Handle Parity:** The `while (i < j)` loop naturally pairs elements correctly and natively ignores the exact middle element if the length of the array is odd.

## 💻 Solution Code

```cpp
// Intuition: The problem describes a strict sequence of operations. A direct simulation using prefix maximum tracking, GCD computation, and a two-pointer pairing approach naturally satisfies all conditions.
// Approach: Iterate through the array to find the running maximum and compute the GCD of the current element and this maximum. Store these in a new array, sort it, and use two pointers to pair the smallest and largest remaining elements. Accumulate their GCD values.
// Time Complexity: O(n log n + n log M), where n is the size of the array (for sorting) and M is the maximum element (for computing GCDs).
// Space Complexity: O(n) to store the computed prefixGcd array.

class Solution {
public:
    long long gcdSum(vector<int>& nums) {
        int n = nums.size();
        
        // Array to store the prefix GCDs
        vector<int> prefixGcd;
        
        // Track the maximum element seen so far
        int maxEl = -1;
        
        // Step 1: Construct the prefixGcd array
        for(int i = 0; i < n; i++) {
            maxEl = max(maxEl, nums[i]); // Update running max
            prefixGcd.push_back(__gcd(nums[i], maxEl)); // Compute and store GCD
        }

        // Step 2: Sort the array in non-decreasing order
        sort(begin(prefixGcd), end(prefixGcd));

        long long result = 0;
        int i = 0;         // Pointer to the smallest element
        int j = n - 1;     // Pointer to the largest element

        // Step 3: Form pairs and calculate sum of their GCDs
        while(i < j) {
            result += __gcd(prefixGcd[i], prefixGcd[j]);
            i++;
            j--;
        }

        return result;
    }
};

/*
*
* Dry Run
*
* Input: nums = [3, 6, 2, 8]
* n = 4
* * Building prefixGcd:
* i = 0: nums[0] = 3, maxEl = max(-1, 3) = 3. gcd(3, 3) = 3. prefixGcd = [3]
* i = 1: nums[1] = 6, maxEl = max(3, 6) = 6. gcd(6, 6) = 6. prefixGcd = [3, 6]
* i = 2: nums[2] = 2, maxEl = max(6, 2) = 6. gcd(2, 6) = 2. prefixGcd = [3, 6, 2]
* i = 3: nums[3] = 8, maxEl = max(6, 8) = 8. gcd(8, 8) = 8. prefixGcd = [3, 6, 2, 8]
* * Sorting prefixGcd:
* prefixGcd = [2, 3, 6, 8]
* * Two Pointers Pairing:
* result = 0, i = 0, j = 3
* 1st iteration: i=0(val=2), j=3(val=8). gcd(2, 8) = 2. result = 0 + 2 = 2. i=1, j=2.
* 2nd iteration: i=1(val=3), j=2(val=6). gcd(3, 6) = 3. result = 2 + 3 = 5. i=2, j=1.
* * Loop terminates since i (2) is not less than j (1).
* Output: 5
*
*/
```

## 📌 Notes & Key Insights
- The `__gcd()` function in C++ is highly optimized and works in logarithmic time relative to the value of the integers.
- Sorting the array scales gracefully with constraints up to `N = 10^5`, taking roughy `~1.6 * 10^6` operations which easily passes the 1-second execution window.
- Using a standard `while(i < j)` loop prevents pointing to the same index. This correctly handles an odd-sized array by stopping before `i` equals `j`, inherently "ignoring" the middle element as instructed.

- Try using custom Euclidean algorithms if standard library implementations are restricted.
- Look into parallel or multithreaded sorting approaches if arrays scale exponentially larger.
