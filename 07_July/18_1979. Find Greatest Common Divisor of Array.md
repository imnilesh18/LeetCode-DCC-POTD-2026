## 📝 Problem Statement

Given an integer array `nums`, return the greatest common divisor (GCD) of the smallest number and largest number in `nums`.

The **greatest common divisor** of two numbers is the largest positive integer that evenly divides both numbers.

---

## 🛑 Constraints

*   `2 <= nums.length <= 1000`
*   `1 <= nums[i] <= 1000`

---

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `nums = [2,5,6,9,10]` | `2` | The smallest number is 2 and the largest is 10. The GCD of 2 and 10 is 2. |
| **2** | `nums = [7,5,6,8,3]` | `1` | The smallest number is 3 and the largest is 8. The GCD of 3 and 8 is 1. |
| **3** | `nums = [3,3]` | `3` | The smallest number is 3 and the largest is 3. The GCD of 3 and 3 is 3. |

---

## 🧠 Approach

The problem boils down to two independent tasks:
1.  **Find the Extremes:** Traverse the array to identify the minimum (`smallest`) and maximum (`largest`) values.
2.  **Calculate the GCD:** Use the Euclidean algorithm to compute the Greatest Common Divisor of these two values.

This can be done using a manual linear scan or by leveraging standard library functions. Both approaches share the same time and space complexity, but utilizing built-in functions leads to cleaner and more readable code.

---

## 💻 Solution Code

### Brute Force / Manual Iteration Approach

```cpp
// Intuition: We need the GCD of the smallest and largest numbers. We can find these extremes by scanning the array once.
// Approach: 
// 1. Initialize `largest` and `smallest` with the first element of the array.
// 2. Iterate through the array, updating `largest` and `smallest` as we find bigger or smaller elements respectively.
// 3. Return the result using the built-in `__gcd` function on the two found values.
// Time Complexity: O(N) where N is the number of elements in the array to find min and max, plus O(log(min(A, B))) for calculating the GCD. Overall Time Complexity is O(N).
// Space Complexity: O(1) since we are only using a few integer variables.

class Solution {
public:
    int findGCD(vector<int>& nums) {
        // Initialize extremes with the first element
        int largest = nums[0];
        int smallest = nums[0];
        
        // Scan array to find the actual min and max
        for(int i = 0; i < nums.size(); i++) {
            if(nums[i] > largest) {
                largest = nums[i];
            }
            if(nums[i] < smallest) {
                smallest = nums[i];
            }
        }
        
        // Compute and return their GCD
        return __gcd(smallest, largest);
    }
};

/*
*
* Dry Run
* Input: nums = [7, 5, 6, 8, 3]
* 
* Initialize:
* largest = 7, smallest = 7
* 
* Iteration:
* i = 0: nums[0] = 7. largest = 7, smallest = 7
* i = 1: nums[1] = 5. largest = 7, smallest = 5
* i = 2: nums[2] = 6. largest = 7, smallest = 5
* i = 3: nums[3] = 8. largest = 8, smallest = 5
* i = 4: nums[4] = 3. largest = 8, smallest = 3
* 
* GCD Calculation:
* __gcd(3, 8) -> 1
* 
* Output: 1
*
*/
```

### Optimized Approach (Standard Library)

```cpp
// Intuition: We can achieve the exact same logic using C++ standard library functions to make the code highly concise and readable.
// Approach:
// 1. Use `*max_element` to find the maximum value in the array.
// 2. Use `*min_element` to find the minimum value in the array.
// 3. Use `gcd` to calculate the answer and return it.
// Time Complexity: O(N) because `max_element` and `min_element` both take linear time. The GCD calculation takes O(log(min(A, B))). Overall O(N).
// Space Complexity: O(1) as no extra memory is allocated.

class Solution {
public:
    int findGCD(vector<int>& nums) {
        // Find the maximum element using STL
        int mx = *max_element(nums.begin(), nums.end());
        // Find the minimum element using STL
        int mn = *min_element(nums.begin(), nums.end());
        
        // Return the GCD of the two extremes
        return gcd(mx, mn);
    }
};

/*
*
* Dry Run
* Input: nums = [2, 5, 6, 9, 10]
* 
* Step 1: mx = *max_element(...) -> finds 10
* Step 2: mn = *min_element(...) -> finds 2
* Step 3: gcd(10, 2) -> calculates the GCD
*   10 % 2 == 0
*   GCD is 2
* 
* Output: 2
*
*/
```

## 📌 Notes & Key Insights

*   **Euclidean Algorithm:** The `gcd(a, b)` function operates on the principle that the GCD of two numbers also divides their difference. It efficiently calculates this using modulo operations: `gcd(a, b) = gcd(b, a % b)` until `b` becomes 0.
*   **C++ Libraries:** In modern C++ (C++17 and above), `std::gcd` is available in the `<numeric>` header. Older codebases might use `std::__gcd` from the `<algorithm>` header.
