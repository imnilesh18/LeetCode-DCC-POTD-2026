# 1291. Sequential Digits

**Problem Link:** [Sequential Digits](https://leetcode.com/problems/sequential-digits/description/)  
**Difficulty:** Medium  
**Tags:** `Array`, `Enumeration`, `Breadth-First Search`

---

## 📝 Problem Statement

An integer has **sequential digits** if and only if each digit in the number is one more than the previous digit.

Return a sorted list of all the integers in the range `[low, high]` inclusive that have sequential digits.

---

## ⚠️ Constraints

- `10 <= low <= high <= 10^9`

---

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **Example 1** | `low = 100`, `high = 300` | `[123, 234]` | Only 123 and 234 fall within the bounds of 100 and 300 while having sequential digits. |
| **Example 2** | `low = 1000`, `high = 13000` | `[1234, 2345, 3456, 4567, 5678, 6789, 12345]` | All these generated numbers possess sequentially increasing digits and fall exactly within the given boundary. |

---

## 🧠 Approach & Explanation

To tackle this problem, we can utilize two distinct approaches:

### 1. Breadth-First Search (BFS) Technique
- Since we know that a sequential number must start with a digit between 1 and 8 (it cannot start with 9 because there is no digit greater than 9), we can inject these starting digits into a Queue.
- By popping an element from the queue, we can check if it lies within our `[low, high]` boundaries. If it does, we append it to our result list.
- Next, we extract the last digit of the popped element. If the last digit is less than 9, we generate the next sequential number by appending `last_digit + 1` to it, and push this newly generated number back into the queue.
- This process organically creates numbers in an increasing order, ensuring that our final result is perfectly sorted.

### 2. Precomputation (O(1) Workaround) Technique
- The constraint specifies `high <= 10^9`. Within this entire universe of integers, the count of numbers that consist exclusively of sequential digits is remarkably small (exactly 36 numbers).
- Because this dataset is entirely fixed and extremely limited, we can hardcode all 36 possibilities into a pre-sorted array.
- We then iterate through this array. We skip numbers smaller than `low`, capture numbers within `[low, high]`, and instantly break the loop the moment we encounter a number strictly greater than `high`.

## 💻 Solution 1: Breadth-First Search (BFS)

```cpp
// Intuition: We can generate sequential digits using a Breadth-First Search (BFS) approach. Starting with single digits 1 to 8, we can continually append the next sequential digit until we exceed the upper bound.
// Approach: 
// 1. Initialize a queue and push the numbers 1 through 8 into it.
// 2. Loop until the queue is empty:
//    a. Pop the front element.
//    b. If it falls within the range [low, high], add it to the result vector.
//    c. Extract the last digit of the popped number (temp % 10).
//    d. If the last digit is less than 9, append (last_digit + 1) to the number and push the new number into the queue.
// 3. Return the result vector.
// Time Complexity: O(N), where N is the number of valid sequential digits (maximum 36). Essentially O(1) constant time.
// Space Complexity: O(N), for the queue storing the sequential digits (maximum 36). Essentially O(1) constant space.

class Solution {
public:
    vector<int> sequentialDigits(int low, int high) {
        queue<int> que;
        
        // Push initial single digits (1 to 8) into the queue
        for(int i = 1; i <= 8; i++) {
            que.push(i);
        }
        
        vector<int> result;
        
        // Process queue elements
        while(!que.empty()) {
            int temp = que.front();
            que.pop();
            
            // Add to result if it satisfies the range condition
            if(temp >= low && temp <= high) {
                result.push_back(temp);
            }
            
            int last_digit = temp % 10;
            
            // Generate the next sequential number and push to queue
            if(last_digit + 1 <= 9) {
                que.push(temp * 10 + (last_digit + 1));
            }
        }
        
        return result;
    }
};

/*
*
* Dry Run
* Input: low = 100, high = 300
* Initial Queue: [1, 2, 3, 4, 5, 6, 7, 8]
* 1st iteration: Pop 1. Not in range. Last digit is 1. Push 1 * 10 + 2 = 12.
* Queue becomes: [2, 3, 4, 5, 6, 7, 8, 12]
* ...
* Pop 12: Not in range. Last digit is 2. Push 123.
* Queue eventually holds 123. 
* Pop 123: In range [100, 300]. Add 123 to result. Push 1234.
* Pop 234: In range [100, 300]. Add 234 to result. Push 2345.
* Numbers > 300 will not be added to the result.
* Result: [123, 234]
*
*/
```

## 🚀 Solution 2: Precomputation (O(1) Optimized)

```cpp
// Intuition: Since the maximum possible number is 10^9, there is a very strictly limited number of sequential digits possible (exactly 36). We can hardcode all of them.
// Approach: 
// 1. Create a precomputed list of all 36 sequential digit numbers.
// 2. Iterate through this list.
// 3. If the current number is less than 'low', skip it.
// 4. If the current number is greater than 'high', break the loop since the array is sorted.
// 5. If it is within the bounds, push it to the result vector.
// Time Complexity: O(1) - Constant time since the array has a fixed size of 36.
// Space Complexity: O(1) - Constant space for the fixed precomputed array.

class Solution {
public:
    vector<int> sequentialDigits(int low, int high) {
        // Precomputed array of all sequential numbers up to 10^9
        vector<int> allPossible = {12, 23, 34, 45, 56, 67, 78, 89,
                                  123, 234, 345, 456, 567, 678, 789,
                                  1234, 2345, 3456, 4567, 5678, 6789,
                                  12345, 23456, 34567, 45678, 56789,
                                  123456, 234567, 345678, 456789,
                                  1234567, 2345678, 3456789,
                                  12345678, 23456789,
                                  123456789};
        
        vector<int> result;
        int n = allPossible.size();
        
        // Iterate and filter within bounds
        for (int i = 0; i < n; i++) {
            if (allPossible[i] < low) continue;
            
            if (allPossible[i] > high) break;
            
            result.push_back(allPossible[i]);
        }
        return result;
    }
};

/*
*
* Dry Run
* Input: low = 1000, high = 13000
* Iteration over allPossible:
* Numbers 12 to 789: Less than 1000, 'continue' triggers.
* Number 1234: Within [1000, 13000], add to result.
* Number 2345: Within range, add to result.
* ...
* Number 12345: Within range, add to result.
* Number 23456: Greater than 13000, 'break' triggers.
* Result: [1234, 2345, 3456, 4567, 5678, 6789, 12345]
*
*/
```

## 🔎 Key Insights

1. **BFS Generates Sorted Answers:** The BFS approach inherently builds the answers in a strictly increasing order because we evaluate the lowest initial digits (1, 2, 3...) first and continuously increment their length sequentially.
2. **Constraint Exploitation:** Noting that the maximum bound is `10^9` is the turning point that reveals the "Precomputation" trick. Whenever the sample set size is universally minuscule (like 36 states), O(1) hardcoding becomes highly viable.
3. **Queue Behavior in BFS:** Pushing numbers digit by digit ensures we don't accidentally skip a valid sequence.
