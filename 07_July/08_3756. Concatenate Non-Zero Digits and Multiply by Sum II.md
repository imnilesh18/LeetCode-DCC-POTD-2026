# 3756. Concatenate Non-Zero Digits and Multiply by Sum II

**Difficulty:** Medium  
**Topics:** `Math`, `String`, `Prefix Sum`  
**Problem Link:** [LeetCode - 3756. Concatenate Non-Zero Digits and Multiply by Sum II](https://leetcode.com/problems/concatenate-non-zero-digits-and-multiply-by-sum-ii/description/)

---

## 📝 Problem Statement

You are given a string `s` of length `m` consisting of digits. You are also given a 2D integer array `queries`, where `queries[i] = [li, ri]`.

For each `queries[i]`, extract the substring `s[li..ri]`. Then, perform the following:
1. Form a new integer `x` by concatenating all the **non-zero digits** from the substring in their original order. If there are no non-zero digits, `x = 0`.
2. Let `sum` be the sum of digits in `x`. 
3. The answer is `x * sum`.

Return an array of integers `answer` where `answer[i]` is the answer to the `ith` query. Since the answers may be very large, return them **modulo 10^9 + 7**.

---

## ⛔ Constraints

- `1 <= m == s.length <= 10^5`
- `s` consists of digits only.
- `1 <= queries.length <= 10^5`
- `queries[i] = [li, ri]`
- `0 <= li <= ri < m`

## 🧠 Explanation & Approach



### Why Brute Force Fails
If we extract the substring for each query and compute `x` and `sum` manually, extracting the substring takes $O(M)$ time in the worst case. For $Q$ queries, the total time complexity would be $O(M \times Q)$. Given $M, Q \le 10^5$, the operations can go up to $10^{10}$, which will definitely result in a **Time Limit Exceeded (TLE)**.

### Optimized Approach (Prefix Computations)
Since the string `s` is completely static, we can precompute the required information so that each query is answered in $O(1)$ time! We need two things for any range `[L, R]`: the **sum** of the digits, and the exact concatenated **number** `x`.

1. **Sum of Digits:** We can maintain a prefix sum array of digits. The sum of digits in the range `[L, R]` is simply `prefixSum[R] - prefixSum[L-1]`.
2. **Concatenated Number (`x`):** We maintain a prefix array `numberUpTo` where `numberUpTo[i]` represents the value formed by all non-zero digits from index `0` to `i`. 
   To extract the number formed *strictly* within the range `[L, R]`, we take the entire number formed up to `R`, and we "chop off" the number formed up to `L-1`. 
   *How do we mathematically chop it off?* If $K$ is the number of non-zero digits in our target range `[L, R]`, we subtract `numberUpTo[L-1] * 10^K` from `numberUpTo[R]`.
3. **Non-Zero Count:** To quickly find $K$ (the number of non-zero digits in `[L, R]`), we use another prefix array `nonZeroCount`. $K = \text{nonZeroCount}[R] - \text{nonZeroCount}[L-1]$.
4. **Powers of 10:** To avoid calculating $10^K \pmod{10^9+7}$ for every query, we precompute all possible powers of 10 in an array.

## 💻 Solution Code

```cpp
// Intuition: A brute-force approach iterating over the substring for each query would take O(M * Q) and yield TLE. Since the string doesn't change, precomputing prefix data allows us to extract sum and concatenated number for any range in O(1) time.
// Approach: 
// 1. Precompute `pow10` to get powers of 10 modulo 1e9+7 in O(1).
// 2. Precompute `nonZeroCount` to quickly find the number of non-zero digits in any range [L, R].
// 3. Precompute `numberUpTo` which evaluates the number formed by non-zero digits from index 0 to i.
// 4. Precompute `digitSumUpTo` for O(1) sum extraction.
// 5. For every query [L, R], evaluate `x = numberUpTo[R] - numberUpTo[L-1] * 10^subStrLen` and calculate `sum = digitSumUpTo[R] - digitSumUpTo[L-1]`.
// Time Complexity: O(M + Q), where M is the length of string `s` (for precomputations) and Q is the number of queries. Each query is answered in O(1).
// Space Complexity: O(M), for storing the prefix arrays (nonZeroCount, numberUpTo, digitSumUpTo, pow10).

class Solution {
public:
    int MOD = 1e9+7;
    typedef long long ll;

    vector<int> sumAndMultiply(string s, vector<vector<int>>& queries) {
        int n = s.length();

        // Arrays for O(1) range extractions
        vector<int> nonZeroCount(n, 0);    
        vector<ll> numberUpTo(n, 0);       
        vector<ll> digitSumUpTo(n, 0);     
        vector<ll> pow10(n + 1, 0);  

        // Precompute powers of 10
        pow10[0] = 1;
        for (int i = 1; i <= n; i++)
            pow10[i] = (pow10[i - 1] * 10) % MOD;

        // Initialize base cases for 0th index
        nonZeroCount[0]      = (s[0] != '0') ? 1 : 0;
        numberUpTo[0]   = s[0] - '0';
        digitSumUpTo[0] = s[0] - '0';

        // Precompute nonZeroCount prefix array
        for (int i = 1; i < n; i++) {
            int digit = s[i] - '0';
            nonZeroCount[i] = nonZeroCount[i - 1] + (digit != 0 ? 1 : 0);
        }

        // Precompute numberUpTo prefix array
        for (int i = 1; i < n; i++) {
            int digit = s[i] - '0';
            if (digit != 0)
                numberUpTo[i] = (numberUpTo[i - 1] * 10 + digit) % MOD;
            else
                numberUpTo[i] = numberUpTo[i - 1]; // carry forward if zero
        }

        // Precompute digitSumUpTo prefix array
        for (int i = 1; i < n; i++) {
            digitSumUpTo[i] = digitSumUpTo[i - 1] + (s[i] - '0');
        }

        int q = queries.size();
        vector<int> result(q);

        // Answer each query in O(1)
        for (int i = 0; i < q; i++) {
            int l  = queries[i][0];
            int r = queries[i][1];

            // Prevent out-of-bounds for prefix subtraction
            int startCount = (l == 0) ? 0 : nonZeroCount[l - 1];
            ll numBefore   = (l == 0) ? 0 : numberUpTo[l - 1];

            int endCount = nonZeroCount[r];
            int subStrLen = endCount - startCount;

            // If no non-zero digits exist in the range, result is 0
            if (subStrLen == 0) {
                result[i] = 0;
                continue;
            }

            // Mathematically chop off the prefix before `L`
            ll x   = (numberUpTo[r] - (numBefore * pow10[subStrLen] % MOD) + MOD) % MOD;
            ll sum = digitSumUpTo[r] - ((l == 0) ? 0 : digitSumUpTo[l - 1]);

            // Final answer is (x * sum) modulo 1e9+7
            result[i] = (int)((x * sum) % MOD);
        }

        return result;
    }
};

/*
*
* Dry Run
*
* String: "1020", Queries: [[0,3], [1,3]]
*
* Precomputations:
* pow10 = [1, 10, 100, 1000, 10000]
* nonZeroCount = [1, 1, 2, 2]
* numberUpTo = [1, 1, 12, 12]
* digitSumUpTo = [1, 1, 3, 3]
*
* Query 1 [0, 3]:
* startCount = 0, numBefore = 0
* endCount = nonZeroCount[3] = 2
* subStrLen = 2 - 0 = 2
* x = (12 - 0 * 100) % MOD = 12
* sum = digitSumUpTo[3] - 0 = 3
* Answer: 12 * 3 = 36
*
* Query 2 [1, 3]:
* startCount = nonZeroCount[0] = 1, numBefore = numberUpTo[0] = 1
* endCount = nonZeroCount[3] = 2
* subStrLen = 2 - 1 = 1
* x = (12 - 1 * pow10[1]) % MOD = (12 - 10) = 2
* sum = digitSumUpTo[3] - digitSumUpTo[0] = 3 - 1 = 2
* Answer: 2 * 2 = 4
*
*/
```
