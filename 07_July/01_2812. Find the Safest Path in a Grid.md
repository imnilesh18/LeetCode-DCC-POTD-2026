## 📝 Problem Statement

You are given a **0-indexed** 2D matrix `grid` of size `n x n`, where `(r, c)` represents:
*   A cell containing a **thief** if `grid[r][c] = 1`.
*   An **empty** cell if `grid[r][c] = 0`.

You are initially positioned at cell `(0, 0)`. In one move, you can move to any adjacent cell in the grid, including cells containing thieves.

The **safeness factor** of a path on the grid is defined as the **minimum** Manhattan distance from any cell in the path to any thief in the grid.

Return the **maximum safeness factor** of all paths leading to cell `(n - 1, n - 1)`.

*Note:*
*   An adjacent cell of cell `(r, c)`, is one of the cells `(r, c + 1)`, `(r, c - 1)`, `(r + 1, c)` and `(r - 1, c)` if it exists.
*   The Manhattan distance between two cells `(a, b)` and `(x, y)` is equal to `|a - x| + |b - y|`, where `|val|` denotes the absolute value of `val`.

---

## 🚦 Constraints

*   `1 <= grid.length == n <= 400`
*   `grid[i].length == n`
*   `grid[i][j]` is either `0` or `1`.
*   There is at least one thief in the `grid`.

## 🛠️ Approach

1. **Precalculate Distances using Multi-Source BFS**:
   A standard BFS from each cell to find the nearest thief would take $O(N^2 \times N^2) = O(N^4)$, resulting in a Time Limit Exceeded (TLE) error. Instead, we insert all thief coordinates into a queue and initiate a **Multi-Source BFS**. This simultaneously computes the shortest Manhattan distance from every cell in the grid to its nearest thief in just $O(N^2)$ time.
2. **Binary Search on the Safeness Factor**:
   The problem asks us to find the "maximum minimum" distance along a path. This is a classic indicator for **Binary Search on Answer**. 
   * The minimum possible safeness factor is `0` (if we step directly on a thief).
   * The maximum possible safeness factor is the maximum dimension of the grid (or `400` based on constraints).
3. **Validation Check**:
   For a chosen safeness factor `mid`, we run a standard BFS starting from `(0, 0)` to `(n-1, n-1)`. We are *only* allowed to step on cells where the precalculated distance to the nearest thief is $\ge$ `mid`. If we reach the target, `mid` is a valid safeness factor. We adjust our binary search bounds to maximize this `mid`.

### Optimized Solution (C++)

```cpp
// Intuition: We need to maximize the minimum distance to any thief across all paths. We can efficiently find the distance of all cells to the nearest thief using a Multi-Source BFS. Then, we can use Binary Search to find the optimal safeness factor.
// Approach:
// 1. Enqueue all thieves and use Multi-Source BFS to precalculate a `distNearestThief` grid.
// 2. Perform Binary Search on the Safeness Factor, setting `l = 0` and `r = 400` (max possible Manhattan distance).
// 3. For each `mid_sf`, use a `check` function (via standard BFS) from (0,0) to (n-1, n-1).
// 4. In the `check` function, only traverse cells where `distNearestThief[r][c] >= mid_sf`.
// Time Complexity: O(log(N) * N^2) - Precomputation takes O(N^2), Binary Search takes O(log N) iterations, and each BFS check takes O(N^2).
// Space Complexity: O(N^2) - For the queue, visited array, and distance array.

class Solution {
public:
    int n;
    
    vector<vector<int>> directions{{1, 0}, {-1, 0}, {0, -1}, {0, 1}};

    bool check(vector<vector<int>>& distNearestThief, int sf) {
        queue<pair<int, int>> que;
        vector<vector<bool>> visited(n, vector<bool>(n, false));
        
        // Push start node
        que.push({0, 0});
        visited[0][0] = true;

        // If the start node itself doesn't satisfy the safeness factor, return false
        if(distNearestThief[0][0] < sf)
            return false;

        while(!que.empty()) {
            int curr_i = que.front().first;
            int curr_j = que.front().second;

            que.pop();

            // Reached destination
            if(curr_i == n-1 && curr_j == n-1) {
                return true;
            }

            for(vector<int>& dir : directions) {
                int new_i = curr_i + dir[0];
                int new_j = curr_j + dir[1];

                // Validate bounds and visited status
                if(new_i >= 0 && new_i < n && new_j >= 0 && new_j < n && visited[new_i][new_j] != true) {
                    // Only traverse if the cell meets the required safeness factor
                    if(distNearestThief[new_i][new_j] < sf) {
                        continue; // Reject this cell
                    }
                    que.push({new_i, new_j});
                    visited[new_i][new_j] = true;
                }
            }
        }

        return false;
    }

    int maximumSafenessFactor(vector<vector<int>>& grid) {
        n = grid.size();

        // Step 1: Precalculation of distNearestThief for each cell using Multi-source BFS
        vector<vector<int>> distNearestThief(n, vector<int>(n, -1));
        queue<pair<int, int>> que;
        vector<vector<bool>> visited(n, vector<bool>(n, false));

        // Push all cells in queue where thieves are present
        for(int i = 0; i < n; i++) {
            for(int j = 0; j < n; j++) {
                if(grid[i][j] == 1) {
                    que.push({i, j});
                    visited[i][j] = true;
                }
            }
        }

        int level = 0;
        while(!que.empty()) {
            int size = que.size();

            while(size--) {
                int curr_i = que.front().first;
                int curr_j = que.front().second;
                que.pop();
                
                // Assign level (distance) to the current cell
                distNearestThief[curr_i][curr_j] = level;
                
                // Explore neighbors
                for(vector<int>& dir : directions) {
                    int new_i = curr_i + dir[0];
                    int new_j = curr_j + dir[1];

                    if(new_i < 0 || new_i >= n || new_j < 0 || new_j >= n || visited[new_i][new_j]) {
                        continue;
                    }

                    que.push({new_i, new_j});
                    visited[new_i][new_j] = true;
                }
            }
            level++; // Increment distance level
        }

        // Step 2: Apply binary search on Safeness Factor (SF)
        int l = 0;
        int r = 400; // Max possible dimension based on constraints
        int result = 0;

        while(l <= r) {
            int mid_sf = l + (r-l)/2;

            if(check(distNearestThief, mid_sf)) {
                // If a path is possible, record it and try to find a larger sf
                result = mid_sf;
                l = mid_sf+1;
            } else {
                // If no path is possible, reduce the sf
                r = mid_sf-1;
            }
        }

        return result;
    }
};

/*
*
* Dry Run
*
* grid:
* [0, 0, 1]
* [0, 0, 0]
* [0, 0, 0]
*
* Step 1: Multi-source BFS computes distNearestThief:
* [2, 1, 0]
* [3, 2, 1]
* [4, 3, 2]
*
* Step 2: Binary Search on SF (Range 0 to 400):
* - Initial: l=0, r=400. mid=200. check(200) -> returns false. r=199.
* - Subsequence iterations reduce the range rapidly.
* - Check(SF=3): Start cell [0][0] has dist 2, which is < 3. check(3) returns false. r=2.
* - Check(SF=2): Start cell [0][0] has dist 2. Valid. Path explores downwards:
*   -> (1,0)[dist=3] -> (2,0)[dist=4] -> (2,1)[dist=3] -> (2,2)[dist=2]. Valid path found. Returns true.
*   result updated to 2. l=3.
* Loop terminates. Final result = 2.
*
*/
```

## 🧠 Key Insights
- Do not fall for buzzwords; break down the problem logically. Instead of finding the nearest thief from every cell (which leads to $O(N^4)$ complexity), flip the logic: find the distance to all cells starting simultaneously from all thieves. This is exactly what Multi-Source BFS achieves perfectly in $O(N^2)$.
- The combination of **Multi-Source BFS** and **Binary Search on the answer** turns an extremely complex, potentially exponential graph-search problem into a highly optimized solution.
