# 2833. Furthest Point From Origin

## 📝 Problem Statement

You are given a string `moves` of length `n` consisting only of characters `'L'`, `'R'`, and `'_'`. The string represents your movement on a number line starting from the origin `0`.

In the $i^{th}$ move, you can choose one of the following directions:
- Move to the left if `moves[i] == 'L'` or `moves[i] == '_'`
- Move to the right if `moves[i] == 'R'` or `moves[i] == '_'`

Return the **distance from the origin** of the **furthest** point you can get to after `n` moves.

---

## 🛑 Constraints

- $1 \le moves.length == n \le 50$
- `moves` consists only of characters `'L'`, `'R'` and `'_'`.

---

## 🧠 Approach

<details>
<summary><b>Click here to view the detailed walk-through</b></summary>

To understand the most optimal approach, let's first eliminate the confusion caused by the dashes (`'_'`). 

1. **Calculate Fixed Position First:** If we temporarily ignore the dashes, our final position is entirely dictated by the fixed `'L'` (left) and `'R'` (right) moves. Moving left takes us in the negative direction, and moving right takes us in the positive direction. Thus, our net distance from the origin based *only* on forced moves is exactly `abs(left_count - right_count)`.
2. **Maximize with Free Moves:** The dashes (`'_'`) give us the freedom to move either left or right. If our goal is to travel as far as possible from the origin, we should always use these "free" moves to continue traveling in the direction we are already leaning towards. 
3. **Final Calculation:** If the forced moves leave us on the left side of the origin, we use all dashes to move further left. If we are on the right side, we use all dashes to move further right. Therefore, the maximum distance is simply the forced distance plus the total number of dashes.
   
**Formula:** `Distance = abs(left - right) + dash`
</details>

## 💻 Solution Code

```cpp
// Intuition: To maximize the distance from the origin, we need to move as far as possible in one direction. The absolute difference between 'L' and 'R' gives our fixed net position, and dashes ('_') allow us to extend that distance further in the same direction.
// Approach: Count the occurrences of 'L', 'R', and '_'. Calculate the absolute difference between 'L' and 'R' to find the net forced distance. Add the count of '_' to this distance to maximize the final reach.
// Time Complexity: O(n) - We iterate through the string of length n exactly once.
// Space Complexity: O(1) - We use only three integer variables for counting, requiring constant extra space.

class Solution {
public:
    int furthestDistanceFromOrigin(string moves) {
        int left  = 0; // Tracks the count of forced left moves
        int right = 0; // Tracks the count of forced right moves
        int dash  = 0; // Tracks the count of free choices

        // Traverse the string to populate direction counts
        for(char &ch : moves) {
            if(ch == 'L') left++;
            else if(ch == 'R') right++;
            else dash++;
        }

        // Net distance + remaining free moves extending the distance
        return abs(left - right) + dash;
    }
};

/*
*
* Dry Run
*
* Input: moves = "L_RL__R"
* * Step 1: Initialization
* left = 0, right = 0, dash = 0
* * Step 2: Iteration
* 'L': left = 1
* '_': dash = 1
* 'R': right = 1
* 'L': left = 2
* '_': dash = 2
* '_': dash = 3
* 'R': right = 2
* * End of loop values: left = 2, right = 2, dash = 3
* * Step 3: Final Calculation
* Result = abs(left - right) + dash
* Result = abs(2 - 2) + 3
* Result = 0 + 3
* Result = 3
* * Output: 3
*
*/
```

## 🔍 Key Insights

- **Greedy Choice:** Always adding the dashes to the dominant direction is a greedy choice that guarantees the maximum distance. Going towards the origin wastes steps.
- **Math Simplification:** Instead of simulating the path or branching for each dash recursively (which would cause exponential time complexity), breaking it down mathematically reduces it to a trivial $O(n)$ counting problem.
