## 📝 Problem Statement

You are given an integer array `nums`.

A tuple `(i, j, k)` of 3 **distinct** indices is **good** if `nums[i] == nums[j] == nums[k]`.

The **distance** of a good tuple is calculated as:
`abs(i - j) + abs(j - k) + abs(k - i)`
where `abs(x)` denotes the absolute value of `x`.

Return an integer denoting the **minimum possible distance** of a good tuple. If no good tuples exist, return `-1`.

<br>

## 🔐 Constraints

- `1 <= n == nums.length <= 10^5`
- `1 <= nums[i] <= n`

> **Note:** The key difference between this problem and Part I is the constraint on `nums.length` which has increased from `100` to `10^5`. This strict constraint completely eliminates `O(N³)` and `O(N²)` approaches, demanding an `O(N)` optimal solution.

## 💡 Explanation & Approach

### The Mathematical Deduction
Let's analyze the given distance formula: `abs(i - j) + abs(j - k) + abs(k - i)`.
Assuming we find the indices sequentially from left to right, we can confidently state that `i < j < k`. 

Because of this order, the distances are strictly non-negative:
- `abs(i - j)` becomes `j - i`
- `abs(j - k)` becomes `k - j`
- `abs(k - i)` becomes `k - i`

If we sum these up:
`(j - i) + (k - j) + (k - i)`
`= j - i + k - j + k - i`
`= 2k - 2i`
`= 2 * (k - i)`

**The Insight:** The distance relies *solely* on the first index `i` and the third index `k`. The middle index `j` completely cancels out of the equation!

### The Strategy
To get the minimum possible distance, we need the difference `k - i` to be as small as possible. This means we are looking for the tightest window that contains exactly 3 identical elements.
1. We can use a Hash Map (`unordered_map` in C++) to store a running list of indices where each element appears.
2. As we iterate through the array, we append the current index `k` to the list of the current element `nums[k]`.
3. If the length of this list becomes `3` or more, it means we have a valid tuple. 
4. To minimize the window, the starting index `i` should be the one exactly two steps behind our current index `k` in our tracked list. We grab this index (`vec[size - 3]`) and calculate `k - i`.
5. We keep track of the minimum `k - i` found globally. At the end, we simply multiply this minimum by `2` to return the actual distance.

## 🚀 Solution Code

### Brute Force Solution (Will give TLE for N = 10⁵)
<details>
<summary>Click to view the O(N³) Brute Force approach</summary>

```cpp
// Intuition: Check every possible triplet combination.
// Approach: Use three nested loops to test all distinct (i, j, k).
// Time Complexity: O(N^3) - Will Result in Time Limit Exceeded (TLE)
// Space Complexity: O(1)

class Solution {
public:
    int minimumDistance(vector<int>& nums) {
        int n = nums.size();
        int minDistance = INT_MAX;
        
        for(int i = 0; i < n; i++) {
            for(int j = i + 1; j < n; j++) {
                for(int k = j + 1; k < n; k++) {
                    if(nums[i] == nums[j] && nums[j] == nums[k]) {
                        int dist = abs(i - j) + abs(j - k) + abs(k - i);
                        minDistance = min(minDistance, dist);
                    }
                }
            }
        }
        
        return minDistance == INT_MAX ? -1 : minDistance;
    }
};
```

### Optimized Solution (Accepted for N = 10⁵)

```cpp
// Intuition: The distance formula abs(i-j) + abs(j-k) + abs(k-i) simplifies mathematically to 2 * (k - i) when i < j < k. The middle index cancels out.
// Approach: Use an unordered_map to maintain a list of seen indices for each number. Whenever a number has at least 3 indices, calculate the difference between the current index and the 3rd-to-last index to find the tightest window.
// Time Complexity: O(N) as we traverse the array only once. Pushing to a vector and hash map lookups are O(1) on average.
// Space Complexity: O(N) to store the indices of the array elements in the map.

class Solution {
public:
    int minimumDistance(vector<int>& nums) {
        int n = nums.size(); // Total number of elements
        
        // Map to map an integer to a vector of its occurrence indices
        unordered_map<int, vector<int>> mp;
        
        int result = INT_MAX; // Track the minimum (k - i) difference

        // Iterate through the array elements
        for(int k = 0; k < n; k++) {
            // Record the current index for the current number
            mp[nums[k]].push_back(k);

            // Check if we have found at least 3 occurrences of this number
            if(mp[nums[k]].size() >= 3) {
                // Get a reference to the vector of indices to avoid copying
                vector<int> &vec = mp[nums[k]];
                int siz = vec.size();

                // Get the starting index 'i' which is the 3rd-to-last occurrence
                int i = vec[siz - 3];
                
                // Update the minimum distance tracker
                result = min(result, k - i);
            }
        }

        // If result wasn't updated, no valid tuple exists, return -1.
        // Otherwise, multiply the minimum difference by 2 as per the simplified formula.
        return result >= INT_MAX ? -1 : 2 * result;
    }
};

/*
*
* Dry Run
* Input: nums = [1, 1, 2, 3, 2, 1, 2]
* Init: mp = {}, result = INT_MAX
*
* k=0, nums[0]=1: mp[1]=[0]
* k=1, nums[1]=1: mp[1]=[0, 1]
* k=2, nums[2]=2: mp[2]=[2]
* k=3, nums[3]=3: mp[3]=[3]
* k=4, nums[4]=2: mp[2]=[2, 4]
* k=5, nums[5]=1: mp[1]=[0, 1, 5] -> Size is 3!
* -> vec = [0, 1, 5], siz = 3
* -> i = vec[3-3] = vec[0] = 0
* -> result = min(INT_MAX, 5 - 0) = 5
* k=6, nums[6]=2: mp[2]=[2, 4, 6] -> Size is 3!
* -> vec = [2, 4, 6], siz = 3
* -> i = vec[3-3] = vec[0] = 2
* -> result = min(5, 6 - 2) = min(5, 4) = 4
*
* Loop ends. Final result = 4.
* Return 2 * 4 = 8.
*
*/
```

## 📌 Notes & Key Insights
- **The "Cancel Out" Trick:** In coordinate geometry and index manipulation problems involving absolute differences of sequential points, elements often cancel out. Recognizing that `j` is irrelevant to the overall distance sum turns a complex grouping problem into a simple two-pointer/hashing window problem.
- **Avoid Repeated Multiplication:** Calculating `2 * (k - i)` inside the loop at every valid step is redundant. Micro-optimization: Find the minimum `k - i` first, and do the multiplication just once at the `return` statement.
- **Reference Passing:** Notice `vector<int> &vec = mp[nums[k]];` inside the loop. Using the `&` reference avoids creating an expensive full copy of the vector during every iteration, preventing potential memory limit or time limit overheads.
