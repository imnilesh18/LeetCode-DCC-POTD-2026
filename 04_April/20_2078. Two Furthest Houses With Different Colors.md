## 📝 Problem Statement

There are `n` houses evenly lined up on the street, and each house is beautifully painted. You are given a **0-indexed** integer array `colors` of length `n`, where `colors[i]` represents the color of the `ith` house.

Return *the **maximum** distance between **two** houses with **different** colors.*

The distance between the `ith` and `jth` houses is `abs(i - j)`, where `abs(x)` is the **absolute value** of `x`.

### 🚨 Constraints
- `n == colors.length`
- `2 <= n <= 100`
- `0 <= colors[i] <= 100`
- Test data are generated such that **at least** two houses have different colors.

## 🧠 Explanation & Approach

To maximize the distance between two differently colored houses, the intuition points towards the extreme edges of our street (the array). 
If the optimal pair were somewhere strictly in the middle, pushing either end of the pair outwards to the edges would either keep the colors different or change them to a color different from the starting house, thus naturally yielding a strictly greater distance. 

Hence, the optimal solution almost always pairs an interior house with one of the boundary houses (either index `0` or index `n-1`).

We can progressively optimize this:
1. **Brute Force `O(N^2)`:** Check the distance between every possible combination of `i` and `j` where their colors differ.
2. **Improved Brute Force `O(N^2)`:** Pick house `i`, and search backward from the end `j = n-1`. Break early as soon as you find a different color because any further checking for that `i` will yield a smaller distance.
3. **Two-Pass Strategy `O(N)`:** Check distances by fixing the rightmost house and finding the leftmost house with a different color. Then, fix the leftmost house and find the rightmost house with a different color. Take the max.
4. **Single-Pass Strategy `O(N)`:** In just one loop, compare the current house against the first house (`colors[0]`) and the last house (`colors[n-1]`). Calculate the distance if they differ, updating the maximum.

## 💻 Solutions

<details>
<summary><b>Approach 1: Brute Force (C++)</b></summary>

```cpp
// Intuition: Check every possible pair of houses to find the maximum distance between two different colored houses.
// Approach: Use nested loops. The outer loop picks the first house (i), and the inner loop picks the second house (j). If colors[i] != colors[j], calculate the distance j - i and update the maximum distance.
// Time Complexity: O(n^2), where n is the number of houses. In the worst case, we check all pairs.
// Space Complexity: O(1), as we only use a few integer variables.
class Solution {
public:
    int maxDistance(vector<int>& colors) {
        int n = colors.size(); // Total number of houses
        int result = 0; // Stores the maximum distance

        // Iterate through each house as the starting point
        for (int i = 0; i < n; ++i) {
            // Check every subsequent house
            for (int j = i + 1; j < n; ++j) {
                // If colors are different, update the max distance
                if (colors[i] != colors[j]) {
                    result = max(result, j - i);
                }
            }
        }
        return result;
    }
};

/*
*
* Dry Run
* colors = [1, 1, 1, 6, 1, 1, 1]
* i = 0, colors[0] = 1
* j = 1, colors[1] = 1 (Same)
* j = 2, colors[2] = 1 (Same)
* j = 3, colors[3] = 6 (Diff) -> result = max(0, 3 - 0) = 3
* ...
* i = 1, colors[1] = 1
* j = 3, colors[3] = 6 (Diff) -> result = max(3, 3 - 1) = 3
* Result remains 3.
* Return 3.
*
*/
```

<summary><b>Approach 2: Improved Brute Force (C++)</b></summary>

```cpp
// Intuition: For a fixed house i, the maximum distance to a differently colored house will naturally be found if we start looking from the furthest possible house (end of the array) backwards.
// Approach: Outer loop picks the first house (i). Inner loop starts from the last house (n-1) and moves backwards to i. The first differently colored house found gives the max distance for house i, so we can break early.
// Time Complexity: O(n^2) in the worst case (if all houses have the same color except one), but practically faster due to early break.
// Space Complexity: O(1), no extra space used.
class Solution {
public:
    int maxDistance(vector<int>& colors) {
        int n = colors.size(); // Total houses
        int result = 0; // Stores max distance
        
        // Iterate through each house
        for (int i = 0; i < n; ++i) {
            // Start checking from the furthest possible house backwards
            for (int j = n-1; j > i; j--) {
                // If colors differ, update result and break to save time
                if (colors[i] != colors[j]) {
                    result = max(result, j - i);
                    break;
                }
            }
        }
        return result;
    }
};

/*
*
* Dry Run
* colors = [1, 8, 3, 8, 3]
* i = 0, colors[0] = 1
* j = 4, colors[4] = 3 (Diff) -> result = max(0, 4 - 0) = 4, break
* i = 1, colors[1] = 8
* j = 4, colors[4] = 3 (Diff) -> result = max(4, 4 - 1) = 4, break
* Returns 4.
*
*/
```

<summary><b>Approach 3: Two-Pass Optimization (C++)</b></summary>

```cpp
// Intuition: The maximum distance will always involve at least one house at the very edges (index 0 or n-1). If both optimal houses were strictly inside, expanding outwards to the edges would only increase distance.
// Approach: 1. Fix the rightmost house (n-1) and traverse from left (i=0) to find the first different colored house. 2. Fix the leftmost house (0) and traverse from right (i=n-1) to find the first different colored house. The maximum of these two distances is the answer.
// Time Complexity: O(n) as we traverse the array at most twice.
// Space Complexity: O(1), no extra space required.
class Solution {
public:
    int maxDistance(vector<int>& colors) {
        int n = colors.size(); // Total houses
        int result = 0; // Stores max distance

        int i = 0;
        // Move i forward until we find a color different from the last house
        while(colors[i] == colors[n-1]) i++;

        // Update result for the fixed rightmost house
        result = max(result, abs(i - (n-1)));

        i = n-1;
        // Move i backwards until we find a color different from the first house
        while(colors[0] == colors[i]) i--;

        // Update result for the fixed leftmost house
        result = max(result, i);

        return result;
    }
};

/*
*
* Dry Run
* colors = [0, 1]
* n = 2
* Pass 1: Fix n-1 (idx 1, color 1). i = 0, colors[0] = 0. Differs! 
* result = max(0, abs(0 - 1)) = 1
* Pass 2: Fix 0 (idx 0, color 0). i = 1, colors[1] = 1. Differs!
* result = max(1, 1) = 1
* Returns 1.
*
*/
```
</details>

<summary><b>Approach 4: Optimal One-Pass (C++)</b></summary>

```cpp
// Intuition: We can optimize the 2-pass approach into a single pass. For each house at index i, we simply check its distance to the leftmost house (0) and the rightmost house (n-1) if their colors differ.
// Approach: Iterate i from 0 to n-1. If colors[i] != colors[0], it's a valid pair, update result with i. If colors[i] != colors[n-1], it's a valid pair, update result with (n - 1) - i. Keep track of the overall maximum.
// Time Complexity: O(n), as we complete the check in exactly one traversal.
// Space Complexity: O(1), utilizing only a constant amount of extra space.
class Solution {
public:
    int maxDistance(vector<int>& colors) {
        int n = colors.size(); // Total houses
        int result = 0; // Stores max distance

        // Single pass through all houses
       for(int i = 0; i < n; i++) {
            // Check against the first house
            if(colors[i] != colors[0]) {
                result = max(result, i);
            }

            // Check against the last house
            if(colors[i] != colors[n-1]) {
                result = max(result, abs(i - (n-1)));
            }
       }

       return result;
    }
};

/*
*
* Dry Run
* colors = [1, 1, 1, 6, 1, 1, 1]
* n = 7
* i = 3, colors[3] = 6
* colors[3] != colors[0] (6 != 1) -> result = max(0, 3) = 3
* colors[3] != colors[6] (6 != 1) -> result = max(3, abs(3 - 6)) = 3
* All other i have colors[i] == 1, so they don't update result.
* Returns 3.
*
*/
```
</details>

## 📌 Notes & Key Insights
* **The "Edge" Strategy:** In geometry/distance problems on 1D arrays, maximizing `|i - j|` logically draws answers toward the array boundaries (`i = 0` and `j = n - 1`). Leveraging this behavior transitions $O(N^2)$ logical comparisons into $O(N)$ limit checks.
* **Refining Code Structure:** Passing through the array only once (as seen in Approach 4) is an excellent micro-optimization and demonstrates profound understanding to an interviewer.
* **Greedy Nature:** The logic heavily hinges on the greedy choice — "if I fix an anchor at the extreme end, checking for the very first different element guarantees the maximum possible stretch for that setup."
