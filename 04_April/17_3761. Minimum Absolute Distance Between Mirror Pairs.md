## 📝 Problem Statement

You are given an integer array `nums`.

A **mirror pair** is a pair of indices `(i, j)` such that:
- `0 <= i < j < nums.length`
- `reverse(nums[i]) == nums[j]`, where `reverse(x)` denotes the integer formed by reversing the digits of `x`. Leading zeros are omitted after reversing (for example, `reverse(120) = 21`).

Return the **minimum absolute distance** between the indices of any mirror pair. The absolute distance between indices `i` and `j` is `abs(i - j)`.

If no mirror pair exists, return `-1`.

---

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `nums = [12, 21, 45, 33, 54]` | `1` | The mirror pairs are:<br>• `(0, 1)` since `reverse(nums[0]) = reverse(12) = 21 = nums[1]`, distance `abs(0 - 1) = 1`.<br>• `(2, 4)` since `reverse(nums[2]) = reverse(45) = 54 = nums[4]`, distance `abs(2 - 4) = 2`.<br>The minimum absolute distance among all pairs is `1`. |

| **3** | `nums = [21, 120]` | `-1` | There are no mirror pairs in the array. |

## ⚙️ Constraints

- `1 <= nums.length <= 10^5`
- `1 <= nums[i] <= 10^9`

---

## 🧠 Approach & Explanation

Instead of using a brute-force approach that checks every possible pair `(i, j)` which would take $\mathcal{O}(N^2)$ time, we can optimize the solution to a single pass $\mathcal{O}(N)$ using a **Hash Map**.

**The Core Idea:**
For any number we process at index `i`, we are looking for a previously seen number whose reverse is equal to our current number `nums[i]`. Instead of searching through the array backwards, we can maintain a mapping of the **reversed values** of numbers we've seen so far, mapped to their indices. 

1. Create a hash map `mp` to store the `reversed_number -> index`.
2. Initialize `result` with a very large value (`INT_MAX`) to track the minimum distance.
3. As we iterate through the array, at each index `i`:
   - Check if `nums[i]` is already a key in our map. If it is, it means we have previously encountered a number whose reverse equals our current number! This forms a valid mirror pair.
   - We calculate the absolute distance: `i - mp[nums[i]]` and update `result` with the minimum distance found so far.
   - Finally, regardless of whether a match was found or not, we compute the reverse of the current number `nums[i]` and store it in our map with its index `i`. If a reverse was already present, updating it with the latest (largest) index is optimal because we want to *minimize* the distance for future matches.
4. If the `result` remains unchanged at the end, return `-1`.

## 💻 Solution Code

```cpp
// Intuition:
// Instead of using a nested loop to check every possible pair which would take O(N^2) time,
// we can use a Hash Map to keep track of the reversed values of the numbers we have seen so far.
// As we iterate, if our current number matches a previously stored reversed number, we've found a mirror pair!

// Approach:
// 1. Initialize an unordered map to store the reverse of each number as the key, and its index as the value.
// 2. Initialize a 'result' variable with infinity (INT_MAX) to keep track of the minimum absolute distance.
// 3. Iterate through the array:
//    a. Check if the current number exists in the map (meaning it is the reverse of a previously seen number).
//    b. If it exists, calculate the distance between the current index and the stored index, and update 'result' if this distance is smaller.
//    c. Reverse the current number and store/update its index in the map. Updating to the latest index ensures the minimum distance for future matches.
// 4. Finally, if 'result' is still infinity, return -1; otherwise, return 'result'.

// Time Complexity: O(n * log10(max_num)), where n is the size of the array. The loop runs n times, and reversing a number takes time proportional to its number of digits.
// Space Complexity: O(n), as we store the reversed numbers and their indices in the hash map.

class Solution {
public:
    // Helper function to reverse the digits of an integer
    int getReverse(int n) {
        int rev = 0;

        // Extract digits from back to front and construct reversed number
        while(n > 0) {
            int rem = n % 10;
            rev = rev * 10 + rem;
            n /= 10;
        }

        return rev;
    }
    
    int minMirrorPairDistance(vector<int>& nums) {
        int n = nums.size();

        // Map to store: reversed_number -> latest_index
        unordered_map<int, int> mp; 

        int result = INT_MAX;

        for(int i = 0; i < n; i++) {
            // If current number matches a previously stored reversed number
            if(mp.count(nums[i])) {
                // Update result with minimum distance
                result = min(result, i - mp[nums[i]]);
            }

            // Store/update the reverse of current number with its index
            mp[getReverse(nums[i])] = i;
        }

        // If no mirror pair found, return -1
        return result == INT_MAX ? -1 : result;
    }
};

/*
*
* Dry Run
*
* Input: nums = [12, 21, 45, 33, 54]
* Initialization: mp = {}, result = INT_MAX
* * i = 0, nums[0] = 12:
* mp doesn't contain 12.
* getReverse(12) = 21. mp becomes {21: 0}.
* * i = 1, nums[1] = 21:
* mp contains 21. Mirror pair found!
* result = min(INT_MAX, 1 - 0) = 1.
* getReverse(21) = 12. mp becomes {21: 0, 12: 1}.
* * i = 2, nums[2] = 45:
* mp doesn't contain 45.
* getReverse(45) = 54. mp becomes {21: 0, 12: 1, 54: 2}.
* * i = 3, nums[3] = 33:
* mp doesn't contain 33.
* getReverse(33) = 33. mp becomes {21: 0, 12: 1, 54: 2, 33: 3}.
* * i = 4, nums[4] = 54:
* mp contains 54. Mirror pair found!
* result = min(1, 4 - 2) = min(1, 2) = 1.
* getReverse(54) = 45. mp becomes {21: 0, 12: 1, 54: 2, 33: 3, 45: 4}.
* * Final Output: result = 1.
*
*/
```

## 📌 Notes & Key Insights

- **Leading Zeros Issue:** The math logic `rev = rev * 10 + rem` gracefully handles trailing zeros in the original number (which become leading zeros in the reversed number) by inherently omitting them (e.g., `120` reversed starts with `0 * 10 + 2 = 2` -> `2 * 10 + 1 = 21`).
- **Hash Map Overwriting:** When assigning `mp[getReverse(nums[i])] = i`, if the reversed value already exists in the map, it gets updated to the newer (and larger) index `i`. This ensures that subsequent matches will yield the smallest possible distance difference.
- **Data Types:** Since `nums[i]` maxes out at $10^9$, reversing it will at most be $9 \times 10^9$ theoretically, but the highest valid reversed integer remains strictly within standard 32-bit signed integer limits. No `long long` is needed.
