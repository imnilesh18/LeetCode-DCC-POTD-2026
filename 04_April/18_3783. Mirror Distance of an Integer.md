## 📝 Problem Statement

You are given an integer `n`. Define its **mirror distance** as: `abs(n - reverse(n))` where `reverse(n)` is the integer formed by reversing the digits of `n`.

Return an integer denoting the mirror distance of `n`. `abs(x)` denotes the absolute value of `x`.

---

## ⚙️ Constraints

| Constraint | Value |
| :--- | :--- |
| **Input Range** | $1 \le n \le 10^9$ |

---

## 📊 Examples

| Input | Output | Explanation |
| :--- | :--- | :--- |
| `n = 25` | `27` | `reverse(25) = 52`. Mirror distance = `abs(25 - 52) = 27`. |
| `n = 10` | `9` | `reverse(10) = 01` (which is `1`). Mirror distance = `abs(10 - 1) = 9`. |
| `n = 7` | `0` | `reverse(7) = 7`. Mirror distance = `abs(7 - 7) = 0`. |

---

## 💡 Approach

<details>
<summary><b>1. Iterative Digit Extraction (Mathematical)</b></summary>
- To reverse an integer, repeatedly extract the last digit using the modulo operator (`% 10`).
- Build the reversed number by shifting existing digits (`result * 10`) and adding the new digit.
- Update the original number by removing the last digit (`n /= 10`).
- Finally, calculate the absolute difference between the original `n` and the reversed value.
</details>

<details>
<summary><b>2. String Manipulation</b></summary>
- Convert the integer `n` to its string representation.
- Use the built-in `reverse` algorithm to reverse the string.
- Convert the reversed string back to an integer.
- Calculate the absolute difference.
</details>

## 🚀 Solution Code

### 🧱 Brute-force / Standard Iterative (Approach 1)

```cpp
// Intuition: The mirror distance requires the reversed version of n. Mathematical extraction of digits is the most space-efficient way to achieve this.
// Approach: Extract digits one by one using modulo 10, append them to a result variable shifted by powers of 10, and calculate the absolute difference. 
// Time Complexity: O(log10(n)) - We iterate through the number of digits in n.
// Space Complexity: O(1) - Only a few integer variables are used.

class Solution {
public:
    int getReverse(int n) {
        int result = 0;

        // Process until all digits are extracted
        while (n) {
            int rem = n % 10;          // Extract the last digit
            result = (result * 10) + rem; // Append digit to the result
            n /= 10;                   // Remove the last digit from n
        }

        return result;
    }

    int mirrorDistance(int n) {
        // Return the absolute difference between n and its reverse
        return abs(n - getReverse(n));
    }
};

/*
* Dry Run:
* Input n = 25
* 1. rem = 25 % 10 = 5, result = 0 * 10 + 5 = 5, n = 2
* 2. rem = 2 % 10 = 2, result = 5 * 10 + 2 = 52, n = 0
* Final result = 52. Difference abs(25 - 52) = 27.
*/
```

### ✨ Optimized / String Conversion (Approach 2)

```cpp
// Intuition: String manipulation can be more readable for reversals, as standard libraries provide optimized methods for string mirroring.
// Approach: Convert the integer to a string, reverse the string, and convert back to integer for the final calculation.
// Time Complexity: O(log10(n)) - String conversion and reversal both depend on the number of digits.
// Space Complexity: O(log10(n)) - A temporary string is created to store the digits of n.

class Solution {
public:
    int mirrorDistance(int n) {
        // Convert integer to string representation
        string s = to_string(n);
        
        // Use STL to reverse the string in-place
        reverse(begin(s), end(s));
        
        // Convert the reversed string back to integer
        int rev = stoi(s);
        
        // Return absolute mirror distance
        return abs(n - rev);
    }
};

/*
* Dry Run:
* Input n = 10
* 1. string s = "10"
* 2. reverse(s) -> s = "01"
* 3. rev = stoi("01") = 1
* 4. abs(10 - 1) = 9
*/
```

## 📌 Key Insights

- **Mathematical Approach** is generally preferred in interviews as it demonstrates core logic without relying on heavy library conversions.
- **Leading Zeros**: When reversing `10`, it becomes `01`. Mathematically, this naturally results in `1`. String conversion via `stoi` also handles this correctly.
- **Constraints**: Since $n \le 10^9$, $n$ and its reverse fit within a standard 32-bit signed `int` (range approx $\pm 2.1 \times 10^9$).

## 🔍 Further Exploration

- **Mirror Numbers**: A number where the mirror distance is `0` is called a Palindrome.
- **Overflow Handling**: If $n$ were larger (e.g., $10^{18}$), we would need `long long` to prevent overflow during the reversal process.
