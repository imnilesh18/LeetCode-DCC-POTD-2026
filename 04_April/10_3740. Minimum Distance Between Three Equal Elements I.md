## 📝 Problem Statement

You are given an integer array `nums`.

A tuple `(i, j, k)` of 3 **distinct** indices is **good** if `nums[i] == nums[j] == nums[k]`.

The **distance** of a good tuple is calculated as:
`abs(i - j) + abs(j - k) + abs(k - i)`
where `abs(x)` denotes the absolute value of `x`.

Return an integer denoting the **minimum possible distance** of a good tuple. If no good tuples exist, return `-1`.

<br>

## 🔐 Constraints

- `1 <= n == nums.length <= 100`
- `1 <= nums[i] <= n`

<br>

## 💡 Explanation & Approach

### The Mathematical Simplification
Whenever a problem involves a complex equation based on indices, the first step is to see if it can be mathematically simplified. 

Let's analyze the distance formula: `abs(i - j) + abs(j - k) + abs(k - i)`
Assuming we process elements from left to right, we can establish that `i < j < k`. 
If we remove the absolute values based on this assumption:
- `abs(i - j)` becomes `(j - i)`
- `abs(j - k)` becomes `(k - j)`
- `abs(k - i)` becomes `(k - i)`

Now, let's add them up:
`Distance = (j - i) + (k - j) + (k - i)`
`Distance = j - i + k - j + k - i`
`Distance = 2k - 2i`
**`Distance = 2 * (k - i)`**

**Wow!** The middle index `j` completely cancels out! This means the distance of any valid tuple only depends on the first index `i` and the last index `k`. The middle index simply needs to exist.

### The Strategy
1. We need to find elements that appear at least **3 times**.
2. We can use a Hash Map where the key is the number, and the value is a list (vector) of indices where this number is found.
3. As we iterate through the array and populate our map, we check if the current element has reached a frequency of 3 or more.
4. If it has, we can form a valid tuple. Since we want the **minimum** distance, and the current index is `k`, we want `i` to be as large as possible (as close to `k` as possible).
5. The closest valid `i` will always be the **3rd-to-last** index added to our list. So, `i = vector[size - 3]`.
6. We calculate `k - i`, keep track of the minimum, and at the very end, multiply the minimum result by `2`.

## 🚀 Solution Code

### Brute Force Solution
<details>
<summary>Click to view the O(N³) Brute Force approach</summary>

```cpp
// Intuition: Check every possible triplet combination to find the minimum distance.
// Approach: Use three nested loops to test all distinct (i, j, k) where i < j < k.
// Time Complexity: O(N^3) where N is the length of the array.
// Space Complexity: O(1) as we only use a few integer variables.

class Solution {
public:
    int minimumDistance(vector<int>& nums) {
        int n = nums.size();
        int minDistance = INT_MAX;
        
        for(int i = 0; i < n; i++) {
            for(int j = i + 1; j < n; j++) {
                for(int k = j + 1; k < n; k++) {
                    if(nums[i] == nums[j] && nums[j] == nums[k]) {
                        int dist = abs(i - j) + abs(j - k) + abs(k - i);
                        minDistance = min(minDistance, dist);
                    }
                }
            }
        }
        
        return minDistance == INT_MAX ? -1 : minDistance;
    }
};

/*
*
* Dry Run
* Array: [1, 2, 1, 1, 3]
* i=0(val=1), j=2(val=1), k=3(val=1)
* Dist = abs(0-2) + abs(2-3) + abs(3-0) = 2 + 1 + 3 = 6
* Min distance is 6.
*
*/
````

### Optimized Solution

```cpp
// Intuition: The distance formula simplifies from abs(i-j) + abs(j-k) + abs(k-i) to 2 * (k - i) assuming i < j < k. Thus, we only need to minimize the difference between the first and last indices of any three identical elements.
// Approach: Use a hash map to store the indices of each element as we iterate. When an element has appeared at least 3 times, we check the difference between the current index (k) and the 3rd-to-last index (i) to update our minimum distance.
// Time Complexity: O(N) as we traverse the array only once. Map insertions and vector size lookups take O(1) time on average.
// Space Complexity: O(N) because we store the indices of elements in the hash map.

class Solution {
public:
    int minimumDistance(vector<int>& nums) {
        int n = nums.size(); // get array size
        
        // Map to store indices for each number
        unordered_map<int, vector<int>> mp;
        int result = INT_MAX; // initialize with max possible value

        // Iterate through the array
        for(int k = 0; k < n; k++) {
            // Add current index to the number's list
            mp[nums[k]].push_back(k);

            // If we have found at least 3 occurrences of this number
            if(mp[nums[k]].size() >= 3) {
                vector<int> &vec = mp[nums[k]];
                int siz = vec.size();

                // i is the 3rd to last index (closest valid starting point)
                int i = vec[siz-3];
                
                // Track minimum difference (k - i)
                result = min(result, k - i);
            }
        }

        // Return -1 if no triplet found, else return 2 * (k-i)
        return result >= INT_MAX ? -1 : 2 * result;
    }
};

/*
*
* Dry Run
* * Input: nums = [1, 2, 1, 1, 3]
* Initialize: mp = {}, result = INT_MAX
*
* k=0, val=1: mp = {1: [0]}
* k=1, val=2: mp = {1: [0], 2: [1]}
* k=2, val=1: mp = {1: [0, 2], 2: [1]}
* k=3, val=1: mp = {1: [0, 2, 3], 2: [1]}
* -> mp[1].size() == 3 (Valid!)
* -> vec = [0, 2, 3], siz = 3
* -> i = vec[3-3] = vec[0] = 0
* -> result = min(INT_MAX, 3 - 0) = 3
* k=4, val=3: mp = {1: [0, 2, 3], 2: [1], 3: [4]}
*
* End Loop. result = 3.
* Return 2 * result = 6. Output: 6.
*
*/
```

## 📌 Notes & Key Insights

  - **Mathematical Reduction is King:** Always write out absolute value/distance formulas on paper to see if terms cancel out. In this case, reducing it to `2 * (k - i)` dropped the complexity immensely.
  - **Dynamic Tracking:** Instead of running nested loops, dynamically tracking indices using an `unordered_map` turns a potential `O(N³)` or `O(N²)` problem into a clean `O(N)` linear scan.
  - **Micro-Optimization:** Rather than multiplying by 2 inside the loop during every check (`2 * (k - i)`), we store the minimum `k - i` and just multiply it by 2 at the very end.
