## 📋 Problem Statement

You are given an `m x n` grid. Each cell of `grid` represents a street. The street of `grid[i][j]` can be:
- **1** which means a street connecting the left cell and the right cell.
- **2** which means a street connecting the upper cell and the lower cell.
- **3** which means a street connecting the left cell and the lower cell.
- **4** which means a street connecting the right cell and the lower cell.
- **5** which means a street connecting the left cell and the upper cell.
- **6** which means a street connecting the right cell and the upper cell.

<div align="center">
  <img src="https://assets.leetcode.com/uploads/2020/03/05/main.png" alt="Street Types" width="600">
</div>

You will initially start at the street of the upper-left cell `(0, 0)`. A valid path in the grid is a path that starts from the upper left cell `(0, 0)` and ends at the bottom-right cell `(m - 1, n - 1)`. The path should only follow the streets.

**Notice** that you are **not allowed** to change any street.

Return `true` *if there is a valid path in the grid or* `false` *otherwise*.

---

## 🎯 Constraints

- `m == grid.length`
- `n == grid[i].length`
- `1 <= m, n <= 300`
- `1 <= grid[i][j] <= 6`

## 🧠 Approach & Explanation

The problem asks us to find if there is a valid sequence of connected streets starting from `(0,0)` to the target `(m-1, n-1)`. This is a classic grid traversal problem that can be effectively solved using **Depth-First Search (DFS)** or **Breadth-First Search (BFS)**.

### Key Insights:
1. **Mapping Directions:** Each street type (1 to 6) dictates the possible directions you can move. For instance, Type 1 allows movement `Left (0, -1)` and `Right (0, 1)`. We can pre-compute this using a hash map.
2. **The "Look-Back" Check (Crucial):** It's not enough that the current cell can lead you to a neighboring cell. To form a valid road, the neighboring cell *must also have an opening pointing right back at you*.
   * *For example:* If you are at `(0,0)` with Type 1 (allows Right) and you step right to `(0,1)`, but `(0,1)` is Type 2 (Up/Down), they don't actually connect. 
   * Before fully committing to exploring a neighbor, we iterate through its allowed directions. If one of those directions correctly reconstructs our current cell's coordinates, the path is bi-directionally valid.
3. **Preventing Cycles:** We use a standard 2D `visited` array to ensure we do not visit the same cell twice and get stuck in infinite loops.

// Intuition:
// The problem can be modeled as a graph traversal where cells are nodes and valid streets are edges.
// We can use DFS to explore paths from the top-left to the bottom-right. The key is to ensure bidirectional connectivity between cells.
//
// Approach:
// 1. Map each street type (1-6) to its allowed movement directions.
// 2. Start DFS from (0, 0). Mark the current cell as visited.
// 3. For the current cell, check its allowed directions and compute the next cell's coordinates.
// 4. Validate the next cell (bounds check, not visited).
// 5. Crucial Step: Check if the next cell's street type allows movement BACK to the current cell.
// 6. If valid, recursively call DFS. If it reaches (m-1, n-1), return true.
//
// Time Complexity: O(m * n)
// We visit each cell at most once.
//
// Space Complexity: O(m * n)
// Space required for the visited matrix and the maximum depth of the recursion stack.

class Solution {
public:
    int m, n;

    // Map each street type to its allowed directional movements
    unordered_map<int, vector<vector<int>>> directions = {
        {1, {{0, -1}, {0, 1}}},   // Left, Right
        {2, {{-1, 0}, {1, 0}}},   // Up, Down
        {3, {{0, -1}, {1, 0}}},   // Left, Down
        {4, {{0, 1}, {1, 0}}},    // Right, Down
        {5, {{0, -1}, {-1, 0}}},  // Left, Up
        {6, {{-1, 0}, {0, 1}}}    // Up, Right
    };

    bool dfs(vector<vector<int>>& grid, int i, int j, vector<vector<bool>>& visited) {
        // If we reach the bottom-right cell, return true
        if(i == m-1 && j == n-1)
            return true;
        
        // Mark current cell as visited
        visited[i][j] = true;

        // Explore all allowed directions from the current street type
        for(auto &dir : directions[grid[i][j]]) {
            int new_i = i + dir[0];
            int new_j = j + dir[1];

            // Boundary and visited checks
            if(new_i < 0 || new_i >= m || new_j < 0 || new_j >= n || visited[new_i][new_j])
                continue;
            
            // IMPORTANT = can you come back to i and j from new_i and new_j
            for(auto &backDir : directions[grid[new_i][new_j]]) {
                if(new_i + backDir[0] == i && new_j + backDir[1] == j) {
                    // Valid path found, recursively explore
                    if(dfs(grid, new_i, new_j, visited)) {
                        return true;
                    }
                }
            }
        }

        return false;
    }

    bool hasValidPath(vector<vector<int>>& grid) {
        m = grid.size();
        n = grid[0].size();

        // Visited matrix to prevent revisiting cells
        vector<vector<bool>> visited(m, vector<bool>(n, false));
        // T.C : O(m*n)

        // Start DFS traversal from (0, 0)
        return dfs(grid, 0, 0, visited);
    }
};

/*
*
* Dry Run
* grid = [[2,4,3],[6,5,2]]
* * Init: m = 2, n = 3, visited = all false
* Call dfs(0, 0)
* i=0, j=0. grid[0][0] = 2. Type 2 directions: (-1, 0), (1, 0).
* Try dir (-1, 0): new_i = -1, new_j = 0 -> Out of bounds. Skip.
* Try dir (1, 0): new_i = 1, new_j = 0. grid[1][0] = 6.
* Type 6 directions: (-1, 0), (0, 1).
* Check back connection:
* 1 + (-1) = 0, 0 + 0 = 0 -> Matches (0,0)! Valid connection.
* Call dfs(1, 0)
* i=1, j=0. grid[1][0] = 6. Directions: (-1, 0), (0, 1).
* Try (-1, 0): new_i=0, new_j=0 -> Already visited. Skip.
* Try (0, 1): new_i=1, new_j=1. grid[1][1] = 5.
* Type 5 directions: (0, -1), (-1, 0).
* Check back connection:
* 1 + 0 = 1, 1 + (-1) = 0 -> Matches (1,0)! Valid.
* Call dfs(1, 1)
* i=1, j=1. grid[1][1] = 5. Directions: (0, -1), (-1, 0).
* Try (0,-1): new_i=1, new_j=0 -> Visited.
* Try (-1,0): new_i=0, new_j=1. grid[0][1] = 4.
* Type 4 directions: (0, 1), (1, 0).
* Check back connection: Matches (1,1)! Valid.
* Call dfs(0, 1)
* i=0, j=1. grid[0][1] = 4. Directions: (0, 1), (1, 0).
* Try (0, 1): new_i=0, new_j=2. grid[0][2] = 3. Valid back connection.
* Call dfs(0, 2)
* i=0, j=2. grid[0][2] = 3. Try (1, 0): new_i=1, new_j=2. grid[1][2] = 2. Valid back connection.
* Call dfs(1, 2)
* i=1, j=2 == m-1, n-1 -> Target Reached. Return True!
* Result: True.
*
*/
