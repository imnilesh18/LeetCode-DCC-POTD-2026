## 📝 Problem Statement

There is a robot starting at the position `(0, 0)`, the origin, on a 2D plane. Given a sequence of its moves, judge if this robot ends up at `(0, 0)` after it completes its moves.

You are given a string `moves` that represents the move sequence of the robot where `moves[i]` represents its `i`th move. Valid moves are:
- `'R'` (right)
- `'L'` (left)
- `'U'` (up)
- `'D'` (down)

Return `true` if the robot returns to the origin after it finishes all of its moves, or `false` otherwise.

> **Note:** The way that the robot is "facing" is irrelevant. `'R'` will always make the robot move to the right once, `'L'` will always make it move left, etc. Also, assume that the magnitude of the robot's movement is the same for each move.

## 🛑 Constraints

- `1 <= moves.length <= 2 * 10^4`
- `moves` only contains the characters `'U'`, `'D'`, `'L'` and `'R'`.

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `moves = "UD"` | `true` | The robot moves up once, and then down once. All moves have the same magnitude, so it ended up at the origin where it started. Therefore, we return true. |
| **2** | `moves = "LL"` | `false` | The robot moves left twice. It ends up two "moves" to the left of the origin. We return false because it is not at the origin at the end of its moves. |

## 🧠 Approach

The problem asks us to determine if the robot returns to its starting point `(0, 0)` after executing a series of commands. 
We can easily solve this by treating the movement as coordinates on a standard 2D Cartesian plane.

1. **Initialize Coordinates:** The robot begins exactly at the origin. We set up two variables, `x = 0` and `y = 0`, to track its current position.
2. **Simulate the Sequence:** We iterate through every character in the `moves` string:
   - Moving `'U'` (Up) means traveling positively along the Y-axis: `y++`.
   - Moving `'D'` (Down) means traveling negatively along the Y-axis: `y--`.
   - Moving `'R'` (Right) means traveling positively along the X-axis: `x++`.
   - Moving `'L'` (Left) means traveling negatively along the X-axis: `x--`.
3. **Validate the Final Position:** After iterating through the entire sequence of moves, the robot's final coordinates will be `(x, y)`. If `x == 0` and `y == 0`, it has returned to the origin, and we return `true`. Otherwise, we return `false`.

## 💻 Solution Code

```cpp
// Intuition: We can simulate the robot's path on a 2D coordinate plane starting at (0, 0).
// Approach: Traverse the moves string. Increment or decrement the x and y coordinates based on the cardinal direction. Finally, check if both coordinates are 0.
// Time Complexity: O(n), where n is the length of the string, since we traverse the sequence exactly once.
// Space Complexity: O(1), as we only use two integer variables to keep track of the current position regardless of the input size.

class Solution {
public:
    bool judgeCircle(string moves) {
        // Initialize starting coordinates at the origin
        int x = 0;
        int y = 0;

        // Iterate through each move command in the sequence
        for(char &ch : moves) {
            // Update y-axis for vertical moves
            if(ch == 'U')       y++;
            else if(ch == 'D')  y--;
            // Update x-axis for horizontal moves
            else if(ch == 'L')  x--;
            else if(ch == 'R')  x++;
        }

        // Return true if the robot ends up exactly where it started
        return x == 0 && y == 0;
    }
};

/*
 * Dry Run
 * * Example: moves = "UD"
 * Initialization:
 * x = 0, y = 0
 * * Loop iteration 1:
 * ch = 'U'
 * Matches condition: ch == 'U'
 * Action: y++
 * State: x = 0, y = 1
 * * Loop iteration 2:
 * ch = 'D'
 * Matches condition: ch == 'D'
 * Action: y--
 * State: x = 0, y = 0
 * * End of loop.
 * * Final Check:
 * Evaluate x == 0 && y == 0
 * 0 == 0 && 0 == 0 is true.
 * Returns true.
 * */
````

## 🎯 Key Insights

  - **Cancellation Property:** For the robot to return to the origin, the number of `'U'` moves must perfectly cancel out the `'D'` moves, and the `'L'` moves must cancel out the `'R'` moves. Coordinate tracking handles this natively.
  - **Simulation:** Since we apply logic rule-by-rule without needing lookaheads or complex state caching, this falls into the classic "Simulation" category of algorithmic problems.
