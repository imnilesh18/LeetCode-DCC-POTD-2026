## 📝 Problem Statement

You are given two string arrays, `queries` and `dictionary`. All words in each array comprise lowercase English letters and have the same length.

In one **edit** you can take a word from `queries`, and change any letter in it to any other letter. Find all words from `queries` that, after a **maximum of two edits**, equal some word from `dictionary`.

Return *a list of all words from* `queries`, *that match with some word from* `dictionary` *after a maximum of **two edits***. Return the words in the **same order** they appear in `queries`.

---

## ⚙️ Constraints

- `1 <= queries.length, dictionary.length <= 100`
- `n == queries[i].length == dictionary[j].length`
- `1 <= n <= 100`
- All `queries[i]` and `dictionary[j]` are composed of lowercase English letters.

## 💡 Approach & Explanation

Based on the problem constraints, the maximum number of words in both arrays is 100, and the maximum length of each word is 100. This relatively small constraint size suggests that a brute-force approach will be highly efficient and perfectly acceptable.

### Step-by-Step Logic:
1. **Iterate Through Queries**: We take each word from the `queries` array one by one.
2. **Compare With Dictionary**: For every selected query word, we iterate through the `dictionary` array to find a potential match.
3. **Character-by-Character Comparison**: We compare the current query word and the dictionary word index by index, keeping track of the mismatched characters using a `diff` counter.
4. **Early Exit Optimization**: As soon as the `diff` counter exceeds 2, we immediately break out of the character comparison loop since making more than 2 edits is invalid.
5. **Record Valid Matches**: If we finish comparing the words and the `diff` is $\le 2$, we add the query word to our result list and `break` out of the dictionary loop to move directly to the next query word.

## 💻 Solution Code

```cpp
// Intuition: The problem requires us to find if a query word can match any dictionary word with at most 2 character changes. Given the small constraints (max 100 words of length 100), a brute-force character-by-character comparison is highly efficient.
// Approach: 
// 1. Iterate through each word in the `queries` array.
// 2. For each query word, iterate through each word in the `dictionary`.
// 3. Compare the characters of the query word and the dictionary word at the same indices.
// 4. Maintain a `diff` counter. If `diff` exceeds 2, break early to optimize.
// 5. If a dictionary word is found with `diff <= 2`, add the query word to the result and move to the next query.
// Time Complexity: O(Q * D * N), where Q is the number of queries, D is the number of dictionary words, and N is the length of each word.
// Space Complexity: O(1) auxiliary space, ignoring the space used for the result array.

class Solution {
public:
    vector<string> twoEditWords(vector<string>& queries,
                                vector<string>& dictionary) {
        vector<string> result;

        // Process each query word
        for (string& query : queries) {
            
            // Compare against every dictionary word
            for (string& s : dictionary) {
                int diff = 0;

                // Count character-by-character differences
                for (int i = 0; i < query.size(); i++) {
                    if (query[i] != s[i]) {
                        ++diff;
                    }

                    // Early exit if edits exceed 2
                    if(diff > 2)
                        break;
                }

                // If match found within allowed edits, add to result
                if (diff <= 2) {
                    result.push_back(query);
                    break; // Move to the next query word
                }
            }
        }
        return result;
    }
};

/*
*
* Dry Run
*
* Input: queries = ["word", "yes"], dictionary = ["wood", "not"]
* * Query 1: "word"
* - Compare with "wood": 
* 'w'=='w', 'o'=='o', 'r'!='o' (diff=1), 'd'=='d'
* diff = 1 (<= 2). Valid! Add "word" to result. Break inner loop.
*
* Query 2: "yes"
* - Compare with "wood": (lengths match per constraints)
* 'y'!='w' (diff=1), 'e'!='o' (diff=2), 's'!='o' (diff=3)
* diff = 3 (> 2). Break character loop early.
* - Compare with "not":
* 'y'!='n' (diff=1), 'e'!='o' (diff=2), 's'!='t' (diff=3)
* diff = 3 (> 2). Break character loop early.
* No valid match found for "yes".
*
* Final Result: ["word"]
*
*/
```

## 📌 Notes & Key Insights

- **Why Not Trie?**: A Trie is an excellent data structure for prefix matching. However, for this specific problem, it acts as an **overkill**. Because we are allowed up to 2 edits, whenever a mismatch occurs in a Trie, we would have to explore multiple branching paths (potentially up to 25 other characters). This adds unnecessary recursive complexity and auxiliary space overhead. The simple nested loop approach with an early exit is far more optimal for these constraints.
- **Early Breaking**: The `if(diff > 2) break;` condition heavily trims down unnecessary iterations, acting as a crucial optimization over standard $O(N)$ string comparison.
