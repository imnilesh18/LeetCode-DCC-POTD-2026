## 📝 Problem Statement

You are given an `m x n` grid. A robot starts at the top-left corner of the grid `(0, 0)` and wants to reach the bottom-right corner `(m - 1, n - 1)`. The robot can move either right or down at any point in time.

The grid contains a value `coins[i][j]` in each cell:
- If `coins[i][j] >= 0`, the robot gains that many coins.
- If `coins[i][j] < 0`, the robot encounters a robber, and the robber steals the **absolute** value of `coins[i][j]` coins.

The robot has a special ability to **neutralize robbers** in at most **2 cells** on its path, preventing them from stealing coins in those cells.

> **Note:** The robot's total coins can be negative.

Return the **maximum** profit the robot can gain on the route.

---

## 🚦 Constraints

- `m == coins.length`
- `n == coins[i].length`
- `1 <= m, n <= 500`
- `-1000 <= coins[i][j] <= 1000`

## 🧠 Approach & Explanation

This is a classic "Dynamic Programming on Grids" problem, similar to finding the minimum/maximum cost path, but with an added state parameter: the ability to skip or neutralize negative values up to two times.

### Step-by-Step Breakdown:
1. **State Definition:** At any cell `(i, j)`, the maximum profit depends not only on the current cell but also on the number of neutralization abilities left (`neu`). Thus, our recursive state is defined by three parameters: `(i, j, neu)`.
2. **Movements:** From the current cell, the robot can move down to `(i + 1, j)` or right to `(i, j + 1)`.
3. **Decisions at Each Cell:**
   - **Take Decision:** We can just add the current cell's value `coins[i][j]` (whether it's positive or negative) and take the maximum profit from the downward or rightward paths.
   - **Skip Decision (Neutralize):** If the current cell has a negative value (`coins[i][j] < 0`) and we have remaining neutralization power (`neu > 0`), we can choose to neutralize the robber. In this case, the cost of the current cell becomes `0`, and our remaining power drops to `neu - 1`. We then take the maximum profit of going down or right.
4. **Base Cases:**
   - If we reach out-of-bounds (`i >= m` or `j >= n`), return a very small value (like `INT_MIN`) to mark it as an invalid path.
   - If we reach the destination cell `(m-1, n-1)`, we simply check if it's a negative value and if we have neutralization power left. If yes, we return `0` (we neutralize it). Otherwise, we return its value.
5. **Memoization:** Since the robot explores overlapping subproblems (many paths lead to the same cell with the same power left), we cache the results in a 3D array `t[i][j][neu]` to drastically reduce the time complexity.

## 💻 Solution Code

```cpp
// Intuition: The problem asks for the maximum profit reaching the bottom-right of a grid from the top-left, moving only right or down. We must deal with negative values (robbers) by optionally using up to 2 neutralizations. This is a classic DP on Grid problem where we explore all paths and cache the overlapping subproblems using memoization.
// Approach:
// 1. Define a recursive function solve(i, j, neu) returning the max money from (i, j) to the destination with `neu` powers left.
// 2. Base case 1: Reaching the destination (m-1, n-1). Return 0 if there's a robber and we can neutralize it, else return its coin value.
// 3. Base case 2: Going out of bounds. Return INT_MIN.
// 4. Choice 1 (Take): Accept the current cell's value and move right or down.
// 5. Choice 2 (Skip): If the cell has a robber (negative) and we have powers left (neu > 0), use one power and move right or down without losing coins.
// 6. Return the maximum of valid choices and memoize the result in t[i][j][neu].
// Time Complexity: O(m * n * 3) ~ O(m * n) since there are m*n*3 unique states, and each takes O(1) transitions.
// Space Complexity: O(m * n * 3) ~ O(m * n) for the 3D memoization array, plus O(m + n) for recursion stack.

class Solution {
public:
    int m;
    int n;
    int t[501][501][3];

    int solve(vector<vector<int>>& coins, int i, int j, int neu) {
        // Base case: Reached destination
        if(i == m-1 && j == n-1) {
            // Neutralize the robber if possible
            if(coins[i][j] < 0 && neu > 0) {
                return 0; // neutralized the robber
            }
            return coins[i][j];
        }

        // Out of bounds validation
        if(i >= m || j >= n) {
            return INT_MIN;
        }

        // Return memoized result
        if(t[i][j][neu] != INT_MIN) {
            return t[i][j][neu];
        }

        // Choice 1: Take the current cell value
        int take = coins[i][j] + max(solve(coins, i+1, j, neu), solve(coins, i, j+1, neu));

        // Choice 2: Skip current value if you can (negative cell & powers > 0)
        int skip = INT_MIN;
        if(coins[i][j] < 0 && neu > 0) {
            int skipDown = solve(coins, i+1, j, neu-1);
            int skipRight = solve(coins, i, j+1, neu-1);
            skip = max(skipDown, skipRight);
        }

        // Save state and return max
        return t[i][j][neu] = max(take, skip);
    }

    int maximumAmount(vector<vector<int>>& coins) {
        m = coins.size();
        n = coins[0].size();

        // Initialize memoization table
        for(int i = 0; i < 501; i++) {
            for(int j = 0; j < 501; j++) {
                for(int k = 0; k < 3; k++) {
                    t[i][j][k] = INT_MIN;
                }
            }
        }

        // Start at top-left with 2 neutralization powers
        return solve(coins, 0, 0, 2);
    }
};

/*
*
* Dry Run
*
* Example 1: coins = [[0,1,-1],[1,-2,3],[2,-3,4]]
* Start at solve(0, 0, 2).
* Cell (0,0) is 0. Take choice: 0 + max(solve(1,0,2), solve(0,1,2)). Skip not possible.
* -> Moving right to (0,1): value 1. Take choice: 1 + max(solve(1,1,2), solve(0,2,2)).
* -> Moving down to (1,1): value -2. 
* At (1,1,2), we encounter a robber (-2).
* Take choice: -2 + max(right, down).
* Skip choice: 0 + max(solve(2,1,1), solve(1,2,1)). (Used 1 power).
* Skip yields a higher result because we avoid losing 2 coins.
* Optimal path taken: 
* (0,0) val=0 -> (0,1) val=1 -> (1,1) val=-2 (neutralized, neu=1) -> (1,2) val=3 -> (2,2) val=4.
* Max profit = 0 + 1 + 0 + 3 + 4 = 8.
* Returns 8.
*
*/
```

## 📊 Key Insights & Notes

- **State Space Simplification:** The problem could sound complex because the total number of paths is exponential ($O(2^{m+n})$). However, observing that we only move Right and Down means we never revisit cells. This DAG structure combined with only 3 possible values for neutralization power allows us to perfectly cache the states.
- **Top-Down to Bottom-Up:** The provided approach uses recursion with memoization (Top-Down). For further exploration and to optimize recursion stack overhead, try converting this into a Bottom-Up tabulation DP approach.
- **Greedy Falls Short:** A greedy approach taking the immediate max at each cell would fail because taking a heavy temporary penalty might open a path to a significantly higher reward later.
