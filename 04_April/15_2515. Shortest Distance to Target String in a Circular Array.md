## 📝 Problem Statement

You are given a **0-indexed** circular string array `words` and a string `target`. A circular array means that the array's end connects to the array's beginning.

Formally, the next element of `words[i]` is `words[(i + 1) % n]` and the previous element of `words[i]` is `words[(i - 1 + n) % n]`, where `n` is the length of `words`.

Starting from `startIndex`, you can move to either the next word or the previous word with `1` step at a time.

Return *the **shortest** distance needed to reach the string `target`*. If the string `target` does not exist in `words`, return `-1`.

### 🎯 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `words = ["hello","i","am","leetcode","hello"], target = "hello", startIndex = 1` | `1` | We start from index 1 and can reach "hello" by moving 1 unit to the left to reach index 0. The shortest distance to reach "hello" is 1. |
| **2** | `words = ["a","b","leetcode"], target = "leetcode", startIndex = 0` | `1` | We start from index 0 and can reach "leetcode" by moving 1 unit to the left to reach index 2. The shortest distance is 1. |
| **3** | `words = ["i","eat","leetcode"], target = "ate", startIndex = 0` | `-1` | Since "ate" does not exist in words, we return -1. |

### 🔗 Constraints

- `1 <= words.length <= 100`
- `1 <= words[i].length <= 100`
- `words[i]` and `target` consist of only lowercase English letters.
- `0 <= startIndex < words.length`

---

## 💡 Approach & Explanation

The problem asks us to find the shortest distance to a `target` string in a circular array, starting from a specific `startIndex`. Since the array is circular, we have two ways to reach any index `i` from `startIndex`:
1. **Moving straight (Forward or Backward directly):** The absolute difference between the indices, i.e., `abs(i - startIndex)`.
2. **Moving around the circle (The opposite way):** If the array length is `n` and the straight distance is `d`, the circular distance to the exact same spot from the other direction will simply be `n - d`.

### Step-by-Step Breakdown:
1. Initialize a variable `result` with a very large value (like `INT_MAX`) to keep track of the minimum distance found.
2. Iterate through every element in the `words` array.
3. Check if the current word matches the `target`.
4. If it does, calculate the **Straight Distance** (`abs(i - startIndex)`) and the **Circular Distance** (`n - straightDistance`).
5. Update `result` with the minimum of its current value, the straight distance, and the circular distance.
6. Once the loop finishes, if `result` is still `INT_MAX`, it implies the target was never found. Return `-1`. Otherwise, return `result`.

## 💻 Solution Code

```cpp
// Intuition: Since the array is circular, we can reach any target index by moving either straightforwardly or by looping around the opposite way. The total length minus the straight distance gives the circular distance. We just need to find the minimum of these two distances for every occurrence of the target.
// Approach: Iterate through the array. Whenever the target is found, calculate both the straight distance `abs(i - startIndex)` and the circular distance `n - straightDist`. Keep track of the absolute minimum distance encountered.
// Time Complexity: O(n * L), where `n` is the number of words in the array and `L` is the average length of the words (string comparison takes O(L) time).
// Space Complexity: O(1), as we only use a few variables for counting and no extra auxiliary space is required.

class Solution {
public:
    int closestTarget(vector<string>& words, string target, int startIndex) {
        int n = words.size();          // Total number of words in the array
        int result = INT_MAX;         // Initialize result to maximum integer

        // Iterate through all words
        for(int i = 0; i < n; i++) {
            if(words[i] == target) {
                // Calculate straightforward distance
                int straightDist = abs(i - startIndex);
                
                // Calculate distance if we move the opposite way around the circle
                int circularDist = n - straightDist;

                // Take the minimum of current result, straight distance, and circular distance
                result = min({result, straightDist, circularDist});
            }
        }
        
        // If result was never updated, the target wasn't found
        return result == INT_MAX ? -1 : result;
    }
};

/*
*
* Dry Run
*
* Input: words = ["hello","i","am","leetcode","hello"], target = "hello", startIndex = 1
* n = 5
* result = INT_MAX
*
* i = 0
* words[0] ("hello") == "hello". Match found!
* straightDist = abs(0 - 1) = 1
* circularDist = 5 - 1 = 4
* result = min(INT_MAX, 1, 4) -> result becomes 1
*
* i = 1
* words[1] ("i") != "hello". No match.
*
* i = 2
* words[2] ("am") != "hello". No match.
*
* i = 3
* words[3] ("leetcode") != "hello". No match.
*
* i = 4
* words[4] ("hello") == "hello". Match found!
* straightDist = abs(4 - 1) = 3
* circularDist = 5 - 3 = 2
* result = min(1, 3, 2) -> result remains 1
*
* End of loop.
* result is not INT_MAX, return 1.
*
*/
```
