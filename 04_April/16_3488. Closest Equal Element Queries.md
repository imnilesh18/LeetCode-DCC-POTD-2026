# 3488. Closest Equal Element Queries

<br>

**Difficulty:** 🟡 Medium  
**Problem Link:** [LeetCode - Closest Equal Element Queries](https://leetcode.com/problems/closest-equal-element-queries/)  
**Tags:** `Array`, `Hash Table`, `Binary Search`

---

## 📝 Problem Statement

You are given a **circular** array `nums` and an array `queries`.

For each query `i`, you have to find the following:
The **minimum** distance between the element at index `queries[i]` and **any** other index `j` in the **circular** array, where `nums[j] == nums[queries[i]]`. If no such index exists, the answer for that query should be `-1`.

Return an array `answer` of the **same** size as `queries`, where `answer[i]` represents the result for query `i`.

---

## 🚫 Constraints

- $1 \le queries.length \le nums.length \le 10^5$
- $1 \le nums[i] \le 10^6$
- $0 \le queries[i] < nums.length$

---

## 💡 Examples

### Example 1
| Input | Output |
| :--- | :--- |
| `nums = [1,3,1,4,1,3,2]`, `queries = [0,3,5]` | `[2,-1,3]` |

> **Explanation:**
> - **Query 0:** The element at `queries[0] = 0` is `nums[0] = 1`. The nearest index with the same value is `2`, and the distance between them is `2`.
> - **Query 1:** The element at `queries[1] = 3` is `nums[3] = 4`. No other index contains `4`, so the result is `-1`.
> - **Query 2:** The element at `queries[2] = 5` is `nums[5] = 3`. The nearest index with the same value is `1`, and the distance between them is `3` (following the circular path: `5 -> 6 -> 0 -> 1`).

### Example 2
| Input | Output |
| :--- | :--- |
| `nums = [1,2,3,4]`, `queries = [0,1,2,3]` | `[-1,-1,-1,-1]` |

> **Explanation:**
> Each value in `nums` is unique, so no index shares the same value as the queried element. This results in `-1` for all queries.

---

## 🚀 Approach

A brute-force solution where we linearly search left and right for every query will yield $O(N \times Q)$ time complexity, leading to a Time Limit Exceeded (TLE) error. We can heavily optimize this using Hash Maps and Binary Search.

1. **Precompute Occurrences:** We store the indices of every unique element in an unordered map of vectors `mp[element] = [idx1, idx2...]`. Because we populate this by traversing from left to right, the stored indices are inherently sorted.
2. **Single Occurrence Verification:** For any query index `qi`, we check the frequency of the element `nums[qi]`. If it is `1`, there is no duplicate element to connect to, so we push `-1` to our results and immediately continue.
3. **Binary Search for Exact Position:** When multiple occurrences exist, we want to find our exact index `qi` in our stored map. We use Binary Search (`lower_bound`) to identify its position `pos` in $O(\log K)$ time.
4. **Identify Neighbors:** In a circular array, the closest matching values can only ever be immediately to the right or immediately to the left.
   - **Right Neighbor:** Located at index `(pos + 1) % size`.
   - **Left Neighbor:** Located at index `(pos - 1 + size) % size`. 
5. **Calculate Absolute & Circular Distances:**
   - Calculate straight distance: `d = abs(qi - neighbor_index)`.
   - Calculate circular distance: `circularDist = N - d`.
   - Track the minimum out of all calculations.

---

## 💻 Solution Code

```cpp
// Intuition: We need to efficiently find the closest matching element in a circular array. Instead of a linear search per query, we can precompute the indices of each element and use binary search to locate the query index, allowing us to quickly check adjacent occurrences.
// Approach: 
// 1. Store all indices of each element in an unordered_map. Since we iterate from 0 to N-1, the indices are naturally sorted.
// 2. For each query, fetch the target element and retrieve its list of indices.
// 3. If the element appears only once, it has no equal elements; return -1.
// 4. Use lower_bound (binary search) to find the position of the query index in the list.
// 5. Check the immediate right and left neighbors in the sorted list (using modulo arithmetic for circularity).
// 6. For both neighbors, calculate the direct distance and the circular wrap-around distance (N - direct_distance), and track the overall minimum.
// Time Complexity: O(N + Q log K) where N is the array length, Q is the number of queries, and K is the maximum occurrences of an element (K <= N). Overall worst case O(N + Q log N).
// Space Complexity: O(N) for storing the indices in the hash map.
class Solution {
public:
    vector<int> solveQueries(vector<int>& nums, vector<int>& queries) {
        int n = nums.size();
        unordered_map<int, vector<int>> mp;

        // Map values to their indices; inherently sorted
        for(int i = 0; i < n; i++) {
            mp[nums[i]].push_back(i);
        }

        vector<int> result;

        // Iterate through all given queries
        for(int qi: queries) {
            int element = nums[qi];
            vector<int> &vec = mp[element];
            int sz = vec.size();
            
            // no more occurence of element
            if(sz == 1) {
                result.push_back(-1);
                continue;
            }

            // Binary search to find current index position
            int pos = lower_bound(begin(vec), end(vec), qi) - begin(vec);
            int res = INT_MAX;

            // Right Neighbout - pos + 1
            int right = vec[(pos + 1) % sz];
            int d = abs(qi - right);
            int circularDist = n - d;

            res = min({res, d, circularDist});

            // Left Neighbout - pos - 1
            int left = vec[(pos - 1 + sz) % sz];
            d = abs(qi - left);
            circularDist = n - d;

            res = min({res, d, circularDist});

            result.push_back(res);
        }

        return result;
    }
};

/*
*
* Dry Run
* nums = [1,3,1,4,1,3,2], queries = [0,3,5]
* n = 7
* Map: {1: [0, 2, 4], 3: [1, 5], 4: [3], 2: [6]}
* * Query 0: qi = 0, element = 1.
* vec = [0, 2, 4], sz = 3.
* pos = lower_bound(0) = 0.
* Right neighbor: vec[(0+1)%3] = vec[1] = 2. Direct dist = abs(0-2) = 2. Circular dist = 7 - 2 = 5. Min = 2.
* Left neighbor: vec[(0-1+3)%3] = vec[2] = 4. Direct dist = abs(0-4) = 4. Circular dist = 7 - 4 = 3. Min = min(2, 4, 3) = 2.
* Result: 2
*
* Query 1: qi = 3, element = 4.
* vec = [3], sz = 1. No other occurrences.
* Result: -1
*
* Query 2: qi = 5, element = 3.
* vec = [1, 5], sz = 2.
* pos = lower_bound(5) = 1.
* Right neighbor: vec[(1+1)%2] = vec[0] = 1. Direct = abs(5-1) = 4. Circular = 7 - 4 = 3. Min = 3.
* Left neighbor: vec[(1-1+2)%2] = vec[0] = 1. Direct = abs(5-1) = 4. Circular = 7 - 4 = 3. Min = min(3, 4, 3) = 3.
* Result: 3
*
* Final Output: [2, -1, 3]
*
*/
````

## 🧠 Key Insights

  - **Circular Array Distance Formula:** The magic trick for finding wrap-around distance on a circular array of size `N` with direct line distance `d` is to compute `N - d`. Always take `min(d, N - d)`.
  - **Pre-Sorted Nature:** Storing array indices sequentially guarantees that the nested map values are strictly increasing, automatically giving us prerequisites required to perform Binary Search efficiently.
