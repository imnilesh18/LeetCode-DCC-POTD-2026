## 📖 Problem Statement

You are given two **non-increasing** 0-indexed integer arrays `nums1` and `nums2`.

A pair of indices `(i, j)`, where `0 <= i < nums1.length` and `0 <= j < nums2.length`, is **valid** if both `i <= j` and `nums1[i] <= nums2[j]`. The **distance** of the pair is `j - i`.

Return *the **maximum distance** of any **valid** pair `(i, j)`*. If there are no valid pairs, return `0`.

> **Note:** An array `arr` is **non-increasing** if `arr[i-1] >= arr[i]` for every `1 <= i < arr.length`.

## ⚠️ Constraints

- `1 <= nums1.length, nums2.length <= 10^5`
- `1 <= nums1[i], nums2[j] <= 10^5`
- Both `nums1` and `nums2` are **non-increasing**.

---

## 💡 Explanation & Approach

### The Core Idea
The problem asks us to maximize the distance `j - i` given that `i <= j` and `nums1[i] <= nums2[j]`. Since both arrays are non-increasing (sorted in descending order), we can leverage a **Two Pointers** approach.

### Step-by-step Approach
1. **Initialize Pointers:** Start with two pointers, `i = 0` for `nums1` and `j = 0` for `nums2`. Set `result = 0`.
2. **Iterate Arrays:** Traverse while `i < nums1.length` and `j < nums2.length`.
3. **Handle Invalid Condition (`nums1[i] > nums2[j]`):** - If the element in `nums1` is larger than the element in `nums2`, it violates our requirement. 
   - Because the arrays are sorted in descending order, moving `j` forward will only give us even smaller elements. Thus, our only choice is to move `i` forward (`i++`) to find a smaller element in `nums1`.
4. **Handle Valid Condition (`nums1[i] <= nums2[j]`):**
   - If the condition is satisfied, we have a potentially valid pair. We calculate the distance `j - i` and update our maximum `result`.
   - To find an even *greater* distance, we move the `j` pointer forward (`j++`). 
   - *Note on `i <= j`:* We don't explicitly need to check `i <= j`. If `i` overtakes `j`, `j - i` becomes negative. Since we initialize `result` at 0 and want the maximum distance, a negative distance is automatically ignored.
5. **Termination:** The loop stops when either pointer runs out of bounds. The `result` holds the maximum distance found.

## 💻 Solution Code

### 🐌 Brute-force Solution (C++)

<details>
<summary>Click to view Brute-force approach</summary>

```cpp
// Intuition: Check every possible pair of (i, j) to find the valid pair with the maximum distance.
// Approach: Use two nested loops. Outer loop for nums1, inner loop for nums2 starting from j = i. Update max distance.
// Time Complexity: O(M * N) where M and N are lengths of the two arrays. This will cause TLE for large inputs.
// Space Complexity: O(1) as we use no extra space.

class Solution {
public:
    int maxDistance(vector<int>& nums1, vector<int>& nums2) {
        int max_dist = 0;
        for (int i = 0; i < nums1.size(); i++) {
            for (int j = i; j < nums2.size(); j++) {
                if (nums1[i] <= nums2[j]) {
                    max_dist = max(max_dist, j - i);
                }
            }
        }
        return max_dist;
    }
};
````

### 🚀 Optimized Solution (C++)

```cpp
// Intuition: Both arrays are non-increasing. Using two pointers allows us to efficiently find valid pairs by traversing both arrays simultaneously.
// Approach: Maintain pointers i for nums1 and j for nums2. If nums1[i] > nums2[j], increment i. Else, update max distance and increment j.
// Time Complexity: O(M + N) where M and N are the lengths of nums1 and nums2, since both pointers only traverse forward.
// Space Complexity: O(1) as we only use a few integer variables.

class Solution {
public:
    int maxDistance(vector<int>& nums1, vector<int>& nums2) {
        int m = nums1.size();
        int n = nums2.size();

        int i = 0; // Pointer for nums1
        int j = 0; // Pointer for nums2

        int result = 0; // Stores the max valid distance

        // Traverse both arrays
        while(i < m && j < n) {
            // If nums1 element is larger, we need a smaller element, so move i
            if(nums1[i] > nums2[j]) {
                i++;
            } else {
                // Valid condition met. Update max distance and advance j to expand distance
                result = max(result, j - i);
                j++;
            }
        }
        
        return result;
    }
};

/*
*
* Dry Run
* nums1 = [55, 30, 5, 4, 2], nums2 = [100, 20, 10, 10, 5]
* Initialize: i = 0, j = 0, result = 0
* * Iteration 1: nums1[0](55) <= nums2[0](100) -> Valid. result = max(0, 0-0) = 0. j++ (j=1)
* Iteration 2: nums1[0](55) > nums2[1](20) -> Invalid. i++ (i=1)
* Iteration 3: nums1[1](30) > nums2[1](20) -> Invalid. i++ (i=2)
* Iteration 4: nums1[2](5) <= nums2[1](20) -> Valid. result = max(0, 1-2) = 0. j++ (j=2)
* Iteration 5: nums1[2](5) <= nums2[2](10) -> Valid. result = max(0, 2-2) = 0. j++ (j=3)
* Iteration 6: nums1[2](5) <= nums2[3](10) -> Valid. result = max(0, 3-2) = 1. j++ (j=4)
* Iteration 7: nums1[2](5) <= nums2[4](5)  -> Valid. result = max(1, 4-2) = 2. j++ (j=5)
* * j reaches 5 (out of bounds). Loop terminates.
* Final result = 2
*
*/
```

## 📝 Notes

  - Since we are calculating `j - i` and evaluating validity, cases where `i > j` produce negative results. These safely do not affect our maximum variable initialized to `0`.
  - The algorithm's linear scaling `O(m + n)` easily handles the upper bound limits of `10^5` within a few milliseconds.

## 🔑 Key Insights

  - **Property of Monotonicity:** Whenever you deal with ordered arrays (non-increasing or non-decreasing), a two-pointer or binary search method is almost always the optimal route over a nested-loop sweep.
  - Because we want to stretch the distance `j - i` as much as possible, advancing `j` when conditions are met is strategically better than scanning randomly.
