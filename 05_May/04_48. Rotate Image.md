## 📝 Problem Statement

You are given an `n x n` 2D `matrix` representing an image. Rotate the image by **90 degrees (clockwise)**.

You have to rotate the image **in-place**, which means you have to modify the input 2D matrix directly. **DO NOT** allocate another 2D matrix and do the rotation.

---

## 🚦 Constraints

* `n == matrix.length == matrix[i].length`
* `1 <= n <= 20`
* `-1000 <= matrix[i][j] <= 1000`

## 💡 Approach

### Brute Force Approach
If we carefully observe the rotation, the first row of our original matrix becomes the last column of the rotated matrix. The second row becomes the second-to-last column, and so forth. 
The simplest way to solve this is to create a new `n x n` 2D array. We can iterate through the given matrix and map `matrix[i][j]` to its new position at `new_matrix[j][n - 1 - i]`. However, this requires allocating extra space, which violates the strict **in-place** constraint requested by the problem.

### Optimized Approach (In-Place)
To achieve the rotation without extra memory, we can use a two-step mathematical trick:
1. **Transpose the Matrix:** Convert all rows into columns by swapping elements across the main diagonal. Specifically, swap `matrix[i][j]` with `matrix[j][i]`. 
2. **Reverse the Rows:** After transposing, we iterate through every row in the matrix and reverse its elements horizontally. 

This sequence (Transpose $\rightarrow$ Reverse) seamlessly rotates the matrix exactly 90 degrees clockwise directly modifying the original memory.

## 💻 Solution Code

<details>
<summary><b>🟢 Brute-force Solution (C++)</b></summary>

```cpp
// Intuition: Create a new matrix and map each row to its corresponding column from right to left.
// Approach: 
// 1. Allocate a dummy matrix of the same size.
// 2. Loop through the grid and assign matrix[i][j] to dummy[j][n - 1 - i].
// 3. Copy the dummy matrix back to the original matrix.
// Time Complexity: O(n^2) - Iterates through the entire matrix twice.
// Space Complexity: O(n^2) - Requires an additional 2D array of the same size.

class Solution {
public:
    void rotate(vector<vector<int>>& matrix) {
        int n = matrix.size();
        vector<vector<int>> dummy(n, vector<int>(n, 0));
        
        // Map rows to columns
        for(int i = 0; i < n; i++) {
            for(int j = 0; j < n; j++) {
                dummy[j][n - 1 - i] = matrix[i][j];
            }
        }
        
        // Copy back to original
        for(int i = 0; i < n; i++) {
            for(int j = 0; j < n; j++) {
                matrix[i][j] = dummy[i][j];
            }
        }
    }
};

/*
*
* Dry Run
* Input: matrix = [[1,2],[3,4]], n = 2
* dummy initialized to [[0,0],[0,0]]
* matrix[0][0] = 1 -> dummy[0][1] = 1
* matrix[0][1] = 2 -> dummy[1][1] = 2
* matrix[1][0] = 3 -> dummy[0][0] = 3
* matrix[1][1] = 4 -> dummy[1][0] = 4
* dummy = [[3,1],[4,2]] -> copy to matrix.
*
*/
```

```cpp
// Intuition: A 90-degree clockwise rotation can be done by first transposing the matrix, then reversing each row.
// Approach: 
// 1. Find the transpose: swap(matrix[i][j], matrix[j][i]).
// 2. Reverse each row horizontally using the built-in reverse function.
// Time Complexity: O(n^2) - We traverse the matrix elements twice (once for transpose, once for reverse).
// Space Complexity: O(1) - All operations are done in-place.

class Solution {
public:
    void rotate(vector<vector<int>>& matrix) {
        int N = matrix.size();
        
        // Find transpose
        for(int i = 0; i < N; i++) {
            for(int j = i; j < N; j++) {
                swap(matrix[i][j], matrix[j][i]);
            }
        }
        
        // Flip horizontally - For clockwise rotation
        for(int i = 0; i < N; i++) {
            reverse(matrix[i].begin(), matrix[i].end());
        }
        
        /*
        // flip vertically - For anti-clockwise rotation
        for(int col = 0; col < N; col++) {
            int row = 0;
            int mid = N / 2;
            while(row < mid) {
                swap(matrix[row][col], matrix[N - row - 1][col]);
                row++;
            }
        }
        */
    }
};

/*
*
* Dry Run
* Input: matrix = [[1,2,3],[4,5,6],[7,8,9]]
* N = 3
*
* Step 1: Transpose
* Swap (0,1) with (1,0) -> 2 and 4
* Swap (0,2) with (2,0) -> 3 and 7
* Swap (1,2) with (2,1) -> 6 and 8
*
* Matrix after transpose:
* [1, 4, 7]
* [2, 5, 8]
* [3, 6, 9]
*
* Step 2: Reverse horizontally
* Row 0: [1,4,7] -> [7,4,1]
* Row 1: [2,5,8] -> [8,5,2]
* Row 2: [3,6,9] -> [9,6,3]
*
* Final Output:
* [[7,4,1],[8,5,2],[9,6,3]]
*
*/
```

## 📌 Notes
* **Anti-Clockwise Rotation:** If an interviewer asks to rotate the matrix by 90 degrees *anti-clockwise*, simply Transpose the matrix first, and then flip it *vertically* (swap elements in a column top-to-bottom) instead of horizontally. (Logic provided in comments).

## 🔑 Key Insights
* The combination of transposing and reversing is a widely applicable mathematical concept for linear algebra grid manipulations. Remembering **Transpose + Reverse rows = 90° Clockwise** makes this an incredibly simple problem.

## 🕳️ Further Exploration
* Try implementing the rotation by shifting groups of 4 elements (the "Layer-by-Layer" rotation approach). It is slightly more complex in index management but acts as an excellent exercise for array boundary handling.
