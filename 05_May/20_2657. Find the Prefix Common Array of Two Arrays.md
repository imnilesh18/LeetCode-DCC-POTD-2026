## 📝 Problem Statement

You are given two **0-indexed** integer permutations `A` and `B` of length `n`.

A **prefix common array** of `A` and `B` is an array `C` such that `C[i]` is equal to the count of numbers that are present at or before the index `i` in both `A` and `B`.

Return *the prefix common array of `A` and `B`*.

> A sequence of `n` integers is called a **permutation** if it contains all integers from `1` to `n` exactly once.

---

## 🛑 Constraints

* `1 <= A.length == B.length == n <= 50`
* `1 <= A[i], B[i] <= n`
* It is guaranteed that `A` and `B` are both a permutation of `n` integers.

## ⚙️ Approach

<details>
<summary><b>Click to expand the walkthrough of our approaches</b></summary>

1. **Brute Force:** The simplest approach is to check every element in the prefix of `A` against every element in the prefix of `B` for each index `i`. This requires three nested loops, leading to an O(n^3) time complexity.
2. **Better Approach:** We can optimize the search by using boolean frequency arrays to keep track of elements we have seen in `A` and `B` so far. For each index `i`, we mark the current elements as seen, and then iterate from 1 to `n` to count how many numbers are marked as seen in both arrays. This reduces the time complexity to O(n^2).
3. **Optimal Approach:** Since we know the arrays are permutations of numbers from 1 to `n`, a number becomes a "common" prefix element exactly when its frequency across both arrays reaches 2. By using a single hash map (or frequency array) and updating a running count whenever a number's frequency hits 2, we can solve this in a single pass, achieving O(n) time complexity.

</details>

## 💻 Solution Code

### Brute Force Solution
```cpp
// Intuition: For every index i, we can simply iterate through all elements in the prefix of A and check if they exist in the prefix of B.
// Approach: Iterate i from 0 to n-1. For each i, loop x from 0 to i to pick an element from A's prefix, then loop y from 0 to i to search for it in B's prefix.
// Time Complexity: O(n^3) due to three nested loops checking prefixes repeatedly.
// Space Complexity: O(1) auxiliary space (excluding the result array).
class Solution {
public:
    vector<int> findThePrefixCommonArray(vector<int>& A, vector<int>& B) {
        int n = A.size();

        vector<int> result(n);

        for(int i = 0 ; i < n; i++) {
            int count = 0; // count for current prefix

            for(int x = 0; x <= i; x++) { //A
                //Find if we have A[x] in B or not till index i
                for(int y = 0; y <= i; y++) { //B
                    if(B[y] == A[x]) {
                        count++; // found common element
                        break;
                    }
                }
            }

            result[i] = count; // store total common for prefix i

        }

        return result;
    }
};

/*
*
* Dry Run
* A = [2,3,1], B = [3,1,2]
* i = 0: x=0 (A[0]=2). B[0]=3. No match. result[0]=0
* i = 1: x=0 (A[0]=2) vs B[0..1] no match. x=1 (A[1]=3) matches B[0]=3. result[1]=1
* i = 2: x=0 (2) matches B[2]. x=1 (3) matches B[0]. x=2 (1) matches B[1]. result[2]=3
* returns [0,1,3]
*
*/
```

### Better Solution
```cpp
// Intuition: Instead of nested searching, we can maintain the presence of elements in both prefixes using boolean arrays.
// Approach: Use two boolean arrays. As we iterate through A and B, mark the current elements as true. Then, iterate from 1 to n to count how many numbers are true in both arrays.
// Time Complexity: O(n^2) because we have an inner loop running n times for each element.
// Space Complexity: O(n) to store the boolean presence arrays.
class Solution {
public:
    vector<int> findThePrefixCommonArray(vector<int>& A, vector<int>& B) {
        int n = A.size();

        vector<int> result(n);

        vector<bool> isPresentA(n+1, false); //isPresent[i] = true, means i is present in A
        vector<bool> isPresentB(n+1, false); //....

        for(int i = 0; i < n; i++) {
            isPresentA[A[i]] = true; // mark A's element seen
            isPresentB[B[i]] = true; // mark B's element seen

            int count = 0;
            for(int num = 1; num <= n; num++) { // check all possible numbers
                if(isPresentA[num] == true && isPresentB[num] == true) {
                    count++; // increment if seen in both
                }
            }

            result[i] = count; // store total common
        }

        return result;
    }
};

/*
*
* Dry Run
* A = [1,3], B = [3,1]
* i = 0: A[0]=1, B[0]=3. isPresentA[1]=T, isPresentB[3]=T. Loop 1 to n finds no common True. result[0]=0
* i = 1: A[1]=3, B[1]=1. isPresentA[3]=T, isPresentB[1]=T. Loop finds 1 and 3 are True in both. result[1]=2
* returns [0,2]
*
*/
```

### Optimal Solution
```cpp
// Intuition: Since both arrays are permutations, any number will appear exactly twice in total across A and B. A number is common in the prefix as soon as we have seen it 2 times.
// Approach: Maintain a frequency map. Process A[i] and B[i] simultaneously. If updating the frequency of either element reaches 2, increment the common counter.
// Time Complexity: O(n) as we only do a single pass and constant time map operations.
// Space Complexity: O(n) to store the frequencies in the hash map.
class Solution {
public:
    vector<int> findThePrefixCommonArray(vector<int>& A, vector<int>& B) {
        int n = A.size();

        vector<int> result(n);

        unordered_map<int, int> mp; // tracks frequency of elements

        int count = 0;
        for(int i = 0; i < n; i++) {
            mp[A[i]]++; // process element from A
            if(mp[A[i]] == 2) {
                count++; // pair found
            }

            mp[B[i]]++; // process element from B
            if(mp[B[i]] == 2) {
                count++; // pair found
            }

            result[i] = count; // record running count
        }

        return result;
    }
};

/*
*
* Dry Run
* A = [1,3], B = [3,1]
* i=0: A[0]=1, mp[1]=1. B[0]=3, mp[3]=1. count=0. result[0]=0
* i=1: A[1]=3, mp[3]=2, count=1. B[1]=1, mp[1]=2, count=2. result[1]=2
* returns [0,2]
*
*/
```

## 🔍 Key Insights

* **Permutation Property:** The fact that both inputs are permutations of the numbers from 1 to `n` is the key to achieving O(n) time. It guarantees that an element can only appear at most twice, meaning exactly when its frequency equals 2, we know it exists in both prefixes.
* **Running State:** Instead of recalculating the common elements from scratch for every prefix, preserving a running count and an aggregate state (like a frequency map) prevents redundant computations.
* **Hash Map vs Array:** In the optimal approach, while an `unordered_map` is used, utilizing a fixed-size `vector<int>` for frequency counting is technically faster due to cache locality and avoiding hashing overhead, though both are O(n) space and time.
