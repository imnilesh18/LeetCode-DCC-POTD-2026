## 📝 Problem Statement

Given the `head` of a linked list, rotate the list to the right by `k` places.

---

## 🛑 Constraints

* The number of nodes in the list is in the range `[0, 500]`.
* `-100 <= Node.val <= 100`
* `0 <= k <= 2 * 10^9`

## 🧠 Approach

<details>
<summary><b>Click to expand detailed walkthrough</b></summary>

1. **Find the Length and Tail:** First, we traverse the linked list to find its total `length` and locate the `tail` node.
2. **Normalize $K$:** Since rotating a list of length $L$ exactly $L$ times results in the identical original list, we can optimize the rotations. We normalize $k$ using modulo arithmetic: `k = k % length`. If the new `k` is 0, no rotation is needed, so we simply return the original `head`.
3. **Form a Circular List:** Connect the `tail` node to the original `head` (`tail->next = head`), making the linked list circular.
4. **Locate the New Tail and Head:** The new tail after rotation will be at the `(length - k)`th node from the start. We traverse `length - k - 1` steps from the current head to reach this new tail.
5. **Break the Circle:** Once at the new tail, the node right after it (`newTail->next`) will be our `newHead`. We sever the circular link by setting `newTail->next = NULL` and return the `newHead`.

</details>

## 💻 Solution Code

### Brute-force Solution (C++)
```cpp
// Intuition: Extract list values into an array, perform a standard array rotation, and then write the values back to the list nodes.
// Approach:
// 1. Traverse the linked list and push all node values into a dynamic array (vector).
// 2. Normalize k by taking k % array.size().
// 3. Create a new array or rotate the existing array elements to the right by k positions.
// 4. Traverse the linked list again from the head, overwriting node values with the rotated array values.
// Time Complexity: O(n) to extract values, O(n) to rotate array, and O(n) to rewrite. Overall O(n).
// Space Complexity: O(n) to store the linked list elements in an array.

class Solution {
public:
    ListNode* rotateRight(ListNode* head, int k) {
        if (!head || !head->next || k == 0) return head;
        
        vector<int> vals;
        ListNode* curr = head;
        while (curr) {
            vals.push_back(curr->val);
            curr = curr->next;
        }
        
        int n = vals.size();
        k = k % n;
        if (k == 0) return head;
        
        vector<int> rotated(n);
        for (int i = 0; i < n; i++) {
            rotated[(i + k) % n] = vals[i];
        }
        
        curr = head;
        for (int i = 0; i < n; i++) {
            curr->val = rotated[i];
            curr = curr->next;
        }
        
        return head;
    }
};

/*
*
* Dry Run
* List: 1 -> 2 -> 3, k = 1
* Array extraction: [1, 2, 3]
* k = 1 % 3 = 1
* Rotated Array: [3, 1, 2]
* Rewrite List: 3 -> 1 -> 2
*
*/
```

### Optimized Solution (C++)
```cpp
// Intuition: Rotating the list by k places moves the last k nodes to the front. We can achieve this by making the list circular and breaking it at the appropriate spot.
// Approach:
// 1. Find the total length of the list and the original tail node.
// 2. Normalize k (k = k % length) to avoid redundant full rotations.
// 3. Connect the tail to the head to create a circular linked list.
// 4. Traverse (length - k) steps from the head to find the new tail of the rotated list.
// 5. Set the new head, break the circular link at the new tail, and return the new head.
// Time Complexity: O(n) where n is the number of nodes in the list. One pass to find the length, and a partial pass to find the new head.
// Space Complexity: O(1) as we are only using a few pointers for traversal and manipulation without taking any extra memory.
class Solution {
public:
    ListNode* rotateRight(ListNode* head, int k) {
        // Handle empty list, single node, or no rotation
        if(!head || !head->next || k == 0) {
            return head;
        }

        int length = 1;
        ListNode* tail = head;
        
        // Find length and tail node
        while(tail->next) {
            tail = tail->next;
            length++;
        }

        // Normalize k
        k = k % length;
        if(k == 0) {
            return head;
        }
        
        // Make it circular
        tail->next = head;

        // Find new tail position
        int steps = length - k;
        ListNode* newTail = head;
        
        // Move to the new tail
        for(int i = 1; i < steps; i++) {
            newTail = newTail->next;
        }

        // Break the circle
        ListNode* newHead = newTail->next;
        newTail->next = NULL;

        return newHead;
    }
};

/*
*
* Dry Run
* List: 1 -> 2 -> 3 -> 4 -> 5, k = 2
* Traverse to find length: length = 5, tail node = 5
* Normalize k: 2 % 5 = 2
* Make circular: node(5)->next = node(1)
* Find target steps: length - k = 5 - 2 = 3
* Move newTail 2 times (i=1 to 2): newTail lands on node(3)
* Set newHead: newHead = node(3)->next = node(4)
* Break circle: node(3)->next = NULL
* Return newHead: 4 -> 5 -> 1 -> 2 -> 3 -> NULL
*
*/
```

## 📌 Key Insights
* **Modulo Optimization:** Bypassing unnecessary full cycle rotations by normalizing `k %= length` reduces operational limits efficiently and ensures no out-of-bounds traversal errors occur.
* **In-Place Manipulation:** Connecting the tail to the head simplifies shifting blocks of nodes without needing secondary data structures or multiple disconnect-reconnect iterations.

---

## 🚀 Further Exploration
* **Doubly Linked List Variation:** Consider how this algorithm might adapt if the linked list had `prev` pointers. The core logic remains identical, but breaking the circle would involve updating two pointers per node instead of one.
* **Left Rotation:** If the problem asked for left rotation instead of right, the calculation for the new tail changes slightly to simply stepping `k` times instead of `length - k`.
