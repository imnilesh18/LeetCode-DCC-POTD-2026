# 788. Rotated Digits

**Difficulty:** Medium  
**Topics:** Math, Dynamic Programming  
**Problem Link:** [LeetCode - Rotated Digits](https://leetcode.com/problems/rotated-digits/description/)

---

## 📝 Problem Statement

An integer `x` is a **good** number if after rotating each digit individually by 180 degrees, we get a valid number that is different from `x`. Each digit must be rotated - we cannot choose to leave it alone.

A number is valid if each digit remains a digit after rotation. For example:
* `0`, `1`, and `8` rotate to themselves.
* `2` and `5` rotate to each other (in this case they are rotated in a different direction, in other words, `2` or `5` gets mirrored).
* `6` and `9` rotate to each other.
* The rest of the numbers (`3`, `4`, `7`) do not rotate to any other number and become **invalid**.

Given an integer `n`, return the number of **good** integers in the range `[1, n]`.

---

## 🚫 Constraints

* `1 <= n <= 10^4`

---

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `n = 10` | `4` | There are four good numbers in the range `[1, 10]` : `2`, `5`, `6`, `9`.<br>Note that `1` and `10` are not good numbers, since they remain unchanged after rotating. |
| **2** | `n = 1` | `0` | `1` rotates to `1`, which is not a different number. |
| **3** | `n = 2` | `1` | `2` rotates to `5`, which is a valid and different number. |

## 🧠 Approach & Explanation

To solve this problem, we need to inspect the digits of every number in the range from `1` to `n`. 

Based on the 180-degree rotation rules, we can divide the digits 0-9 into three distinct categories:
1. **Invalid Digits (`3`, `4`, `7`)**: If a number contains any of these digits, rotating it will result in an invalid number. Thus, the entire number becomes invalid.
2. **Valid but Unchanged Digits (`0`, `1`, `8`)**: Rotating these digits results in the exact same digit. If a number consists *only* of these digits (e.g., `10`, `801`), it will rotate into itself. It is valid, but it is not a *different* number, so it is not a "good" number.
3. **Valid and Changing Digits (`2`, `5`, `6`, `9`)**: Rotating these yields a valid, completely different digit. 

**Core Logic:**
A number is considered "good" if and only if:
1. It contains **zero** invalid digits (`3, 4, 7`).
2. It contains **at least one** changing digit (`2, 5, 6, 9`).

### 1. Brute Force Simulation
We can iterate through all numbers from `1` to `n`. For each number, we extract its digits one by one using the modulo (`% 10`) operator and check the categories. If we spot an invalid digit, we immediately discard the number. If we spot a changing digit, we flag it as changed. If the loop finishes without hitting an invalid digit, and the flag is true, we increment our answer count.

### 2. Recursion with Memoization (Digit DP Concept)
We can optimize by recognizing overlapping subproblems. For any number `num`, its validity depends on:
* Its last digit (`num % 10`).
* The remaining prefix of the number (`num / 10`).
Instead of checking every digit from scratch, we can recursively find the state of `num / 10` and combine it with the state of `num % 10`. We define three states:
* `0`: Valid but unchanged.
* `1`: Valid and changed (Good).
* `2`: Invalid.
We use an array to memoize these states up to `n`.

### 3. Bottom-Up Dynamic Programming
Taking the recursive logic a step further, we can build the states from `1` to `n` iteratively. We maintain a DP array `t` of size `n + 1`. For each index `i`, its state is derived directly from `t[i / 10]` and the classification of `i % 10`. This eliminates recursion overhead entirely.

```cpp
// Intuition: We can determine if a number is "good" by inspecting each of its digits. A number is good if it doesn't contain any invalid digits (3, 4, 7) and contains at least one digit that changes its value upon rotation (2, 5, 6, 9).
// Approach: Iterate through all numbers from 1 to n. For each number, extract digits one by one using modulo 10. If an invalid digit is found, the number is bad. If a changing digit is found, track it. If no invalid digits are found and at least one changing digit exists, increment the count.
// Time Complexity: O(n * log10(n)) where n is the given range. For each number, we process its digits which takes log10(num) time.
// Space Complexity: O(1) as we only use a few variables for counting and digit extraction.
class Solution {
public:

    bool isGood(int num) {
        bool changed = false; // Flag to check if the number changes upon rotation

        while(num > 0) {
            int digit = num % 10; // Extract the last digit

            // If digit is 3, 4, or 7, rotation makes it invalid
            if(digit == 3 || digit == 4 || digit == 7) return false;
            // If digit is 2, 5, 6, or 9, rotation creates a new valid number
            if(digit == 2 || digit == 5 || digit == 6 || digit == 9) changed = true;

            num /= 10; // Move to the next digit
        }
        return changed; // Valid only if at least one digit changed
    }

    int rotatedDigits(int n) {
        int count = 0; // Initialize good number count

        // Check every number in the range [1, n]
        for(int i = 1; i <= n; i++) {
            if(isGood(i)) {
                count++; // Increment if the number is good
            }
        }
        return count;
    }
};

/*
*
* Dry Run
* Input: n = 10
* i = 1: isGood(1) -> digit: 1 -> changed: false -> returns false
* i = 2: isGood(2) -> digit: 2 -> changed: true -> returns true -> count = 1
* i = 3: isGood(3) -> digit: 3 -> invalid -> returns false
* i = 4: isGood(4) -> digit: 4 -> invalid -> returns false
* i = 5: isGood(5) -> digit: 5 -> changed: true -> returns true -> count = 2
* i = 6: isGood(6) -> digit: 6 -> changed: true -> returns true -> count = 3
* i = 7: isGood(7) -> digit: 7 -> invalid -> returns false
* i = 8: isGood(8) -> digit: 8 -> changed: false -> returns false
* i = 9: isGood(9) -> digit: 9 -> changed: true -> returns true -> count = 4
* i = 10: isGood(10) -> digits: 0, 1 -> changed: false -> returns false
* Output: 4
*
*/
```

<summary><b>Approach 2: Recursion with Memoization (C++)</b></summary>

```cpp
// Intuition: Many numbers share common prefixes/bases (e.g., 12, 120, 125). We can break down the problem into subproblems: the validity of a number depends on its last digit and the validity of the remaining prefix (num / 10).
// Approach: Use recursion to find the state of `num / 10` and combine it with the state of `num % 10`. We define 3 states: 0 (valid but same), 1 (valid and changed/good), 2 (invalid). Memoize results in an array `t` to avoid redundant calculations.
// Time Complexity: O(n) because we memoize the state of each number from 1 to n, computing each state in O(1) time once its subproblems are solved.
// Space Complexity: O(n) for the memoization array `t` and the recursive call stack depth.
class Solution {
public:
    vector<int> t; // Memoization array to store states

    int solve(int num) {
        // Return cached result if already computed
        if(t[num] != -1)
            return t[num];

        // Base case: 0 remains 0 upon rotation (valid but unchanged)
        if(num == 0)
            return t[num] = 0;
        
        int remain = solve(num / 10); // Solve for the remaining part of the number
        
        // If the remaining part is invalid, the whole number is invalid
        if(remain == 2)
            return t[num] = 2;

        int digit_check;
        int d = num % 10; // Extract current digit

        // Classify the current digit into 3 states
        if (d == 0 || d == 1 || d == 8) 
            digit_check = 0;       // same
        else if (d == 2 || d == 5 || d == 6 || d == 9) 
            digit_check = 1;       // good
        else 
            return t[num] = 2;     // invalid

        // If both the remaining part and current digit are unchanged
        if(remain == 0 && digit_check == 0) {
            return t[num] = 0;
        }

        // Otherwise, it must be valid and changed (since invalid cases are caught above)
        return t[num] = 1;
        
    }

    int rotatedDigits(int n) {
        int count = 0;
        t.resize(n + 1, -1); // Initialize memo array with -1

        // 0 : same
        // 1 : Good
        // 2 : invalid
        for(int i = 1; i <= n; i++) {
            if(solve(i) == 1) { // Check if the number falls in the "Good" state
                count++;
            }
        }

        return count;
    }
};

/*
*
* Dry Run
* Input: n = 10, t initialized to -1
* solve(1): num=1 -> remain=solve(0)=0. d=1 -> digit_check=0. remain==0 && check==0 -> t[1]=0.
* solve(2): num=2 -> remain=solve(0)=0. d=2 -> digit_check=1. returns 1. t[2]=1. count=1.
* solve(3): num=3 -> remain=solve(0)=0. d=3 -> returns 2. t[3]=2.
* ...
* solve(5): num=5 -> remain=0. d=5 -> digit_check=1. returns 1. t[5]=1. count=2.
* ...
* solve(10): num=10 -> remain=solve(1)=0. d=0 -> digit_check=0. remain==0 && check==0 -> t[10]=0.
* Output: 4
*
*/
```

<summary><b>Approach 3: Bottom-Up Dynamic Programming (C++)</b></summary>

```cpp
// Intuition: Since the recursive approach processes numbers sequentially and depends purely on `i / 10` and `i % 10`, we can directly build the solution iteratively using a 1D DP array.
// Approach: Create a DP array `t` where `t[i]` stores the state of number `i` (0=same, 1=good, 2=invalid). For each `i` from 1 to n, its state is determined by combining the pre-computed state of `i / 10` and the nature of its last digit `i % 10`. Count the numbers that end up in state 1.
// Time Complexity: O(n) as we iterate from 1 to n exactly once, performing O(1) operations per number.
// Space Complexity: O(n) to store the states of numbers up to n in the DP array `t`.
class Solution {
public:
    int rotatedDigits(int n) {
        vector<int> t(n + 1, -1); // DP array to store states

        t[0] = 0; // Base case: 0 is valid but unchanged
        int count = 0;

        for(int i = 1; i <= n; i++) {
            int remain = t[i / 10]; // Get state of the number without the last digit

            // If prefix is invalid, current number is invalid
            if(remain == 2) {
                t[i] = 2;
                continue;
            }

            int d = i % 10; // Extract last digit
            int digit_check;

            // Evaluate the current digit
            if (d == 0 || d == 1 || d == 8) 
                digit_check = 0;       // same
            else if (d == 2 || d == 5 || d == 6 || d == 9) 
                digit_check = 1;       // good
            else {
                t[i] = 2;              // invalid
                continue;
            }

            // If both prefix and current digit are unchanged, number is unchanged
            if(remain == 0 && digit_check == 0)
                t[i] = 0;
            else
                t[i] = 1; // Otherwise, it's a good number

            if(t[i] == 1) {
                count++; // Increment count if state is "Good"
            }
        }

        return count;
    }
};

/*
*
* Dry Run
* Input: n = 10, t initialized to -1
* t[0] = 0
* i = 1: remain=t[0]=0, d=1 -> check=0. t[1]=0. count=0
* i = 2: remain=t[0]=0, d=2 -> check=1. t[2]=1. count=1
* i = 3: remain=t[0]=0, d=3 -> invalid. t[3]=2. count=1
* i = 4: remain=t[0]=0, d=4 -> invalid. t[4]=2. count=1
* i = 5: remain=t[0]=0, d=5 -> check=1. t[5]=1. count=2
* i = 6: remain=t[0]=0, d=6 -> check=1. t[6]=1. count=3
* i = 7: remain=t[0]=0, d=7 -> invalid. t[7]=2. count=3
* i = 8: remain=t[0]=0, d=8 -> check=0. t[8]=0. count=3
* i = 9: remain=t[0]=0, d=9 -> check=1. t[9]=1. count=4
* i = 10: remain=t[1]=0, d=0 -> check=0. remain==0 & check==0 -> t[10]=0. count=4
* Output: 4
*
*/
```

## 📌 Key Insights
* **State Transition:** The problem perfectly models a state machine. Every number inherits the baseline validity of its prefix (`num / 10`), combined with the property of its current tail digit (`num % 10`). 
* **Short Circuiting:** By immediately checking for invalid digits (`3, 4, 7`), we save computation time. If a prefix is ruined, any number built upon it is irreparably invalid.
* **Optimization Progression:** The thought process naturally flows from naive simulation -> observing overlapping subproblems (Memoization) -> iteratively building from base cases (Tabulation DP).
