# 796. Rotate String

**Difficulty:** Easy  
**Topics:** String, String Matching  

## 📝 Problem Statement
Given two strings `s` and `goal`, return `true` if and only if `s` can become `goal` after some number of **shifts** on `s`.

A **shift** on `s` consists of moving the leftmost character of `s` to the rightmost position.
* For example, if `s = "abcde"`, then it will be `"bcdea"` after one shift.

---

## 🚦 Constraints
* `1 <= s.length, goal.length <= 100`
* `s` and `goal` consist of lowercase English letters.

---

## 🎯 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **Example 1** | `s = "abcde", goal = "cdeab"` | `true` | Rotating `s` two times to the left results in `"cdeab"`, which matches the goal. |
| **Example 2** | `s = "abcde", goal = "abced"` | `false` | No matter how many times `s` is rotated, it will never equal `"abced"`. |

---

## 🧠 Approach

Based on the provided breakdown, there are two primary ways to approach this problem:

### 1. Brute Force (Simulation)
The most straightforward method is to physically rotate the string and check it against the `goal`. 
* A string of length $n$ can be rotated at most $n$ times before it repeats its original state.
* We can use a loop to shift the first character to the back $n$ times, verifying if the current state of the string matches `goal` at each step.

### 2. Optimized (String Concatenation)
There is a clever mathematical property of strings we can leverage. 
* If you concatenate a string with itself (i.e., `s + s`), the newly formed string will inherently contain **all possible valid rotations** of the original string `s` as continuous substrings.
* For instance, if `s = "abc"`, then `s + s = "abcabc"`. Notice how all rotations (`"bca"`, `"cab"`, `"abc"`) exist perfectly within this new string.
* By simply creating `s + s` and checking if `goal` is a substring (while ensuring both original strings have identical lengths), we can solve the problem in a single line of logic.

---

## 💻 Solution Code

### Brute Force Solution

```cpp
// Intuition: We can simulate the process of rotating the string one character at a time. A string of length 'm' returns to its original form after 'm' rotations, so checking 'm' variations is exhaustive.
// Approach: Iterate 'm' times. In each step, rotate the string left by one character using std::rotate. Check if the newly rotated 's' equals 'goal'.
// Time Complexity: O(n^2), where 'n' is the length of the string. std::rotate takes O(n) time, and we execute it 'n' times.
// Space Complexity: O(1), as we modify the string in-place without using extra auxiliary memory.
class Solution {
public:
    bool rotateString(string s, string goal) {
        int m = s.length();
        int n = goal.length();

        // Lengths must be equal for a valid rotation
        if (m != n) 
            return false;

        // Try all possible rotations of the string
        for (int rotationCount = 1; rotationCount <= m; ++rotationCount) {
            // Perform one rotation
            rotate(s.begin(), s.begin() + 1, s.end()); //it says that I want (s.begin()+1)th character to become the first character now. i.e. shifting left by one
            
            // Check for match after shifting
            if (s == goal) 
                return true;
        }
        return false;
    }
};

/*
*
* Dry Run
* s = "abcde", goal = "cdeab"
* m = 5, n = 5
* Iteration 1: rotate "abcde" -> "bcdea". "bcdea" != "cdeab"
* Iteration 2: rotate "bcdea" -> "cdeab". "cdeab" == "cdeab" -> Returns true!
*
*/
```

### Optimized Solution

```cpp
// Intuition: If we concatenate a string to itself (s + s), the resulting string contains all possible valid rotations of 's' as contiguous substrings.
// Approach: First, verify lengths are equal. Then, concatenate 's' with 's'. Finally, use the find() method to check if 'goal' exists as a substring within the concatenated result.
// Time Complexity: O(n), where 'n' is the length of the string. String concatenation takes O(n), and substring search (like KMP internally) operates in linear time.
// Space Complexity: O(n), as we create a new string of length 2n to store the concatenation of 's + s'.
class Solution {
public:
    bool rotateString(string s, string goal) {
        int m = s.length();
        int n = goal.length();
        
        // Match lengths and search for goal in the concatenated string
        if(m == n && (s+s).find(goal) != string::npos)
            return true;
        
        return false;
    }
};

/*
*
* Dry Run
* s = "abcde", goal = "cdeab"
* m = 5, n = 5 (Lengths match)
* Create concatenated string: s + s = "abcde" + "abcde" = "abcdeabcde"
* Search for goal: "cdeab" is found inside "ab[cdeab]cde" at index 2.
* Returns true.
*
*/
```

## 📓 Notes
* It is critical to check `s.length() == goal.length()` in the concatenation approach. If `s = "a"` and `goal = "aa"`, `goal` would be found in `s + s` (`"aa"`), but it is obviously not a valid rotation.

## 💡 Key Insights
* **The Concatenation Trick**: The `s + s` technique is a standard string manipulation paradigm that elegantly flattens circular array/string problems into linear substring searches. Understanding this will help heavily with harder cyclic string problems.
* **STL Tools**: The C++ standard library includes `std::rotate`, which is highly optimized for shifting elements in arrays and strings. Familiarity with internal implementations of STL functions is a hallmark of expert programming.

## 🚀 Further Exploration
* Consider how you would manually write the substring search algorithm (like KMP or Rabin-Karp) instead of relying on the built-in `.find()` method to ensure a strict $O(N)$ worst-case time complexity.
