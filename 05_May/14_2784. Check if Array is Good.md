## 📝 Problem Statement

You are given an integer array `nums`. We consider an array **good** if it is a permutation of an array `base[n]`.

`base[n] = [1, 2, ..., n - 1, n, n]` (in other words, it is an array of length `n + 1` which contains `1` to `n - 1` exactly once, plus two occurrences of `n`). 

For example, `base[1] = [1, 1]` and `base[3] = [1, 2, 3, 3]`.

Return `true` if the given array is **good**, otherwise return `false`.

> **Note:** A permutation of integers represents an arrangement of these numbers.

---

## ⚠️ Constraints

- `1 <= nums.length <= 100`
- `1 <= nums[i] <= 200`

## 🧠 Approach & Explanation

Understanding the core concept of a `base[n]` array is crucial. A valid `base[n]` array must always satisfy three conditions:
1. **Size check:** Its length must be exactly `n + 1` (where `n` is the maximum element in the array).
2. **Uniqueness constraint:** Every number from `1` to `n - 1` must appear exactly **once**.
3. **Duplicate constraint:** The maximum element `n` must appear exactly **twice**.

We can solve this using three different logic paths, ranging from straightforward frequency counting to highly optimized in-place indexing:

### 1. Frequency Count (Hash Map / Array)
Find the maximum element `maxEl` in the array. If the size of the array is not `maxEl + 1`, we can instantly return `false`. Otherwise, iterate through the array and use a frequency map (or size-based array) to count occurrences. Any element other than `maxEl` appearing more than once, or `maxEl` appearing more than twice, invalidates the array.

### 2. Sorting Method
By sorting the input array, we align its structure to ideally match `1, 2, 3, ..., n, n`. Once sorted, any element at index `i` (from `0` to `n-2`) must logically equal `i + 1`. The final element at the end of the array must be the same as the element right before it, which should be equal to the length of the array minus one. 

### 3. In-place Marker Technique (Most Optimal)
Because elements represent an unbroken sequence from `1` to `n-1` (and `n` twice), we can use the elements themselves as valid indices. We can traverse the array, taking the absolute value of the current element, and flipping the sign of the value located at that index to mark it as "visited." If we hit an index that is *already* negative, it means a duplicate has been found. Only the maximum element is allowed to be duplicated.

## 💻 Solution Codes

### Approach 1: Frequency Count

```cpp
// Intuition: A valid array has elements 1 to max-1 exactly once, and the max element exactly twice. We can track this using a frequency count.
// Approach: 
// 1. Find the maximum element (`maxEl`).
// 2. Validate the size of the array against `maxEl + 1`.
// 3. Keep a tally of element occurrences.
// 4. Return false if boundaries are violated (i.e., extra duplicates).
// Time Complexity: O(N) where N is the size of the array. Finding the max and counting frequencies both take linear time.
// Space Complexity: O(N) since we allocate a frequency array of size maxEl + 1.

class Solution {
public:
    bool isGood(vector<int>& nums) {
        // Find the maximum element in the array
        int maxEl = *max_element(begin(nums), end(nums));

        // Array size must strictly be max + 1
        if(nums.size() != maxEl + 1)
            return false;

        // Tally vector to track occurrences
        vector<int> count(maxEl + 1, 0);

        for(int i = 0; i < nums.size(); i++) {
            count[nums[i]]++;

            // If non-max element appears more than once, it's invalid
            if(nums[i] != maxEl && count[nums[i]] > 1)
                return false;
            // If max element appears more than twice, it's invalid
            else if(nums[i] == maxEl && count[nums[i]] > 2)
                return false;
        }

        return true;
    }
};

/*
* Dry Run
* Input: nums = [1, 3, 3, 2]
* maxEl = 3, size = 4. Size validation passes (4 == 3 + 1).
* frequency array initialized: count = [0, 0, 0, 0]
* i=0: nums[0]=1 -> count[1]=1. Valid.
* i=1: nums[1]=3 -> count[3]=1. Valid.
* i=2: nums[2]=3 -> count[3]=2. Valid (since 3 is maxEl, up to 2 is allowed).
* i=3: nums[3]=2 -> count[2]=1. Valid.
* End of loop, no violations. Returns true.
*/

```

### Approach 2: Sorting

```cpp
// Intuition: If the array is "good", sorting it should perfectly yield the sequence [1, 2, ..., n-1, n, n].
// Approach:
// 1. Sort the input array in ascending order.
// 2. Iterate up to the second to last index. Verify that value at index `i` equals `i + 1`.
// 3. Verify the final element matches the expected duplicated max element (n - 1).
// Time Complexity: O(N log N) dominated by the sorting operation.
// Space Complexity: O(1) auxiliary space (excluding the recursion stack of the sorting algorithm).

class Solution {
public:
    bool isGood(vector<int>& nums) {
        int n = nums.size();
        
        // Sort array to naturally align values
        sort(begin(nums), end(nums));

        // Check 1 to n-1 condition (up to the second to last element)
        for(int i = 0; i <= n - 2; i++) {
            if(nums[i] != i + 1)
                return false;
        }

        // Final element must duplicate the previous (which is n-1 based on length)
        return nums[n - 1] == n - 1;
    }
};

/*
* Dry Run
* Input: nums = [2, 1, 3] -> sort -> nums = [1, 2, 3]
* size n = 3. Loop goes up to i <= 1.
* i=0: nums[0] (1) == 0+1. Valid.
* i=1: nums[1] (2) == 1+1. Valid.
* Loop finishes. 
* Final return checks if nums[2] (3) == 3-1 (2). 
* 3 == 2 is false. Return false.
*/

```

### Approach 3: In-Place Marking (Optimal Space)

```cpp
// Intuition: We can leverage the input values as literal indices. Modifying the sign of values at those indices allows us to detect duplicates with constant extra space.
// Approach:
// 1. Establish expected max element based on array size (n - 1).
// 2. Iterate through nums. Take `abs(val)` to safely use as an index mapping.
// 3. If value exceeds `maxEl`, fail immediately.
// 4. If we hit an index containing a negative value, we found a duplicate. Allow it ONLY if it's the `maxEl` AND it's only the second occurrence.
// 5. Otherwise, flip the sign of the element at the mapped index.
// Time Complexity: O(N) as we traverse the elements only a single time.
// Space Complexity: O(1) space since array markings modify the array directly.

class Solution {
public:
    bool isGood(vector<int>& nums) {
        int n = nums.size();
        
        // Based on length `n`, expected max element is `n - 1`
        int maxEl = n - 1;
        int maxElCount = 0;

        for(int num : nums) {
            int val = abs(num);

            // Any value strictly larger than max expected is invalid
            if(val > maxEl) {
                return false;
            } else if(val == maxEl) {
                maxElCount++;
            }

            // If the element at this mapped index is already negative, we've visited it!
            if(nums[val] < 0) {
                // If the repeated element is NOT the max element, invalid.
                if(val != maxEl) {
                    return false;
                } 
                // If it IS max element but we've seen it > 2 times, invalid.
                else if(maxElCount > 2) {
                    return false;
                }
            } else {
                // Mark the index as visited by negating its value
                nums[val] *= -1;
            }
        }

        return true;
    }
};

/*
* Dry Run
* Input: nums = [1, 3, 3, 2], n = 4, maxEl = 3, maxElCount = 0
* i=0: num=1, val=1. nums[1] is 3. Mark visited: nums[1] = -3.
* i=1: num=-3, val=3. maxElCount=1. nums[3] is 2. Mark visited: nums[3] = -2.
* i=2: num=3, val=3. maxElCount=2. nums[3] is -2 (Negative! Visited!). 
* Duplicate check: val == maxEl(3) & maxElCount <= 2. Valid.
* i=3: num=-2, val=2. nums[2] is 3. Mark visited: nums[2] = -3.
* End of array reached with no violations. Return true.
*/

```

## 📝 Key Insights & Further Exploration

* **Time vs. Space Trade-offs:** The frequency array limits time complexity to strictly linear `O(N)` while demanding `O(N)` space. The sorting approach guarantees `O(1)` space but slows down execution to `O(N log N)`.
* **Index-Mapping Magic:** Using input array elements as structural bounds (array marking) is a highly prominent trick in array sequence algorithms (e.g., `Find All Duplicates in an Array`, `First Missing Positive`). Whenever a problem constrains element bounds strictly to the length array sizes, keep in-place state manipulation in mind!
