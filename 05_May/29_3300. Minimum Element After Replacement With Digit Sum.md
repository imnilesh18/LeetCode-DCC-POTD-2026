## 🔗 Problem Link
**[3300. Minimum Element After Replacement With Digit Sum](https://leetcode.com/problems/minimum-element-after-replacement-with-digit-sum/)**

## 📝 Problem Statement
You are given an integer array `nums`.

You replace each element in `nums` with the sum of its digits.

Return the minimum element in `nums` after all replacements.

## 🎯 Examples

| Example | Input | Output | Explanation |
| :---: | :--- | :---: | :--- |
| **1** | `nums = [10,12,13,14]` | `1` | `nums` becomes `[1, 3, 4, 5]` after all replacements, with minimum element `1`. |
| **2** | `nums = [1,2,3,4]` | `1` | `nums` becomes `[1, 2, 3, 4]` after all replacements, with minimum element `1`. |
| **3** | `nums = [999,19,199]` | `10` | `nums` becomes `[27, 10, 19]` after all replacements, with minimum element `10`. |

## 🔒 Constraints
- `1 <= nums.length <= 100`
- `1 <= nums[i] <= 10^4`

---

## 💡 Approach & Explanation

The goal is to calculate the sum of the digits for every integer inside the array and find the smallest derived value.

We can achieve this using two main approaches:
1.  **Iterative Mathematical Approach:** Standard modulo and division operations dynamically peel off each digit from the number until it hits `0`. We accumulate these digits to form the sum.
2.  **Constraint-Optimized Approach:** Leveraging the tight constraints (`nums[i] <= 10^4`), we definitively know the number has at most 5 digits. We can extract these five digits explicitly in $O(1)$ constant time using hard-coded positional checks instead of executing a `while` loop.

For both approaches, note that the maximum sum achievable is `36` (which occurs for the number `9999` under the constraint $10^4$). We safely initialize our minimum tracker variable to `37`.

## 💻 Brute-force / Iterative Solution 
**Language:** C++

```cpp
// Intuition: We can find the sum of digits of a number by repeatedly extracting the last digit using modulo 10 and removing it using division by 10.
// Approach: Iterate over each number in the array. For each number, calculate its digit sum using a loop. Keep track of the minimum digit sum found so far. We initialize the minimum with 37 since the maximum possible sum for nums[i] <= 10^4 is 9+9+9+9 = 36.
// Time Complexity: O(n * log10(m)) where n is the number of elements and m is the maximum element in nums. The while loop runs log10(num) times for each number.
// Space Complexity: O(1) as we are only using a few variables for tracking the sum and the minimum result.

class Solution {
public:
    // Helper function to calculate the sum of digits of a number
    int digitSum(int n) {
        int sum = 0;

        // Extract and add digits until the number becomes 0
        while(n > 0) {
            sum += n % 10; // Get the last digit
            n /= 10;       // Remove the last digit
        }

        return sum;
    }

    int minElement(vector<int>& nums) {
        // Note that 1 <= nums[i] <= 10^4 and biggest sum can be obtained by 9999 (whose digit sum = 36)
        int result = 37; 

        for(int num : nums) {
            int s = digitSum(num); // Calculate digit sum, O(log10(num))

            // Update the minimum element found so far
            result = min(result, s);
        }

        return result;
    }
};

/*
*
* Dry Run
* Input: nums = [10, 12, 13, 14]
* Initial result = 37
* Loop 1: num = 10, digitSum(10) = 1 + 0 = 1. result = min(37, 1) = 1.
* Loop 2: num = 12, digitSum(12) = 1 + 2 = 3. result = min(1, 3) = 1.
* Loop 3: num = 13, digitSum(13) = 1 + 3 = 4. result = min(1, 4) = 1.
* Loop 4: num = 14, digitSum(14) = 1 + 4 = 5. result = min(1, 5) = 1.
* Final Result: 1
*
*/
```

## 🚀 Optimized Solution
**Language:** C++

```cpp
// Intuition: Since the problem constraints strictly limit the maximum value of nums[i] to 10^4, a number will have at most 5 digits. We can extract these digits directly using fixed division and modulo operations in constant time.
// Approach: Instead of using a loop, we extract the units, tens, hundreds, thousands, and ten-thousands digits sequentially by combining integer division and modulo 10. Sum these parts up and track the minimum sum across the array.
// Time Complexity: O(n) where n is the number of elements in the array. The digit sum calculation is strictly O(1) for every element without loop overhead.
// Space Complexity: O(1) since we do not require any extra auxiliary space.

class Solution {
public:
    // O(1) function to calculate digit sum given the constraint num <= 10^4
    int digitSum(int n) {
        return
            (n / 10000) +           // ten-thousands digit
            (n % 10000 / 1000) +    // thousands digit
            (n % 1000 / 100) +      // hundreds digit
            (n % 100 / 10) +        // tens digit
            (n % 10);               // units digit
    }

    int minElement(vector<int>& nums) {
        // Note that 1 <= nums[i] <= 10^4 and biggest sum can be obtained by 9999 (whose digit sum = 36)
        int result = 37; 

        for(int num : nums) {
            int s = digitSum(num); // Calculate digit sum in O(1)

            // Update the global minimum sum
            result = min(result, s);
        }

        return result;
    }
};

/*
*
* Dry Run
* Input: nums = [999, 19, 199]
* Initial result = 37
* Loop 1: num = 999. digitSum(999) = 0 + 0 + 9 + 9 + 9 = 27. result = min(37, 27) = 27.
* Loop 2: num = 19. digitSum(19) = 0 + 0 + 0 + 1 + 9 = 10. result = min(27, 10) = 10.
* Loop 3: num = 199. digitSum(199) = 0 + 0 + 1 + 9 + 9 = 19. result = min(10, 19) = 10.
* Final Result: 10
*
*/
```

## 🔍 Key Insights
- Always keep an eye on problem constraints! Optimizations like bypassing `while` loops completely are only viable because we know the boundary conditions (`10^4`). 
- Using `37` as the default minimum variable effectively handles all boundary test cases securely without using generic infinity limits like `INT_MAX`.
