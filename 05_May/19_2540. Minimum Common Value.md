# 2540. Minimum Common Value

## 📝 Problem Statement

Given two integer arrays `nums1` and `nums2`, sorted in **non-decreasing order**, return the *minimum integer common to both arrays*. If there is no common integer amongst `nums1` and `nums2`, return `-1`.

Note that an integer is said to be common to `nums1` and `nums2` if both arrays have at least one occurrence of that integer.

---

## 🛑 Constraints

* `1 <= nums1.length, nums2.length <= 10^5`
* `1 <= nums1[i], nums2[j] <= 10^9`
* Both `nums1` and `nums2` are sorted in **non-decreasing order**.

---

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `nums1 = [1,2,3], nums2 = [2,4]` | `2` | The smallest element common to both arrays is 2, so we return 2. |
| **2** | `nums1 = [1,2,3,6], nums2 = [2,3,4,5]` | `2` | There are two common elements in the array 2 and 3 out of which 2 is the smallest, so 2 is returned. |

---

## 🧠 Approach

Based on the problem requirements, there are three primary ways to approach the solution, each optimizing the search process differently:

### 1. Hash Set (Brute Force / Naive Optimized)
The simplest way to find common elements is to check if an element from `nums1` exists in `nums2`. To do this efficiently, we can store all elements of `nums1` in an `unordered_set`. Then, we iterate through `nums2` from left to right. Since `nums2` is sorted, the first element we find that also exists in the set is guaranteed to be the minimum common value. 

### 2. Binary Search
Since both arrays are already sorted, storing elements in a Set uses unnecessary extra space. Instead, we can iterate through the elements of `nums1` and use **Binary Search** to find if that element exists in `nums2`. Since we process `nums1` from left to right (smallest to largest), the first element we successfully find in `nums2` will be the answer. 

### 3. Two Pointers (Most Optimized)
The most optimal solution takes advantage of the fact that *both* arrays are sorted. We can place a pointer at the beginning of each array (`i` for `nums1`, `j` for `nums2`). We then compare the elements at both pointers:
* If the elements are **equal**, we have found our minimum common value.
* If `nums1[i] < nums2[j]`, the element in `nums1` is too small, so we increment `i`.
* If `nums1[i] > nums2[j]`, the element in `nums2` is too small, so we increment `j`.
This approach completely avoids extra space and reduces the time complexity to a strict linear traversal.

## 💻 Solution Code

### Approach 1: Hash Set

```cpp
// Intuition: Since we need to find a common element efficiently, we can store all elements of one array in a Hash Set to achieve O(1) average time complexity for lookups.
// Approach: Insert all elements of `nums1` into an unordered_set. Then, iterate through `nums2` and check if the current element exists in the set. The first matched element is the minimum common value since `nums2` is sorted in non-decreasing order.
// Time Complexity: O(m + n), where m and n are the sizes of nums1 and nums2. We traverse both arrays.
// Space Complexity: O(m), as we store the elements of nums1 in the hash set.
class Solution {
public:
    int getCommon(vector<int>& nums1, vector<int>& nums2) {
        // Store all elements of nums1 in a hash set
        unordered_set<int> st(begin(nums1), end(nums1));
        
        // Iterate through the sorted nums2 array
        for(int &num : nums2) {
            // Check if the current element exists in the set
            if(st.find(num) != st.end()) {
                return num; // First match is the minimum common value
            }
        }
        
        return -1; // If no common element is found
    }
};

/*
 * Dry Run
 * nums1 = [1,2,3], nums2 = [2,4]
 * 1. Set `st` stores {1, 2, 3}
 * 2. Loop nums2:
 *    - num = 2: st.find(2) != st.end() is true. Return 2.
 * 3. Result: 2
 */
```

### Approach 2: Binary Search

```cpp
// Intuition: Since both arrays are already sorted, we can leverage Binary Search to find elements of `nums1` inside `nums2` without using extra space.
// Approach: Iterate through each element of `nums1`. For each element, perform a binary search on `nums2`. If the element is found, return it immediately. This works optimally because `nums1` is sorted, guaranteeing the first match is the minimum.
// Time Complexity: O(m log n), where m is the size of nums1 and n is the size of nums2. We do a binary search of O(log n) for each of the m elements.
// Space Complexity: O(1), as no auxiliary space is used.
class Solution {
public:
    // Standard binary search helper function
    bool binarySearch(vector<int>& nums, int target) {
        int l = 0, r = nums.size()-1;
        
        while(l <= r) {
            int mid = l + (r-l)/2;
            
            // Target found
            if(nums[mid] == target) {
                return true;
            } else if(nums[mid] < target) {
                l = mid+1; // Search right half
            } else {
                r = mid-1; // Search left half
            }
        }
        
        return false; // Target not found
    }
    
    int getCommon(vector<int>& nums1, vector<int>& nums2) {
        // Iterate through the first sorted array
        for(int &num : nums1) {
            // Search for the element in the second array
            if(binarySearch(nums2, num))
                return num; // Return the first common element found
        }
        
        return -1; // No common element exists
    }
};

/*
 * Dry Run
 * nums1 = [1,2,3], nums2 = [2,4]
 * 1. num = 1 from nums1:
 *    - binarySearch(nums2, 1): l=0, r=1, mid=0. nums2[0]=2 > 1. r=-1. Returns false.
 * 2. num = 2 from nums1:
 *    - binarySearch(nums2, 2): l=0, r=1, mid=0. nums2[0]=2 == 2. Returns true.
 * 3. Match found, return 2.
 */
```

### Approach 3: Two Pointers (Optimized)

```cpp
// Intuition: Both arrays are sorted, which makes them perfect candidates for a Two-Pointer approach. We can compare elements from both arrays simultaneously.
// Approach: Initialize two pointers `i` and `j` to 0. Compare `nums1[i]` and `nums2[j]`. If they are equal, we found the minimum common value. If `nums1[i] < nums2[j]`, increment `i` to catch up. Otherwise, increment `j`.
// Time Complexity: O(m + n), where m and n are the sizes of the arrays. In the worst case, we traverse both arrays completely.
// Space Complexity: O(1), as we only use two integer pointers.
class Solution {
public:
    int getCommon(vector<int>& nums1, vector<int>& nums2) {
        int m = nums1.size();
        int n = nums2.size();
        
        int i = 0; // Pointer for nums1
        int j = 0; // Pointer for nums2
        
        // Traverse both arrays simultaneously
        while(i < m && j < n) {
            if(nums1[i] == nums2[j]) {
                return nums1[i]; // Common element found
            } else if(nums1[i] < nums2[j]) {
                i++; // Increment pointer i to find a larger value
            } else {
                j++; // Increment pointer j to find a larger value
            }
        }
        
        return -1; // No common element exists
    }
};

/*
 * Dry Run
 * nums1 = [1,2,3], nums2 = [2,4]
 * 1. i=0, j=0: nums1[0]=1, nums2[0]=2. 1 < 2, so i++ (i=1).
 * 2. i=1, j=0: nums1[1]=2, nums2[0]=2. 2 == 2. Match found!
 * 3. Return nums1[i] which is 2.
 */
```

## 🔑 Key Insights

* **Exploiting Sorted Order:** The non-decreasing order of the arrays is the biggest hint in this problem. It allows us to bypass entirely brute-force quadratic $O(m \times n)$ nested loop checks.
* **Space vs. Time Trade-offs:** The Hash Set method is excellent for unsorted array lookups but wastes memory here. Binary Search saves space but operates at $O(m \log n)$. The Two-Pointer method achieves the best of both worlds with $O(m+n)$ time and $O(1)$ space.
