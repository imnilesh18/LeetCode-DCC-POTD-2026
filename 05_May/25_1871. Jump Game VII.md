# 1871. Jump Game VII
**Difficulty: Medium**

---
**Problem Link:** [Jump Game VII](https://leetcode.com/problems/jump-game-vii/description/)
**Problem Name:** 1871. Jump Game VII
**Tags:** String, Dynamic Programming, Sliding Window, Prefix Sum
---

## 📝 Problem Statement

You are given a **0-indexed** binary string `s` and two integers `minJump` and `maxJump`. In the beginning, you are standing at index `0`, which is equal to `'0'`. You can move from index `i` to index `j` if the following conditions are fulfilled:

1. `i + minJump <= j <= min(i + maxJump, s.length - 1)`
2. `s[j] == '0'`

Return `true` if you can reach index `s.length - 1` in `s`, or `false` otherwise.

---

## ⚠️ Constraints

- `2 <= s.length <= 10^5`
- `s[i]` is either `'0'` or `'1'`.
- `s[0] == '0'`
- `1 <= minJump <= maxJump < s.length`

---

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `s = "011010", minJump = 2, maxJump = 3` | `true` | In the first step, move from index 0 to index 3. In the second step, move from index 3 to index 5. |
| **2** | `s = "01101110", minJump = 2, maxJump = 3` | `false` | It is impossible to reach the last index. |

## 🧠 Approach & Explanation

The problem asks whether we can reach the end of a binary string by jumping between indices that contain `'0'` within a specific range `[minJump, maxJump]`.

### Method 1: Recursion + Memoization (Brute Force)
From a current index `i`, we can try jumping to all possible indices `j` in the range `[i + minJump, i + maxJump]`. If `s[j]` is `'0'`, we make a recursive call from `j`. We can memoize the results to avoid re-evaluating the same index. However, since the jump range can be large, exploring every valid jump leads to a Time Complexity of `O(N * (maxJump - minJump))`, resulting in a Time Limit Exceeded (TLE) error.

### Method 2: Bottom-Up Dynamic Programming
Instead of recursion, we can iteratively check from the second-to-last index back to `0`. For each index `i`, we check if any valid jump lands on an index `j` from which the end is reachable. This is simply the tabular form of Method 1 and still suffers from the same `O(N * (maxJump - minJump))` complexity, causing TLE.

### Method 3: Sliding Window + DP (Optimized)
Instead of asking *"Where can I jump to from index `i`?"*, we shift our perspective and ask *"Which indices can jump to the current index `j`?"*
For an index `j`, any index `i` in the range `[j - maxJump, j - minJump]` can potentially jump to `j`. 
We maintain a sliding window of reachable indices in this range. A running `count` tracks how many valid jump sources exist for the current `j`. 
- As `j` increments, `j - minJump` enters the valid window, so we add its reachability status to `count`.
- Similarly, `j - maxJump - 1` leaves the window, so we subtract its status from `count`.
If `count > 0` and the current character `s[j]` is `'0'`, it means `j` is reachable. This brings the time complexity down to a highly optimized `O(N)`.

## 💻 Solution Codes

### Approach 1: Recursion + Memoization

```cpp
// Intuition: We can try every possible valid jump from the current index. If any jump leads to the end, return true. We use memoization to cache results.
// Approach: Recursively explore jumps from `idx + minJump` to `idx + maxJump`. If we land on '0', explore further. Memoize the reachability of each index.
// Time Complexity: O(n * (maxJump - minJump)), as we check a range of jumps for every index.
// Space Complexity: O(n) for the memoization array and the recursion stack.

class Solution {
public:
    int n;
    int solve(int idx, string& s, int minJump, int maxJump, vector<int>& t) {
        // reached last index successfully
        if (idx == n - 1) {
            return true;
        }
        // return already computed result
        if (t[idx] != -1) {
            return t[idx];
        }
        // try every possible jump within the allowed range
        for (int jump = minJump; jump <= maxJump; jump++) {
            int j = idx + jump;
            // out of bounds check
            if (j >= n) {
                break;
            }
            // can only jump on '0'
            if (s[j] == '0') {
                if (solve(j, s, minJump, maxJump, t)) {
                    return t[idx] = true;
                }
            }
        }
        // mark as false if no path leads to the end
        return t[idx] = false;
    }

    bool canReach(string s, int minJump, int maxJump) {
        n = s.length();
        vector<int> t(n, -1);
        return solve(0, s, minJump, maxJump, t);
    }
};

/*
*
* Dry Run
* s = "011010", minJump = 2, maxJump = 3
* Call solve(0) -> tries jump 2 (j=2, s[2]='1' invalid) -> tries jump 3 (j=3, s[3]='0' valid).
* Call solve(3) -> tries jump 2 (j=5, s[5]='0' valid).
* Call solve(5) -> Reached end! Returns true.
* Propagates true to solve(3) and solve(0). Final output: true.
*
*/
```

### Approach 2: Bottom-Up DP

```cpp
// Intuition: Instead of recursive top-down exploration, build the solution backwards.
// Approach: Iterate from n-2 down to 0. For each index, check all valid forward jumps to see if they can reach the end.
// Time Complexity: O(n * (maxJump - minJump)), due to the nested loops checking future reachable indices.
// Space Complexity: O(n) for the DP array `t`.

class Solution {
public:
    bool canReach(string s, int minJump, int maxJump) {
        int n = s.length();
        vector<int> t(n, false);
        // base case: last index is trivially reachable from itself
        t[n - 1] = true;
        
        // fill from back to front
        for (int i = n - 2; i >= 0; i--) {
            // try every possible jump
            for (int jump = minJump; jump <= maxJump; jump++) {
                int j = i + jump;
                // out of bounds, stop checking further jumps
                if (j >= n) {
                    break;
                }
                // can only jump on '0'
                if (s[j] == '0') {
                    // if target index is reachable to end, current is too
                    if (t[j]) {
                        t[i] = true;
                        break;
                    }
                }
            }
        }
        return t[0];
    }
};

/*
*
* Dry Run
* s = "011010", minJump = 2, maxJump = 3
* Initial t = [F, F, F, F, F, T]
* i = 4 (s[4]='1'): skips logic
* i = 3 (s[3]='0'): jumps to 5 (s[5]='0', t[5]=T). t[3] becomes T.
* i = 2 (s[2]='1'): skips
* i = 1 (s[1]='1'): skips
* i = 0 (s[0]='0'): jumps to 2 (invalid) and 3 (s[3]='0', t[3]=T). t[0] becomes T.
* Returns t[0] -> true.
*
*/
```

### Approach 3: Sliding Window + Bottom Up

```cpp
// Intuition: The nested loop causes TLE. Instead of checking forward jumps, check backward valid sources using a sliding window.
// Approach: For each index j, maintain a count of reachable indices in the window [j - maxJump, j - minJump]. Update this count as the window slides.
// Time Complexity: O(n), since each index is processed and added/removed from the window count in constant time.
// Space Complexity: O(n), to maintain the reachability status of each index.

class Solution {
public:
    bool canReach(string s, int minJump, int maxJump) {
        int n = s.length();

        vector<int> t(n, 0);
        // t[i] > 0 means it's possible to reach index i
        // t[i] == 0 means not possible

        t[0] = 1; // start index is always reachable
        int count = 0; // tracks number of reachable sources in the current window

        for(int j = 1; j <= n - 1; j++) {
            // include the new index entering our valid jump window
            if(j - minJump >= 0) {
                count += t[j - minJump];
            }

            // remove the index that just fell out of our valid jump window
            if(j - maxJump - 1 >= 0) {
                count -= t[j - maxJump - 1];
            }

            // if we have valid sources and current position is '0', mark reachable
            if(count > 0 && s[j] == '0') {
                t[j] = 1;
            }
        }

        return t[n - 1] > 0;
    }
};

/*
*
* Dry Run
* s = "011010", minJump = 2, maxJump = 3
* t = [1, 0, 0, 0, 0, 0]
* j = 1: window [-2, -1], count = 0. t[1] = 0.
* j = 2: window [-1, 0], count = t[0] = 1. s[2] = '1', so t[2] = 0.
* j = 3: window [0, 1], add t[1]=0, remove out-of-bound. count = 1. s[3] = '0', t[3] = 1.
* j = 4: window [1, 2], add t[2]=0, remove t[0]=1. count = 0. t[4] = 0.
* j = 5: window [2, 3], add t[3]=1, remove t[1]=0. count = 1. s[5] = '0', t[5] = 1.
* Output t[5] > 0 -> true.
*
*/
```
