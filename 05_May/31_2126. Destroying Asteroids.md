## 📖 Problem Statement

You are given an integer `mass`, which represents the original mass of a planet. You are further given an integer array `asteroids`, where `asteroids[i]` is the mass of the `i`th asteroid.

You can arrange for the planet to collide with the asteroids in any **arbitrary order**. If the mass of the planet is greater than or equal to the mass of the asteroid, the asteroid is destroyed and the planet gains the mass of the asteroid. Otherwise, the planet is destroyed.

Return `true` if all asteroids can be destroyed. Otherwise, return `false`.

---

## 🚦 Constraints

- `1 <= mass <= 10^5`
- `1 <= asteroids.length <= 10^5`
- `1 <= asteroids[i] <= 10^5`

## 🧠 Approach & Explanation

Even the smallest step forward taken with a quiet and honest heart is something to be proud of! Let's approach this problem step by step.

The most intuitive way to solve this is to tackle the smallest asteroids first. Think about it: if our planet has a mass of 10 and we face a massive asteroid of 21 right away, our planet gets destroyed. But if we fight the smaller ones first (like 3, then 5, then 9), our planet's mass continuously increases. By the time we reach the larger asteroids, our planet has grown massive enough to easily destroy them. 

This leads us to a classic **Greedy Algorithm**:
1. **Sort** the `asteroids` array in ascending order.
2. Iterate through each asteroid, starting from the smallest.
3. Compare the current planet `mass` with the `asteroid` mass. 
   - If `mass < asteroid`, the planet gets destroyed (return `false`).
   - If `mass >= asteroid`, the asteroid is destroyed, and its mass is added to the planet.

**Handling Integer Overflow:**
Because we are constantly adding the masses of asteroids, the planet's mass can exceed the maximum value an integer can hold. We can prevent this by either:
1. Using a `long long` variable to accumulate the mass.
2. Observing the constraints: The maximum possible mass of any asteroid is $10^5$. If our planet reaches a mass of $\ge 10^5$, it becomes invincible and can destroy *any* remaining asteroid. We can return `true` immediately to avoid computing huge sums!

## 💻 Brute-force / Basic Greedy Solution (C++)

<details>
<summary>Click to view the Standard Greedy Solution with Long Long</summary>

```cpp
// Intuition: To maximize our chances of survival, we greedily collide with the smallest asteroids first to increase our mass safely.
// Approach: 
// 1. Sort the asteroids in ascending order.
// 2. Use a long long data type for mass to prevent integer overflow during addition.
// 3. Linearly traverse the sorted array. Return false if the planet's mass is smaller than the current asteroid.
// 4. Otherwise, add the asteroid's mass to the planet.
// Time Complexity: O(n log n) - Sorting takes O(n log n) where n is the number of asteroids.
// Space Complexity: O(1) or O(log n) depending on the internal sorting algorithm stack space.

class Solution {
public:
    bool asteroidsDestroyed(int mass, vector<int>& asteroids) {
        // Sort asteroids to face smallest ones first
        sort(begin(asteroids), end(asteroids));

        // Use long long to handle potential massive sums safely
        long long totalMass = mass;

        // Traverse each asteroid
        for(int &asteroid : asteroids) {
            // If planet is too small, it's destroyed
            if(totalMass < asteroid)
                return false;

            // Planet absorbs asteroid mass
            totalMass += asteroid;
        }

        return true;
    }
};

/*
*
* Dry Run
* mass = 10, asteroids = [3, 9, 19, 5, 21]
* Sorted: [3, 5, 9, 19, 21]
* 1. 10 >= 3 -> mass = 13
* 2. 13 >= 5 -> mass = 18
* 3. 18 >= 9 -> mass = 27
* 4. 27 >= 19 -> mass = 46
* 5. 46 >= 21 -> mass = 67
* Result: true
*
*/
```

## 🚀 Optimized Solution (C++ with Early Exit)

```cpp
// Intuition: The maximum asteroid mass is bounded at 10^5. If our planet reaches a mass of 10^5 or higher, it is virtually invincible against any remaining asteroid.
// Approach:
// 1. Sort the asteroids ascendingly.
// 2. Iterate through the array checking collisions.
// 3. If mass < asteroid, return false.
// 4. If mass >= 1e5 (100,000), return true instantly to bypass unnecessary addition and potential integer overflow.
// Time Complexity: O(n log n) - Due to the initial sorting step.
// Space Complexity: O(1) - Modifying variables in-place without extra data structures.

class Solution {
public:
    bool asteroidsDestroyed(int mass, vector<int>& asteroids) {
        // Sort the array in ascending order
        sort(begin(asteroids), end(asteroids));

        for(int &asteroid : asteroids) {
            // Planet is destroyed by a larger asteroid
            if(mass < asteroid)
                return false;
            // Early exit: planet is invincible (>= max possible asteroid size)
            else if(mass >= 1e5)
                return true;

            // Safely accumulate mass
            mass += asteroid;
        }

        // Survived all asteroids
        return true;
    }
};

/*
*
* Dry Run
* mass = 5, asteroids = [4, 9, 23, 4]
* Sorted: [4, 4, 9, 23]
* 1. mass 5 >= 4 -> mass = 9
* 2. mass 9 >= 4 -> mass = 13
* 3. mass 13 >= 9 -> mass = 22
* 4. mass 22 < 23 -> return false
*
*/
```

## 📝 Notes & Key Insights
- **Greedy Choice Property:** This problem perfectly exhibits the greedy choice property. By continuously absorbing the smallest possible mass, we maximize our chances of absorbing the next immediate element.
- **Constant Limitations Trick:** Taking advantage of constraint boundaries (like `mass >= 10^5`) is a highly effective optimization technique in competitive programming to avoid data type scaling (e.g., bypassing the need for `long long`).
