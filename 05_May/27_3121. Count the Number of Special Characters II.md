## 📜 Problem Statement

You are given a string `word`. A letter `c` is called **special** if it appears both in lowercase and uppercase in `word`, and **every** lowercase occurrence of `c` appears before the **first** uppercase occurrence of `c`.

Return the number of special letters in `word`.

---

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `word = "aaAbcBC"` | `3` | The special characters are 'a', 'b', and 'c'. Their lowercase forms all appear before their first uppercase occurrences. |
| **2** | `word = "abc"` | `0` | There are no special characters in the word since no uppercase letters exist. |
| **3** | `word = "AbBCab"` | `0` | There are no special characters. For instance, lowercase 'a' and 'b' appear after their uppercase counterparts. |

---

## ⚙️ Constraints

- `1 <= word.length <= 2 * 10^5`
- `word` consists of only lowercase and uppercase English letters.

---

## 🧠 Approach

A brute-force approach (checking the left side of every uppercase letter to make sure no lowercase letters exist on its right) would result in an $O(N^2)$ time complexity, which will result in a Time Limit Exceeded (TLE) error given the constraints ($N \le 2 \times 10^5$). 

Instead, we can solve this optimally in $O(N)$ time:
1. **Track the Indicies:** The problem requires that *all* lowercase occurrences of a character appear strictly before its *first* uppercase occurrence. This essentially means the **last index** of the lowercase character must be smaller than the **first index** of the uppercase character.
2. **Use Frequency/Index Arrays:** Since there are only 26 English letters, we can use two arrays of size 26 initialized to `-1`:
   - `lastSmall` to store the last observed index of each lowercase letter.
   - `firstCapital` to store the first observed index of each uppercase letter.
3. **Iterate and Update:** Traverse the string once. If a character is lowercase, continuously update its index in `lastSmall`. If a character is uppercase, update its index in `firstCapital` *only if* it is currently `-1` (meaning we've found its first occurrence).
4. **Evaluate:** After the traversal, loop through the alphabet (0 to 25). If both arrays have valid indices (`!= -1`) for a given character, check if `lastSmall[i] < firstCapital[i]`. If true, we found a special character!

## 💻 Solution Code

```cpp
// Intuition: We need every lowercase instance to appear before the very first uppercase instance. By recording the last index of a lowercase letter and comparing it with the first index of its uppercase counterpart, we can efficiently verify this condition.
// Approach: 
// 1. Initialize two size-26 arrays (`lastSmall` and `firstCapital`) with -1.
// 2. Traverse the string. If the char is lowercase, continuously update its index in `lastSmall` (capturing the last occurrence).
// 3. If the char is uppercase, update its index in `firstCapital` only if it hasn't been set yet (capturing the first occurrence).
// 4. Iterate through 0 to 25. If both arrays have a valid index for a character and the `lastSmall` index is less than the `firstCapital` index, increment the count.
// Time Complexity: O(n) - where n is the length of the string, as we do a single pass to fill the arrays and a constant 26-step loop to count.
// Space Complexity: O(1) - we use two fixed-size arrays of length 26, irrespective of the input string length.

class Solution {
public:
    int numberOfSpecialChars(string word) {
        
        // Track the last occurrence of lowercase characters
        vector<int> lastSmall(26, -1);
        // Track the first occurrence of uppercase characters
        vector<int> firstCapital(26, -1);

        for(int i = 0; i < word.length(); i++) {
            
            char ch = word[i];

            if(islower(ch)) {
                // Continuously update to keep the last seen index
                lastSmall[ch - 'a'] = i;
            }
            else {
                // Update only if it's the very first time we see this uppercase letter
                if(firstCapital[ch - 'A'] == -1) {
                    firstCapital[ch - 'A'] = i;
                }
            }
        }

        int count = 0;

        for(int i = 0; i < 26; i++) {
            // Check if both forms exist and if the last lowercase is before the first uppercase
            if(lastSmall[i] != -1 && firstCapital[i] != -1 &&
               lastSmall[i] < firstCapital[i]) {
                count++;
            }
        }

        return count;
    }
};

/*
*
* Dry Run
* Input: word = "aaAbcBC"
*
* Initial State:
* lastSmall = [-1, -1, ..., -1]
* firstCapital = [-1, -1, ..., -1]
* count = 0
*
* Iteration over word:
* i=0, ch='a', lastSmall['a'] = 0
* i=1, ch='a', lastSmall['a'] = 1 (updates)
* i=2, ch='A', firstCapital['A'] = 2
* i=3, ch='b', lastSmall['b'] = 3
* i=4, ch='c', lastSmall['c'] = 4
* i=5, ch='B', firstCapital['B'] = 5
* i=6, ch='C', firstCapital['C'] = 6
*
* Final Array Evaluation (0 to 25):
* i=0 ('a'): lastSmall=1, firstCapital=2 => 1 < 2 is True  (count = 1)
* i=1 ('b'): lastSmall=3, firstCapital=5 => 3 < 5 is True  (count = 2)
* i=2 ('c'): lastSmall=4, firstCapital=6 => 4 < 6 is True  (count = 3)
* Other indices remain -1 and are ignored.
* * Result: 3
*
*/
```

## 📝 Notes & Key Insights

* **Fixed Size Advantage:** When working with the English alphabet (lowercase or uppercase), leveraging an array of size 26 is much faster and more space-efficient than using a dynamic Hash Map (`std::unordered_map`). 
* **Constant Space Validation:** Even though we use `vector<int>` objects, their size is fixed at 26 regardless of how large the input `word` grows. Thus, the space complexity firmly remains $O(1)$.
* **Condition Breakdown:** The phrase "every lowercase... before first uppercase" seamlessly translates to mathematical indices: `max(lowercase_indices) < min(uppercase_indices)`.
