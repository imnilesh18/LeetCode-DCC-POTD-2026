## 📝 Problem Statement

You are given an integer array `nums` of length `n`. You start at index `0`, and your goal is to reach index `n - 1`.

From any index `i`, you may perform one of the following operations:
1. **Adjacent Step:** Jump to index `i + 1` or `i - 1`, if the index is within bounds.
2. **Prime Teleportation:** If `nums[i]` is a **prime number** `p`, you may instantly jump to any index `j != i` such that `nums[j] % p == 0`.

Return the **minimum** number of jumps required to reach index `n - 1`.

## ⚠️ Constraints

* $1 \le n == nums.length \le 10^5$
* $1 \le nums[i] \le 10^6$

---

## 🧠 Approach & Explanation

### Brute Force Idea
A naive approach would involve running a standard Breadth-First Search (BFS) where, at every step, we look at `i-1`, `i+1`, and if `nums[i]` is prime, we scan the **entire array** to find elements divisible by `nums[i]` to push into the queue. This would result in $O(N^2)$ time complexity, leading to a Time Limit Exceeded (TLE) error given $N \le 10^5$.

### Optimized Solution (BFS + Sieve of Eratosthenes)
To find the shortest path in an unweighted graph, **BFS** is the most optimal approach. We can heavily optimize the prime teleportation step to achieve passing execution times.

1. **Precompute Primes:** We find the maximum element in `nums` and use the Sieve of Eratosthenes to quickly answer whether any number is prime.
2. **Index Mapping:** We use a Hash Map to store all indices where a specific value occurs. This prevents us from scanning the whole array during teleportations.
3. **Optimized BFS:** We start our BFS queue at index `0`.
   - We check valid adjacent indices (`i - 1` and `i + 1`) and push them if unvisited.
   - If `nums[i]` is prime, we generate all its multiples up to the maximum element in the array. Using our Hash Map, we grab the indices of these multiples and push them to the queue.
4. **The `seen` Set:** To prevent processing the multiples of the *same prime number* multiple times across different indices, we add the processed prime to a `seen` set. This guarantees that prime teleportation edges are computed exactly once per unique prime, optimizing the time complexity significantly.

## 💻 Solution Code

```cpp
// Intuition: Since we need the minimum number of jumps to reach a target, Breadth-First Search (BFS) is the optimal algorithm. The graph edges are formed by adjacent indices and prime divisibility.
// Approach: 
// 1. Map every value to its corresponding indices in the array.
// 2. Use the Sieve of Eratosthenes to precompute primes up to the maximum array element.
// 3. Run a BFS starting from index 0. For each popped index, attempt adjacent steps (i-1, i+1).
// 4. If the value at the current index is prime, generate all its multiples up to the max element. If a multiple exists in the array, push all its indices to the BFS queue.
// 5. Crucially, track processed primes using a `seen` set to avoid repeating the expensive multiple-generation step for the same prime.
// Time Complexity: O(N + maxEl * log(log(maxEl))) where N is the array size and maxEl is the maximum element. BFS processes each node once, and Sieve/multiples are bounded by harmonic series.
// Space Complexity: O(N + maxEl) to store the BFS queue, visited array, frequency map, and the Sieve array.

class Solution {
public:
    vector<bool> isPrime;
    
    // Function to precompute primes using Sieve of Eratosthenes
    void buildSieve(int maxEl) {
        // fill isPrime array, assuming all are prime initially
        isPrime.resize(maxEl+1, true);

        isPrime[0] = false;
        isPrime[1] = false;

        // Standard Sieve logic
        for(int num = 2; num*num <= maxEl; num++) {
            if(isPrime[num]) {
                for(int multiple = num*num; multiple <= maxEl; multiple += num) {
                    isPrime[multiple] = false;
                }
            }
        }
    }

    int minJumps(vector<int>& nums) {
        int n = nums.size();

        unordered_map<int, vector<int>> mp; // element -> list of indices
        int maxEl = 0;
        
        // Map elements to their indices and find the maximum element
        for(int i = 0; i < n; i++) {
            mp[nums[i]].push_back(i);
            maxEl = max(maxEl, nums[i]);
        }

        // build isPrime vector using Sieve
        buildSieve(maxEl); // O(maxEL * loglog(maxEl));

        queue<int> que;
        vector<bool> visited(n, false);
        
        // Start BFS from index 0
        que.push(0);
        visited[0] = true;

        unordered_set<int> seen; // To avoid processing the same prime multiple times

        int steps = 0;
        
        // We visit each node only once - O(n)
        // Overall T.C: O(n + maxEL * loglog(maxEl))
        while(!que.empty()) {
            int size = que.size();
            
            // Process the current BFS level
            while(size--) {
                int i = que.front();
                que.pop();

                // Target reached
                if(i == n-1) {
                    return steps;
                }

                // Adjacent Step: Jump backward
                if(i-1 >= 0 && !visited[i-1]) {
                    que.push(i-1);
                    visited[i-1] = true;
                }

                // Adjacent Step: Jump forward
                if(i+1 <= n-1 && !visited[i+1]) {
                    que.push(i+1);
                    visited[i+1] = true;
                }

                // Skip if current number is not prime or has already been used for teleportation
                if(!isPrime[nums[i]] || seen.count(nums[i])) {
                    continue;
                }

                // Prime Teleportation: Find all multiples up to maxEl
                for(int multiple = nums[i]; multiple <= maxEl; multiple += nums[i]) {
                    // If the multiple doesn't exist in our array, skip it
                    if(!mp.contains(multiple)) {
                        continue;
                    }

                    // Push all indices containing this multiple into the queue
                    for(int &j : mp[multiple]) {
                        if(!visited[j]) {
                            que.push(j);
                            visited[j] = true;
                        }
                    }
                }

                // Mark this prime as processed
                seen.insert(nums[i]);
            }

            steps++; // Increment jump count after processing a full level
        }

        return steps;
    }
};

/*
*
* Dry Run
* Input: nums = [1, 2, 4, 6]
* maxEl = 6
* mp = {1: [0], 2: [1], 4: [2], 6: [3]}
* isPrime precomputed: 1=false, 2=true, 3=true, 4=false, 5=true, 6=false
* 
* BFS Initialization:
* que = [0], visited[0] = true, steps = 0
* 
* Level 0:
* - Pop 0. i = 0, nums[0] = 1.
* - Check i+1: i=1 is unvisited. Push 1, visited[1] = true.
* - nums[0] is not prime. Continue.
* que = [1], steps = 1
* 
* Level 1:
* - Pop 1. i = 1, nums[1] = 2.
* - Check i-1: i=0 is visited.
* - Check i+1: i=2 is unvisited. Push 2, visited[2] = true.
* - nums[1]=2 is prime and not seen.
* - Check multiples of 2: 2, 4, 6.
*   - Multiple 2: exists at index 1 (visited).
*   - Multiple 4: exists at index 2 (already pushed/visited).
*   - Multiple 6: exists at index 3. Push 3, visited[3] = true.
* - Add 2 to seen.
* que = [2, 3], steps = 2
* 
* Level 2:
* - Pop 2. i = 2, nums[2] = 4. Target not met.
* - Pop 3. i = 3. Target reached! (i == n-1).
* - Return steps (2).
*
*/
```

## 🔍 Key Insights & Notes

- **Level-by-Level Graph Traversal:** When a problem asks for the "minimum number of operations/jumps," a graph traversal mapping (specifically BFS) is almost always the intended solution framework.
- **Deduplication:** The `seen` hash set is the cornerstone of the optimization here. Without it, having thousands of occurrences of the same prime number would result in executing the inner `multiple` loop repetitively, causing an inevitable TLE. 
- **Time-Space Tradeoff:** By leveraging `O(maxEl)` space for the Sieve array and Hash Map, we vastly compress the runtime. Space complexity remains entirely reasonable as `nums[i] <= 10^6`.
