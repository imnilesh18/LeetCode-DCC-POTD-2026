## 📖 Problem Statement

You are given an `m x n` integer matrix `grid`, where `m` and `n` are both **even** integers, and an integer `k`.

The matrix is composed of several layers, which is shown in the below image, where each color is its own layer:

<div align="center">
  <img src="https://assets.leetcode.com/uploads/2021/06/10/ringofgrid.png" alt="Grid Layers">
</div>

A cyclic rotation of the matrix is done by cyclically rotating **each layer** in the matrix. To cyclically rotate a layer once, each element in the layer will take the place of the adjacent element in the **counter-clockwise** direction. An example rotation is shown below:

<div align="center">
  <img src="https://assets.leetcode.com/uploads/2021/06/22/explanation_grid.jpg" alt="Rotation Example">
</div>

Return the matrix after applying `k` cyclic rotations to it.

---

## 🚫 Constraints

* `m == grid.length`
* `n == grid[i].length`
* `2 <= m, n <= 50`
* Both `m` and `n` are **even** integers.
* `1 <= grid[i][j] <= 5000`
* `1 <= k <= 10^9`

## 🧠 Approach

Instead of trying to rotate the 2D grid in-place through complex coordinate math, we can drastically simplify the problem by flattening each concentric layer into a 1D array.

1. **Calculate the Number of Layers:**
   Because both the number of rows (`m`) and columns (`n`) are guaranteed to be even, each layer consumes exactly 2 rows and 2 columns. Therefore, the total number of concentric layers in the grid will always be `min(m, n) / 2`.
2. **Layer Traversal:**
   We can iterate through the grid layer by layer (from the outermost layer 0 to the innermost layer). For every layer, we maintain boundaries: `top`, `bottom`, `left`, and `right`.
3. **Extraction:**
   We traverse the perimeter of the current layer in a counter-clockwise direction:
   * Traverse the top row from left to right.
   * Traverse the right column from top to bottom.
   * Traverse the bottom row from right to left.
   * Traverse the left column from bottom to top.
   All extracted elements are pushed into a 1D array `nums`.
4. **Rotation:**
   Since `k` can be as large as 1,000,000,000, rotating `k` times is inefficient. If a layer has `L` elements, rotating it `L` times yields the exact same array. Thus, we normalize `k` using the modulo operator: `normalized_k = k % L`. 
   We then use the C++ standard library function `std::rotate` to shift our 1D array counter-clockwise by `normalized_k` steps.
5. **Re-insertion:**
   Finally, we loop through the exact same boundaries of the current layer (in the exact same counter-clockwise order) and overwrite the grid values with our newly rotated elements from the 1D array.

```cpp
// Intuition:
// Rotating a matrix directly is mathematically complex. Instead, flattening each concentric layer
// into a 1D array makes the rotation trivial. After rotating the 1D array, we can safely map the 
// elements back to their original layer positions.

// Approach:
// 1. Calculate total layers using min(m, n) / 2.
// 2. Iterate layer by layer, defining top, bottom, left, and right boundaries.
// 3. Extract elements of the current layer in counter-clockwise order into a 1D vector `nums`.
// 4. Normalize the rotation `k` using `k % nums.size()` to avoid redundant cycles.
// 5. Use `std::rotate` to effectively shift the 1D array.
// 6. Traverse the grid layer again in the same counter-clockwise order to inject the rotated elements.

// Time Complexity: O(m * n)
// Every element in the m x n grid is visited exactly twice (once to extract, once to replace).

// Space Complexity: O(m + n)
// The space is dominated by the 1D array `nums` used to store the elements of the current layer.
// The outermost layer contains the most elements, which is bounded by O(m + n).

class Solution {
public:
    vector<vector<int>> rotateGrid(vector<vector<int>>& grid, int k) {
        int m = grid.size();
        int n = grid[0].size();

        // Each layer requires two rows and two columns
        int layers = min(m, n) / 2; 

        for (int layer = 0; layer < layers; layer++) {

            vector<int> nums;

            int top = layer;
            int bottom = m - layer - 1;
            int left = layer;
            int right = n - layer - 1;

            // Extract Top row (left -> right)
            for (int j = left; j <= right; j++) {
                nums.push_back(grid[top][j]);
            }

            // Extract Right column (top -> bottom)
            // Skip corners because they are already included in top/bottom rows
            for (int i = top + 1; i <= bottom - 1; i++) {
                nums.push_back(grid[i][right]);
            }

            // Extract Bottom row (right -> left)
            for (int j = right; j >= left; j--) {
                nums.push_back(grid[bottom][j]);
            }

            // Extract Left column (bottom -> top)
            // Skip corners because they are already included in top/bottom rows
            for (int i = bottom - 1; i >= top + 1; i--) {
                nums.push_back(grid[i][left]);
            }

            int len = nums.size();

            // Normalize k to prevent unnecessary redundant rotations
            int nomralized_k = k % len;

            // Rotate the 1D vector using STL
            rotate(begin(nums), begin(nums) + nomralized_k, end(nums));

            int idx = 0;

            // Inject rotated elements back into Top row
            for (int j = left; j <= right; j++) {
                grid[top][j] = nums[idx++];
            }

            // Inject rotated elements back into Right column
            for (int i = top + 1; i <= bottom - 1; i++) {
                grid[i][right] = nums[idx++];
            }

            // Inject rotated elements back into Bottom row
            for (int j = right; j >= left; j--) {
                grid[bottom][j] = nums[idx++];
            }

            // Inject rotated elements back into Left column
            for (int i = bottom - 1; i >= top + 1; i--) {
                grid[i][left] = nums[idx++];
            }
        }

        return grid;
    }
};

/*
*
* Dry Run
*
* Input: grid = [[40,10],[30,20]], k = 1
* m = 2, n = 2
* layers = min(2, 2) / 2 = 1
*
* Layer 0 Execution:
* top = 0, bottom = 1, left = 0, right = 1
* 
* Extraction:
* Top row (j=0 to 1): nums gets 40, 10
* Right col (i=1 to 0): doesn't run
* Bottom row (j=1 to 0): nums gets 20, 30
* Left col (i=0 to 1): doesn't run
* Array nums = [40, 10, 20, 30], len = 4
*
* Rotation:
* normalized_k = 1 % 4 = 1
* rotate(begin, begin + 1, end) transforms nums to [10, 20, 30, 40]
*
* Re-insertion (idx = 0):
* Top row: grid[0][0] = 10, grid[0][1] = 20
* Bottom row: grid[1][1] = 30, grid[1][0] = 40
*
* Return grid: [[10, 20], [40, 30]]
*
*/
```

## 📝 Notes

* A Brute Force approach would literally shift elements in the matrix 1 step at a time and repeat this `k` times. Due to `k <= 10^9`, this would result in a Time Limit Exceeded (TLE) error. The optimized approach prevents this by using mathematical normalization (`k % length_of_layer`).
* Standard Template Library (STL) algorithms like `std::rotate` are heavily optimized and highly recommended for array manipulation to maintain clean and readable code.

## 💡 Key Insights

1. **Problem Decomposition:** Breaking down 2D operations into 1D arrays simplifies boundary and indexing logic substantially.
2. **Modulo Arithmetic:** Large constraints usually hint at cyclic repetitions. Leveraging the modulo operator `k % length` drops runtime from $O(K \times M \times N)$ down to $O(M \times N)$.
3. **Corner Overlaps:** When flattening grid layers manually, extreme care must be taken to not duplicate the corners. Notice how the row loops cover the corners completely, allowing the column loops to safely skip the top and bottom indices.
