# 3043. Find the Length of the Longest Common Prefix

Problem Link: [LeetCode - Find the Length of the Longest Common Prefix](https://leetcode.com/problems/find-the-length-of-the-longest-common-prefix/)
Problem Name: Find the Length of the Longest Common Prefix
Difficulty: Medium
Tags: Array, Hash Table, String, Trie

---

## 📝 Problem Statement

You are given two arrays with positive integers `arr1` and `arr2`.

A **prefix** of a positive integer is an integer formed by one or more of its digits, starting from its leftmost digit. For example, `123` is a prefix of the integer `12345`, while `234` is not.

A **common prefix** of two integers `a` and `b` is an integer `c`, such that `c` is a prefix of both `a` and `b`. For example, `5655359` and `56554` have common prefixes `565` and `5655` while `1223` and `43456` do not have a common prefix.

You need to find the length of the **longest common prefix** between all pairs of integers `(x, y)` such that `x` belongs to `arr1` and `y` belongs to `arr2`.

Return the length of the longest common prefix among all pairs. If no common prefix exists among them, return `0`.

---

## ⛔ Constraints

* `1 <= arr1.length, arr2.length <= 5 * 10^4`
* `1 <= arr1[i], arr2[i] <= 10^8`

---

## 💡 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `arr1 = [1,10,100]`, `arr2 = [1000]` | `3` | There are 3 pairs: (1, 1000) longest prefix is 1. (10, 1000) longest prefix is 10. (100, 1000) longest prefix is 100. The longest common prefix is 100 with a length of 3. |
| **2** | `arr1 = [1,2,3]`, `arr2 = [4,4,4]` | `0` | There exists no common prefix for any pair (arr1[i], arr2[j]), hence we return 0. |

> **Note:** Common prefixes between elements of the same array do not count.

## 🧠 Approach & Explanation

This problem requires us to find the longest common prefix between numbers in two different arrays. Let's break down two different approaches to solve this efficiently.

### Approach 1: Hash Set (Brute Force / Prefix Hashing)
The simplest way to solve this is to generate every possible prefix for all the numbers in `arr1` and store them in a Hash Set. 
1. **Store Prefixes:** For a number like `123`, its prefixes are `123`, `12`, and `1`. We can easily extract these by repeatedly dividing the number by `10` and inserting them into an `unordered_set`.
2. **Find Matches:** Next, we iterate through each number in `arr2`. For each number, we generate its prefixes (again, by dividing by `10`) and check if that prefix exists in our Hash Set. 
3. **Maximize Length:** The first prefix we find that exists in the set is the longest common prefix for that specific number. We calculate its length using the mathematical formula `log10(num) + 1` and continuously update our global maximum result.

### Approach 2: Prefix Tree (Trie)
Whenever a problem revolves around finding or matching prefixes, a **Trie (Prefix Tree)** is naturally the most optimal data structure.
1. **Insert into Trie:** We treat the numbers in `arr1` as strings (or sequences of digits). We build a Trie where each node represents a digit from `0` to `9`. We insert all numbers from `arr1` into this Trie.
2. **Search for Longest Prefix:** Once the Trie is built, we iterate through the numbers in `arr2`. We convert each number to a string and traverse the Trie character by character. 
3. **Calculate Depth:** As long as the current digit exists as a child node in the Trie, we increment our prefix length and move down the tree. The moment we encounter a digit that doesn't exist in the Trie, we stop the search for that number. We keep track of the maximum depth reached across all numbers in `arr2`.

## 💻 Solution Code

### Brute Force (Hash Set)
```cpp
// Intuition: Storing all possible prefixes of the first array in a Hash Set allows for quick O(1) lookups when checking the prefixes of the second array.
// Approach: Iterate through arr1, extract all prefixes by repeatedly dividing by 10, and store them in a HashSet. Then, iterate through arr2, extract its prefixes, check against the HashSet, and maximize the matched length using log10.
// Time Complexity: O(m*log10(M) + n*log10(N)) where m, n are array sizes and M, N are the max values in the arrays.
// Space Complexity: O(m*log10(M)) to store all prefixes from arr1 in the set.

class Solution {
public:
    int longestCommonPrefix(vector<int>& arr1, vector<int>& arr2) {
        unordered_set<int> st;

        // Insert all prefixes of arr1 elements into the set
        for(int val : arr1) {
            while(!st.count(val) && val > 0) {
                st.insert(val);
                val = val / 10; // Remove last digit to get next prefix
            }
        }

        int result = 0; // Tracks longest common prefix length
        
        // Check prefixes of arr2 elements against the set
        for(int num : arr2) {
            while(!st.count(num) && num > 0) {
                num /= 10; // Reduce number until a matching prefix is found
            }

            // If a valid prefix is found, calculate its length
            if(num > 0) {
                result = max(result, static_cast<int>(log10(num)) + 1);
            }
        }

        return result;
    }
};

/*
*
* Dry Run
* arr1 = [1, 10, 100], arr2 = [1000]
*
* 1. Process arr1:
*    - val = 1: insert 1
*    - val = 10: insert 10, insert 1
*    - val = 100: insert 100, insert 10, insert 1
*    Set now contains: {1, 10, 100}
*
* 2. Process arr2:
*    - num = 1000:
*      - 1000 is not in set. num = 1000 / 10 = 100
*      - 100 is in the set!
*      - Calculate length: log10(100) + 1 = 2 + 1 = 3
*      - result = max(0, 3) = 3
*
* Return result: 3
*
*/
```

### Optimized Solution (Trie)
```cpp
// Intuition: Tries are inherently designed for prefix matching. Storing digits of arr1 in a Trie provides an optimal structure to find the longest common prefix for elements in arr2.
// Approach: Build a Trie using the digits of every number in arr1. For every number in arr2, traverse the Trie digit by digit to find the maximum depth (length) we can match.
// Time Complexity: O(m*d + n*d) where m and n are array sizes, and d is the max digits per number.
// Space Complexity: O(m*d) to store the Trie nodes for arr1.

struct TrieNode {
    TrieNode* children[10]; // Array for digits 0-9
};

class Solution {
public:
    // Helper to initialize a new TrieNode
    TrieNode* getTrieNode() {
        TrieNode* node = new TrieNode();
        for(int i = 0; i < 10; i++) {
            node->children[i] = nullptr;
        }
        return node;
    }

    // Insert a number into the Trie
    void insert(int num, TrieNode* root) {
        TrieNode* crawl = root;
        string numStr = to_string(num); // Convert to string for easy iteration

        for(char ch : numStr) {
            int idx = ch - '0';
            if(!crawl->children[idx]) {
                crawl->children[idx] = getTrieNode(); // Create node if missing
            }
            crawl = crawl->children[idx]; // Move to child
        }
    }

    // Returns the length of the longest prefix matched in the Trie
    int search(int num, TrieNode* root) {
        TrieNode* crawl = root;
        string numStr = to_string(num);
        int length = 0;

        for(char ch : numStr) {
            int idx = ch - '0';
            if(crawl->children[idx]) {
                length++; // Prefix matches, increment length
                crawl = crawl->children[idx];
            } else {
                break; // Mismatch found, stop searching
            }
        }

        return length;
    }

    int longestCommonPrefix(vector<int>& arr1, vector<int>& arr2) {
        TrieNode* root = getTrieNode(); // Initialize TRIE root

        // Insert all numbers from arr1
        for(int num : arr1) {
            insert(num, root);
        }

        int result = 0;
        
        // Search all numbers from arr2 and maximize the prefix length
        for(int num : arr2) {
            result = max(result, search(num, root));
        }

        return result;
    }
};

/*
*
* Dry Run
* arr1 = [1, 10, 100], arr2 = [1000]
*
* 1. Insert arr1 into Trie:
*    - 1: root -> [1]
*    - 10: root -> [1] -> [0]
*    - 100: root -> [1] -> [0] -> [0]
*
* 2. Search arr2 in Trie:
*    - num = 1000, string = "1000"
*      - char '1': found in children, length = 1, move down
*      - char '0': found in children, length = 2, move down
*      - char '0': found in children, length = 3, move down
*      - char '0': NOT found in children. Break.
*      - search returns 3.
*
* result = max(0, 3) = 3
*
* Return result: 3
*
*/
```

## 🔍 Key Insights
*   **Logarithmic Length Calculation:** When working directly with integers, the number of digits can be calculated mathematically using `log10(number) + 1` without allocating string memory.
*   **Hash Set Pruning:** In the brute-force approach, stopping the division loop once a number is already found in the set (`!st.count(val)`) prevents redundant operations and optimizes performance.
*   **Trie Efficiency:** Tries naturally deduplicate shared prefixes, meaning memory scales strictly with unique digit paths rather than the total number of characters across all arrays.
