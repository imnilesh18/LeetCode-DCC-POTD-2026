# 1306. Jump Game III

**Difficulty:** Medium 🟡  
**Topics:** `Array`, `Depth-First Search`, `Breadth-First Search`

---

## 📝 Problem Statement

Given an array of non-negative integers `arr`, you are initially positioned at `start` index of the array. When you are at index `i`, you can jump to `i + arr[i]` or `i - arr[i]`, check if you can reach any index with value `0`.

Notice that you can not jump outside of the array at any time.

---

## 📊 Examples

| Example | Input | Output | Explanation |
| :--- | :--- | :--- | :--- |
| **1** | `arr = [4,2,3,0,3,1,2], start = 5` | `true` | All possible ways to reach at index 3 with value 0 are:<br>index 5 -> index 4 -> index 1 -> index 3 <br>index 5 -> index 6 -> index 4 -> index 1 -> index 3 |
| **2** | `arr = [4,2,3,0,3,1,2], start = 0` | `true` | One possible way to reach at index 3 with value 0 is: <br>index 0 -> index 4 -> index 1 -> index 3 |
| **3** | `arr = [3,0,2,1,2], start = 2` | `false` | There is no way to reach at index 1 with value 0. |

---

## 🔐 Constraints

*   $1 \le \text{arr.length} \le 5 \cdot 10^4$
*   $0 \le \text{arr}[i] < \text{arr.length}$
*   $0 \le \text{start} < \text{arr.length}$

---

## 💡 Approach

The problem presents us with a classic graph traversal scenario. At any given index `i`, we have exactly two choices (options) for our next move: jump left to `i - arr[i]` or jump right to `i + arr[i]`. Because we have choices that branch out, this can be efficiently solved using either **Depth-First Search (DFS)** or **Breadth-First Search (BFS)**.

**Step-by-Step Breakdown:**
1.  **Avoid Infinite Loops**: A major edge case in this problem is falling into an infinite loop (e.g., bouncing back and forth between two indices). To prevent this, we must track the indices we have already visited.
2.  **In-Place Visited Array**: Instead of using an external `visited` array (which takes extra space), we can utilize the given array. Since the constraints guarantee that all values are non-negative, we can mark a visited index by negating its value (`arr[i] *= -1`). If we ever encounter a negative number, we know we've been there and should return `false`.
3.  **Boundary Checks**: Ensure that the next jump `i + arr[i]` and `i - arr[i]` stays strictly within the bounds of the array (`>= 0` and `< n`).
4.  **Target Condition**: If at any point the value at our current index is `0`, we have successfully reached our destination and immediately return `true`.

## 💻 Solution Code

### Standard Approach (DFS)
```cpp
// Intuition: The problem requires exploring all valid jumps from a starting index to find a target value (0). Since we have two branching choices at each step, we can use a Depth-First Search (DFS) to explore paths until we find the target or hit a dead end.
// Approach: Recursively explore the left and right jumps. Handle out-of-bounds and previously visited indices (marked as negative) as base cases returning false. If a 0 is found, return true. Mark the current index as visited in-place by multiplying by -1 to save space.
// Time Complexity: O(n) where n is the length of the array, as each index is visited at most once.
// Space Complexity: O(n) for the maximum depth of the recursion stack.
class Solution {
public:
    int n;

    bool dfs(vector<int>& arr, int i) {
        // Base case: Out of bounds or already visited
        if(i < 0 || i >= n || arr[i] < 0) {
            return false;
        }
        
        // Target found
        if(arr[i] == 0)
            return true;

        // Mark as visited in-place
        arr[i] *= -1;
        
        // Explore left and right jumps
        int left  = dfs(arr, i - arr[i]);
        int right = dfs(arr, i + arr[i]);
        
        return left || right;
    }

    bool canReach(vector<int>& arr, int start) {
        n = arr.size();
        return dfs(arr, start);
    }
};

/*
*
* Dry Run
* Input: arr = [4,2,3,0,3,1,2], start = 5
* DFS(5): arr[5] is 1. Mark arr[5] = -1. 
*   Left jump: 5 - 1 = 4. DFS(4).
*     DFS(4): arr[4] is 3. Mark arr[4] = -3.
*       Left jump: 4 - 3 = 1. DFS(1).
*         DFS(1): arr[1] is 2. Mark arr[1] = -2.
*           Left jump: 1 - 2 = -1 (Out of bounds -> false).
*           Right jump: 1 + 2 = 3. DFS(3).
*             DFS(3): arr[3] is 0. Target found! Returns true.
* Returns true all the way up the recursion stack.
* Output: true
*
*/
```

### Iterative Approach (BFS)
```cpp
// Intuition: Instead of going deep recursively, we can explore all reachable indices level by level using a queue. This Breadth-First Search (BFS) prevents deep recursion stacks and evaluates all immediate jump options simultaneously.
// Approach: Start by pushing the initial index into a queue. While the queue is not empty, pop the current index. Check for the target value (0). If found, return true. Calculate valid left and right jumps, push them into the queue, and mark the current index as visited (negative) in-place.
// Time Complexity: O(n) because each node (index) is added to the queue at most once.
// Space Complexity: O(n) for the queue storing the indices to be processed.
class Solution {
public:
    bool bfs(vector<int>& arr, int start, int n) {
        queue<int> que;
        que.push(start);

        while(!que.empty()) {
            int curr = que.front();
            que.pop();

            //reached 0
            if(arr[curr] == 0)
                return true;

            //This was visited before. Ignore
            if(arr[curr] < 0)
                return false; // Slightly safer to continue or return depending on structure, but the code skips before pushing.

            int left  = curr + arr[curr];
            int right = curr - arr[curr];

            // Push valid unvisited neighbors
            if(left >= 0 && left < n)
                que.push(left);
            if(right >= 0 && right < n)
                que.push(right);

            //curr is visited and its neighbours are explored. We don't need it
            arr[curr] = -arr[curr];
        }

        return false;
    }

    bool canReach(vector<int>& arr, int start) {
        int n = (int) arr.size();
        return bfs(arr, start, n);
    }
};

/*
*
* Dry Run
* Input: arr = [4,2,3,0,3,1,2], start = 0
* Queue initializes with [0].
* Pop 0: arr[0] is 4. 
*   Valid jumps: 0 + 4 = 4. Push 4 to queue. Queue: [4]. Mark arr[0] = -4.
* Pop 4: arr[4] is 3. 
*   Valid jumps: 4 - 3 = 1, 4 + 3 = 7 (Out of bounds). Push 1 to queue. Queue: [1]. Mark arr[4] = -3.
* Pop 1: arr[1] is 2.
*   Valid jumps: 1 + 2 = 3. Push 3 to queue. Queue: [3]. Mark arr[1] = -2.
* Pop 3: arr[3] is 0. Target found!
* Output: true
*
*/
```

## 📌 Notes
- Focus on recognizing cyclic dependencies in graph-like structures within arrays. 
- Marking elements in-place (`arr[i] = -arr[i]`) is a fantastic trick to save $O(N)$ extra space, but ensure you clarify with interviewers if modifying the input array is acceptable.

## 🗝️ Key Insights
- The jump options explicitly form a **Directed Graph**.
- The core challenge is completely handling **cycles** without getting trapped. DFS combined with an in-place visited set solves this gracefully.
- Recursive DFS and iterative BFS both yield the same optimal O(N) time bounds, giving you the flexibility to code whichever pattern you're more comfortable with.

## 🚀 Further Exploration
- How would you modify the algorithm if you needed to find the *minimum number of jumps* to reach any zero? (Hint: BFS is inherently better for shortest paths on unweighted graphs).
- What if the array values could be negative initially? (You'd need a separate `visited` boolean array).
