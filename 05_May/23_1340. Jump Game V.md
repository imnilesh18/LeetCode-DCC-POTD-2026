## 📝 Problem Statement

Given an array of integers `arr` and an integer `d`. In one step, you can jump from index `i` to index:

- `i + x` where: `i + x < arr.length` and `0 < x <= d`.
- `i - x` where: `i - x >= 0` and `0 < x <= d`.

In addition, you can only jump from index `i` to index `j` if `arr[i] > arr[j]` and `arr[i] > arr[k]` for all indices `k` between `i` and `j` (More formally `min(i, j) < k < max(i, j)`).

You can choose any index of the array and start jumping. Return the **maximum number of indices** you can visit.

> **Notice** that you can not jump outside of the array at any time.

---

## ⚙️ Constraints

- `1 <= arr.length <= 1000`
- `1 <= arr[i] <= 10^5`
- `1 <= d <= arr.length`

## 💡 Approach & Key Insights

To solve this problem, we need to find the longest chain of valid jumps. 

1. **Jumping Rules**: From a building of height `H` at index `i`, we can look left (up to `d` steps) and right (up to `d` steps).
2. **Visibility Blocker**: If we encounter any building that is taller than or equal to `H` in our path, our line of sight is blocked. We must instantly stop evaluating further steps in that specific direction.
3. **Overlapping Subproblems**: The maximum jumps from index `j` will remain the same regardless of where we came from. This perfectly aligns with **Dynamic Programming**.

### Approach 1: Recursion + Memoization (Top-Down)
We can start a simulation from every single index. A recursive function `solve(i)` will look up to `d` steps left and right. If a valid jump to index `j` is found, it recursively evaluates `1 + solve(j)`. To avoid recomputing, we memoize the answer for each index `i` using a 1D array.

### Approach 2: Sorting + DP (Bottom-Up)
In a bottom-up DP approach, we need to process states such that a problem is solved only after its subproblems are solved. Because we can only jump to **strictly smaller values**, the answer for smaller values must be computed *before* taller values.
By grouping elements with their indices `(value, index)` and **sorting them in ascending order**, we guarantee that when we evaluate a tall building, all valid smaller buildings it could potentially jump to have already been fully processed.

## 💻 Solution Code

### Solution 1: Top-Down Recursion with Memoization

```cpp
// Intuition: We need to find the maximum jumps possible starting from any index. Since we have multiple choices (left and right up to d steps) and overlapping subproblems (an index might be reached from multiple paths), this is a perfect candidate for Dynamic Programming (Memoization).
// Approach:
// 1. Iterate through every index to simulate starting the jump from there.
// 2. For a given index `i`, explore all valid jumps to the left (i-1 down to i-d) and right (i+1 up to i+d).
// 3. Stop checking further in a direction if a height greater than or equal to arr[i] is encountered.
// 4. Memoize the result to avoid redundant calculations.
// Time Complexity: O(N * d), where N is the number of elements. Each index is evaluated once, iterating up to 'd' neighbors.
// Space Complexity: O(N), for the memoization array and recursion call stack depth.
class Solution {
public:
    int n;
    vector<int> t;

    int solve(vector<int>& arr, int d, int i) {
        // Return memoized result if already computed
        if (t[i] != -1)
            return t[i];

        int result = 1; // count current index also

        // move left
        for (int j = i - 1; j >= max(0, i - d); j--) {
            // can't jump further once taller/equal element appears
            if (arr[j] >= arr[i])
                break;
            
            // recursively find the max path length
            result = max(result, 1 + solve(arr, d, j));
        }

        // move right
        for (int j = i + 1; j <= min(n - 1, i + d); j++) {
            // can't jump further once taller/equal element appears
            if (arr[j] >= arr[i])
                break;
            
            // recursively find the max path length
            result = max(result, 1 + solve(arr, d, j));
        }

        // memoize and return
        return t[i] = result;
    }

    int maxJumps(vector<int>& arr, int d) {
        n = arr.size();
        t.resize(n, -1);

        int maxJump = 1;
        // We need to try from every index
        for (int i = 0; i < n; i++) {
            maxJump = max(maxJump, solve(arr, d, i));
        }

        return maxJump;
    }
};

/*
*
* Dry Run
* Input: arr = [3, 3, 3], d = 1
* i = 0: solve(0). Left out of bounds. Right(j=1) arr[1] >= arr[0] (3 >= 3), break. t[0] = 1.
* i = 1: solve(1). Left(j=0) arr[0] >= arr[1] (break). Right(j=2) arr[2] >= arr[1] (break). t[1] = 1.
* i = 2: solve(2). Left(j=1) arr[1] >= arr[2] (break). Right out of bounds. t[2] = 1.
* Max jumps = 1.
*
*/
```

### Solution 2: Bottom-Up DP using Sorting (Optimized Traversal)

```cpp
// Intuition: In a bottom-up DP, we must solve subproblems before the main problem. Since a jump can only go to strictly smaller elements, calculating the answer for shorter heights first ensures that when we evaluate a taller height, all its potential jump targets are already solved.
// Approach:
// 1. Create a list of pairs (value, index) and sort them in ascending order of values.
// 2. Iterate through this sorted list. For each element at original index `i`, simulate jumps to the left and right up to distance `d`.
// 3. Stop if an element >= arr[i] is encountered.
// 4. Update the maximum jumps for `i` using the pre-calculated answers of the smaller targets.
// Time Complexity: O(N log N + N * d), due to initial sorting and checking up to 'd' neighbors for each index.
// Space Complexity: O(N), for the DP array and the vector of value-index pairs.
class Solution {
public:
    int maxJumps(vector<int>& arr, int d) {
        int n = arr.size();
        // t[i] = maximum jumps starting from index i
        vector<int> t(n, 1);

        /*
            In recursion + memoization:
            solve(i) depends on smaller elements only.
            So in bottom-up, process smaller values first.
        */
        vector<pair<int, int>> vec;
        for (int i = 0; i < n; i++) {
            vec.push_back({arr[i], i});
        }
        
        // sort in ascending order based on values
        sort(begin(vec), end(vec));

        for (auto& it : vec) {
            int val = it.first;
            int i   = it.second;

            // move left
            for (int j = i - 1; j >= max(0, i - d); j--) {
                // same breaking condition
                if (arr[j] >= arr[i])
                    break;
                // update current state using pre-computed smaller target
                t[i] = max(t[i], 1 + t[j]);
            }

            // move right
            for (int j = i + 1; j <= min(n - 1, i + d); j++) {
                // same breaking condition
                if (arr[j] >= arr[i])
                    break;
                // update current state using pre-computed smaller target
                t[i] = max(t[i], 1 + t[j]);
            }
        }

        // Return the global maximum from the dp array
        return *max_element(begin(t), end(t));
    }
};

/*
*
* Dry Run
* Input: arr = [7, 6, 5], d = 1
* Vector of pairs: [(7,0), (6,1), (5,2)]. Sorted: [(5,2), (6,1), (7,0)]
* 1. i=2 (val=5): Left(1) arr[1]=6 > 5 (break). t[2]=1.
* 2. i=1 (val=6): Left(0) arr[0]=7 > 6 (break). Right(2) arr[2]=5 < 6. t[1] = max(1, 1+t[2]) = 2.
* 3. i=0 (val=7): Right(1) arr[1]=6 < 7. t[0] = max(1, 1+t[1]) = 3.
* max_element(t) = 3.
*
*/
```

## 📌 Notes & Further Exploration
* **Dependency Mapping in DP**: Identifying dependencies is critical for DP. When state changes depend purely on a measurable attribute (like height/value), sorting transforms a seemingly chaotic state-transition graph into a strictly sequential one.
* Try visualizing this as a **Directed Acyclic Graph (DAG)**. Valid jumps represent directed edges. We are finding the Longest Path in this DAG. Since edges only point to smaller elements, no cycles can exist.
